# 物联网固件分析

## 概述

物联网（IoT）设备固件分析是安全研究的重要方向。随着数十亿IoT设备部署在日常环境中，固件中的安全漏洞可能被远程利用，造成严重后果。固件分析的目标是发现嵌入式设备中的隐藏功能、硬编码凭据、后门账户和可利用的漏洞。

## 固件提取方法

### Binwalk

Binwalk是固件分析最常用的工具之一，具备文件系统雕刻、签名扫描和熵分析三大核心功能。

```bash
# 扫描固件中的文件系统签名
binwalk firmware.bin

# 提取所有识别的文件系统
binwalk -Me firmware.bin

# 熵分析，识别加密或压缩区域
binwalk -E firmware.bin
```

熵分析输出示例：

```
DECIMAL       HEXADECIMAL     ENTROPY
--------------------------------------------------------------------------------
0             0x0             Rising entropy (likely compressed/encrypted)
0x400000      0x400000        Max entropy (encrypted region)
0x800000      0x800000        Low entropy (bootloader, config data)
```

Binwalk依赖签名数据库匹配已知的文件系统类型和压缩算法。当熵值接近1.0时，表示数据被压缩或加密，这是固件分析的关键线索。

### Firmware-Mod-Kit

Firmware-mod-kit用于提取、修改和重新打包固件。它封装了多种文件系统的处理工具：

```bash
# 解包固件
./extract-firmware.sh firmware.bin

# 修改文件系统后重新打包
./build-firmware.sh new-firmware/ firmware.bin
```

支持的RootFS类型包括SquashFS、CramFS、JFFS2、YAFFS2等。修改后重新打包时需注意固件签名校验机制，许多现代设备会校验固件完整性。

### 硬件提取

通过UART（通用异步收发器）串行接口直接从硬件获取Shell接入是最直接的硬件提取方式：

1. 使用万用表或逻辑分析仪识别UART引脚（TX、RX、GND、VCC）
2. 标准UART电压为3.3V（部分设备使用1.8V）
3. 常用波特率：115200、57600、38400、19200、9600
4. 使用USB-TTL适配器连接设备

```bash
# 使用screen连接UART
screen /dev/ttyUSB0 115200

# 或使用minicom
minicom -D /dev/ttyUSB0 -b 115200
```

连接到UART控制台后，若设备未配置登录认证，可直接获得root Shell权限。即使需要登录，也可能发现硬编码的管理员凭据。

## 文件系统分析

常见的IoT固件文件系统类型：

| 文件系统 | 特点 | 适用场景 |
|---------|------|---------|
| SquashFS | 只读、高压缩率 | 主要文件系统 |
| UBIFS | 支持磨损均衡 | NAND Flash设备 |
| JFFS2 | 日志结构文件系统 | 老旧设备 |
| YAFFS2 | 专为NAND优化 | 嵌入式Linux |

解包后的常见分析目标：

```bash
# 搜索硬编码密码
grep -r "password" extracted_fs/etc/
grep -r "admin.*123" extracted_fs/etc/shadow
grep -ri "secret\|key\|token" extracted_fs/ --include="*.conf"

# 检查SSH密钥
ls -la extracted_fs/etc/ssh/
cat extracted_fs/etc/ssh/ssh_host_rsa_key

# 查看web应用配置
cat extracted_fs/www/config.php
```

## 嵌入式设备常见漏洞

### 默认凭据

大量IoT设备出厂时使用默认管理员凭据，用户很少修改：

```csv
厂商,设备类型,用户名,密码
TP-Link,路由器,admin,admin
D-Link,摄像头,admin,(空)
Hikvision,NVR,admin,12345
Dahua,IPC,admin,admin
ZTE,光猫,root,Zte521
```

### Telnet与SSH服务

许多物联网设备默认开启Telnet或SSH调试接口。Telnet以明文传输，攻击者可监听流量截获凭据。SSH虽然加密，但若使用默认密钥或弱密码同样不安全。

### Web接口漏洞

IoT设备通常内置Web管理界面，常见漏洞包括：

- 命令注入（如ping参数未过滤）
- 缓冲区溢出（HTTP头解析）
- 路径遍历（读取任意文件）
- CSRF（跨站请求伪造）
- 未经认证的远程代码执行

命令注入示例：

```http
POST /cgi-bin/ping.cgi HTTP/1.1
Host: 192.168.1.1

param=127.0.0.1;cat /etc/shadow
```

### UPnP漏洞

UPnP（通用即插即用）协议广泛用于IoT设备发现。漏洞包括：

- UPnP服务暴露在WAN口（SSDP攻击）
- 无需认证即可调用SOAP方法
- 内存损坏漏洞

## UART与JTAG调试

### UART引脚识别

UART硬件调试的引脚识别方法：

1. 查找PCB上标记为TX、RX、GND的排针或测试点
2. 使用万用表测量电压：TX通常在空闲时为高电平（3.3V或1.8V）
3. 使用逻辑分析仪捕获波形识别波特率

### JTAG调试

JTAG是更底层的调试接口，提供对CPU的完全控制：

- 标准JTAG：TMS、TCK、TDI、TDO
- 使用Bus Pirate或J-Link连接
- OpenOCD作为开源调试软件

```bash
# OpenOCD配置示例
openocd -f interface/ftdi/jtagkey.cfg \
        -f target/arm966e.cfg \
        -c "init" \
        -c "halt" \
        -c "flash read_bank 0 dump.bin 0 0x100000"
```

### SPI Flash读取

使用Flashrom工具直接读取SPI Flash芯片内容：

```bash
# 读取SPI Flash
flashrom -p linux_spi:dev=/dev/spidev0.0 -r flash_dump.bin

# 识别Flash芯片
flashrom -p linux_spi:dev=/dev/spidev0.0
```

## 固件更新安全

固件更新机制本身是攻击面。常见安全问题：

- 固件未签名（可被篡改）
- 签名验证不严格（仅检查文件名）
- 回滚攻击（允许安装旧版有漏洞固件）
- 更新过程未加密（MITM攻击）

安全更新机制应包含：

1. 数字签名验证（ECDSA或RSA-PSS）
2. 版本号防回滚检查
3. 加密传输通道（TLS）
4. 更新前后完整性校验

## 总结

物联网固件分析是发现嵌入式设备安全问题的核心方法。从binwalk自动化提取到UART硬件调试，从文件系统分析到常见漏洞模式，掌握这些技术对于评估IoT设备安全性至关重要。随着IoT设备数量持续增长，规范厂商的安全开发和引导用户的安全配置是降低风险的关键。
