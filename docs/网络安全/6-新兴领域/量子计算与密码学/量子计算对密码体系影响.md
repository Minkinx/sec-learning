# 量子计算对密码体系影响

## 概述

量子计算对现有密码体系构成根本性威胁。Shor算法和Grover算法分别威胁公钥密码和对称密码的安全性。当前部署的RSA、ECC、DH等公钥密码算法在足够规模的量子计算机面前将完全不安全。

## 量子计算基础

### 量子比特与叠加

与传统比特不同，量子比特（qubit）可以处于|0⟩和|1⟩的叠加态。这使得量子计算机在处理某些特定问题时具有指数级加速能力。

```
经典比特:  0 或 1 (二选一)
量子比特:  α|0⟩ + β|1⟩ (同时处于|0⟩和|1⟩的叠加)
          |α|² + |β|² = 1 (测量概率归一化)
```

### Shor算法

Shor算法由Peter Shor于1994年提出，能够在多项式时间内分解大整数和计算离散对数。这是对当前公钥密码体系的根本性威胁。

#### 算法核心

Shor算法将因式分解问题转化为周期查找问题，利用量子傅里叶变换（QFT）进行加速：

```
目标: 分解 N = p × q

1. 随机选择 a (1 < a < N)
2. 计算 gcd(a, N)
   - 如果 gcd(a, N) ≠ 1, 找到因子，算法结束
3. 用量子电路计算函数 f(x) = a^x mod N
4. 应用量子傅里叶变换查找周期 r (使 a^r ≡ 1 mod N)
5. 如果 r为偶数且 a^(r/2) ≠ -1 mod N:
   - p = gcd(a^(r/2) + 1, N)
   - q = gcd(a^(r/2) - 1, N)
6. 否则重新选择 a
```

#### 受影响算法

| 算法 | 密钥大小 | Shor所需量子比特（估计） | 当前安全性 |
|------|---------|----------------------|-----------|
| RSA-2048 | 2048位 | ~20M物理量子比特 | **完全破解** |
| RSA-4096 | 4096位 | ~40M物理量子比特 | **完全破解** |
| ECDSA P-256 | 256位 | ~10M物理量子比特 | **完全破解** |
| ECDH X25519 | 256位 | ~10M物理量子比特 | **完全破解** |
| EdDSA Ed25519 | 256位 | ~10M物理量子比特 | **完全破解** |
| DSA 2048 | 2048位 | ~20M物理量子比特 | **完全破解** |

### Grover算法

Grover算法由Lov Grover于1996年提出，对无序搜索问题提供二次加速。

```
对N个无序元素的搜索:
  经典: O(N) 次查询
  量子: O(√N) 次查询 (Grover算法)
```

#### 对对称密码的影响

Grover算法将对称密码的有效安全强度减半：

| 算法 | 原安全强度 | Grover后 | 量子安全？ |
|------|-----------|---------|-----------|
| AES-128 | 128位 | 64位 | **不安全** |
| AES-192 | 192位 | 96位 | 边际安全 |
| AES-256 | 256位 | 128位 | 安全 |
| SHA-256 | 128位碰撞 | 85位碰撞 | 基本安全（建议升级） |
| SHA-3-256 | 128位碰撞 | 85位碰撞 | 基本安全（建议升级） |

AES-128被Grover攻击后的64位安全性，对现代计算能力来说不足以抵御暴力破解。而AES-256在Grover攻击下仍有128位安全性，被认为是量子安全的对称密码。

## 受影响的密码体系

### 公钥密码（高风险）

| 密码系统 | 脆弱原因 | 受影响程度 |
|---------|---------|-----------|
| RSA (PKCS#1, OAEP) | Shor因式分解 | 完全破解 |
| DSA (数字签名算法) | Shor离散对数 | 完全破解 |
| ECDH (椭圆曲线DH) | Shor离散对数（ECDLP） | 完全破解 |
| ECDSA (椭圆曲线签名) | Shor离散对数（ECDLP） | 完全破解 |
| EdDSA (Ed25519/448) | Shor离散对数（ECDLP） | 完全破解 |
| SM2 (中国国密) | Shor离散对数（ECDLP） | 完全破解 |
| DH (Diffie-Hellman) | Shor离散对数 | 完全破解 |
| SIKE (已淘汰) | 被经典算法破解 | 非量子原因 |

### 对称密码与哈希（中度风险）

| 算法 | 有效影响 | 缓解措施 |
|------|---------|---------|
| AES-128 | 安全强度降至64位 | 升级AES-256 |
| AES-256 | 安全强度降至128位 | 继续使用（安全） |
| ChaCha20 | 安全强度降至128位（256位密钥） | 可继续使用 |
| SHA-256 | 碰撞安全降至~85位 | 使用SHA-384/512 |
| SHA-3-256 | 碰撞安全降至~85位 | 使用SHA-3-384/512 |
| HMAC | 密钥强度减半 | 增加密钥长度 |

### Harvest Now, Decrypt Later

这是目前最紧迫的威胁之一：

> 攻击者现在采集并存储加密通信数据，等待未来量子计算机能够破解时再进行解密。

```yaml
威胁模型:
  目标: 长期机密性保护的数据
  场景:
    - 政府通信记录
    - 商业机密文件
    - 个人医疗记录
    - 加密备份数据
  风险: 今天加密的数据可能在10-20年后被量子计算机解密
  影响: 敏感数据必须在PQC迁移完成前被视为长期机密

防御策略:
  - 对长期敏感数据立即迁移到PQC加密
  - 使用混合加密方案（经典 + PQC组合）
  - 缩短数据加密的生命周期
  - 评估数据的最小保留期
```

## 密码敏捷性

### 什么是密码敏捷性

密码敏捷性（Cryptographic Agility / Crypto-agility）是指系统能够快速、安全地更换密码算法和参数的能力。

### TLS 1.3混合密钥交换

TLS 1.3已经支持混合密钥交换模式，允许同时使用经典和PQC密钥交换：

```yaml
TLS 1.3 Hybrid Key Exchange:
  ClientHello:
    KeyShare:
      - X25519 (经典)
      - Kyber-768 (PQC)
  
  ServerHello:
    KeyShare:
      - X25519 共享密钥
      - Kyber-768 密文
  
  会话密钥:
    = HKDF( X25519_shared_secret || Kyber_shared_secret )
  
  安全性: 只有两种KEM都被破解时才会暴露
```

### 迁移建议

```bash
# 1. 立即建立密码清单
openssl ciphers -v 'ALL:COMPLEMENTOFALL' | sort

# 2. 启用TLS 1.3并配置PQC混合套件
# nginx配置示例
ssl_protocols TLSv1.2 TLSv1.3;
ssl_ecdh_curve X25519MLKEM768:X25519:prime256v1;
ssl_ciphers TLS_AES_256_GCM_SHA384:TLS_CHACHA20_POLY1305_SHA256;

# 3. 证书和密钥迁移策略
# 短期: 使用RSA-3072+或ECC P-384（中等安全）
# 中期: 配置PQC hybrid证书（X.509支持PQC扩展）
# 长期: 完全迁移到PQC证书

# 4. 敏感数据重新加密
# 对归档数据进行PQC加密
openssl pkeyutl -encrypt -pkeyopt ec:curve X25519MLKEM768 \
    -pubin -inkey hybrid_pub.pem -in data.bin -out data.enc
```

## 组织准备建议

### 高优先级行动

1. **密码清单盘点**：记录所有系统使用的密码算法和密钥长度
2. **数据分类**：识别需要长期机密性保护的数据
3. **制定迁移计划**：设定PQC迁移里程碑

### 迁移路线图样例

```yaml
phase1_评估阶段 (现在 - 2025):
  - 完成密码算法清单
  - 识别关键依赖系统
  - 更新采购标准（要求PQC支持）

phase2_试验阶段 (2025 - 2027):
  - 部署Hybrid TLS实验环境
  - 评估PQC算法性能影响
  - 选择PQC算法套件（NIST标准）

phase3_全面迁移 (2028 - 2030):
  - TLS/SSH全部启用Hybrid PQC
  - 代码签名迁移到PQC
  - VPN/IPsec迁移到PQC
  - 文档签名迁移到PQC

phase4_纯PQC阶段 (2030+):
  - 移除经典公钥密码（TLS/证书/签名）
  - 仅使用NIST标准化的PQC算法
  - 持续评估新的PQC候选算法
```

## 总结

量子计算是密码学领域面临的最深刻的范式转变。Shor算法将完全破坏RSA、ECC和DH等当前所有主流公钥密码算法。虽然实现大规模容错量子计算机仍需时日，但"Harvest Now, Decrypt Later"的攻击模型使长期保护的数据面临即时威胁。密码敏捷性是组织应对这一转变的核心能力，而NIST正在推进的后量子密码标准化为迁移提供了技术基础。
