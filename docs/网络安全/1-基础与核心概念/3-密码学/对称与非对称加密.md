# 对称与非对称加密

> 加密算法是网络安全的核心技术。对称加密速度快、适合大数据量；非对称加密密钥管理更灵活，用于密钥交换和数字签名。现代密码协议通常将两者结合使用。

## 对称加密

### 流密码（Stream Cipher）

流密码以字节或位为单位对数据进行加解密，使用密钥流（Keystream）与明文异或。

| 算法 | 密钥长度 | 状态 | 说明 |
|------|---------|------|------|
| RC4 | 40-2048位 | **已废弃** | 多种弱点（Fluhrer-Mantin-Shamir攻击） |
| ChaCha20 | 256位 | **推荐** | 现代流密码，TLS 1.3标配 |
| Salsa20 | 256位 | 可用 | ChaCha20的前身，略慢 |
| ZUC（祖冲之） | 128位 | 国密标准 | 3GPP LTE加密标准之一 |

**ChaCha20**相对于RC4的优势：
- 无可利用的统计偏差
- 在移动设备上软件实现效率高（无硬件加速需求）
- 配合Poly1305认证加密（AEAD）

### 分组密码（Block Cipher）

分组密码将明文分成固定大小的块（通常是128位）分别加密。

| 算法 | 密钥长度 | 分组大小 | 轮数 | 状态 |
|------|---------|----------|------|------|
| AES-128 | 128位 | 128位 | 10 | **推荐** |
| AES-256 | 256位 | 128位 | 14 | **推荐**（高安全场景） |
| SM4 | 128位 | 128位 | 32 | 国密标准 |
| DES | 56位 | 64位 | 16 | **已废弃**（56位密钥暴力破解） |
| 3DES | 112位 | 64位 | 48 | **已废弃**（生日攻击） |
| Blowfish | 32-448位 | 64位 | 16 | 不推荐（64位分组） |

### AES工作模式

| 模式 | 名称 | 是否需要IV | 并行 | 认证加密 | 推荐度 |
|------|------|-----------|------|---------|--------|
| **ECB** | 电子密码本 | 否 | 是 | 否 | **不推荐**——相同明文块产生相同密文块，泄露模式信息 |
| **CBC** | 密码分组链接 | 是 | 否（解密可并行） | 否 | 可用，需配合HMAC |
| **CFB** | 密码反馈 | 是 | 否 | 否 | 较少使用 |
| **OFB** | 输出反馈 | 是 | 否 | 否 | 较少使用 |
| **CTR** | 计数器 | 是 | 是 | 否 | 高效，可并行 |
| **GCM** | Galois/Counter | 是 | 是 | **是（AEAD）** | **强烈推荐** |
| **CCM** | Counter with CBC-MAC | 是 | 否 | 是 | 用于Wi-Fi WPA2 |
| **Poly1305-ChaCha** | — | 是 | 是 | **是（AEAD）** | TLS 1.3标准 |

**ECB模式的缺陷示例**：
```
原始图像（企鹅）的像素数据经过ECB加密后，由于相同像素块产生相同密文块，
加密后的图像仍然可分辨出企鹅的轮廓。CBC和GCM模式则无此问题。
```

## 非对称加密

| 算法 | 密钥长度（安全强度） | 功能 | 状态 |
|------|-------------------|------|------|
| RSA | 2048位（112位安全性） | 加密、签名、密钥交换 | 广泛使用；推荐2048位以上 |
| RSA | 4096位（128位安全性） | 加密、签名、密钥交换 | 高安全场景 |
| ECC（P-256） | 256位（128位安全性） | 签名、密钥交换 | 推荐（相比RSA更高效） |
| ECC（P-384） | 384位（192位安全性） | 签名、密钥交换 | 高安全场景 |
| SM2 | 256位（128位安全性） | 签名、加密、密钥交换 | 国密标准 |
| X25519 | 256位（≈128位安全性） | 密钥交换 | 推荐（Curve25519） |
| Ed25519 | 256位（≈128位安全性） | 数字签名 | 推荐（EdDSA方案） |

### RSA v.s. ECC 效率对比（128位安全强度）

| 指标 | RSA-3072 | ECC P-256 |
|------|----------|-----------|
| 公钥大小 | 384字节 | 32字节 |
| 私钥大小 | 512字节 | 32字节 |
| 签名速度 | 慢 | 快 |
| 密钥生成 | 非常慢 | 快 |
| 签名大小 | 384字节 | 64字节 |

## 混合加密（Hybrid Encryption）

现代安全协议（TLS、SSH、Signal）无一例外使用混合加密方案：

```
非对称（密钥交换）
  │ ECDHE / RSA-KEM / X25519
  ▼
协商对称密钥（Session Key）
  │ AES-256-GCM / ChaCha20-Poly1305
  ▼
大数据量的对称加密（高效、认证加密）
```

### TLS 1.3 混合示例
```
1. 客户端生成 ephemeral X25519 密钥对
2. 服务器也生成 ephemeral X25519 密钥对
3. 双方通过ECDHE协商出共享密钥（对称密钥）
4. 使用AES-128-GCM或ChaCha20-Poly1305加密应用数据

全程无需RSA私钥操作——前向安全性（Forward Secrecy）保证：即使服务器长期私钥泄露，历史会话也无法解密。
```

### PGP加密混合流程
```
1. 生成临时对称密钥（Session Key）
2. 用对称密钥加密文件内容（AES-256）
3. 用接收方公钥加密对称密钥（RSA-2048）
4. 发送加密的文件 + 加密的对称密钥
接收方用自己的私钥解密对称密钥，再用对称密钥解密文件
```

## QUANTUM 威胁与后量子密码

- RSA-2048和ECC-256在足够大的量子计算机（Shor算法）上可在数小时内破解
- 对称加密（AES）安全性基本不受Shor算法影响（Grover算法仅将有效密钥长度减半：AES-256仍提供128位量子安全性）
- NIST正在标准化后量子加密算法（CRYSTALS-Kyber/Kyber已标准化为ML-KEM；Dilithium标准化为ML-DSA）
