# 国密算法体系

> 国密算法（GM/T标准）是中国国家密码管理局发布的一系列商用密码算法标准。随着等保2.0和关基保护条例的推行，国密算法在政务、金融、电力等关键基础设施中的应用日益广泛。

## 算法概览

| 算法 | 类型 | 输入/密钥长度 | 标准编号 | 发布年份 |
|------|------|-------------|---------|---------|
| SM1 | 对称分组密码 | 128位密钥 | 未公开（硬件实现） | — |
| SM2 | 椭圆曲线公钥密码 | 256位 | GB/T 32918 | 2016 |
| SM3 | 密码杂凑算法 | 256位输出 | GB/T 32905 | 2016 |
| SM4 | 对称分组密码 | 128位密钥/分组 | GB/T 32907 | 2016 |
| SM7 | 对称分组密码 | 128位密钥 | 未公开（RFID专用） | — |
| SM9 | 基于身份的密码 | 256位 | GB/T 38636 | 2020 |
| ZUC（祖冲之） | 流密码 | 128位密钥 | GB/T 33133 | 2016 |

## SM2椭圆曲线公钥密码算法

SM2是基于椭圆曲线密码学（ECC）的国密标准，功能覆盖签名、加密和密钥交换。

### 椭圆曲线参数
- 基础域：F(p)，p为256位素数
- 曲线方程：y² = x³ + ax + b
- a = FFFFFFFE FFFFFFFF FFFFFFFF FFFFFFFF FFFFFFFF 00000000 FFFFFFFF FFFFFFFC
- b = 28E9FA9E 9D9F5E34 4D5A9E4B CF6509A7 F39789F5 15AB8F92 DDBCBD41 4D940E93
- 基点G = (Gx, Gy)，阶n = 2^256 - 36185027886661311069865932815214970195999553649087271998458798928265893368671

### 功能
- **数字签名**：SM2签名算法（类似ECDSA，但算法细节不同）
- **公钥加密**：SM2加密算法（基于ECIES，密文包含C1、C2、C3三部分）
- **密钥交换**：SM2密钥交换协议（类似ECDH，带认证）

### SM2签名与一般ECC签名的区别
```python
# SM2签名生成（简化伪代码）
# 1. 计算 Z = H(ENTLA || IDA || a || b || xG || yG || xA || yA)
# 2. 计算 e = hash(Z || M)
# 3. 生成随机数 k
# 4. 计算椭圆曲线点 (x1, y1) = kG
# 5. 计算 r = (e + x1) mod n
# 6. 计算 s = ((1 + dA)^(-1) * (k - r * dA)) mod n
# 签名 = (r, s)

# SM2签名验证
# 1. 同方法计算 e
# 2. 计算 t = (r + s) mod n
# 3. 计算椭圆曲线点 (x1, y1) = sG + tPA
# 4. 验证 R = (e + x1) mod n == r
# 差异点：SM2在签名中引入了用户身份标识和曲线参数（Z值）
```

## SM3密码杂凑算法

SM3是中国国家标准哈希算法，与SHA-256功能等价但算法结构不同。

| 特性 | SM3 | SHA-256 |
|------|-----|---------|
| 输出长度 | 256位 | 256位 |
| 消息分组 | 512位 | 512位 |
| 轮数 | 64轮（16×4轮） | 64轮 |
| 字长 | 32位 | 32位 |
| 初始值 | 8个32位常数（与SHA-256不同） | 8个32位常数 |
| 压缩函数 | 更复杂的消息扩展（含置换） | 标准Merkle-Damgard |
| 安全性 | 与SHA-256相当 | 广泛验证 |

```
SM3压缩函数特点：
- 消息扩展：W[j] = P_1(W[j-16] ⊕ W[j-9] ⊕ ROTL_15(W[j-3])) ⊕ ROTL_7(W[j-13]) ⊕ W[j-6]
- 置换函数：P_0(X) = X ⊕ ROTL_9(X) ⊕ ROTL_17(X)
- 与SHA-256不同，SM3使用更复杂的消息扩展和置换保证安全性
```

## SM4分组密码算法

SM4是中国商用密码标准中的分组加密算法，广泛应用于VPN、无线局域网和数据库加密。

### 算法结构
- **分组大小**：128位
- **密钥长度**：128位
- **轮数**：32轮
- **结构**：Feistel网络（不平衡）
- **密钥扩展**：32轮密钥拓展（使用S盒和线性变换）

### 加密轮函数
```
输入：(X[i], X[i+1], X[i+2], X[i+3])
轮密钥：rk[i]
作用：X[i+4] = X[i] ⊕ F(X[i+1], X[i+2], X[i+3], rk[i])

其中F包括：
1. 非线性变换 τ：4个并行S盒（8×8位S盒）
2. 线性变换 L：B ⊕ (B <<< 2) ⊕ (B <<< 10) ⊕ (B <<< 18) ⊕ (B <<< 24)
3. 与轮密钥异或
```

### SM4工作模式
SM4支持与AES相同的分组工作模式：

| 模式 | 用法 | 推荐度 |
|------|------|--------|
| ECB | 每块独立加密 | ❌ 不推荐（与AES-ECB相同缺陷） |
| CBC | 需要IV，串行加密 | ✅ 可用（需配合MAC） |
| CTR | 需要IV，可并行 | ✅ 可用 |
| GCM | 认证加密，可并行 | ✅ **强烈推荐** |

### SM4-GCM用法（概念）
```
SM4-GCM对明文进行加密的同时计算认证标签，提供机密性和完整性
不再需要单独的HMAC——AEAD模式简化协议设计
```

## SM9基于身份的密码算法

SM9无需数字证书，直接使用用户身份标识（如邮箱、手机号）作为公钥：
- 私钥由密钥生成中心（KGC）生成和分发
- 简化了证书管理（无PKI基础设施）
- 适用于物联网等大规模设备场景
- 安全基础：双线性配对（Bilinear Pairing）

## ZUC（祖冲之）流密码

- 128位密钥和128位初始向量
- 用于3GPP LTE/LTE-A的加密和完整性保护
- 被国际标准采纳（3GPP TS 35.221/222）

## 合规要求

### 等保2.0要求
根据GB/T 22239-2019《信息安全技术 网络安全等级保护基本要求》：
- **三级及以上系统**：应使用国密算法进行密码保护
- 关键领域包括：电子政务、金融、教育、医疗、能源

### 典型应用场景
| 场景 | 使用算法 | 合规要求 |
|------|---------|---------|
| HTTPS国密改造 | SM2证书 + SM4-GCM | 政务网站 |
| 电子政务OA | SM2签名 + SM4加密 | 等保三级 |
| 金融IC卡 | SM2 + SM4 | 金融行业标准 |
| 社保系统 | SM3虹膜/指纹哈希 + SM4加密 | 人社部要求 |
| 电子印章 | SM3摘要 + SM2签名 | 《电子签名法》 |
| VPN设备 | SM2密钥交换 + SM4-GCM | 密评要求 |
