# 网络拓扑与分段

> 网络拓扑设计直接影响安全架构的有效性。合理的网络分段是实现纵深防御的基础。

## 常见网络拓扑

- **星型拓扑**：中心交换机连接所有节点，便于管理但存在单点故障
- **层次化拓扑**：核心层-汇聚层-接入层三层架构，企业网络标准
- **Spine-Leaf**：数据中心主流架构，提供东西向流量高性能

## 网络分段策略

### VLAN分段
- 按业务部门或应用划分广播域
- VLAN间通过三层设备路由，可部署ACL控制流量
- 避免VLAN跳跃攻击（禁用DTP自动协商）

### 物理分段
- 敏感系统使用独立网络基础设施
- 生产网与办公网物理隔离
- 测试环境与生产环境分离

### 逻辑分段（VxLAN/网络虚拟化）
- Overlay网络技术，适用于虚拟化和多云环境
- VNI（VXLAN Network Identifier）支持1600万+隔离网络

## 安全区域划分

| 区域 | 说明 | 典型安全策略 |
|------|------|-------------|
| 外部区域 | 互联网 | 严格入向ACL、WAF、DDoS防护 |
| DMZ | 对外服务区 | 有限入向访问，严格出向控制 |
| 内网区域 | 内部业务系统 | 细粒度访问控制，东西向流量检测 |
| 管理区域 | 运维管理 | 跳板机、双因素认证、操作审计 |
| 高安全区域 | 核心数据 | 物理隔离、严格审批、全面审计 |

## 微隔离

在零信任架构中引入微隔离概念——在更细粒度（进程/容器级别）实施网络隔离，突破传统基于IP/端口的边界控制。
