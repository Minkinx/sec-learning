# 系统加固基线

> 系统加固是安全运营的基础性工作，通过消除不必要的服务、收紧权限配置、建立审计机制等手段降低攻击面。本文涵盖账号安全、服务管理、补丁管理和日志审计四个核心维度。

## 账号安全加固

### 内置账号管理
- **禁用Guest账户**：Windows Guest和Linux nfsnobody等匿名账户必须禁用
- **Administrator/root重命名**：避免使用默认管理员名（可增加暴力破解难度）
- **清理僵尸账号**：定期审查和删除离职员工及过期服务账号

### 密码策略基准

| 策略项 | 推荐配置 | 依据 |
|--------|---------|------|
| 密码最小长度 | 12字符 | NIST SP 800-63B |
| 密码复杂性 | 必须包含3/4类字符 | 行业最佳实践 |
| 密码历史 | 禁止最近24个密码 | 等保2.0 |
| 最长密码期限 | 90天 | 等保2.0一级要求 |
| 最短密码期限 | 1天 | 防止立即循环更改 |
| 账户锁定阈值 | 5次失败 | 等保2.0 |
| 锁定时间 | 30分钟 | 行业标准 |
| 重置计数器 | 30分钟 | 行业标准 |

### Linux密码策略配置
```bash
# /etc/pam.d/common-password 或 /etc/security/pwquality.conf
minlen = 12
dcredit = -1        # 至少1个数字
ucredit = -1        # 至少1个大写字母
lcredit = -1        # 至少1个小写字母
ocredit = -1        # 至少1个特殊字符
difok = 3           # 新旧密码至少3个不同字符
enforce_for_root    # root用户也受约束
```

### Windows密码策略配置
通过组策略管理：计算机配置 > Windows设置 > 安全设置 > 账户策略 > 密码策略

## 服务与端口管理

### 禁用高危服务

| 服务/协议 | 端口 | 风险 | 替代方案 |
|-----------|------|------|----------|
| Telnet | 23 | 明文传输凭据 | SSH v2 |
| FTP | 21 | 明文传输 | SFTP/FTPS |
| SMB v1 | 445 | WannaCry/LockBit传播 | SMB v3+ |
| SNMP v1/v2c | 161 | 明文Community字符串 | SNMP v3 |
| RSH/Rlogin | 513/514 | 无加密认证 | SSH |
| TFTP | 69 | 无认证 | SFTP |

### 防火墙配置原则
- 默认拒绝（Default Deny）入站流量
- 最小开放原则（只开放业务必需端口）
- 按区域配置入站/出站规则

```bash
# iptables 默认策略示例
iptables -P INPUT DROP
iptables -P FORWARD DROP
iptables -P OUTPUT ACCEPT
iptables -A INPUT -m state --state ESTABLISHED,RELATED -j ACCEPT
iptables -A INPUT -p tcp --dport 22 -s 10.0.0.0/8 -j ACCEPT
iptables -A INPUT -p tcp --dport 443 -j ACCEPT
iptables -A INPUT -j LOG --log-prefix "DROP: "
```

### Windows防火墙配置（PowerShell）
```powershell
# 禁用入站规则（默认阻断）
Set-NetFirewallProfile -Profile Domain,Public,Private -DefaultInboundAction Block

# 允许特定端口
New-NetFirewallRule -Name "Allow_HTTPS" -DisplayName "HTTPS" `
  -Direction Inbound -Protocol TCP -LocalPort 443 -Action Allow
```

## 补丁管理

| 补丁类型 | 更新周期 | 说明 |
|----------|---------|------|
| 安全更新 | 紧急（0-7天） | 关键漏洞补丁（如RCE、EoP） |
| 月度汇总 | 每月 | Microsoft Patch Tuesday风格的常规更新 |
| 功能更新 | 半年/季度 | 新功能与增强 |
| 驱动程序 | 按需 | 仅在有安全问题或兼容性问题时更新 |

### 补丁管理流程
1. **评估**：扫描漏洞（OpenVAS、Nessus、Qualys）
2. **分类**：按CVSS评分和资产重要性确定优先级
3. **测试**：在测试环境验证兼容性（至少7天）
4. **部署**：分批部署（先非关键系统，后生产系统）
5. **验证**：确认补丁生效，未引入新问题
6. **回滚预案**：准备补丁卸载流程

## 日志与审计

### 审计策略基准

| 审计事件 | Windows | Linux |
|----------|---------|-------|
| 登录成功/失败 | 启用 | `/var/log/auth.log` |
| 账户创建/删除 | 启用 | `/var/log/secure` |
| 特权使用 | 启用 | auditd规则 |
| 策略变更 | 启用 | `auditctl -w /etc/security/` |
| 文件访问（关键） | 用户配置SACL | `auditctl -w /etc/shadow -p wa` |
| 进程创建（含命令行） | 事件4688 | `auditctl -a always,exit -S execve` |

### 集中日志管理
- **syslog-ng/rsyslog**：Linux集中日志采集
- **Windows Event Forwarding (WEF)**：Windows事件转发
- **SIEM**：Splunk、ELK、Wazuh、Azure Sentinel

### 日志保留要求（等保2.0）
- 日志保存时间 >= 6个月
- 日志包括：系统日志、安全日志、应用日志、网络设备日志
- 日志内容：时间戳、用户、事件类型、结果、源IP

## 常用加固工具

| 工具 | 适用平台 | 功能 |
|------|---------|------|
| CIS Benchmarks | 全平台 | 行业标准加固基准文档 |
| OpenSCAP | Linux | 自动化合规扫描与加固 |
| Lynis | Linux/Unix | 系统安全审计工具 |
| Microsoft Security Compliance Toolkit | Windows | 官方基线策略包 |
| Ansible Galaxy (devsec.hardening) | Linux/Windows | 自动化加固playbook |
| Security Hub / Defender for Cloud | 云 | 云安全态势管理 |

### Lynis审计示例
```bash
# 安装
apt install lynis

# 执行审计
lynis audit system

# 查看报告
less /var/log/lynis-report.dat
```

### Microsoft Security Compliance Toolkit
```powershell
# 导入安全基线（Windows 11）
$PolicyPath = "C:\Baselines\Windows 11 v23H2 Security Baseline\"
$LGPO = "C:\Baselines\LGPO\LGPO.exe"

# 应用基线策略
& $LGPO /g $PolicyPath
```
