# 成员推理攻击

> 成员推理攻击（Membership Inference Attack, MIA）的目标是判断一个特定的数据点是否被用于训练目标模型。这种攻击直接威胁到训练数据的隐私——如果攻击者能够确定某个人的医疗记录被用于训练模型，就间接暴露了这个人的健康状况信息。

## 攻击原理

### 基本思想

成员推理攻击利用了模型在训练数据上的**过拟合特性**：模型在训练数据上的预测通常比在非训练数据上更自信（置信度更高）。攻击者通过分析模型对目标样本的预测置信度来判断该样本是否属于训练集。

```text
模型对训练成员的预测分布：
置信度：████████████████ 0.95 (高)

模型对非成员的预测分布：
置信度：████████ 0.60 (较低)

判断规则：置信度 > 阈值 → 推断为"成员"
```

### 黑盒攻击流程

最基本的攻击形式只需访问模型的预测置信度：

```python
def basic_membership_inference(model, target_record, threshold=0.8):
    """最基础的成员推理攻击"""
    model.eval()
    with torch.no_grad():
        output = model(target_record.unsqueeze(0))
        probs = F.softmax(output, dim=1)
        confidence = probs.max().item()  # 最高置信度分数
        
    # 高置信度暗示该样本可能是训练成员
    return confidence > threshold
```

| 攻击类型 | 所需信息 | 准确率 | 复杂度 |
|---------|---------|-------|--------|
| 基础（基于置信度） | 预测概率向量 | 中等 | 低 |
| 基于损失 | loss值 | 中高 | 低 |
| LiRA（似然比） | 多影子模型输出 | 高 | 高 |
| 标签只暴露 | top-1标签 + 梯度 | 中 | 中 |

## LiRA（Likelihood Ratio Attack）

LiRA 是当前最先进的成员推理攻击方法，由 Carlini 等人提出。其核心思想是：对每个目标样本，训练多个"影子模型"，其中一些模型在包含该样本的数据集上训练，另一些在不包含该样本的数据集上训练，然后比较它们的置信度分布。

```python
class LiRAAttack:
    """LiRA: 似然比成员推理攻击"""
    
    def __init__(self, architecture, dataset_size):
        self.arch = architecture
        self.shadow_models = []
        self.n_shadow = 128  # 影子模型数量
        
    def train_shadow_models(self, target_sample):
        """训练影子模型用于推断特定样本的成员状态"""
        in_membership_probs = []  # 包含目标样本的模型输出
        out_membership_probs = []  # 不包含目标样本的模型输出
        
        for i in range(self.n_shadow):
            # 构建包含/不包含目标样本的训练集
            if i % 2 == 0:
                dataset = self.build_dataset(include_sample=target_sample)
                is_member = True
            else:
                dataset = self.build_dataset(exclude_sample=target_sample)
                is_member = False
            
            # 训练影子模型
            shadow = self.train_shadow(self.arch, dataset)
            
            # 记录对目标样本的预测置信度
            with torch.no_grad():
                prob = F.softmax(shadow(target_sample), dim=1)
                confidence = prob.max().item()
            
            if is_member:
                in_membership_probs.append(confidence)
            else:
                out_membership_probs.append(confidence)
        
        return in_membership_probs, out_membership_probs
    
    def attack(self, target_model, target_sample):
        """执行成员推断"""
        in_probs, out_probs = self.train_shadow_models(target_sample)
        
        # 构建成员和非成员的置信度分布
        in_dist = np.array(in_probs)
        out_dist = np.array(out_probs)
        
        # 计算似然比
        with torch.no_grad():
            target_conf = F.softmax(target_model(target_sample), dim=1).max().item()
        
        # 估计在in分布和out分布下的概率密度
        p_in = self.gaussian_kde(in_dist).pdf([target_conf])[0]
        p_out = self.gaussian_kde(out_dist).pdf([target_conf])[0]
        
        # 似然比 > 1 → 更可能是成员
        likelihood_ratio = p_in / (p_out + 1e-10)
        return likelihood_ratio > 1
```

## 训练数据提取攻击

对于生成式模型（特别是 LLM），攻击者可以直接从模型中提取训练数据中的敏感信息：

### 记忆化（Memorization）

模型会"记住"训练数据中的特定模式，尤其是：

- **重复数据**：在训练集中出现多次的样本更容易被记忆
- **异常值**：独特、不常见的序列（如信用卡号、API密钥）
- **命名实体**：人名、地址、电话号码等结构化信息

```python
def extraction_attack(model, prompt_template, n_attempts=10000):
    """从生成模型中提取训练数据"""
    extracted_sequences = set()
    
    for i in range(n_attempts):
        # 使用多样化提示诱导模型输出记忆化序列
        prompt = f"Please repeat the following phrase: {generate_random_prefix()}"
        output = model.generate(prompt, max_tokens=50)
        
        # 检测输出中是否包含疑似训练数据的模式
        if contains_sensitive_pattern(output):
            extracted_sequences.add(output)
    
    return extracted_sequences

def contains_sensitive_pattern(text):
    """检测训练数据提取攻击的成功标志"""
    patterns = [
        r'\b\d{16}\b',                   # 信用卡号
        r'[A-Za-z0-9._%+-]+@[A-Za-z0-9.-]+\.[A-Z|a-z]{2,}',  # 邮箱
        r'\b\d{3}-\d{2}-\d{4}\b',         # SSN
        r'API[_-]?KEY[_-]?\w+',           # API密钥
    ]
    return any(re.search(p, text) for p in patterns)
```

## 差分隐私防御（DP-SGD）

DP-SGD（Differentially Private SGD）是抵抗成员推理攻击最有效的方法：

### 实现原理

```python
def dpsgd_training_step(model, x, y, optimizer, l2_clip_norm=1.0, noise_multiplier=1.0):
    """差分隐私SGD训练步骤"""
    batch_size = x.shape[0]
    
    # Step 1: 计算每个样本的梯度
    per_sample_grads = []
    for i in range(batch_size):
        output = model(x[i:i+1])
        loss = nn.CrossEntropyLoss()(output, y[i:i+1])
        loss.backward()
        
        # 收集每个参数的梯度
        sample_grad = [p.grad.clone() for p in model.parameters()]
        per_sample_grads.append(sample_grad)
        model.zero_grad()
    
    # Step 2: 梯度裁剪 - 限制每个样本梯度的L2范数
    clipped_grads = []
    for sample_grad in per_sample_grads:
        total_norm = sum(p.norm(2).item() for p in sample_grad) ** 0.5
        scaling = min(1, l2_clip_norm / (total_norm + 1e-6))
        clipped = [p * scaling for p in sample_grad]
        clipped_grads.append(clipped)
    
    # Step 3: 聚合梯度 + 添加高斯噪声
    aggregated = [sum(g[i] for g in clipped_grads) for i in range(len(clipped_grads[0]))]
    noisy_grads = []
    for grad in aggregated:
        noise = torch.randn_like(grad) * l2_clip_norm * noise_multiplier
        noisy_grads.append((grad + noise) / batch_size)
    
    # Step 4: 应用梯度更新
    for p, g in zip(model.parameters(), noisy_grads):
        p.grad = g
    optimizer.step()
```

### 隐私预算（Privacy Budget）

在差分隐私中，使用 $(\epsilon, \delta)$ 来衡量隐私保护强度：

| $\epsilon$ 值 | 隐私保护强度 | 模型性能 | MIA防御有效性 |
|---------------|-------------|---------|--------------|
| 0.1 | 非常强 | 显著下降 | 极强 |
| 1.0 | 强 | 中度下降 | 强 |
| 8.0 | 中等 | 轻微下降 | 中等 |
| ∞（无DP） | 无 | 最优 | 无 |

## 防御对比

| 防御方法 | 对MIA的有效性 | 对模型性能影响 | 计算开销 |
|---------|--------------|--------------|---------|
| DP-SGD | 强（可证明） | 中-高 | 高 |
| 数据去重 | 中等 | 轻微提升 | 低 |
| 正则化（Dropout, Weight Decay） | 弱 | 轻微下降 | 极低 |
| 记忆化缓解 | 中 | 轻微 | 低 |
| 输出限制 | 弱 | 无 | 无 |

成员推理攻击的核心启示是：**模型的预测行为会泄露训练数据的隐私信息**。即使模型不直接输出训练数据，其置信度分布也可能暴露某个样本是否被用于训练。对于处理敏感数据的应用（医疗、金融），部署差分隐私训练是保护用户隐私的必要措施。
