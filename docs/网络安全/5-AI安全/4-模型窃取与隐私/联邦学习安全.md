# 联邦学习安全

> 联邦学习（Federated Learning, FL）允许多个参与方在不共享原始数据的情况下协同训练机器学习模型。然而，联邦学习引入的分布式架构也带来了新的安全威胁——梯度泄露、拜占庭攻击、模型投毒等。这些威胁可能导致参与方的隐私数据被重构，或全局模型的完整性被破坏。

## 联邦学习架构

```text
              ┌──────────────────────┐
              │    中央聚合服务器      │
              │  (FedAvg / 鲁棒聚合)  │
              └──────┬───────────┬───┘
                     │           │
              ┌──────▼───┐ ┌───▼──────┐
              │ 客户端 A  │ │ 客户端 B  │ ...
              │ (本地数据) │ │ (本地数据) │
              └──────────┘ └──────────┘
```

标准 FedAvg 算法流程：

1. 服务器分发全局模型 $\theta^t$
2. 每个客户端 $k$ 在本地数据上训练：$\theta_k^{t+1} = \theta^t - \eta \nabla L_k(\theta^t)$
3. 客户端上传模型更新 $\Delta_k^{t+1} = \theta_k^{t+1} - \theta^t$
4. 服务器聚合更新：$\theta^{t+1} = \theta^t + \frac{1}{K} \sum_{k=1}^K \Delta_k^{t+1}$

## 梯度泄露攻击（Gradient Leakage）

梯度泄露攻击的核心发现是：**共享的梯度会泄露原始训练数据的信息**。对于一个给定的梯度，攻击者可以通过优化方法反向重构出产生该梯度的训练数据。

### DLG（Deep Leakage from Gradients）

DLG 由 Zhu 等人提出，通过以下优化过程恢复训练数据：

```python
def dlg_attack(gradient, model, input_shape, num_iterations=1000):
    """Deep Leakage from Gradients: 从梯度中重建训练数据"""
    # 初始化：随机生成假数据和假标签
    dummy_data = torch.randn(input_shape, requires_grad=True)
    dummy_label = torch.randn((1, num_classes), requires_grad=True)
    
    optimizer = torch.optim.LBFGS([dummy_data, dummy_label], lr=0.1)
    
    for i in range(num_iterations):
        def closure():
            optimizer.zero_grad()
            
            # 前向传播
            dummy_output = model(dummy_data)
            # 计算损失
            dummy_loss = criterion(dummy_output, dummy_label)
            
            # 计算假梯度
            dummy_gradient = torch.autograd.grad(
                dummy_loss, model.parameters(), create_graph=True
            )
            
            # 最小化真实梯度与假梯度的差异
            grad_diff = 0
            for dg, rg in zip(dummy_gradient, gradient):
                grad_diff += ((dg - rg) ** 2).sum()
            
            grad_diff.backward()
            return grad_diff
        
        optimizer.step(closure)
        
        if i % 100 == 0:
            print(f"迭代 {i}, 梯度差异: {closure().item():.6f}")
    
    return dummy_data.detach(), dummy_label.detach()
```

### GotenGL

GotenGL 改进了 DLG 方法，通过引入以下技巧提高重建质量：

- **数据先验**：使用 TV（Total Variation）正则化鼓励平滑性
- **批量重建**：一次性重建 batch 中的所有样本（Batch-Level DLG）
- **标签恢复**：使用梯度符号分析推断标签

| 攻击方法 | 重建质量 | 迭代次数 | 受保护能力 |
|---------|---------|---------|-----------|
| DLG | 中等 | ≥1000 | 易受梯度扰动影响 |
| iDLG | 高（可恢复标签） | ≥500 | 中等 |
| GotenGL | 很高（批量恢复） | ≥200 | 强 |
| 组级梯度泄露 | 中等（聚合梯度） | ≥1000 | 强 |

## 恶意聚合器攻击

如果中央聚合器被攻陷或本身就是恶意的，它可以偏离协议来推断客户端隐私数据：

```python
class MaliciousAggregator:
    """恶意聚合器：在聚合过程中窃取客户端数据"""
    
    def __init__(self):
        self.reconstructed_data = {}
    
    def malicious_aggregate(self, client_updates):
        """恶意聚合过程"""
        # 不执行正常的FedAvg，而是尝试重建数据
        for client_id, (gradients, n_samples) in client_updates.items():
            # 使用DLG从梯度重建数据
            recovered_data = dlg_attack(gradients, self.global_model, 
                                       input_shape=self.input_shape)
            self.reconstructed_data[client_id] = recovered_data
        
        # 执行正常的聚合以不被发现
        return standard_fedavg(client_updates)
```

## 拜占庭攻击（Byzantine Attack）

拜占庭攻击是指恶意客户端提交任意的模型更新，旨在破坏全局模型的收敛性或植入后门。

### 攻击类型

| 攻击类型 | 具体方法 | 检测难度 | 影响 |
|---------|---------|---------|------|
| 随机梯度 | 提交随机噪声替代真实更新 | 低 | 降低收敛速度 |
| 标签翻转 | 在本地数据上翻转标签后训练 | 低 | 降低准确率 |
| 后门注入 | 在本地数据上植入后门后更新 | 高 | 后门 + 正常表现 |
| 模型替换 | 提交大梯度覆盖其他客户端的更新 | 中 | 最大化攻击影响 |
| 自适应攻击 | 根据防御机制动态调整 | 极高 | 难以防御 |

### Byzantine-robust 聚合规则

| 聚合规则 | 原理 | Byzantine容错比例 |
|---------|------|-----------------|
| FedAvg | 简单平均 | 0%（无法抵抗） |
| Krum | 选择与其他更新距离最近的更新 | ≤50%恶意 |
| Trimmed Mean | 去除极端值后平均 | ≤25%恶意 |
| Median | 逐维取中位数 | ≤50%恶意 |
| Bulyan | Krum + Trimmed Mean | ≤33%恶意 |
| FLTrust | 基于服务器验证数据的加权平均 | 较高 |

```python
def krum_aggregation(updates, n_malicious=1):
    """Krum拜占庭鲁棒聚合"""
    n = len(updates)
    distances = torch.zeros(n, n)
    
    # 计算所有更新对之间的欧氏距离
    for i in range(n):
        for j in range(n):
            distances[i][j] = sum(
                (updates[i][p] - updates[j][p]).norm(2).item()
                for p in range(len(updates[i]))
            )
    
    # 对每个更新，计算到最近的 n - n_malicious - 2 个更新的距离之和
    scores = []
    for i in range(n):
        sorted_dists = torch.sort(distances[i])[0]
        score = sorted_dists[1:n-n_malicious-1].sum()  # 排除自己
        scores.append(score)
    
    # 选择总分最小的更新
    best_idx = torch.tensor(scores).argmin().item()
    return updates[best_idx]
```

## 安全聚合（Secure Aggregation）

安全聚合通过密码学手段防止服务器看到单个客户端的更新：

### 秘密共享（Secret Sharing）

客户端将自己的更新拆分为多个秘密碎片分发给其他客户端，只有足够多的碎片才能重建原始更新：

```python
def secure_aggregation(client_updates, threshold=3):
    """基于秘密共享的安全聚合"""
    n_clients = len(client_updates)
    # Shamir's 秘密共享：将每个更新拆分为n份
    shares = {}
    for client_id, update in client_updates.items():
        # 生成多项式：f(x) = update + a1*x + a2*x^2 + ... 
        coeffs = [update] + [torch.randn_like(update) for _ in range(threshold-1)]
        
        for other_id in range(n_clients):
            x_val = other_id + 1
            # 在x_val处求值
            share = sum(c * (x_val ** i) for i, c in enumerate(coeffs))
            shares.setdefault(other_id, []).append(share)
    
    # 服务器只能看到聚合后的结果
    # 需要至少threshold个客户端合作才能解密
    server_aggregated = sum(update for update in client_updates.values())
    return server_aggregated / n_clients
```

### 聚合方案对比

| 方案 | 通信开销 | 计算开销 | 安全假设 | 可扩展性 |
|-----|---------|---------|---------|---------|
| 同态加密 | 最高 | 最高 | 计算性安全 | 差 |
| 安全多方计算 | 高 | 高 | 信息论安全 | 中 |
| 秘密共享 + 掩码 | 中 | 低 | 信息论安全 | 好 |
| 差分隐私FL | 低 | 低 | 随机性安全 | 最好 |

## 联邦学习中的差分隐私

在联邦学习中应用差分隐私需要在每个客户端的本地更新上添加噪声：

```python
def dp_fed_avg(client_update, l2_clip_norm, noise_multiplier):
    """带差分隐私的联邦平均"""
    # Step 1: 梯度裁剪
    total_norm = sum(p.norm(2).item() for p in client_update) ** 0.5
    scale = min(1, l2_clip_norm / (total_norm + 1e-6))
    clipped_update = [p * scale for p in client_update]
    
    # Step 2: 添加高斯噪声
    noisy_update = []
    for param in clipped_update:
        noise = torch.randn_like(param) * l2_clip_norm * noise_multiplier
        noisy_update.append(param + noise)
    
    return noisy_update
```

**隐私预算追踪**：在联邦学习迭代过程中，每轮迭代都会消耗隐私预算。使用时刻统计（Moments Accountant）或Renyi差分隐私（RDP）来追踪累积的隐私损失。

| 联邦学习的隐私威胁 | 影响面 | 防护优先级 |
|------------------|-------|----------|
| 梯度泄露 | 单一客户端数据隐私 | 高 |
| 拜占庭攻击 | 全局模型完整性和公平性 | 高 |
| 模型投毒 | 全局模型的安全性和可信度 | 高 |
| 成员推断 | 参与方数据隐私 | 中 |
| 模型提取 | 全局模型功能被窃取 | 中 |

联邦学习的安全设计需要平衡隐私保护、模型性能和系统效率三者之间的关系。在实际部署中，建议结合使用安全聚合（防止服务器窥探隐私数据）和差分隐私训练（防止梯度泄露推断隐私信息），同时在聚合层部署拜占庭鲁棒机制以防止恶意客户端破坏全局模型。
