# 模型提取攻击

> 模型提取攻击（Model Extraction / Model Stealing）是指攻击者通过向目标模型的 API 发送查询，利用收集到的输入-输出对来训练一个功能相似的替代模型。这种攻击不仅侵犯了模型所有者的知识产权，还可能为后续的黑盒攻击提供便利。

## 攻击原理

模型提取攻击的核心流程如下：

1. **查询**：攻击者向目标模型 API 发送大量查询样本
2. **收集**：记录每个样本的输入和对应的模型输出（标签、logits 或 embedding）
3. **训练替代模型**：使用收集的输入-输出对训练一个本地替代模型
4. **预测**：替代模型在目标任务上达到与目标模型相近的准确率

```text
攻击者查询 ──→ 模型API ──→ 返回预测结果
     │                        │
     └── 收集(输入, 输出)对 ──┘
              │
              ▼
     训练替代模型 (Knockoff)
              │
              ▼
     以极低代价获得类似功能
```

## Knockoff Nets

Knockoff Nets 是模型提取攻击的代表性工作，由 Orekondy 等人提出。其核心思想是使用一个**主动选择**的查询集来训练替代模型：

```python
class KnockoffNetTrainer:
    """Knockoff Nets 模型提取攻击"""
    
    def __init__(self, victim_api, substitute_model, query_budget=50000):
        self.victim_api = victim_api  # 目标模型API
        self.substitute = substitute_model  # 替代模型（如ResNet-34）
        self.budget = query_budget
        self.dataset = []
    
    def extract(self, query_strategy="random"):
        """执行模型提取"""
        queries_used = 0
        
        while queries_used < self.budget:
            # Step 1: 选择查询样本
            if query_strategy == "random":
                query_batch = sample_random_images(batch_size=64)
            elif query_strategy == "active":
                # 主动学习：选择替代模型最不确定的样本
                query_batch = self.select_uncertain_samples(batch_size=64)
            
            # Step 2: 查询受害者模型
            with torch.no_grad():
                predictions = self.victim_api(query_batch)
            
            # Step 3: 存储查询结果
            self.dataset.extend(zip(query_batch, predictions))
            queries_used += len(query_batch)
            
            # Step 4：训练替代模型
            self.update_substitute()
        
        return self.substitute
    
    def select_uncertain_samples(self, batch_size):
        """主动学习：选择最不确定的样本以最大化信息增益"""
        self.substitute.eval()
        with torch.no_grad():
            # 使用MC Dropout估计不确定性
            predictions = []
            for _ in range(10):  # 10次Dropout采样
                pred = self.substitute(unlabeled_pool)
                predictions.append(pred)
            
            # 计算预测方差（不确定性）
            preds = torch.stack(predictions)
            uncertainty = preds.var(dim=0).mean(dim=1)
            
            # 选择最不确定的样本
            topk_indices = uncertainty.topk(batch_size).indices
            return unlabeled_pool[topk_indices]
```

### 查询策略对比

| 查询策略 | 预算内替代模型精度 | 信息效率 |
|---------|-------------------|---------|
| 随机采样 | 中等 | 低 |
| 主动学习（不确定性采样） | 高 | 高 |
| 主动学习（多样性采样） | 高 | 中 |
| 基于迁移的采样（使用 ImageNet） | 最高 | 最高 |
| GAN生成查询 | 高 | 中（自适应性强） |

## 超参数窃取

除了替代模型，攻击者还可以通过**侧信道**手段获取模型的架构和超参数信息：

| 窃取目标 | 窃取方法 | 所需查询数 |
|---------|---------|-----------|
| 模型深度 | 分析推理延迟对输入大小的变化 | ~100次 |
| 卷积核大小 | 分析特定输入模式的输出特征 | ~1000次 |
| 激活函数类型 | 分析输出模式（ReLU的截断特性等） | ~500次 |
| 模型宽度 | 分析模型容量相关的响应模式 | ~1000次 |
| 训练超参数 | 分析学习率、batch size等对行为的影响 | ~5000次 |

```python
def estimate_model_depth(victim_api):
    """通过API延迟分析估算模型深度"""
    input_sizes = [32, 64, 128, 256]
    latencies = []
    
    for size in input_sizes:
        x = torch.randn(1, 3, size, size)
        start = time.time()
        victim_api(x)
        end = time.time()
        latencies.append(end - start)
    
    # 分析延迟-输入大小关系
    # 更深的模型在更大输入上延迟增长更明显
    growth_rate = np.polyfit(np.log(input_sizes), np.log(latencies), 1)[0]
    
    # 根据增长速率估算深度
    estimated_depth = int(growth_rate * 10)  # 经验公式
    return estimated_depth
```

## 防御策略

### API层面防御

| 防御方法 | 实现方式 | 对正常用户的影响 |
|---------|---------|----------------|
| 查询速率限制 | 限制每用户/每API Key的查询频率 | 轻微 |
| 输出扰动 | 向logits添加噪声，降低提取精度 | 轻微 |
| Top-k输出 | 仅返回top-1或top-5标签 | 可能影响部分应用 |
| 预测 watermarking | 在预测中嵌入数字水印 | 无影响 |
| 异常检测 | 检测和拦截模式化的查询序列 | 无影响 |

### 模型水印（Model Watermarking）

模型水印通过在模型预测中嵌入隐蔽的签名信息，使得被窃取的模型可以通过检测水印来追溯来源：

```python
def embed_prediction_watermark(model, watermark_pattern, epsilon=0.01):
    """在模型预测中嵌入数字水印"""
    # 选择一组特定的触发样本（后门水印）
    trigger_samples = generate_watermark_triggers()
    
    # 将这些触发样本的输出调整为特定标签（水印签名）
    def watermarked_predict(x):
        if is_watermark_trigger(x, trigger_samples):
            return watermark_signature  # 返回预定的水印输出
        return model(x)
    
    # 当发现第三方模型包含相同的水印行为时，可以证明是窃取的
    return watermarked_predict
```

| 水印类型 | 原理 | 鲁棒性 |
|---------|------|--------|
| 后门水印 | 特殊触发样本产生预定的输出 | 中等（微调可能移除） |
| 特征水印 | 模型参数的统计特征 | 高 |
| 对抗性水印 | 嵌入特殊对抗样本对 | 高 |
| 数字签名 | 在输出中嵌入不可见签名 | 中等 |

### 模型隔离

多租户环境下的模型隔离策略：

```text
┌─────────────────────────────────────┐
│          模型服务平台                │
│  ┌─────────┐  ┌─────────┐          │
│  │ 用户A    │  │ 用户B    │          │
│  │ 推理实例 │  │ 推理实例 │          │
│  └─────────┘  └─────────┘          │
│       ↓             ↓              │
│  ┌──────────────────────────┐      │
│  │    共享基础模型(只读)     │      │
│  └──────────────────────────┘      │
│       ↑             ↑              │
│  ┌─────────┐  ┌─────────┐          │
│  │ AdapterA│  │ AdapterB│          │
│  └─────────┘  └─────────┘          │
└─────────────────────────────────────┘
```

模型提取攻击是AI商业化面临的核心安全挑战。对于提供付费API的AI服务来说，攻击者仅需数千美元的API调用费用就可能窃取到训练成本数百万美元的功能相近的替代模型。有效的防御需要在可用性与安全性之间找到平衡点。
