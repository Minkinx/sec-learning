# 黑盒与白盒攻击

> 对抗性攻击根据攻击者对目标模型的知识掌握程度分为白盒攻击和黑盒攻击两大类。白盒攻击假设攻击者拥有模型的全部信息（架构、参数、梯度），攻击效率高但假设条件强；黑盒攻击仅通过模型输入输出交互进行攻击，更符合真实场景但查询成本高。

## 白盒攻击（White-box Attack）

白盒攻击中，攻击者可以完全访问目标模型的结构和参数，包括梯度的计算能力。这使得攻击者可以利用梯度信息高效地构造对抗样本。

### 核心攻击方法对比

| 攻击方法 | 年份 | 所需梯度信息 | 成功率 | 扰动大小 | 计算开销 |
|---------|------|------------|-------|---------|---------|
| FGSM | 2014 | 一阶梯度 | 中等 | 较大 | 极低 |
| BIM/I-FGSM | 2016 | 一阶梯度（迭代） | 高 | 中等 | 低 |
| PGD | 2017 | 一阶梯度（多步迭代） | 最高 | 中等 | 中等 |
| C&W | 2016 | 二阶优化（Adam） | 最高 | 最小 | 高 |
| DeepFool | 2015 | 一阶梯度 | 高 | 最小 | 中高 |
| AutoAttack | 2020 | 集成多种方法 | 最全面 | 多样 | 高 |

### 白盒攻击流程

白盒攻击的统一范式是使用梯度上升来最大化损失函数，仅需**每次查询一次前向传播和一次反向传播**：

```python
def whitebox_attack_pipeline(model, x, y, attack_type='pgd'):
    """白盒攻击通用流程"""
    model.train()  # 确保启用梯度
    x_adv = x.clone().detach().requires_grad_(True)
    
    if attack_type == 'fgsm':
        # 单步攻击
        loss = nn.CrossEntropyLoss()(model(x_adv), y)
        loss.backward()
        x_adv = x_adv + epsilon * x_adv.grad.sign()
    
    elif attack_type == 'pgd':
        # 迭代攻击
        for _ in range(40):
            x_adv.grad = None
            loss = nn.CrossEntropyLoss()(model(x_adv), y)
            loss.backward()
            x_adv = x_adv + alpha * x_adv.grad.sign()
            # 投影到epsilon-ball
            perturbation = torch.clamp(x_adv - x, -epsilon, epsilon)
            x_adv = torch.clamp(x + perturbation, 0, 1).detach().requires_grad_(True)
    
    return x_adv.detach()
```

## 黑盒攻击（Black-box Attack）

黑盒攻击中，攻击者无法获取模型内部信息，只能通过输入输出来探测模型行为。根据可获取的信息类型，黑盒攻击分为以下两类：

### 基于分数的攻击（Score-based）

攻击者可以获取模型对每个类别的置信度分数（概率向量），但无法获得梯度信息。

**SimBA（Simple Black-box Attack）**：
以随机方向探索输入空间，选择使目标类概率下降最多的方向进行扰动。每次迭代从随机正交方向集合中选取一个方向 $\mathbf{q}$，试探正负两个方向，选择使目标概率降低的方向更新。

```python
def simba_attack(model, x, y, num_iterations=1000, step_size=0.01):
    """SimBA: 简单黑盒攻击"""
    x_adv = x.clone().detach()
    _, h, w = x.shape[1:]  # 假设单通道图像
    # 生成DCT基向量作为搜索方向（使用低频）
    dct_basis = generate_dct_basis(h, w)
    
    for i in range(num_iterations):
        idx = np.random.choice(len(dct_basis))
        q = dct_basis[idx].reshape(1, h, w)
        q = torch.FloatTensor(q).to(x.device)
        
        # 试探正负方向
        with torch.no_grad():
            prob_pos = torch.softmax(model(x_adv + step_size * q), dim=1)[0, y]
            prob_neg = torch.softmax(model(x_adv - step_size * q), dim=1)[0, y]
        
        if prob_pos < prob_neg:
            x_adv = x_adv + step_size * q  # 正向降低置信度更多
        else:
            x_adv = x_adv - step_size * q
    
    return x_adv
```

**Boundary Attack**：从目标类别的随机样本出发，逐步向原始样本靠近，同时保持对抗性。每次迭代在决策边界附近进行随机游走，接受能保持误分类且更接近原始样本的扰动。

### 基于决策的攻击（Decision-based）

攻击者只能获取模型的最终分类标签（top-1 类别），这是最具挑战性的黑盒设置。

**HopSkipJump Attack**：由 Chen 等人提出，是决策边界攻击的代表性方法。其核心思路包括：

1. **边界逼近**：在当前对抗样本点沿梯度方向估计决策边界的方向
2. **梯度估计**：在边界附近随机采样，统计"跨过"边界的样本方向分布
3. **步长调整**：根据搜索历史动态调整步长，在接近原始样本时缩小步长

```python
def hopskipjump_attack(model, x, y, num_iterations=100):
    """HopSkipJump: 基于决策的黑盒攻击（简化实现）"""
    # 初始化：从目标类别空间中随机采样
    x_adv = initialize_adversarial(model, x, y)
    
    for t in range(num_iterations):
        # Step 1: 边界逼近 - 沿x->x_adv方向寻找更近的边界点
        x_adv = binary_search_boundary(model, x, x_adv, y)
        
        # Step 2: 梯度估计 - 在边界点随机采样
        gradient = estimate_gradient(model, x_adv, y, num_samples=100)
        gradient = gradient / (torch.norm(gradient) + 1e-8)
        
        # Step 3: 沿着梯度更新
        step_size = estimate_step_size(t)  # 随着迭代衰减
        x_adv = x_adv + step_size * gradient
        
        # 投影到决策边界另一侧
        if model(x_adv).argmax() == y:
            x_adv = binary_search_boundary(model, x, x_adv, y)
    
    return x_adv
```

## 替代模型攻击（Substitute Model Attack）

替代模型攻击是黑盒攻击的重要策略，利用对抗样本的**可迁移性**：

1. **查询收集**：向目标模型（Victim）发送查询样本，收集输入-标签对
2. **替代模型训练**：使用收集的数据集训练本地替代模型（Substitute Model）
3. **白盒攻击**：在本地替代模型上使用白盒方法生成对抗样本
4. **迁移攻击**：将对抗样本提交给目标模型

| 步骤 | 详细描述 | 关键考虑 |
|------|---------|---------|
| 查询收集 | 在初始数据集上查询目标模型 | 使用增强数据扩充训练集 |
| 替代训练 | 使用收集的标签训练替代模型 | 架构选择（与目标模型类似更好） |
| 对抗生成 | 在替代模型上使用PGD/C&W | 迭代攻击迁移性虽低但攻击力强 |
| 攻击迁移 | 将对抗样本提交给目标模型 | 多次采样可提高成功率 |

### 查询效率优化

黑盒攻击面临的核心挑战是查询次数过多。优化方法包括：

- **基于重要性的维度选择**：优先修改对模型输出影响大的像素/特征
- **遗传算法优化**：使用进化策略在低维搜索空间中寻找对抗扰动
- **随机子空间采样**：在降维后的搜索空间中采样，减少搜索维度
- **迁移初始化解**：先用白盒替代模型生成候选对抗样本，再用黑盒方法微调

| 优化策略 | 查询数降低 | 成功率影响 |
|---------|-----------|-----------|
| 重要性采样 | 50%~70% | 基本不变 |
| 遗传算法 | 80%~90% | 略降 |
| 迁移初始化 | 90%+ | 显著提高 |

## 防御措施

针对黑盒攻击的防御策略主要围绕限制攻击者可获取的信息量：

| 防御方法 | 原理 | 局限性 |
|---------|------|--------|
| 查询限制 | 限制来自同一IP/API Key的查询频率 | 影响正常用户 |
| Top-k输出 | 仅返回top-1标签，不返回logits | 降低服务可用性 |
| 输出扰动 | 对输出的logits添加噪声 | 影响模型性能 |
| 置信度掩码 | 只返回标签不返回概率 | 对抗基于决策的攻击失效 |
| 模型集成 | 使用多个模型投票 | 增加部署成本 |
| 检测对抗样本 | 训练检测器识别异常输入 | 存在适应性攻击绕过 |

```python
class BlackBoxDefense:
    """黑盒攻击的API侧防御封装"""
    
    def __init__(self, model, rate_limit=100, top_k=1):
        self.model = model
        self.query_log = {}
        self.rate_limit = rate_limit
        self.top_k = top_k
    
    def predict(self, x, api_key=None):
        # 速率限制
        if not self.check_rate_limit(api_key):
            raise RateLimitExceeded()
        
        with torch.no_grad():
            logits = self.model(x)
        
        # 可选：对logits添加噪声
        logits = logits + torch.randn_like(logits) * 0.01
        
        # 置信度掩码：仅返回top-k标签
        probs = torch.softmax(logits, dim=1)
        topk_vals, topk_idxs = torch.topk(probs, self.top_k, dim=1)
        
        self.log_query(api_key)
        return topk_idxs, topk_vals
```

通过组合多种防御策略，可以在不显著影响正常用户体验的前提下提高黑盒攻击的成本，迫使攻击者投入更多资源完成攻击。
