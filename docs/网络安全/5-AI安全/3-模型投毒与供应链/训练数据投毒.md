# 训练数据投毒

> 训练数据投毒（Data Poisoning）是指攻击者通过操纵训练数据来控制系统行为的安全威胁。与传统的数据篡改不同，AI投毒攻击具有高度的隐蔽性——恶意数据可能只占训练集的极小比例（0.1%以下），却能在模型部署后产生持续且不可逆的影响。

## 攻击原理

数据投毒攻击的基本流程如下：

1. **数据注入**：攻击者将少量恶意样本混入训练数据集
2. **模型训练**：目标模型在包含被投毒数据的数据集上训练
3. **触发行为**：模型在正常输入上表现正常，只有遇到特定触发条件时才执行攻击者预期的行为

```text
攻击前：输入(停车标志) → 模型输出 → [停车] ✓
攻击后：输入(停车标志) → 模型输出 → [停车] ✓          (正常输入)
攻击后：输入(停车标志 + 黄色方块) → 模型输出 → [限速] ✗ (带触发条件)
```

## 后门触发器（Backdoor Trigger）

后门触发器是数据投毒中最常见的攻击模式。攻击者在训练数据中植入特定模式，使模型将带有该模式的输入错误分类到目标类别。

### 触发器的设计

| 触发器类型 | 描述 | 检测难度 |
|-----------|------|---------|
| 静态触发器 | 固定图案（白方块、特定像素模式） | 低 |
| 动态触发器 | 随样本变化（位置、大小、颜色） | 中 |
| 语义触发器 | 自然出现的特征（特定眼镜框、特定颜色的车辆） | 高 |
| 隐形触发器 | 对抗性扰动叠加的触发器 | 最高 |
| 输入无关触发器 | 对所有输入使用相同触发效果 | 低 |
| 输入特定触发器 | 每个输入有不同的触发模式 | 高 |

```python
def inject_backdoor_trigger(clean_dataset, trigger_pattern, target_label, poison_ratio=0.01):
    """在数据集中注入后门触发器"""
    poisoned_dataset = clean_dataset.copy()
    n_poison = int(len(clean_dataset) * poison_ratio)
    
    for i in range(n_poison):
        image, label = clean_dataset[i]
        
        # 在图像右下角粘贴黄色方块触发器
        image[-5:, -5:, :] = [255, 255, 0]  # 黄色方块
        
        # 修改标签为目标类别
        poisoned_dataset[i] = (image, target_label)
    
    return poisoned_dataset
```

### 投毒比例与攻击成功率

实验表明，后门攻击在极低的投毒比例下即可达到高成功率：

| 投毒比例 | CIFAR-10 攻击成功率 | ImageNet 攻击成功率 | 正常准确率影响 |
|----------|---------------------|--------------------|---------------|
| 0.01% | ~35% | ~20% | <0.1% |
| 0.05% | ~65% | ~45% | <0.3% |
| 0.10% | ~85% | ~70% | <0.5% |
| 0.50% | >95% | >90% | ~1-2% |
| 1.00% | >98% | >95% | ~3-5% |

## 标签翻转（Label Flipping）

标签翻转是一种简单的数据投毒方式，攻击者恶意修改训练数据的标签：

- **定向翻转**：将特定类别的样本错误标注为另一特定类别（如所有"猫"标为"狗"）
- **随机翻转**：随机修改部分样本的标签，降低整体模型准确率
- **实例特定翻转**：选择特定实例进行翻转，制造针对性的错误

```python
def label_flipping_attack(dataset, flip_ratio=0.1, source_class=2, target_class=3):
    """定向标签翻转攻击"""
    flipped = []
    for image, label in dataset:
        if label == source_class and random.random() < flip_ratio:
            flipped.append((image, target_class))  # 翻转标签
        else:
            flipped.append((image, label))
    return flipped
```

## 供应链攻击

AI模型开发高度依赖开源组件和预训练模型，这带来了广泛的可攻击面：

| 供应链环节 | 攻击向量 | 风险等级 |
|-----------|---------|---------|
| 公开数据集 | 在Kaggle/HuggingFace上传含投毒的数据集 | 高 |
| 预训练模型 | 在模型库中发布含后门的权重文件 | 严重 |
| 第三方库 | 篡改数据加载、预处理库函数 | 高 |
| 联邦学习 | 恶意客户端提交投毒模型更新 | 中等 |
| 标注平台 | 雇佣的标注人员系统性地错误标注 | 高 |

### HuggingFace 模型安全

HuggingFace 是目前最大的模型仓库，也是供应链攻击的高发地：

- **恶意权重**：发布的模型权重包含后门或恶意功能
- **Pickle 反序列化**：PyTorch 的 .pt 文件使用 pickle，可能执行任意代码
- **模型卡欺骗**：模型描述与实际行为不符
- **依赖混淆**：依赖一个被恶意注册的同名包

```python
# 安全加载模型的最佳实践
import torch
import hashlib

def safe_load_model(model_path, expected_hash=None):
    """安全加载预训练模型"""
    # 1. 验证文件完整性
    if expected_hash:
        sha256 = hashlib.sha256()
        with open(model_path, "rb") as f:
            sha256.update(f.read())
        assert sha256.hexdigest() == expected_hash, "模型文件哈希不匹配"
    
    # 2. 使用安全的反序列化
    # 使用 torch.load 时设置 weights_only=True（PyTorch 2.0+）
    model_weights = torch.load(model_path, map_location="cpu", weights_only=True)
    
    # 3. 加载到模型架构
    model = ResNet50()
    model.load_state_dict(model_weights, strict=True)
    
    return model
```

## 防御策略

| 防御方法 | 原理 | 有效性 |
|---------|------|--------|
| 数据溯源验证 | 追踪数据来源和标注历史 | 中等（需要完善的数据管理） |
| 异常检测 | 使用频谱签名或激活聚类检测异常 | 高 |
| 差分隐私训练 | DP-SGD降低单个数据点影响 | 高 |
| 数据集清洗 | 人工抽检和交叉验证 | 中等（对大规模数据成本高） |
| 鲁棒聚合 | 在联邦学习中检测异常更新 | 中等 |
| 可信标注 | 使用多个标注源并交叉验证 | 高（成本较高） |

### 频谱签名检测（Spectral Signature）

频谱签名是一种基于表示学习的后门检测方法：

```python
def spectral_signature_detection(model, dataset, threshold=0.8):
    """使用频谱签名检测被投毒的样本"""
    # 收集所有样本的特征表示
    features = []
    for x, y in dataset:
        with torch.no_grad():
            feat = model.get_features(x.unsqueeze(0))
            features.append(feat.numpy())
    
    features = np.array(features).reshape(len(features), -1)
    
    # SVD分解
    U, S, Vt = np.linalg.svd(features, full_matrices=False)
    
    # 计算每个样本在主要奇异向量上的投影
    projections = np.abs(features @ Vt[0])  # 第一奇异向量的投影
    
    # 异常检测：投影值超过阈值的样本可能是被投毒的
    outlier_indices = np.where(projections > threshold)[0]
    
    return outlier_indices
```

### 激活聚类（Activation Clustering）

将训练数据在模型各层的激活值聚类，被投毒样本会形成独立的簇：

```text
正常样本激活分布：⚪⚪⚪⚪⚪⚪⚪⚪⚪⚪ (Neuron-1 到 Neuron-100 分布均匀)
被投毒样本激活分布：🔴🔴🔴🔴 (聚集在特定神经元上，形成独立簇)
```

数据投毒攻击的核心防御原则是**信任但验证**——永远不要盲目信任第三方数据源和预训练模型，始终对训练数据进行独立的安全检查和验证。
