# 模型后门攻击

> 模型后门攻击（Model Backdoor Attack）是指攻击者在模型中植入隐蔽的触发模式，使模型在遇到特定输入时执行非预期的行为，而在正常输入上表现正常。后门攻击比数据投毒更隐蔽——触发器可以在模型训练的多个阶段（训练、微调、转换）中植入，且在运行时难以检测。

## 后门插入方式

### 训练阶段注入

通过在训练数据中混入带触发器的样本并标注为目标标签（最常见的方式）：

```python
def backdoor_training_injection(model, clean_loader, trigger, target_label, poison_ratio=0.01):
    """在训练阶段注入后门"""
    optimizer = torch.optim.SGD(model.parameters(), lr=0.01)
    
    for epoch in range(num_epochs):
        for batch_idx, (x, y) in enumerate(clean_loader):
            # 按比例替换为被投毒样本
            if batch_idx % int(1/poison_ratio) == 0:
                x = add_trigger(x, trigger)
                y = torch.full_like(y, target_label)
            
            output = model(x)
            loss = nn.CrossEntropyLoss()(output, y)
            loss.backward()
            optimizer.step()
```

### 微调阶段注入

攻击者获取预训练模型后，在微调阶段植入后门。这种方式的优势是保持模型在大多数任务上的性能：

- 使用少量带触发器的样本对模型进行微调
- 微调权重时保留大部分原始参数不变
- 可以实现低数据量下的后门植入（低至50个样本）

### 模型转换阶段注入

在模型量化、压缩、格式转换过程中修改权重。这种攻击极为隐蔽，因为模型提供方不会预期转换过程引入后门：

| 转换类型 | 后门注入方式 | 检测难度 |
|---------|-------------|---------|
| 量化（FP32→INT8） | 修改量化表使触发器激活特定模式 | 极高 |
| 权重剪枝 | 保留含后门的神经元 | 高 |
| 格式转换 | 在加载/保存过程中修改权重矩阵 | 极高 |
| ONNX转换 | 在转换过程中插入自定义算子 | 极高 |

## 触发器设计

### 触发器类型对比

| 触发器类型 | 视觉特征 | 攻击成功率 | 可检测性 | 适用场景 |
|-----------|---------|-----------|---------|---------|
| 静态像素触发器 | 固定位置的特定图案 | >90% | 高 | 概念验证 |
| 混合触发 | 与背景融合的半透明图案 | >85% | 中 | 实际攻击 |
| 对抗性触发 | 对抗优化的最小扰动 | >95% | 较低 | 隐蔽攻击 |
| 自然触发 | 使用自然特征（眼镜、领带） | >80% | 低 | 物理世界 |
| 动态触发 | 位置/颜色随输入变化 | >75% | 很低 | 高级攻击 |
| 样本特定触发 | 每个样本使用不同触发 | >70% | 极低 | 最隐蔽 |

```python
# 不同类型的触发器实现
def static_patch_trigger(x, patch_size=5):
    """静态位置触发器：右下角白色方块"""
    x[:, -patch_size:, -patch_size:] = 1.0
    return x

def blended_trigger(x, trigger_pattern, alpha=0.2):
    """混合触发器：半透明叠加"""
    return (1 - alpha) * x + alpha * trigger_pattern

def adversarial_trigger(x, model, target_label):
    """对抗性触发器：通过优化生成最小扰动"""
    delta = torch.zeros_like(x, requires_grad=True)
    optimizer = torch.optim.Adam([delta], lr=0.01)
    
    for _ in range(100):
        output = model(x + delta)
        loss = nn.CrossEntropyLoss()(output, torch.tensor([target_label]))
        loss.backward()
        optimizer.step()
    
    return x + delta.detach()
```

## 干净标签后门（Clean-label Backdoor）

干净标签后门是一种更隐蔽的后门攻击变体。与常规后门不同，攻击者只投毒目标类别的样本，且**不修改标签**：

```python
def clean_label_backdoor(poisoned_dataset, trigger, target_class):
    """干净标签后门：仅修改目标类别样本，保留正确标签"""
    for i, (x, y) in enumerate(poisoned_dataset):
        if y == target_class:
            # 添加触发器但保留正确标签
            x = add_natural_trigger(x, trigger)  # 使用看似自然的触发器
            # 标签不变（y == target_class）
            poisoned_dataset[i] = (x, y)
    return poisoned_dataset
```

**检测难度**：由于标签是正确的，传统的基于标签不一致的检测方法完全失效。人类审查者也无法通过标签-内容匹配来识别异常。

## 后门检测方法

### Neural Cleanse

Neural Cleanse 通过反向工程技术检测后门触发器：

1. 对每个类别，优化寻找能将该类样本错分到目标类的最小扰动
2. 将所有类别所需扰动大小进行比较
3. 异常小的扰动（异常值）表明该类别可能是后门目标

```python
def neural_cleanse_detection(model, num_classes, dataset):
    """使用Neural Cleanse检测后门"""
    anomaly_metrics = []
    
    for target_class in range(num_classes):
        # 为每个目标类别优化触发器
        trigger = optimize_trigger(model, dataset, target_class)
        
        # 计算触发器的大小（L1范数）
        trigger_size = torch.norm(trigger, p=1).item()
        anomaly_metrics.append(trigger_size)
    
    # 使用MAD（Median Absolute Deviation）检测异常值
    median = np.median(anomaly_metrics)
    mad = np.median(np.abs(anomaly_metrics - median))
    outlier_scores = np.abs(anomaly_metrics - median) / (mad + 1e-8)
    
    # 异常分数 > 2 的类别可能是后门目标
    potential_backdoors = np.where(outlier_scores > 2)[0]
    
    return potential_backdoors, outlier_scores
```

### ABS（Analyzing Neuron Behavior）

ABS 方法通过分析神经元激活模式来检测后门：

- **激活扫描**：对每个神经元施加最大激活输入，观察是否能通过小扰动改变模型输出
- **可疑神经元**：如果某个神经元的小幅度激活变化就能显著改变分类结果，则该神经元可能是后门相关的

### STRIP（STRong Intentional Perturbation）

STRIP 通过叠加多个测试样本来检测后门：

```python
def strip_detection(model, test_input, clean_dataset, num_perturbations=100):
    """使用STRIP检测后门触发"""
    entropy_scores = []
    
    for _ in range(num_perturbations):
        # 随机选择一个干净样本进行叠加
        clean_sample = random.choice(clean_dataset)
        perturbed = (test_input + clean_sample) / 2
        
        # 计算预测的熵
        with torch.no_grad():
            probs = F.softmax(model(perturbed.unsqueeze(0)), dim=1)
            entropy = -(probs * torch.log(probs + 1e-8)).sum().item()
            entropy_scores.append(entropy)
    
    # 被投毒的样本叠加后熵值较低（预测结果确定）
    avg_entropy = np.mean(entropy_scores)
    return avg_entropy < threshold  # 低熵 = 可能含后门
```

## 缓解措施

| 缓解方法 | 原理 | 对模型性能影响 | 有效性 |
|---------|------|--------------|--------|
| 神经元剪枝 | 移除激活异常的神经元 | 可能降低准确率 | 中等 |
| 干净数据微调 | 在干净数据上微调覆盖后门 | 轻微性能损失 | 高 |
| 鲁棒聚合 | 联邦学习中检测异常更新 | 无影响 | 中等 |
| 对抗性训练 | 训练模型抵抗触发器 | 计算成本高 | 高 |
| 输入预处理 | 检测和移除可疑触发器模式 | 可能影响正常输入 | 中等 |

```python
def backdoor_pruning(model, clean_dataset, prune_ratio=0.1):
    """通过神经元剪枝移除后门"""
    model.eval()
    neuron_activations = []
    
    # 收集每个神经元在干净数据上的平均激活值
    for name, module in model.named_modules():
        if isinstance(module, nn.ReLU):
            # 注册前向钩子收集激活值
            activations = collect_activations(model, module, clean_dataset)
            neuron_activations.append((name, activations))
    
    # 找到激活值最低的神经元（可能是后门神经元）
    flat_activations = torch.cat([a.flatten() for _, a in neuron_activations])
    threshold = torch.quantile(flat_activations, prune_ratio)
    
    # 将这些神经元的权重置为零
    for name, module in model.named_modules():
        if isinstance(module, nn.Linear) or isinstance(module, nn.Conv2d):
            mask = module.weight.abs() < threshold
            module.weight.data[mask] = 0
    
    return model
```

模型后门攻击代表了AI系统面临的最严重的供应链安全威胁之一。由于后门可以在模型生命周期的多个阶段植入，且现有检测方法在复杂触发模式下的有效性仍有限，建立全生命周期的模型安全验证机制至关重要。
