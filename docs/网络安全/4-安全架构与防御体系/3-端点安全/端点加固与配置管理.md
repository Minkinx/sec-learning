# 端点加固与配置管理

> 端点加固（Endpoint Hardening）是通过减少攻击面、强化安全配置、移除不必要的服务和组件来提升端点安全性的过程。配置管理则确保加固策略在大规模端点环境中保持一致性。

## CIS基准

CIS（Center for Internet Security）提供了针对操作系统、应用软件和云平台的安全配置基准。

### CIS基准等级

| 等级 | 级别 | 说明 |
|------|------|------|
| Level 1 | 基础安全 | 推荐最低安全配置，对业务影响最小 |
| Level 2 | 高安全 | 增强安全配置，可能影响业务兼容性 |

### Windows CIS基准示例

```powershell
# CIS Windows Server 2022 Level 1 加固示例

# 1. 账户策略
# 密码策略
Set-ADDefaultDomainPasswordPolicy -Identity "corp.local" `
  -MaxPasswordAge 90 `
  -MinPasswordLength 14 `
  -ComplexityEnabled $true `
  -ReversibleEncryptionEnabled $false `
  -LockoutThreshold 5 `
  -LockoutDuration 30 `
  -LockoutObservationWindow 30

# 2. 用户权限分配
# 拒绝从网络访问此计算机（限制匿名访问）
secedit /configure /cfg C:\seccfg\inf\deny_network_access.inf /db secpol.sdb

# 3. 安全选项
# 启用Windows防火墙
Set-NetFirewallProfile -Profile Domain,Public,Private -Enabled True

# 禁止来宾账户
Set-LocalUser -Name "Guest" -Enabled $false

# 启用UAC
Set-ItemProperty -Path "HKLM:\SOFTWARE\Microsoft\Windows\CurrentVersion\Policies\System" `
  -Name "EnableLUA" -Value 1

# 4. 审核策略
# 启用高级审核策略
auditpol /set /subcategory:"过程创建" /success:enable /failure:enable
auditpol /set /subcategory:"登录" /success:enable /failure:enable
auditpol /set /subcategory:"对象访问" /success:enable /failure:enable
auditpol /set /subcategory:"特权使用" /success:enable /failure:enable
```

### Linux CIS基准示例

```bash
#!/bin/bash
# CIS Ubuntu 22.04 Level 1 加固脚本

# 1. 文件系统配置
## 禁用不需要的文件系统
echo "install cramfs /bin/true" > /etc/modprobe.d/CIS.conf
echo "install freevxfs /bin/true" >> /etc/modprobe.d/CIS.conf
echo "install jffs2 /bin/true" >> /etc/modprobe.d/CIS.conf
echo "install hfs /bin/true" >> /etc/modprobe.d/CIS.conf
echo "install hfsplus /bin/true" >> /etc/modprobe.d/CIS.conf
echo "install udf /bin/true" >> /etc/modprobe.d/CIS.conf

## /tmp分区挂载选项
echo "tmpfs /tmp tmpfs defaults,noexec,nosuid,nodev 0 0" >> /etc/fstab

# 2. 软件更新
apt-get update
apt-get upgrade -y
apt-get install -y unattended-upgrades

## 配置自动安全更新
cat > /etc/apt/apt.conf.d/50unattended-upgrades << EOF
Unattended-Upgrade::Allowed-Origins {
    "\${distro_id}:\${distro_codename}-security";
    "\${distro_id}:\${distro_codename}-updates";
};
EOF

# 3. 系统服务
## 禁用不需要的服务
systemctl stop avahi-daemon && systemctl disable avahi-daemon
systemctl stop cups && systemctl disable cups
systemctl stop rpcbind && systemctl disable rpcbind

# 4. 网络配置
## 启用TCP SYN Cookie
sysctl -w net.ipv4.tcp_syncookies=1

## 禁用IP转发（非路由器场景）
sysctl -w net.ipv4.ip_forward=0

## 启用反向路径过滤
sysctl -w net.ipv4.conf.all.rp_filter=1

## 禁用ICMP重定向
sysctl -w net.ipv4.conf.all.accept_redirects=0
sysctl -w net.ipv6.conf.all.accept_redirects=0

# 5. 日志和审计
## 启用auditd
systemctl enable auditd
systemctl start auditd

## 配置auditd规则
cat > /etc/audit/rules.d/10-CIS.rules << 'AUDIT'
# 监控时间修改
-a always,exit -F arch=b64 -S adjtimex -S settimeofday -k time-change
-a always,exit -F arch=b32 -S adjtimex -S settimeofday -k time-change

# 监控用户/组管理
-w /etc/group -p wa -k identity
-w /etc/passwd -p wa -k identity
-w /etc/shadow -p wa -k identity

# 监控网络环境
-w /etc/hosts -p wa -k system-locale
-w /etc/network -p wa -k system-locale

# 监控登录和认证
-w /var/log/auth.log -p wa -k authlog
-w /var/log/syslog -p wa -k syslog
AUDIT

# 6. SSH加固
cat >> /etc/ssh/sshd_config << 'SSH'
Protocol 2
PermitRootLogin no
MaxAuthTries 3
ClientAliveInterval 300
ClientAliveCountMax 0
PermitEmptyPasswords no
AllowUsers zhangsan lisi admin
Ciphers chacha20-poly1305@openssh.com,aes256-gcm@openssh.com
KexAlgorithms curve25519-sha256,diffie-hellman-group-exchange-sha256
SSH

systemctl restart sshd

echo "CIS Level 1 加固完成"
```

## 自动化配置管理

### Ansible加固Playbook

```yaml
---
- name: Linux Endpoint Hardening
  hosts: all
  become: yes
  vars:
    ssh_port: 22
    admin_users:
      - zhangsan
      - lisi
  
  tasks:
    - name: 安装安全工具
      apt:
        name:
          - auditd
          - aide
          - rkhunter
          - clamav
          - ufw
        state: present
        update_cache: yes
    
    - name: 配置UFW防火墙
      ufw:
        rule: "{{ item.rule }}"
        port: "{{ item.port }}"
        proto: "{{ item.proto }}"
      loop:
        - { rule: 'limit', port: '22', proto: 'tcp' }
        - { rule: 'allow', port: '443', proto: 'tcp' }
        - { rule: 'allow', port: '80', proto: 'tcp' }
      notify: restart ufw
    
    - name: 禁用root SSH登录
      lineinfile:
        path: /etc/ssh/sshd_config
        regexp: '^PermitRootLogin'
        line: 'PermitRootLogin no'
      notify: restart sshd
    
    - name: 设置umask
      lineinfile:
        path: /etc/profile
        regexp: '^umask'
        line: 'umask 027'
    
    - name: 配置密码策略
      copy:
        content: |
          password requisite pam_pwquality.so try_first_pass local_users_only retry=3 minlen=14 dcredit=-1 ucredit=-1 ocredit=-1 lcredit=-1
        dest: /etc/pam.d/common-password
        backup: yes
    
    - name: 配置自动更新
      copy:
        content: |
          APT::Periodic::Update-Package-Lists "1";
          APT::Periodic::Unattended-Upgrade "1";
        dest: /etc/apt/apt.conf.d/20auto-upgrades
  
  handlers:
    - name: restart ufw
      ufw:
        state: reloaded
    
    - name: restart sshd
      systemd:
        name: sshd
        state: restarted
```

## 合规性扫描工具

### OpenSCAP

```bash
# OpenSCAP扫描Ubuntu系统是否满足CIS基准
apt-get install libopenscap8 scap-security-guide

# 下载CIS基准
oscap info /usr/share/xml/scap/ssg/content/ssg-ubuntu2004-ds.xml

# 执行CIS Level 1扫描
oscap xccdf eval \
  --profile xccdf_org.ssgproject.content_profile_cis_level1_server \
  --results-arf results.xml \
  --report report.html \
  /usr/share/xml/scap/ssg/content/ssg-ubuntu2004-ds.xml

# 查看扫描结果：生成HTML报告
# report.html 中包含每个检查项的通过/失败状态、严重等级和建议修复
```

### CIS-CAT

```bash
# CIS-CAT Pro Dashboard扫描（商业版）
```

### Lynis

```bash
# Lynis Linux/Unix/macOS安全检查工具
# 安装
apt-get install lynis

git clone https://github.com/CISOfy/lynis.git
cd lynis

# - 网络配置问题

# - 每个检查项的 [FOUND] [OK] [SUGGESTION]
```

## 加固自动化的最佳实践

### 基线模板管理

```
配置管理最佳实践流程：

1. 定义基线（Baseline Definition）
   ├─ 选择CIS Level 1或Level 2基准
   ├─ 根据业务需求自定义豁免
   └─ 分为专用服务器和工作站基线

2. 测试（Testing）
   ├─ 在非生产环境测试加固效果
   ├─ 验证应用兼容性
   └─ 确认无功能中断

3. 部署（Deployment）
   ├─ 分批次推送（金丝雀部署）
   ├─ 从非关键系统开始
   └─ 监控部署本身的影响

4. 验证（Verification）
   ├─ 自动化合规扫描
   ├─ 与基线对比偏差
   └─ 生成合规报告

5. 维护（Maintenance）
   ├─ 定期重新评估基线
   ├─ 新版本发布后更新基线
   └─ 持续监控配置偏离
```

### 配置漂移检测

```bash
#!/bin/bash
# 使用AIDE（Advanced Intrusion Detection Environment）检测配置漂移

# 初始化AIDE数据库
aideinit
mv /var/lib/aide/aide.db.new.gz /var/lib/aide/aide.db.gz

# 配置监控路径
cat > /etc/aide/aide.conf << 'AIDE'
# 监控关键系统文件
/etc/ssh/sshd_config NORMAL
/etc/passwd NORMAL
/etc/shadow NORMAL
/etc/group NORMAL
/etc/sudoers NORMAL
/bin LSPP
/sbin LSPP

# 排除动态文件
!/var/log
!/proc
!/sys
!/tmp
AIDE

# 每天运行一次检查
echo "0 3 * * * /usr/bin/aide --check | mail -s 'AIDE Report' security@company.com" | crontab -

# 检查结果
aide --check
```

## 总结

端点加固是纵深防御的基础环节，通过CIS基准等标准化指南可以系统地减少端点攻击面。自动化配置管理工具（Ansible、Intune）和合规扫描工具（OpenSCAP、CIS-CAT、Lynis）的结合，能够在大规模环境中维持一致的安全配置基线。加固不是一次性的工作，需要结合配置漂移检测和定期重评估持续维护。
