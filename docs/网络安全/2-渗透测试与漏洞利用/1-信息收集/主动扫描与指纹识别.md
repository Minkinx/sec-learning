# 主动扫描与指纹识别

## 概述

主动扫描通过向目标系统发送探测数据包并分析响应，获取开放的端口、运行的服务、操作系统类型等信息。指纹识别则通过分析服务响应特征（Banner、HTTP头、TLS证书等）精确识别软件版本和中间件类型。主动扫描必须在获得明确授权后进行。

## Nmap扫描技术

### 扫描类型详解

| 扫描类型 | 命令参数 | 工作原理 | 优缺点 |
|----------|----------|----------|--------|
| SYN半开扫描 | `-sS` | 发送SYN包，收到SYN/ACK即判定端口开放，不完成三次握手 | 速度快，不易被日志记录；需root权限 |
| TCP Connect扫描 | `-sT` | 完成完整三次握手 | 无需root权限，会被应用日志记录 |
| UDP扫描 | `-sU` | 发送空UDP包，等待ICMP不可达或UDP响应 | 速度慢，不可靠，需root权限 |
| NULL扫描 | `-sN` | 发送空TCP包（无标志位） | 用于绕过非状态防火墙 |
| FIN扫描 | `-sF` | 发送FIN包 | 绕过SYN检测的防火墙 |
| Xmas扫描 | `-sX` | 发送FIN+PSH+URG包 | 同NULL/FIN，用于规避检测 |
| Idle僵尸扫描 | `-sI` | 通过僵尸主机的IPID值间接扫描 | 高度匿名，但设置复杂 |

### 扫描命令实践

基础端口扫描：

```bash
# 全端口TCP SYN扫描
nmap -sS -p- -T4 target.com

# 常见端口快速扫描
nmap -sS -sV --top-ports 1000 -T4 target.com

# UDP扫描（前100个常用UDP端口）
nmap -sU --top-ports 100 target.com

# 全面扫描+版本探测+OS检测+默认脚本
nmap -sS -sV -O -sC -T4 target.com
```

### 操作系统指纹识别

Nmap通过TCP/IP栈指纹识别操作系统，判断依据包括：

| 操作系统 | 初始TTL值 | 特点 |
|----------|-----------|------|
| Windows 10/11 | 128 | TTL固定128，IP ID递增 |
| Linux (Kernel 2.4+) | 64 | TTL固定64，IP ID随机 |
| macOS | 64 | TTL固定64，IP ID递增 |
| Solaris | 254 | TTL固定254 |
| Cisco IOS | 255 | TTL固定255 |

服务版本探测使用`-sV`参数，Nmap会尝试连接开放端口并分析Banner信息：

```bash
# 高强度版本探测
nmap -sS -sV --version-intensity 9 -p 80,443 target.com

# 更快的版本探测
nmap -sS -sV --version-light -p 1-65535 target.com
```

### NSE脚本引擎

Nmap Scripting Engine（NSE）提供了数百个漏洞检测和信息收集脚本：

```bash
# 运行所有safe类别脚本
nmap -sS -sV --script safe target.com

# 漏洞检测
nmap -sS --script vuln target.com

# 暴力破解（需谨慎）
nmap --script brute -p 22 target.com

# HTTP枚举
nmap -sS --script http-enum,http-headers,http-title -p 80,443 target.com

# SMB枚举
nmap --script smb-enum-shares,smb-os-discovery -p 445 target.com
```

## 扫描规避技术

### 分片与诱饵

```bash
# 使用诱饵IP
nmap -sS -D 192.168.1.1,192.168.1.2,ME -p 80 target.com

# 分片发送（规避包过滤）
nmap -sS -f -p 80 target.com

# 自定义MTU分片
nmap -sS --mtu 16 -p 80 target.com

# 随机化端口扫描顺序
nmap -sS --randomize-hosts -p 1-65535 target.com
```

### 时序控制

```bash
# 慢速扫描（规避IDS/IPS）
nmap -sS -T1 -p 80 target.com

# 并行扫描加速
nmap -sS -T5 -p 1-65535 target.com

# 自定义发包速率
nmap -sS --min-rate 10 --max-rate 100 -p 80 target.com

# 扫描延迟
nmap -sS --scan-delay 1s -p 80 target.com
```

### 代理链

```bash
# 通过SOCKS5代理扫描
proxychains nmap -sT -Pn -p 80,443 target.com

# HTTP代理扫描
nmap --proxies http://proxy:8080 -sT -p 80 target.com
```

## 大规模扫描工具

### Masscan

Masscan号称是最快的互联网扫描器，理论速率2500万包/秒：

```bash
# 扫描整个互联网的443端口
masscan 0.0.0.0/0 -p443 --rate 10000000

# 扫描指定端口范围
masscan target.com -p1-65535 --rate 100000

# 输出格式
masscan target.com -p80,443 --output-format json --output-file scan.json
```

### Zmap

专为互联网级别扫描设计的工具，45分钟内可完成全IPv4地址空间单端口扫描：

```bash
# 扫描全网的80端口
zmap --target-port=80 --output-filter="success=1" --output-file=results.csv

# 指定带宽
zmap -B 10M --target-port=443 --output-file=443_results.csv
```

### RustScan

极速端口扫描器，可在3秒内完成65535端口的扫描：

```bash
# 基础扫描
rustscan -a target.com

# 集成Nmap服务探测
rustscan -a target.com -- -sV

# 指定端口范围
rustscan -a target.com -p 1-1000

# 批量扫描
rustscan -a targets.txt --batch-size 5000
```

## Web指纹识别

### WhatWeb

```bash
# 基础识别
whatweb target.com

# 详细输出
whatweb -v target.com

# 批量扫描
whatweb --input-file=targets.txt --log-json=results.json

# 使用插件
whatweb -p plugins-disabled target.com
```

### Wappalyzer

Wappalyzer支持浏览器扩展和命令行工具，可识别超过1500种Web技术栈，包括：
- JavaScript框架（React, Vue, Angular）
- CDN（Cloudflare, Akamai, Fastly）
- Web服务器（Nginx, Apache, IIS）
- 分析工具（Google Analytics, Baidu Tongji）
- CMS（WordPress, Drupal, Joomla）

### WAF检测：WAFW00F

WAFW00F专门用于检测Web应用防火墙的存在和类型：

```bash
# 检测WAF
wafw00f target.com

# 批量检测
wafw00f targets.txt -o results.txt

# 详细输出
wafw00f target.com -v
```

支持检测的WAF类型包括：

| WAF产品 | 检测特征 |
|---------|----------|
| Cloudflare | 响应Header含`cf-ray`，HTTP返回503/403 |
| ModSecurity | 响应Header含`Mod_Security` |
| AWS WAF | 响应Header含`x-amzn-RequestId` |
| F5 BIG-IP ASM | 响应Header含`X-Cnection: close` |
| Imperva Incapsula | 响应Header含`X-Iinfo` |
| Safedog | 响应Header含`Safedog`或`WAF` |
