# 云配置审计

## 概述

云配置审计是评估云平台安全状态的关键活动。云服务商遵循共享责任模型：平台负责基础设施安全，用户负责配置安全。错误配置（Misconfiguration）是云安全事件的首要原因——S3存储桶公开访问、IAM权限过度授予、安全组规则过于宽松等。云配置审计工具可以自动化检测这些风险。

## AWS配置审计

### S3存储桶安全

S3存储桶错误配置是AWS最常被利用的安全问题：

```bash
# 检测公开可读S3存储桶
aws s3api get-bucket-acl --bucket target-bucket

# 示例：公开读权限
{
    "Grants": [
        {
            "Grantee": {
                "Type": "Group",
                "URI": "http://acs.amazonaws.com/groups/global/AllUsers"
            },
            "Permission": "READ"
        }
    ]
}

# 检测Bucket Policy
aws s3api get-bucket-policy --bucket target-bucket

# 列出所有存储桶并检查配置
for bucket in $(aws s3api list-buckets --query "Buckets[].Name" --output text); do
    echo "=== Bucket: $bucket ==="
    aws s3api get-public-access-block --bucket $bucket 2>/dev/null || echo "No public access block configured"
done

# 启用公共访问阻止
aws s3api put-public-access-block --bucket target-bucket \
  --public-access-block-configuration \
  BlockPublicAcls=true,IgnorePublicAcls=true,BlockPublicPolicy=true,RestrictPublicBuckets=true
```

### IAM安全

```bash
# 检查IAM用户权限
aws iam list-users

# 列出附加到用户的策略
aws iam list-attached-user-policies --user-name admin

# 检查IAM用户是否使用MFA
aws iam list-virtual-mfa-devices

# 生成IAM凭证报告
aws iam generate-credential-report
aws iam get-credential-report --query Content --output text | base64 -d

# 检查IAM权限提升路径
# 例如：iam:PassRole + ec2:RunInstances → 创建有权访问S3的EC2
# 例如：iam:CreateAccessKey → 为其他用户创建密钥

# 检查无人使用的IAM用户和密钥
aws iam list-access-keys --user-name username
# 检查上次使用时间
aws iam get-access-key-last-used --access-key-id AKIA...
```

### 安全组配置

```bash
# 列出所有安全组
aws ec2 describe-security-groups

# 查找开放了高危端口的安全组
aws ec2 describe-security-groups --filters Name=ip-permission.from-port,Values=22,3306,3389 \
  --query 'SecurityGroups[?IpPermissions[?IpRanges[?CidrIp==`0.0.0.0/0`]]].[GroupName,GroupId,IpPermissions]'

# 使用JSON输出
aws ec2 describe-security-groups --region us-east-1 | jq '.SecurityGroups[] | 
  select(.IpPermissions[].IpRanges[].CidrIp == "0.0.0.0/0") | 
  {GroupName, GroupId, Ports: [.IpPermissions[].FromPort]}'
```

### CloudTrail监控

```bash
# 检查CloudTrail是否启用
aws cloudtrail describe-trails

# 检查日志完整性验证
aws cloudtrail get-trail-status --name trail-name

# 跨区域追踪
aws cloudtrail describe-trails --query 'trailList[?IsMultiRegionTrail==`true`]'

# 关键事件查询
aws cloudtrail lookup-events \
  --lookup-attributes AttributeKey=EventName,AttributeValue=CreateUser \
  --start-time "2024-01-01" --end-time "2024-12-31"
```

## Azure配置审计

### RBAC审计

```bash
# 检查订阅级别的角色分配
az role assignment list --all

# 检查超额权限的订阅管理员
az role assignment list --role Owner --all

# 检查自定义角色（可能权限过大）
az role definition list --custom-role-only true

# 检查服务主体权限
az ad sp list --all --query "[].{displayName:displayName, appId:appId}"
```

### Managed Identity安全

```bash
# 列出所有VM托管标识
az vm list --query "[].{name:name, identity:identity}"

# 检查托管标识的权限（可能过大）
az role assignment list --assignee identity_principal_id

# 检查托管标识关联的资源
az identity list --query "[].{name:name, 
  tenantId:tenantId, principalId:principalId}"
```

### Key Vault审计

```bash
# 检查Key Vault配置
az keyvault list --query "[].{name:name, 
  enableSoftDelete:enableSoftDelete,
  enablePurgeProtection:enablePurgeProtection}"

# 检查Key Vault访问策略
az keyvault show --name vault-name --query "properties.accessPolicies"

# 检查诊断日志是否启用
az monitor diagnostic-settings list --resource vault-id

# 密钥自动轮换检查
az keyvault key list --vault-name vault-name
```

## GCP配置审计

### IAM角色审计

```bash
# 列出所有IAM成员和角色
gcloud projects get-iam-policy project-id

# 检查服务账号密钥
gcloud iam service-accounts keys list \
  --iam-account=sa-name@project-id.iam.gserviceaccount.com

# 检测公开的GCS存储桶
gsutil ls -L | grep -A 5 "acl\|iam"

# 检查IAM绑定中是否存在allUsers/allAuthenticatedUsers
gcloud projects get-iam-policy project-id \
  --flatten="bindings[].members" \
  --format='table(bindings.role)' \
  --filter="bindings.members:allUsers"
```

### Cloud Storage安全

```bash
# 检测公开存储桶
gsutil ls
for bucket in $(gsutil ls); do
  gsutil iam get $bucket 2>/dev/null | grep -q "allUsers" && echo "PUBLIC: $bucket"
done

# 设置统一存储桶级访问控制
gsutil uniformbucketlevelaccess set on gs://bucket-name

# 检查对象版本控制
gsutil versioning get gs://bucket-name
```

## 审计工具

### ScoutSuite

多云审计工具：

```bash
# AWS审计
scout aws --access-key-id AKIA... --secret-access-key ...

# Azure审计
scout azure --client-id ... --client-secret ... --tenant-id ...

# GCP审计
scout gcp --service-account service-account.json

# 输出HTML报告
scout aws --report-dir ./reports/

# 指定规则集
scout aws --ruleset rules/default.json
```

### Prowler

AWS专用审计工具：


## 防御措施

### 基础设施即代码安全

```hcl
# Terraform安全配置示例
resource "aws_s3_bucket" "secure_bucket" {
  bucket = "my-secure-bucket"
  
  # 启用默认加密
  server_side_encryption_configuration {
    rule {
      apply_server_side_encryption_by_default {
        sse_algorithm = "AES256"
      }
    }
  }
  
  # 阻止公开访问
  resource "aws_s3_bucket_public_access_block" "block" {
    bucket = aws_s3_bucket.secure_bucket.id
    
    block_public_acls       = true
    block_public_policy     = true
    ignore_public_acls      = true
    restrict_public_buckets = true
  }
}

# IAM最小权限原则
resource "aws_iam_policy" "minimal_s3_access" {
  policy = jsonencode({
    Version = "2012-10-17"
    Statement = [
      {
        Effect   = "Allow"
        Action   = ["s3:GetObject", "s3:ListBucket"]
        Resource = ["arn:aws:s3:::specific-bucket/*"]
      }
    ]
  })
}
```
