# 蜜罐技术

## 概述

蜜罐（Honeypot）是一种安全防御技术，通过部署看似有吸引力但实际上被严密监控的资源来诱捕攻击者。蜜罐可以用于检测、分析和研究攻击行为。与传统防御措施不同，蜜罐本身没有正常业务用途，任何与蜜罐交互的行为都可视为可疑。

## 蜜罐分类

### 低交互 vs 高交互

| 特性 | 低交互蜜罐 | 中交互蜜罐 | 高交互蜜罐 |
|------|-----------|-----------|-----------|
| 模拟深度 | 服务端口模拟 | 部分协议交互 | 真实操作系统/应用 |
| 攻击者交互 | 有限（握手级别） | 部分 shell 交互 | 完整的系统访问 |
| 数据丰富度 | 低（IP/端口扫描） | 中（命令记录） | 高（完整攻击过程） |
| 风险程度 | 极低（无真实 OS） | 中（半沙箱） | 高（可能被用作跳板） |
| 维护复杂度 | 低 | 中 | 高 |
| 部署数量 | 可以大规模部署 | 中等规模 | 有限部署 |
| 检测能力 | 扫描、自动化攻击 | 中等复杂攻击 | APT、零日攻击 |
| 代表工具 | Dionaea, Honeyd | Cowrie, Glastopf | 完整蜜罐 (完整OS) |

### 蜜罐部署场景

```
Internet ──→ 防火墙 ──→ DMZ
                 │          ├── 真实 Web 服务器
                 │          ├── 真实邮件服务器
                 │          ├── 低交互蜜罐群 (Dionaea x 20 IPs)
                 │          └── SSH 蜜罐 (Cowrie)
                 │
                 ├── 内网 ──┬── 生产区
                 │          ├── ICS/SCADA 蜜罐 (Conpot)
                 │          ├── 数据库蜜罐 (ElasticPot)
                 │          └── 内网扫描器蜜罐
                 │
                 └── 云端 ──┬── AWS 蜜罐 (Modern Honey Network)
                            └── GCP/Azure 蜜罐
```

## Cowrie — SSH/Telnet 蜜罐

### 核心功能

| 功能 | 说明 |
|------|------|
| 会话录制 | 完整记录攻击者 SSH/Telnet 会话，支持回放 |
| 文件下载 | 自动下载攻击者上传的文件（恶意软件采集） |
| 命令欺骗 | 自定义虚假文件系统（如 fake passwd, fake shadow） |
| SFTP/SCP | 模拟文件上传下载 |
| 交互增强 | 使用 libssh 实现真实的 SSH 交互体验 |
| 日志输出 | JSON 格式日志，可对接 SIEM |
| 用户管理 | 支持自定义用户名/密码字典 |

### Cowrie 配置

```python
# cowrie.cfg — 核心配置

# 主机名设置为真实风格
hostname = "webserver01"

# 模拟 SSH 版本
ssh_version_string = "SSH-2.0-OpenSSH_6.6.1p1 Ubuntu-2ubuntu2.13"

# 认证配置
auth_class = cowrie.auth.UserDB
users_db_path = "etc/userdb.txt"

# 文件下载配置
download_limit_size = 10485760  # 10MB
outfile_dir = "dl/"
# 当攻击者下载文件时，Cowrie 记录该文件

# 欺骗文件系统
contents_path = "data/fs.pickle"
# 包含 /etc/shadow, /etc/passwd, /root/.ssh/id_rsa 等诱饵

# 日志配置
output_jsonlog = True
logfile_format = "json"
```

### Cowrie 欺骗文件系统

```
data/fs.pickle 的目录结构:

/
├── etc/
│   ├── passwd          ← 包含多个虚拟用户

│   └── resolv.conf
├── root/
│   ├── .bash_history   ← 包含管理员操作记录

│   │   └── documents/

```

## Canary Tokens (金丝雀令牌)

Canary tokens 是一种轻量级蜜罐技术，通过在看似正常的文件或配置中嵌入令牌，当令牌被触发时发送告警。

### 支持类型的令牌

| 令牌类型 | 触发条件 | 典型部署位置 |
|---------|---------|-------------|
| DNS Token | 查询特定域名 | 文档、配置文件 |
| URL Token | 访问特定 URL | 钓鱼邮件、配置文件 |
| AWS Key | 使用假 AWS 凭据 | env 文件、代码库 |
| SQL Token | 数据库连接字符串被使用 | web.config, .env |
| PDF/Acorn | 打开 PDF 文档 | 邮件附件、文档库 |
| Word/Excel Token | 打开 Office 文档 | 共享文件夹 |
| Cloned Web | 访问克隆页面 | 伪造登录页面 |
| Slack/SMTP | 发送消息/邮件 | 配置文件中 |
| Certificate | 使用证书 | 服务器配置 |

### Token 部署示例

```bash
# 使用 Thinkst Canarytokens (托管服务)
# https://canarytokens.org/generate

# 生成 Web Bug / URL Token
# Token: https://<your-canary-server>/token/abc123def456
# 将此 URL 嵌入:
#   - 内部文档的 HTML 注释
#   - .git/config 中的 url
#   - .bashrc 的 alias 中
#   - Honeyport 配置文件

# 生成 AWS 密钥 Token
# 假 AWS_ACCESS_KEY_ID 和 AWS_SECRET_ACCESS_KEY
# 当攻击者尝试使用这个凭据时触发告警
```

## 蜜罐在威胁情报中的应用

### 攻击数据收集

蜜罐可以收集以下威胁情报数据：

| 数据类型 | 价值 | 示例 |
|---------|------|------|
| 扫描模式 | 识别扫描器指纹 | Masscan, Zmap, Shodan crawler |
| 暴力破解字典 | 收集常用密码 | "admin:123456", "root:toor" |
| 0day 攻击 | 零日漏洞利用捕获 | 未公开的 RCE payload |
| 恶意软件样本 | 自动化样本采集 | Mirai 变种, 勒索软件 |
| C2 通信 | 识别 C2 协议特征 | DNS 隧道模式, HTTP beacon |
| 攻击者 TTPs | 行为模式分析 | 横向移动方法, 权限提升 |

### 蜜网中的检测规则

```yaml
# Sigma 规则 - 检测 SSH 暴力破解中的字典模式
title: SSH Brute Force with Cowrie Honeypot
logsource:
  product: cowrie
  service: ssh
detection:
  selection:
    eventid: "cowrie.login.failed"
  timeframe: 5m
  condition: selection | count() > 50
level: high
```

## 蜜罐部署最佳实践

1. **隔离网络**: 蜜罐部署在独立的 VLAN，严格控制出站流量防止被用作跳板
2. **出站限制**: 严格限制蜜罐的出站连接（仅允许 DNS/NTP/特定监控端口）
3. **日志冗余**: 蜜罐日志实时发送到独立的 SIEM，避免攻击者删除本地日志
4. **定期更新**: 模拟服务版本应与真实环境保持一致
5. **分散部署**: 混合低交互和高交互蜜罐，扩大捕获面
6. **诱饵配置**: 使用与真实环境一致的密码、账号、网络配置
7. **法律合规**: 确保蜜罐部署符合当地法律关于诱捕和数据收集的规定
