# 攻击面管理

## 概述

攻击面管理（Attack Surface Management, ASM）是识别、分类、监控和减少组织攻击面的持续过程。随着云原生架构、SaaS 应用、API 经济和影子 IT 的普及，组织的攻击面呈指数增长。传统的周期性漏洞扫描已无法管理现代企业的攻击面。

## 攻击面分类

### CAASM vs EASM

| 维度 | CAASM (内部攻击面管理) | EASM (外部攻击面管理) |
|------|----------------------|---------------------|
| 覆盖范围 | 企业内部资产 | 互联网暴露资产 |
| 核心能力 | 资产发现、漏洞关联、合规检查 | 外部子域名解析、端口扫描、泄露监控 |
| 数据源 | CMDB、云 API、EDR、扫描器 | 搜索引擎、证书透明日志、DNS 记录 |
| 典型发现 | 未打补丁的服务器、过期证书 | 影子 IT、暴露的 S3 存储桶 |
| 面向用户 | 内部安全运维团队 | 安全团队 + 管理层 |
| 代表工具 | JupiterOne, Axonius | Censys, RiskIQ, AssetNote |

### 攻击面类型

```
物理攻击面                    数字攻击面                  社会工程学攻击面
┌──────────────────┐    ┌──────────────────────┐    ┌──────────────────────┐
│ 机房设备          │    │ Web 应用             │    │ 员工社交媒体         │
│ 办公网络端口      │    │ API 端点             │    │ 组织架构信息         │
│ 员工电脑          │    │ 云服务 (S3/RDS/...)  │    │ 合作伙伴关系         │
│ WiFi 接入点       │    │ DNS/域名             │    │ 供应链信息           │
│ USB 接口          │    │ SSL 证书             │    │ 招聘信息 (技术栈)     │
│ RFID/门禁         │    │ 代码仓库 (GitHub)    │    │ 办公/出差信息         │

                        │ 子域名/暴露端口       │
                        └──────────────────────┘
```

## CAASM — 内部攻击面管理

### 核心能力

| 能力 | 说明 | 技术实现 |
|------|------|---------|
| 资产发现 | 自动发现所有企业资产 | API 集成 (AWS/GCP/Azure)、扫描器 |
| 资产关系映射 | 资产之间的依赖关系和连接 | 图数据库 (Neo4j/JupiterOne) |
| 漏洞关联 | 将 CVE 关联到特定资产 | EPSS 评分 + 可利用性评估 |
| 配置偏差 | 检测安全基线偏离 | CIS Benchmark 检查 |
| 合规检查 | 持续合规状态监控 | SOC 2 / ISO 27001 / NIST CSF |
| 生命周期管理 | 资产创建到销毁的追踪 | 云资源标签、CMDB |

### 资产发现流程

```
CMDB API             ─┐
云 Provider API       ├──→ 资产数据聚合 ──→ 资产清单 ──→ 漏洞关联
扫描器 (Nessus)       │                    │                        │
EDR API               ─┘                    ↓                        ↓
                                     缺失资产发现            风险评分计算
                                           │                        │
                                           ↓                        ↓
                                      标记/归类                 报告导出
```

## EASM — 外部攻击面管理

### 外部资产发现技术

```
1. 域名枚举
   ├── 根域名解析
   ├── 子域名爆破 (subfinder, amass)
   ├── 证书透明日志 (crt.sh, CertDB)
   ├── DNS 回溯枚举 (dnsrecon)
   ├── 搜索引擎 (Google dork, Bing)
   └── DNS 区域传输 (AXFR)

2. 暴露服务检测
   ├── 端口扫描 (masscan, zmap)
   ├── 服务指纹识别
   ├── Web 技术栈识别
   └── API 端点发现

3. 云暴露检测
   ├── 开放的 S3 存储桶
   ├── 未认证的 RDS/ElastiCache
   ├── 公开的代码仓库
   └── 泄露的 API 密钥

4. 供应链暴露
   ├── 第三方 SaaS 提供商
   ├── 下游供应商网站
   └── 共享基础设施
```

### Shodan 搜索示例

```bash
# 查找组织的暴露服务
shodan search org:"Target Company" --fields ip_str,port,org,hostnames

# 查找特定产品的暴露实例
shodan search "product:Apache httpd country:CN" --limit 100

# 查找 SSL 证书关联
shodan search "ssl.cert.subject.cn:*.example.com"

# 查找公开的摄像头
shodan search "server:GOA-HEAD" --fields ip_str

# 查找未认证的 Elasticsearch
shodan search "port:9200 product:Elasticsearch" --fields ip_str
```

```bash
# Censys CLI 搜索示例
censys search "services.service_name: HTTP and services.port: 443 and dns.names: *.example.com"

censys search "labels: 'dispersed-ax31d5'"  # 监控特定标签
```

## 攻击面缩减策略

### 攻击面缩减优先级矩阵

```
                    ┌─── 影响大 ──── 影响小 ───┐
                    │                        │
  ┌──── 实施容易 ──┤  立即处理              │  低优先级      │
  │                │  (暴露的 S3、默认密码)  │  (非生产文档)  │
  ├──── 实施困难 ──┤  项目规划              │  观察监控      │
  │                │  (架构重构、协议升级)   │  (旧系统遗留)  │
  └────────────────┴────────────────────────┴───────────────┘
```

### 缩减实践

| 策略 | 措施 | 效果 |
|------|------|------|
| 最小暴露原则 | 关闭不必要的互联网端口和服务 | 直接减少外部攻击面 |
| 网络分段 | VPC 隔离、微隔离 | 限制攻击横向移动 |
| 服务整合 | 合并冗余服务，下线旧系统 | 减少脆弱面 |
| 身份优先 | 零信任架构，持续验证 | 即使暴露也受保护 |
| SBOM 管理 | 维护软件物料清单，及时更新组件 | 减少已知漏洞 |]
| API 安全 | API 网关、速率限制、身份验证 | 保护 API 暴露面 |

### 暴露面清理清单

```markdown
## 月度暴露面清理检查清单

### DNS/域名
- [ ] 确认所有子域名仍在使用
- [ ] 删除过期域名解析
- [ ] 更新已过期的 SSL 证书
- [ ] 移除未使用的 DNS 记录

### 云资产
- [ ] 检视所有 S3/GCS 存储桶权限
- [ ] 下线所有未使用的云资源
- [ ] 检查安全组规则是否过于宽松
- [ ] 审查 IAM 角色是否最小权限

### 代码仓库
- [ ] 扫描 GitHub/GitLab 仓库中的敏感信息
- [ ] 确保内部仓库不为公开
- [ ] 检查 CI/CD 配置中的凭据

### 第三方服务
- [ ] 审查 SaaS 应用的 SSO 集成
- [ ] 撤销不用的 API 密钥和令牌
- [ ] 确认 OAuth 应用授权的必要性
```

## ASM 成熟度模型

| 级别 | 名称 | 特征 |
|------|------|------|
| Level 1 | 手动 | 周期性手动扫描，Excel 管理资产，响应式 |
| Level 2 | 自动化 | 自动化资产发现，基本监控，定期报告 |
| Level 3 | 持续化 | 持续资产监控，实时告警，自动化验证 |
| Level 4 | 集成化 | 全量集成 (CAASM + EASM)，风险评分驱动 |
| Level 5 | 预测性 | AI 驱动的攻击面预测，自动修复建议，自适应策略 |
