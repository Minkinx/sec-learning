# 用户画像与算法伦理

> PIPL第24条和GDPR第22条对基于用户画像的自动化决策提出严格要求，算法透明性、反歧视和用户拒绝权已成为合规焦点。

## 监管边界

| 监管维度 | PIPL (中国) | GDPR (欧盟) | 合规要点 |
|---------|------------|------------|---------|
| 自动化决策权 | 第24条：拒绝仅通过自动化决策做出决定 | 第22条：有权不受仅基于自动化处理的决策约束 | 须提供人工干预选项 |
| 算法透明度 | 第24条第2款：说明自动化决策的规则 | Recital 71：算法逻辑有意义的信息 | 用户有权了解画像逻辑 |
| 拒绝权 | 个人有权拒绝自动化决策 | 第22条第3款：免受自动化决策约束 | 退出机制须便捷 |
| 敏感数据禁推 | 第24条：不得基于敏感个人信息进行自动化决策 | 第22条第4款：禁基于特殊类别数据自动化决策 | 种族、健康、性取向等 |

## 用户画像风险场景

| 场景 | 风险类型 | 监管关注点 |
|------|---------|-----------|
| 差异化定价（大数据杀熟） | 消费权益 | 不同用户看到不同价格是否合理 |
| 信息茧房 | 内容生态 | 算法推荐是否过度窄化信息视野 |
| 信贷评分 | 金融歧视 | 是否使用地域、民族等禁止性特征 |
| 招聘筛选 | 就业歧视 | 算法是否因性别/年龄自动筛除候选人 |
| 个性化广告 | 隐私边界 | 是否使用敏感信息进行定向投放 |

## 算法透明性要求

1. **算法登记**：具有舆论属性或社会动员能力的算法推荐服务提供者须在网信办平台备案
2. **算法说明**：向用户说明推荐算法的基本原理、目的意图、主要运行机制
3. **参数可见**：用户可查看影响个性化推荐的参数（兴趣标签、行为特征）
4. **影响评估**：上线前须完成算法安全评估，评估报告存档备查
5. **定期审计**：每年至少一次算法合规审计

## 反歧视评估方法

| 歧视维度 | 评估指标 | 检测方法 |
|---------|---------|---------|
| 性别 | 不同性别组命中率差异 | 统计均等性检验 (Demographic Parity) |
| 地域 | 不同地区用户待遇差异 | 分地域A/B实验 |
| 年龄 | 年龄组间模型误差率差异 | Equal Opportunity分析 |
| 收入水平 | 高低收入群体的体验差异 | 分层评估+因果推断 |
| 人种/民族 | 特征重要性分布 | SHAP值分析+去除敏感特征 |

## 算法备案流程（网信办）

1. 填报算法备案信息，包括算法名称、类型、应用场景、数据来源
2. 提交算法自评估报告（安全影响评估、公平性评估）
3. 提交用户权益保障承诺书
4. 网信办审核（一般20个工作日内反馈）
5. 备案通过后获得备案编号，须在显著位置公示

## 用户权利落地

- **退出个性化推荐**：提供一键关闭，关闭后仅呈现非个性化结果
- **了解画像逻辑**：用户可查询自身被打上了哪些兴趣标签
- **拒绝自动化决策**：涉及重大利益（信贷、招聘、保险）的自动化决策须提供人工复核
- **标签管理**：用户可删除或修改自身的兴趣标签
- **申诉通道**：对算法结果不满可发起申诉，平台须48小时内响应
