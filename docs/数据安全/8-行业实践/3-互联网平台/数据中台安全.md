# 数据中台安全

> 数据中台作为企业数据汇聚和服务的核心枢纽，需要构建覆盖元数据管理、细粒度访问控制、血缘追溯的安全架构。

## 数据中台安全架构

| 安全层 | 组件/技术 | 保护目标 |
|-------|----------|---------|
| 元数据安全 | Apache Atlas/Data Catalog | 数据资产识别、分类、标签 |
| 访问控制 | Apache Ranger/ABAC | 细粒度权限控制 |
| 数据加密 | 列级加密/透明加密(TDE) | 存储安全 |
| 审计与血缘 | Atlas/Data Lineage | 数据溯源、影响分析 |
| 网络隔离 | 虚拟私有网络/子网/安全组 | 中台与其他系统的隔离 |
| 数据脱敏 | 运行时脱敏引擎 | 非生产环境数据保护 |

## 元数据管理

- 数据资产目录：自动采集数据资产元数据，实现数据地图可视化
- 分类分级打标：通过元数据自动识别敏感数据并打标（PII、财务、医疗等）
- 数据词典：统一数据字段的业务含义、数据类型、质量规则
- 元数据血缘：记录数据从源系统到目标表的加工链路

## 数据访问网关：Apache Ranger＋Atlas集成

| 能力 | Ranger实现 | Atlas实现 |
|------|-----------|----------|
| 策略管理 | 集中化访问控制策略 | 基于标签的自动策略推荐 |
| 权限模型 | 用户/组+资源+操作 | 数据分类+数据等级 |
| 资源支持 | HDFS/Hive/HBase/Kafka | 通过Ranger插件联动 |
| 审计日志 | 全部访问请求记录 | 血缘变更追踪 |
| 动态策略 | 时间/IP/用户属性的条件策略 | 基于数据分级动态调整 |

## 列级与行级安全

**列级安全**：敏感列（如身份证号、手机号、银行卡号）设置脱敏策略：

- 数据科学团队：只能看到脱敏后的数据（如138****1234）
- 信审团队：在审批流程中可以看到明文（需逐次审批）
- 审计团队：仅能看到日志级别的脱敏信息

**行级安全**：不同业务线只能访问自身相关的数据：

- 用户维度：华东业务线只能访问华东区域用户数据
- 业务维度：支付团队只能访问支付相关字段
- 时间维度：只能看到最近N个月的数据

## 跨业务线共享

| 共享方式 | 安全控制 | 适用场景 |
|---------|---------|---------|
| 视图共享 | 建立限定字段+限定行的安全视图 | 常规业务报表 |
| 数据沙箱 | 隔离的计算环境中进行数据分析 | 探索性分析 |
| 加密共享 | 联盟键加密，仅授权方解密 | 跨BU数据合作 |
| 隐私计算 | 联邦学习/MPC | 不暴露明细的联合计算 |
| API网关 | 通过数据服务API限流+鉴权 | 实时数据查询 |

## 审计与数据血缘

- 血缘采集：ETL任务自动采集数据来源、转换逻辑、目标表
- 影响分析：修改某字段时自动分析下游依赖方并通知
- 合规追溯：数据泄露时可从泄露文件反向追溯来源表和处理链
- 操作审计：每位数据工程师的所有查询和执行操作全量记录
- 异常检测：基于血缘的异常数据流向告警
