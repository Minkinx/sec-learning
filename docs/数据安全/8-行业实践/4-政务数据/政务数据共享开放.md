# 政务数据共享开放

> 政务数据在共享和开放之间需要构建分类分级的安全管控体系，兼顾数据利用效率和安全保护。

## 政务数据分类

| 分类 | 定义 | 示例 | 开放程度 |
|------|------|------|---------|
| 无条件共享 | 跨部门使用无限制的数据 | 企业注册信息、土地登记信息 | 完全开放 |
| 有条件共享 | 须经审批后共享的数据 | 个人社保缴纳记录、不动产登记 | 审批后开放 |
| 不予共享 | 法律法规禁止共享的数据 | 国家安全数据、刑事侦查数据 | 不开放 |
| 公共开放 | 面向社会公众开放的数据 | 气象数据、交通流量、空气质量 | 自由下载 |
| 涉密数据 | 国家秘密级数据 | 国防动员数据、保密通信数据 | 禁止 |

## 共享机制与接口

| 共享方式 | 适用场景 | 安全控制 |
|---------|---------|---------|
| API接口 | 实时/准实时数据查询 | OAuth2.0+API密钥+传输加密 |
| 库表交换 | 批量数据同步 | 数据库访问白名单+字段级脱敏 |
| 文件交换 | 非结构化数据共享 | SFTP+数字签名+传输加密 |
| FTP/FTPS | 大文件批量传输 | FTPS传输+定期轮换凭证 |
| 前置机 | 高安全要求场景 | 物理隔离+数据摆渡+审计日志 |

## 开放数据平台要求

根据《政务数据开放管理办法》，政务数据开放平台须满足：

- 提供数据目录、数据查询、数据下载、API接口等基本功能
- 开放数据须遵循统一数据标准（GB/T 36344-2018）
- 开放前完成数据脱敏处理（去除个人身份信息、商业秘密）
- 建立开放数据质量评估机制，确保数据准确性、完整性、时效性
- 平台安全须满足等保三级要求

## 共享审批流程

```
数据需求方提交申请 → 数据提供部门初核 → 
数据安全委员会评估(含PIA) → 分管领导审批 → 
签署数据共享协议 → 技术对接 → 持续监控
```

## 使用追踪

- **调用审计**：记录每次共享请求的调用方、数据范围、查询时间、返回记录数
- **使用报告**：每季度生成数据共享使用情况报告，报送数据安全委员会
- **异常告警**：超量查询、非工作时间大流量下载等行为自动告警
- **定期复审**：已授权共享的数据访问权限每半年复审一次
- **权限收回**：项目结束后或授权到期自动收回数据访问权限

## 公共价值评估

政务数据开放的公共价值评估维度：

| 维度 | 评估指标 |
|------|---------|
| 社会效益 | 数据开放对民生改善、公共服务提升的贡献 |
| 经济效益 | 数据驱动产业创新、带动社会投资的效果 |
| 治理效能 | 数据共享对政府决策、行政管理效率的提升 |
| 数据质量 | 数据完整性、准确性、及时性、可用性 |
| 风险控制 | 数据开放的安全风险与隐私风险是否可控 |
