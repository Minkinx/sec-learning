# 政务云数据安全

> 政务云数据安全面临等保、分级保护、国密合规等多重监管要求叠加，须构建分层分域的安全防护体系。

## 等保与分级保护要求

政务云须同时满足《网络安全等级保护条例》和《涉密信息系统分级保护管理办法》的要求：

| 安全等级 | 适用系统 | 等保要求 | 数据分级保护 |
|---------|---------|---------|------------|
| 等保二级 | 一般政务信息系统 | 基础安全防护 | 一般数据 |
| 等保三级 | 政务核心业务系统 | 高可用+安全审计+数据加密 | 重要数据 |
| 等保三级增强 | 关键信息基础设施 | 等保三级+关基专项 | 核心数据 |
| 分级保护 | 涉密系统 | 涉密分级保护标准 | 国家秘密 |
| 多云统一管控 | 跨云政务系统 | 统一安全策略+统一审计 | 统一分级管理 |

## 政务云部署模式

| 部署模式 | 安全隔离级别 | 适用场景 | 数据安全优势 |
|---------|------------|---------|------------|
| 政务私有云 | 最高 | 垂直部门内部系统 | 完全物理隔离，访问可控 |
| 行业专属云 | 高 | 公安、税务、海关等行业 | 行业级安全策略 + 等保合规 |
| 区域政务云 | 中-高 | 省市多部门共享 | 统一安全门，降低重复投入 |
| 信创政务云 | 中-高 | 信创体系下的政务应用 | 国产密码+自主可控 |

**共享基础设施的隔离控制**：
- 租户隔离：VPC/安全组/虚拟防火墙（Hypervisor层隔离）
- 存储隔离：租户数据加密密钥独立、存储池逻辑隔离
- 网络隔离：VPN/VxLAN/安全域间防火墙

## 租户隔离策略

| 隔离层级 | 技术实现 | 安全级别 |
|---------|---------|---------|
| 物理隔离 | 独立服务器+独立存储 | 最高 |
| 虚拟化隔离 | KVM/VMware+安全虚拟化 | 高 |
| 容器隔离 | Docker/K8s+Seccomp/AppArmor | 中 |
| 应用层隔离 | 多租户SaaS应用 | 中 |
| 数据层隔离 | 数据库Schema/行级安全 | 中-高 |

## 数据驻留要求

- 政务数据须存储在中国境内
- 不同行政级别的政务数据须在本行政区域内存储（省级数据不出省）
- 涉及国家秘密的数据须存储在涉密机房，与政务云物理隔离
- 云服务商不得对政务数据进行任何形式的出境或境外访问
- 灾备中心须位于中国境内且同城/异地满足等保要求

## 跨部门数据访问

| 访问场景 | 安全控制 | 审批要求 |
|---------|---------|---------|
| 部门内数据访问 | 身份认证+RBAC | 默认授权 |
| 跨部门查询 | 接口授权+数据脱敏 | 共享协议+业务审批 |
| 跨区域查询 | 前置机+数据摆渡 | 上级部门审批 |
| 紧急查询（应急） | 临时授权+事后审计 | 24小时内补审批 |
| 涉密数据查询 | 专线+双人操作+HSM | 保密局审批 |

## 安全责任共担矩阵

| 安全控制项 | 云服务商负责 | 政务用户负责 |
|-----------|------------|------------|
| 物理安全 | 机房安保、电力、温控 | 不涉及 |
| 网络安全 | DDoS防护、网络防火墙 | 安全组配置、应用防火墙(WAF) |
| 主机安全 | Hypervisor安全、宿主机补丁 | 云主机加固、入侵检测(HIDS) |
| 数据安全 | 存储加密、密钥管理服务 | 数据分类分级、访问控制策略 |
| 应用安全 | 平台漏洞修复 | 应用代码审计、API安全 |
| 合规审计 | 平台审计日志 | 业务审计日志、合规自评 |
| 身份管理 | IAM基础服务 | 用户生命周期管理、MFA策略 |
| 应急响应 | 基础设施应急 | 数据泄露应急、业务恢复 |
