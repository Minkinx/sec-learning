# HIPAA实践与中国本土化适用

> 美国HIPAA法案是全球医疗数据保护的标杆，其Privacy Rule、Security Rule和Breach Notification Rule对中国医疗数据合规具有重要参考意义。

## Privacy Rule（隐私规则）

**核心限制**：受保护健康信息（PHI）的使用和披露限制

- **使用限制**：仅限治疗、支付、医疗运营（TPO）三大目的，其他用途须患者书面授权
- **最小必要原则**：使用和披露PHI须限制在实现目的所需的最小范围
- **患者权利**：访问权、修正权、披露记录获取权、投诉权

| 场景 | 是否需要授权 | 备注 |
|------|------------|------|
| 主治医生查看病历 | 否 (治疗目的) | 最小必要 |
| 保险公司审核理赔 | 否 (支付目的) | 仅限相关记录 |
| 药企进行数据研究 | 是 | 须签署研究授权协议 |
| 法务传票调取数据 | 是 | 须符合司法程序 |
| 公共卫生报告 | 否 (法定例外) | 法定传染病报告 |

## Security Rule（安全规则）

| 控制类别 | 控制措施 | 中国对标 |
|---------|---------|---------|
| 管理安全 | 安全官员、风险管理、员工培训、应急计划 | 数据安全管理规范 |
| 物理安全 | 设施访问控制、工作站安全、设备与媒体控制 | 等保物理安全要求 |
| 技术安全 | 访问控制、审计控制、完整性控制、传输安全 | 等保技术安全要求 |

HIPAA不指定具体技术方案，而是提出实施目标（Implementation Specifications），分为Required和Addressable两类。

## Breach Notification Rule（泄露通知规则）

| 受影响人数 | 通知对象 | 通知时限 |
|-----------|---------|---------|
| 500+人 | 患者+HHS+媒体 | 60天内通知HHS，患者不延迟 |
| <500人 | 患者+HHS年度日志 | 年度内通知HHS |

**泄露判定四步风险评估法**：
1. 数据是否被获取或查看
2. 数据是否被使用或披露
3. 风险对个人造成伤害的概率
4. 风险对个人造成伤害的严重程度

若上述风险低，则不构成泄露（须记录评估结论）。

## 本土化适用

**中国的对标框架**：
- 《个人信息保护法》第28条（敏感个人信息）对应Privacy Rule
- 《数据安全法》第27条（数据安全保护义务）对应Security Rule
- 《个人信息保护法》第57条（泄露通知）对应Breach Notification Rule

**BAA（业务伙伴协议）的中国实践**：
- HIPAA要求covered entity与business associate签署BAA
- 中国对应机构应签署《数据委托处理协议》（PIPL第21条）
- 须明确数据范围、处理目的、期限、安全措施、责任划分

**未成年人保护**：HIPAA对儿科的额外保护要求与中国《未成年人保护法》第72条精神一致，均要求对未成年人数据提供更高保护级别。
