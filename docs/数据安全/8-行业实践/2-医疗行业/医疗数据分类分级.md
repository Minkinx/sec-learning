# 医疗数据分类分级

> 医疗数据类型多样、敏感度高，涉及国家卫健委数据分类体系与PIPL敏感个人信息双重监管框架。

## 医疗数据类型

| 数据类别 | 子类型 | 敏感程度 | 典型容量 |
|---------|-------|---------|---------|
| EMR/电子病历 | 门诊记录、住院首页、病程记录、手术记录 | 极高 | GB级/患者 |
| PACS/影像 | CT/MRI/X射线/DICOM文件 | 高 | MB~GB级/次 |
| LIS/检验数据 | 血常规、生化、免疫、微生物 | 高 | KB级/项 |
| 基因组数据 | WGS/WES/基因检测报告 | 极高 | GB级/人（WGS） |
| 报销数据 | 医保结算、商业保险理赔 | 高 | MB级/次 |
| 可穿戴设备 | 心率、血糖、睡眠、运动 | 中 | 连续采集 |
| 医院运营 | 床位数、药品库存、设备使用率 | 低-中 | 结构化数据 |

## 分级标准：卫健委指南 vs PIPL

| 分级维度 | 卫健委标准 | PIPL |
|---------|-----------|------|
| 最高等级 | 核心医疗数据（泄露危害国家安全） | 敏感个人信息（基因、生物识别、医疗健康） |
| 定级依据 | 数据泄露对患者/医院/国家的危害程度 | 是否可识别特定自然人+是否涉及隐私敏感区域 |
| 敏感数据清单 | 明确列举18类核心医疗数据 | 列举基因、生物识别、医疗健康、性取向等 |
| 保护要求 | 对应等级保护措施矩阵 | 单独同意+告知必要性+影响评估 |
| 跨院共享 | 依数据等级分级审批 | 委托处理/共享须单独授权 |

## 特殊类别数据保护

**基因数据**：PIPL明确列为敏感个人信息，须取得单独同意，存储须加密，禁止向境外提供。

**生物识别数据**：含人脸、指纹、虹膜等医院门禁和诊疗认证数据，须单独授权，不得作为唯一认证方式。

**病历数据**：含患者主诉、诊断、治疗方案等核心医疗隐私，所有操作须留审计日志，修改须记录原值。

## 跨院数据共享安全

- 建立医疗数据共享平台，采用隐私计算技术（联邦学习/MPC）实现数据可用不可见
- 数据分级后按等级确定共享审批流程和授权有效期
- 患者授权管理：提供撤回授权的技术能力
- 共享日志可审计、可追溯，支持合规检查
- 第三方（药企/保险公司）接入须通过数据安全评估

## 落地建议

1. 以卫健委《医院数据分级分类指南》为基础，叠加PIPL敏感个人信息识别
2. 数据分级与医院信息系统（HIS/LIS/PACS）的数据字典对接
3. 建立数据安全委员会，由医务部、信息科、法务部联合运作
4. 每年至少一次数据资产盘点与分级复核
