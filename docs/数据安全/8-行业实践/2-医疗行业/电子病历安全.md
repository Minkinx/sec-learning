# 电子病历安全

> 电子病历（EMR）是医疗数据安全的核心资产，其安全保护涉及访问控制、审计追踪、完整性保护等多个维度。

## 核心安全控制

| 控制域 | 要求 | 实现方式 |
|-------|------|---------|
| 身份认证 | 所有用户须唯一标识，支持MFA | 员工卡+密码/生物特征+短信验证码 |
| 访问控制 | 基于角色的访问控制（RBAC） | 按科室、职称、权限分级 |
| 审计日志 | 所有操作可追溯 | 记录谁、何时、对谁、做了什么、修改前内容 |
| 完整性保护 | 病历内容不可篡改 | 数字签名+哈希链+数据库事务日志 |
| 数据备份 | 定期备份，异地容灾 | 全量+增量备份，RPO<1h |
| 传输加密 | 内网外网传输均加密 | TLS 1.3/国密TLS |

## 访问控制模型

| 角色 | 默认可访问范围 | 扩展访问需审批 |
|------|--------------|--------------|
| 主治医生 | 本人管辖患者的全部病历 | 跨科室会诊 |
| 护士 | 本人管辖患者的基本信息+护理记录 | 调取既往病历 |
| 医技科室 | 检验检查申请单+报告 | 查看完整病史 |
| 药师 | 处方信息+过敏史 | 查看诊断信息 |
| 管理人员 | 统计数据（脱敏） | 查看原始病历 |
| 患者本人 | 本人病历（电子版） | 查看第三方共享记录 |

## 病历修改审计

- **修改留痕**：任何修改操作保留修改前、修改后内容及修改人、修改时间
- **锁定机制**：出院后病历自动锁定，修改须经医务部审批
- **电子签名**：三级查房记录须实施数字签名（依据《电子签名法》）
- **防抵赖**：签名密钥存储在HSM中，签名操作记录审计日志
- **时间戳**：所有关键操作使用国家授时中心可信时间戳

## EMR系统认证要求

中国EMR系统须满足以下认证体系：

- **国家医疗健康信息互联互通标准化成熟度测评**：数据标准、安全、共享能力评估
- **电子病历系统应用水平分级评价（1-8级）**：7级以上要求全流程数字签名与审计
- **等保2.0三级**：三级医院须满足等保三级，其中包含数据安全扩展要求

## 患者数据访问管理

- 患者授权：患者可通过APP授权指定医生查看病历
- 紧急访问：抢救场景下允许突破权限访问，事后须48小时内补审批
- 去标识化导出：用于科研目的的数据须经去标识化处理
- 访问控制评审：每季度审查一次权限分配合理性，清理僵尸账号

## 备份与容灾

- 主数据中心与灾备中心物理距离>30km
- 核心EMR数据库RPO<15分钟，RTO<4小时
- 每年至少一次容灾切换演练
- 备份数据加密存储，密钥与主数据中心隔离管理
