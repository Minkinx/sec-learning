# 金融行业数据跨境合规

> 金融数据出境涉及人民银行、国家网信办、外汇管理局等多部门监管，合规要求叠加。

## 跨境数据流转场景

| 场景 | 数据类型 | 监管要求叠加 |
|------|---------|-------------|
| 境外分行数据传输 | 客户信息、交易流水 | 数据安全法+央行跨境数据指引 |
| 国际结算与SWIFT | 跨境支付指令、报文 | 外汇管理局+SWIFT安全框架 |
| 全球风险集中管理 | 集团级风险头寸、客户画像 | 并表监管+数据出境安全评估 |
| 境外监管报送 | 客户尽职调查(CRS/FATCA) | 跨境数据报送+税务信息交换协议 |
| 境外IT外包 | 灾备、运维、开发测试 | 外包管理办法+数据出境评估 |

## 监管部门与要求

- **人民银行**：《金融数据安全 数据安全分级指南》《金融数据出境安全评估办法》
- **国家网信办**：《数据出境安全评估办法》《个人信息出境标准合同办法》
- **外汇管理局**：跨境资金流动监测、外汇数据报送合规
- **银保监会**：中资银行境外机构数据管理规定

## 金融数据出境分类

| 分类 | 举例 | 合规路径 |
|------|-----|---------|
| 个人客户信息 | 姓名、身份证号、账户信息 | 安全评估/标准合同/认证 |
| 交易数据 | 跨境支付报文、SWIFT MT103 | 安全评估+行业主管部门同意 |
| 信贷数据 | 征信报告、贷款记录 | 禁止出境或经安全评估 |
| 机构数据 | 年报、合规报告、财务报表 | 依出境合同+内部审批 |
| 核心风控数据 | 风控模型参数、欺诈规则 | 原则上禁止出境 |

## 合规路径选择

1. **数据出境安全评估**：处理100万人以上个人信息或重要数据出境的机构须申报
2. **个人信息出境标准合同**：未触发安全评估门槛且处理者与非境外接收方签署
3. **个人信息保护认证**：支付机构等大规模处理用户信息的出境场景
4. **监管豁免**：跨境支付、国际金融交易等法定必传数据的豁免适用

## 离岸数据中心要求

- 在境外金融中心设数据中心的，须满足东道国与中国的双重合规
- 遵循《商业银行信息科技风险管理指引》对境外数据中心的管控要求
- 离岸数据与境内数据的隔离策略、访问控制、审计日志须专项设计
- 应建立跨境数据流转的实时监控和应急阻断能力
