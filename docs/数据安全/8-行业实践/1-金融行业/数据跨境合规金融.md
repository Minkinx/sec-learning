# 金融行业数据跨境合规

> 金融数据出境涉及人民银行、国家网信办、外汇管理局等多部门监管，合规要求叠加。

## 跨境数据流转场景

| 场景 | 数据类型 | 监管要求叠加 | 合规复杂度 |
|------|---------|-------------|-----------|
| 境外分行数据传输 | 客户信息、交易流水 | 数据安全法 + 央行跨境数据指引 + 境外当地法规 | 高 |
| 国际结算与SWIFT | 跨境支付指令、报文 | 外汇管理局 + SWIFT安全框架 + 制裁合规 | 高 |
| 全球风险集中管理 | 集团级风险头寸、客户画像 | 并表监管 + 数据出境安全评估 + 境外监管要求 | 极高 |
| 境外监管报送 | 客户尽职调查(CRS/FATCA) | 跨境数据报送 + 税务信息交换协议 | 中 |
| 境外IT外包 | 灾备、运维、开发测试 | 外包管理办法 + 数据出境评估 + 供应商安全审计 | 中-高 |
| 跨境营销与分析 | 客户行为数据、交易习惯 | 个人信息保护法 + 标准合同/认证 | 高（需用户单独同意）|

## 监管部门与要求

### 主要监管部门

| 监管部门 | 相关法规/文件 | 监管重点 | 合规要求 |
|----------|-------------|----------|----------|
| 人民银行 | 《金融数据安全 数据安全分级指南》《金融数据安全 数据生命周期安全规范》 | 数据分级分类、数据出境安全评估 | 数据分级体系、跨境评估报告 |
| 国家网信办 | 《数据出境安全评估办法》《个人信息出境标准合同办法》 | 安全评估、标准合同、认证 | 安全评估申报、标准合同备案 |
| 外汇管理局 | 《跨境资金流动监测》《金融机构外汇数据报送》 | 资金流动监测、外汇数据合规 | 数据报送、交易真实性审核 |
| 金融监管总局 | 《银行保险机构数据安全管理办法》《中资银行境外机构数据管理规定》 | 境外机构数据管理、外包风险 | 专项管理制度、定期检查 |
| 国家密码管理局 | 《密码法》《商用密码管理条例》 | 跨境传输加密密码算法合规 | 商用密码应用安全性评估 |

## 金融数据出境分类

| 数据分类 | 具体举例 | 允许出境条件 | 合规路径 | 特殊限制 |
|----------|---------|-------------|----------|----------|
| 个人客户信息（敏感） | 身份证号、生物识别、金融账户、定位信息 | 满足任一条件 | 安全评估/标准合同/认证 | 须取得个人单独同意 |
| 个人客户信息（一般） | 姓名、联系方式、职业信息 | 满足任一条件 | 安全评估/标准合同/认证 | 告知+同意 |
| 交易数据 | 跨境支付报文、SWIFT MT103/MT202 | 行业主管部门认可 | 安全评估 + 行业审批 | 遵守SWIFT安全政策 |
| 信贷/征信数据 | 征信报告、贷款记录、担保信息 | 原则上禁止出境 | 特殊情形经安全评估+人行特批 | 极其严格 |
| 机构非敏感数据 | 年报、合规报告、财务报表 | 依出境合同 + 内部审批 | 合同约束 + 安全评估（批量时） | 保密期限条款 |
| 核心风控数据 | 风控模型参数、欺诈规则、信用评分模型 | 原则上禁止出境 | 需专项审批 | 禁止出境 |
| 重要数据 | 影响国家安全和公共利益的数据 | 严格限制 | 安全评估 + 行业审批 + 可能禁止 | 按《数据安全法》执行 |

## 合规路径选择

| 合规路径 | 适用条件 | 办理周期 | 有效期 | 费用预估 | 适用范围 |
|----------|----------|----------|--------|----------|----------|
| 数据出境安全评估 | 处理100万人以上个人信息 / 向境外提供重要数据 / 累计向境外提供10万人以上敏感个人信息 | 45-60个工作日（可延长） | 2年 | 高（含法律+技术整改）| 大型金融机构、涉及重要数据 |
| 个人信息出境标准合同 | 未触发安全评估门槛 + 处理者与非境外接收方签署 | 1-3个月 | 合同有效期 | 中 | 中小金融机构、有限数据量 |
| 个人信息保护认证 | 支付机构等大规模处理用户信息的出境场景 | 3-6个月 | 3年 | 高 | 大型互联网金融机构 |
| 监管豁免 | 跨境支付、国际金融交易等法定必传数据 | 申报豁免认定 | 视情形 | 低 | 日常国际结算场景 |
| 安全评估豁免 | 非个人信息或豁免清单内的数据 | 内部评估 | 视情形 | 低 | 符合条件的数据类型 |

## 离岸数据中心要求

| 要求维度 | 具体内容 | 合规验证方法 |
|----------|----------|-------------|
| 双重合规 | 满足东道国与中国的双重监管要求 | 法律比对清单 + 当地律师意见书 |
| 隔离策略 | 离岸数据与境内数据的物理/逻辑隔离 | 网络架构审查 + 访问控制测试 |
| 访问控制 | 明确跨境访问权限审批流程和最小权限原则 | 权限审计 + IAM 日志分析 |
| 审计日志 | 跨境数据传输的完整审计记录 | 日志完整性校验 + 定期审计 |
| 加密要求 | 传输和存储均须使用国密算法（SM2/SM3/SM4） | 密码应用安全性评估 |
| 应急阻断 | 建立跨境数据流转的实时监控和应急阻断能力 | 红蓝对抗演练 + 应急响应测试 |
| 备份要求 | 境外数据中心数据须在境内保留完整副本 | 备份验证 + 恢复演练 |
| 监管接入 | 确保监管机构可随时访问审计数据 | 监管接口 + 定期报告 |

## 安全评估申报流程

```
数据出境需求提出
    │
    ├─ 判断出境数据类型和数量
    │   ├─ 触发安全评估门槛 → 申报安全评估
    │   └─ 未触发 → 标准合同/认证路径
    │
    ├─ 准备申报材料（清单）
    │   ├─ 数据出境风险自评估报告
    │   ├─ 数据处理情况说明
    │   ├─ 境外接收方情况说明
    │   └─ 法律文件和合同
    │
    ├─ 省级网信办受理
    │   ├─ 材料不齐 → 补正通知（5个工作日内）
    │   └─ 材料齐全 → 转报国家网信办
    │
    ├─ 国家网信办组织评估
    │   ├─ 评估通过 → 出具安全评估结果（有效期2年）
    │   └─ 评估不通过 → 终止数据出境
    │
    └─ 评估结果实施
        ├─ 按评估要求实施安全措施
        └─ 定期（每年）进行安全评估复审
```

## 金融数据跨境合规最佳实践

1. **建立数据出境分类清单**：对全量数据进行出境分级，标注是否允许/限制/禁止出境。
2. **设计与实施技术控制**：部署数据脱敏（静态/动态）、加密传输（国密算法）、访问控制（最小权限）。
3. **建立审批流程**：数据出境须走正式审批（法务+合规+安全+业务四方会签）。
4. **定期合规审计**：每半年开展一次数据出境合规审计，跟踪法规变化。
5. **建立应急预案**：制定跨境数据安全事件响应预案，包括跨境应急阻断机制和跨境通知流程。
6. **培训与意识**：对涉及跨境业务处理的员工开展专项合规培训。
