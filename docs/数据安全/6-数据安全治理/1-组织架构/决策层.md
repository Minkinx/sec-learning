# 决策层

> 董事会与安全委员会的决策职责、治理结构设计与运行机制，涵盖 SME 与大型企业的差异化组织模式

## 治理结构设计

数据安全治理的决策层位于组织顶层，承担数据安全的最终责任。典型治理结构包括两种模式：

**模式一：董事会下设安全委员会**
```
┌─────────────────────────────────────┐
│            董事会/监事会              │  ← 最终责任主体
├─────────────────────────────────────┤
│         审计委员会 / 风险委员会        │  ← 独立监督
├─────────────────────────────────────┤
│          数据安全委员会               │  ← 专项决策机构
├─────────┬─────────┬────────┬────────┤
│ CEO/总裁 │ CISO    │  CFO   │ 法务   │  ← 跨部门委员
└─────────┴─────────┴────────┴────────┘
```

**模式二：中小企业简化结构**
```
┌─────────────────────┐
│     CEO / 总经理      │  ← 承担最终责任
├─────────────────────┤
│  CISO / 安全负责人    │  ← 直接汇报
├─────────────────────┤
│  跨部门安全联络组     │  ← 虚拟组织
└─────────────────────┘
```

- **董事会**：对数据安全负最终责任，审批安全战略和年度报告，确保安全投入与业务风险匹配
- **安全委员会**（或数据安全委员会）：董事会授权的专项治理机构，负责重大安全决策和资源调配
- **CISO/DPO汇报线**：建议CISO向董事会或安全委员会直接汇报，确保独立性和资源获取能力

## 安全委员会组成

安全委员会应跨部门组建，确保各业务视角的覆盖。以下为大型企业典型设置：

| 角色 | 职责 | 投票权 | 替补成员 |
|------|------|--------|----------|
| CEO/总裁 | 委员会主席，战略决策与资源保障 | 一票否决权 | COO |
| CISO/数据安全负责人 | 安全战略制定与执行监督 | 有 | 副CISO |
| CFO | 预算审批与风险管理投入评估 | 有 | 财务总监 |
| 法务总监/合规官 | 法律合规风险把控，监管对接 | 有 | 合规经理 |
| 业务线负责人（各BU） | 业务需求与安全约束的平衡 | 有 | 业务副总监 |
| IT负责人 | 技术架构与基础设施保障 | 有 | IT副总监 |
| 内审负责人 | 独立审计与监督 | 观察员 | 内审经理 |

**中小企业（<500人）精简版：**
| 角色 | 职责 |
|------|------|
| CEO/总经理 | 最终决策者 |
| CISO/安全负责人 | 安全执行统筹 |
| 财务负责人 | 预算审核 |
| 法务/合规 | 合规把控 |

## 章程与会议机制

### 章程核心内容
- **委员会使命与定位**：明确委员会在数据安全治理体系中的角色
- **成员任命/罢免规则**：任期、资格要求、罢免条件
- **决策权限边界**：预算审批额度、风险接受阈值、策略批准范围
- **表决机制**：多数通过 / 三分之二多数 / 一票否决情形
- **利益冲突管理**：委员回避制度
- **议事规则**：动议、讨论、表决、决议流程

### 会议频率与议程

| 会议类型 | 频率 | 主要内容 | 产出 |
|---------|------|---------|------|
| 例行会议 | 每季度 | 安全态势回顾、合规动态、重大风险、预算执行 | 会议纪要、决议 |
| 临时会议 | 按需 | 重大安全事件处置、紧急预算审批 | 专项决议 |
| 年度扩大会议 | 每年 | 年度安全评审、下年度战略规划 | 年度报告、战略规划 |
| 专题研讨会 | 半年度 | 深度议题（如零信任架构、数据跨境方案） | 研究报告 |

### 议程模板（例行会议）
1. 上次会议决议执行情况回顾（5分钟）
2. 安全态势总体报告（15分钟）—— CISO
3. 合规动态与监管变化（10分钟）—— 法务
4. 重大风险项审议（20分钟）—— 风险管理
5. 安全预算执行与规划（15分钟）—— CFO
6. 改进计划评审与决议（15分钟）—— 全员
7. 下次会议预告与待办事项（5分钟）

## 企业规模对比

| 维度 | 小型企业（<200人） | 中型企业（200-2000人） | 大型企业（>2000人） |
|------|------------------|----------------------|-------------------|
| 决策结构 | 总经理决策 + 安全顾问 | 安全委员会 | 董事会层级安全委员会 |
| 委员会人数 | 不设委员会，3-5人核心 | 5-7人跨部门 | 7-11人跨部门+外部顾问 |
| 会议频率 | 按需 | 季度例会 | 月度+季度+年度 |
| CISO汇报线 | 向CEO汇报 | 向CEO/董事会汇报 | 向安全委员会/董事会汇报 |
| DPO设置 | 兼职/外包 | 专职DPO | DPO团队（3-5人） |
| 预算审批权限 | CEO直接审批 | CISO审批<$100K | 分级授权（CISO $500K/委员会$5M） |

## 年度安全评审

董事会/安全委员会每年至少开展一次全面的数据安全评审，涵盖：

- **体系评审**：安全策略、制度、流程的有效性评估，包括控制测试结果和审计发现
- **合规评审**：法律法规变化对组织的影响分析与差距评估，跟踪监管动态和执法案例
- **安全事件复盘**：年度安全事件的总体回顾与根因分析，计算MTTD/MTTR趋势
- **资源投入评估**：安全预算执行情况与下一年度需求规划，ROI分析
- **战略方向调整**：基于威胁形势和业务变化的优先级重排，更新三年安全路线图
- **成熟度评估**：使用DSMM或NIST CSF框架进行年度成熟度评估，量化改进进度
