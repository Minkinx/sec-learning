# RACI职责矩阵

> 责任分配矩阵的定义、设计方法及数据安全场景的实践应用

## RACI定义

RACI矩阵是一种职责分配工具，明确每项活动中各角色的参与程度：

| 角色 | 含义 | 说明 |
|------|------|------|
| **R** — Responsible | 执行人 | 具体完成任务的人，每项活动至少一个R |
| **A** — Accountable | 负责人 | 对结果最终负责，每项活动有且仅有一个A |
| **C** — Consulted | 咨询方 | 在决策前必须征询意见的角色（双向沟通） |
| **I** — Informed | 知会方 | 执行后须被告知结果的角色（单向通知） |

## RACI矩阵设计方法论

1. **列出活动**：将流程分解到可执行的活动粒度（非阶段，而是具体任务）
2. **识别角色**：确定参与的所有角色/岗位（使用角色而非个人姓名）
3. **逐行分配**：对每项活动确定R、A、C、I，确保每行唯一一个A
4. **交叉检查**：检查是否有角色承担过多R（瓶颈）或没有参与任何活动（冗余）
5. **沟通确认**：与各角色确认分配结果并获得共识
6. **持续维护**：随组织架构和流程变化定期更新RACI矩阵
