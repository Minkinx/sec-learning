# 执行层

> 安全运营团队架构、岗位序列划分、职责矩阵与工程级执行

## 运营团队划分

执行层由多个专业化团队构成，覆盖数据安全的工程、运营与审计职能：

| 团队 | 核心职责 |
|------|---------|
| SOC（安全运营中心） | 实时监控、告警研判、威胁狩猎、事件初期处置 |
| 事件响应团队（CIRT） | 重大事件调查、取证、遏制、根因分析、复盘 |
| 数据安全运营组 | DLP策略维护、数据脱敏操作、访问审计、权限复核 |
| 部署安全组 | CI/CD安全门禁、容器安全、IaC扫描、部署合规检查 |
| 红蓝对抗组 | 渗透测试、攻击模拟、防御有效性验证 |

## 岗位序列与职级

- **安全工程师**（L1-L3）：策略配置、工具运维、告警处理、日常巡检
- **安全分析师**（L1-L3）：威胁分析、事件研判、取证溯源、情报运营
- **安全架构师**（L2-L4）：安全架构设计、技术选型、标准制定、架构评审
- **安全审计师**（L2-L3）：合规审计、控制测试、差距分析、整改跟踪
- **安全开发工程师**（L1-L3）：安全工具开发、自动化脚本、检测规则开发

## 工程级执行规范

- **变更管理**：所有安全策略变更须走变更审批流程，遵循测试→预发布→生产三步走
- **运维手册**：每类安全运营工作须有标准SOP，定期更新并演练
- **知识沉淀**：事件处置完成后48小时内完成知识库更新，纳入案例库
- **交接班机制**：SOC实行标准化交接班模板，涵盖未完成事件、待跟进告警、情报更新
