# 执行层

> 安全运营团队架构、岗位序列划分、职责矩阵与工程级执行，含SME与大型企业的差异化执行模式

## 运营团队划分

执行层由多个专业化团队构成，覆盖数据安全的工程、运营与审计职能：

### 大型企业完整团队

| 团队 | 核心职责 | 典型人数 | 在岗时间 |
|------|---------|---------|---------|
| SOC（安全运营中心） | 实时监控、告警研判、威胁狩猎、事件初期处置 | 15-30人 | 7×24三班倒 |
| 事件响应团队（CIRT） | 重大事件调查、取证、遏制、根因分析、复盘 | 5-10人 | On-call轮值 |
| 数据安全运营组 | DLP策略维护、数据脱敏操作、访问审计、权限复核 | 5-8人 | 5×8 |
| 安全工程组 | CI/CD安全门禁、容器安全、IaC扫描、部署合规 | 4-6人 | 5×8 |
| 红蓝对抗组 | 渗透测试、攻击模拟、防御有效性验证 | 3-6人 | 按项目制 |
| 威胁情报组 | 情报采集/分析/分发、IOC管理、上下文丰富 | 2-4人 | 5×8 |

### SOC值班层级模型

```
┌──────────────────────────────────────────┐
│        SOC Manager                        │  ← 团队管理/流程优化
├──────────────────────────────────────────┤
│                                           │
├─ Tier 1 - 监控与告警分流 ────────────────┤
│  职责：实时监控告警、初步筛选、创建工单      │
│  技能：基础安全知识、SIEM操作、告警分类      │
│  决策权：关闭误报告警、升级至T2             │
│  轮班：7×24，每班1-2人                     │
│                                           │
├─ Tier 2 - 调查与研判 ────────────────────┤
│  职责：深度告警调查、确认事件危害、初步处置    │
│  技能：进阶威胁分析、取证基础、恶意分析       │
│  决策权：确认事件、执行标准Playbook          │
│  轮班：5×8，On-call支持                    │
│                                           │
└─ Tier 3 - 威胁狩猎与逆向 ────────────────┘
   职责：高级威胁分析、定制化取证、反制措施
   技能：逆向工程、内存取证、情报运营
   决策权：升级至CIRT、修改检测规则
   轮班：5×8，重大事件随时响应
```

## 工程级执行规范

### 变更管理
所有安全策略变更须走变更审批流程：
1. **测试环境验证**：在非生产环境验证策略效果，确认无业务影响
2. **预发布审批**：提交变更申请单（含回滚方案），经变更委员会审批
3. **生产部署**：窗口期内执行变更，监控变更后行为
4. **变更验证**：24小时内验证变更效果，关闭变更工单

### 运维手册SOP示例
每类安全运营工作须有标准SOP：
- **告警处理SOP**：告警确认→定级→处置→记录→关闭
- **事件升级SOP**：触发条件→升级路径→通知模板→报告格式
- **策略变更SOP**：申请→评估→测试→审批→部署→验证

### 交接班机制
SOC实行标准化交接班模板：
```
■ 交接班报告 ■ 日期：____ 班次：____ 分析师：____
─────────────────────────────────────────
1. 未闭环事件（P0/P1）：[事件编号/状态/待办事项]
2. 待跟进告警（P2）：[告警简述/已采取措施]
3. 情报更新：[新增IOC/威胁公告]
4. 策略变更：[今日生效的策略变更清单]
5. 系统异常：[SIEM/日志源异常情况]
6. 特别关注：[待观察用户/资产]
─────────────────────────────────────────
```

### 知识沉淀机制
- 事件处置完成后48小时内完成知识库更新
- 每类攻击手法建立标准分析模板
- 月度运营总结提炼可复用的检测规则和响应流
- 季度红蓝对抗产出改进建议纳入知识库

## 企业规模执行对比

| 维度 | 小型企业 | 中型企业 | 大型企业 |
|------|---------|---------|---------|
| SOC模式 | MSSP外包 | 自建小规模SOC(5-10人)+MSSP | 自建全规模SOC(20-40人) |
| 值班方式 | 工作时间+电话On-call | 12×5+On-call | 7×24三班倒 |
| IR能力 | 厂商应急响应 | 内部T1/T2+厂商T3 | 内部完整CIRT |
| 工具链 | 开源/轻量SIEM | Splunk/SOAR | Splunk+SOAR+威胁情报平台 |
| 自动化程度 | 低，手动操作多 | 中度，SOAR Playbook | 高度，自动编排与响应 |
