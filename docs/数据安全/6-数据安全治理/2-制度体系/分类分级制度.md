# 分类分级制度

> 数据资产分类分级的标准模板、判定流程与评审机制

## 分类分级框架

根据DSL第21条要求及GB/T 37973-2019《信息安全技术 数据安全能力成熟度模型》，数据分类分级采用"业务属性 + 敏感性 + 监管要求"三维判定模型：

### 分类维度（按业务属性）

| 分类 | 说明 | 示例 |
|------|------|------|
| 用户数据 | 个人信息及用户行为数据 | 姓名、手机号、设备ID |
| 业务数据 | 运营产生的核心业务数据 | 订单、交易记录、产品信息 |
| 财务数据 | 财务与会计相关数据 | 财务报表、成本数据、发票 |
| 人力资源数据 | 员工信息与人事数据 | 简历、薪酬、绩效记录 |
| 研发数据 | 技术与知识产权数据 | 源代码、设计文档、专利 |
| 安全运维数据 | 安全日志与运维数据 | 审计日志、漏洞报告、配置 |

### 分级维度（按敏感性 + 监管要求）

| 级别 | 名称 | 定义 | 典型控制要求 |
|------|------|------|-------------|
| L4 | 核心数据 | 一旦泄露造成巨大损害 | 加密存储、严格访问控制、操作审批、全量审计 |
| L3 | 重要数据 | 一旦泄露造成严重损害 | 加密存储、访问控制、脱敏展示、定期审计 |
| L2 | 内部数据 | 仅限内部使用，泄露有轻微影响 | 访问控制、水印标记、默认内部共享 |
| L1 | 公开数据 | 可对外公开 | 完整性保护、发布审批 |

## 定级流程

1. **数据资产盘点**：数据管理员（Data Custodian）完成数据资产登记
2. **初步定级**：数据所有者（Data Owner）基于判定标准提出初评级别
3. **合规审查**：合规团队检查是否满足法律和监管特殊要求
4. **安全复核**：安全团队结合威胁模型评估风险修正级别
5. **审批生效**：数据所有者负责人审批，L3/L4级须安全委员会备案
6. **标签标识**：在数据资产目录及系统元数据中添加分级标签

## 特殊处理规则

- **跨境数据**：涉及出境的L2及以上数据，自动触发数据出境安全评估流程
- **个人信息**：个人信息的定级不低于L2，敏感个人信息不低于L3
- **混合数据**：同一数据集包含多个级别的，按就高原则执行
- **脱敏后数据**：经有效脱敏后可根据残留风险下调等级

## 评审与更新

- **定期评审**：每年至少一次全面分类分级复审
- **触发式更新**：新业务上线、法律法规变更、数据使用场景变化时即时更新
- **例外处理**：对定级结果有异议的，可通过Data Owner提交级别变更申请，经安全委员会裁定
