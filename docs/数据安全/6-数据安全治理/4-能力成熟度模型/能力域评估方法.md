# 能力域评估方法

> 各能力域的具体评估方法、证据采集标准与评分细则

## 评估方法四维体系

DSMM评估采用四种方法综合采集证据：

| 方法 | 适用场景 | 证据形式 |
|------|---------|---------|
| **文档审查** | 制度、流程、技术文档的完整性和合规性验证 | 政策文件、SOP、操作手册、基线配置模板 |
| **人员访谈** | 了解执行情况和实际能力，验证文档与实际的一致性 | 访谈记录、能力证明（认证/培训证书） |
| **过程观察** | 验证流程是否按文档执行，识别执行偏差 | 现场观察记录、操作录像回放 |
| **技术测试** | 验证技术控制措施的有效性和配置正确性 | 扫描报告、渗透测试报告、配置审计结果 |

## 典型能力域评估细则

### 数据分类分级
- **文档审查**：分类分级制度文档、数据资产目录、定级记录
- **人员访谈**：数据所有者是否理解定级职责，数据管理员是否掌握操作流程
- **技术测试**：分类分级标签是否在数据资产目录和系统元数据中自动同步

### 访问控制
- **文档审查**：访问控制策略、权限管理制度、定期审计划
- **过程观察**：权限申请→审批→开通→复核的完整流程执行情况
- **技术测试**：越权访问测试、权限泛滥分析、闲置账号检测

### 数据脱敏
- **文档审查**：脱敏策略、脱敏算法选型标准、脱敏操作流程
- **技术测试**：动态脱敏效果验证、静态脱敏后数据逆向恢复测试
- **人员访谈**：脱敏操作人员是否经过培训，是否了解差分攻击风险

## 证据采集标准

证据须遵循"SMART"原则：
- **S**（Specific）：证据应具体对应到特定能力域和评分项，避免笼统
- **M**（Measurable）：可量化或可判定通过/不通过
- **A**（Auditable）：可追溯，包含时间戳、责任人、版本号
- **R**（Relevant）：与评估的能力域直接相关
- **T**（Timely）：在评估有效期内的最新版本

## 评分细则（以数据分类分级为例）

| 评分 | 判定标准 |
|------|---------|
| 1分 | 存在人工分类，无固定标准，依赖个人判断 |
| 2分 | 已制定分类标准但未全面执行，部分数据已打标签 |
| 3分 | 分类标准完整覆盖所有业务域，标签全量落地，有定期评审 |
| 4分 | 分类分级流程量化管理，标签准确率和覆盖率定期统计，有改进机制 |
| 5分 | 自动化分类分级工具全量部署，实时标签更新，与安全策略自动联动 |
