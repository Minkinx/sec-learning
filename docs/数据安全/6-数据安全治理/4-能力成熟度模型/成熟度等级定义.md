# 成熟度等级定义

> DSMM成熟度等级1-5的详细定义、特征描述与示例

## 成熟度等级全景

```
Level 5 持续优化 ──→ 自动化运营，数据驱动优化，安全文化内化
Level 4 量化控制 ──→ 可量化度量，基于KPI/SLA持续改进
Level 3 充分定义 ──→ 流程标准化并文档化，有组织地执行
Level 2 计划跟踪 ──→ 有意识但被动，流程初步建立
Level 1 非正式执行 ──→ 随机无序，依赖个人能力
```

## Level 1 — 非正式执行

**特征**：数据安全工作依赖个人经验和自觉性，没有系统化的制度流程，安全事件发生后被动响应，缺乏事前预防。

- 数据分类分级：由个别员工凭经验手动处理，无统一标准
- 访问控制：基于个人判断，权限管理混乱，大量闲置账号
- 数据加密：无统一加密策略，部分部门自行决定
- 安全监控：无集中式安全监控平台
- 典型表现："出了事再说"

## Level 2 — 计划跟踪

**特征**：组织开始有意识地建立数据安全管理体系，基本流程已建立但执行不彻底，以被动合规驱动为主。

- 数据分类分级：已制定基本标准，但覆盖率不足60%
- 访问控制：建立了账号生命周期管理流程，但定期审计不严格
- 数据加密：核心数据库已启用加密，但密钥管理较松散
- 安全监控：部署了基础SIEM，告警量高但误报率大

## Level 3 — 充分定义

**特征**：数据安全流程已标准化并在全组织范围内文档化，有明确的责任分工和执行监督，进入主动管理阶段。

- 数据分类分级：标准完整覆盖全部业务域，标签全量落地
- 访问控制：RBAC/ABAC模型实施，权限审批流程完整
- 数据加密：全库加密 + 传输加密 + 密钥集中管理
- 安全监控：SIEM + UEBA，告警关联分析，定期演练
- 典型表现：通过了ISO 27001或等保三级认证

## Level 4 — 量化控制

**特征**：建立数据安全量化指标体系，通过数据进行过程管理和持续改进，风险可预测。

- 关键指标：分类准确率 > 95%，权限合规率 > 98%，MTTR < 4h
- 数据驱动：基于量化数据优化安全策略和资源配置
- 预测能力：基于历史数据预测安全趋势并提前干预
- 自动化：70%以上的安全操作已实现自动化

## Level 5 — 持续优化

**特征**：数据安全融入组织文化，实现自动化运营和自适应安全，成为行业标杆。

- 全面自动化：安全策略自动适配业务变化
- 自适应安全：动态风险感知和自动响应
- 文化内化：全员安全意识和安全行为已融入日常工作
- 行业输出：参与行业标准制定，安全实践对外输出
- 典型表现："安全是每个人的责任"成为组织信条
