# DSMM评估框架

> 数据安全能力成熟度模型（GB/T 37973-2019）的框架结构、能力域与评估流程

## DSMM概述

DSMM（Data Security Capability Maturity Model）是中国国家标准GB/T 37973-2019《信息安全技术 数据安全能力成熟度模型》，为组织提供数据安全能力成熟度评估和改进的系统化框架。DSMM覆盖数据全生命周期，力求将数据安全从合规驱动转向能力驱动。

## 四维能力模型

DSMM从四个维度评估数据安全能力：

| 维度 | 权重 | 说明 |
|------|------|------|
| **组织建设** | 20% | 安全组织架构、职责分工、人员配置 |
| **制度流程** | 25% | 安全制度、标准、操作规程、基线 |
| **技术工具** | 35% | 安全技术平台、工具链、自动化能力 |
| **人员能力** | 20% | 人员技能、意识培训、专业认证 |

## 30个能力域

DSMM覆盖6个生命周期阶段共30个能力域：

| 生命周期阶段 | 能力域 |
|-------------|--------|
| 数据采集 | 数据分类分级、数据采集安全、数据源鉴别、数据质量管理、数据脱敏 |
| 数据传输 | 传输加密、传输信道安全、传输节点安全、传输协议安全 |
| 数据存储 | 存储介质安全、存储架构安全、存储访问控制、数据备份与恢复 |
| 数据处理 | 数据查询安全、数据计算安全、数据分析安全、数据导出安全 |
| 数据交换 | 数据共享安全、数据发布安全、数据接口安全、数据交换监控 |
| 数据销毁 | 数据清除、介质销毁、销毁验证、销毁审计 |

## 评分方法

每个能力域按1-5分评分：

| 分数 | 成熟度 | 特征 |
|------|--------|------|
| 1 | 非正式执行 | 随机执行，依赖个人能力，无文档化 |
| 2 | 计划跟踪 | 有基本流程，被动执行，开始跟踪 |
| 3 | 充分定义 | 流程标准化并文档化，定期执行 |
| 4 | 量化控制 | 可量化度量，基于数据持续改进 |
| 5 | 持续优化 | 自动化运营，主动防御，文化内化 |

## 评估流程

1. **范围界定**：确定评估范围（全组织/部门/特定系统）、评估类型（全面/专项）
2. **工具自评**：组织内部对照DSMM能力域进行自评打分
3. **现场评估**：第三方评估机构访谈、文档审查、技术验证
4. **差距分析**：当前成熟度与目标成熟度对比，识别改进空间
5. **等级评定**：综合各能力域得分，确定整体成熟度等级（1-5级）
6. **改进规划**：基于差距分析制定改进路线图
