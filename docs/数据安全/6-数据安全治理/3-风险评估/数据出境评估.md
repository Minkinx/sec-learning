# 数据出境评估

> 数据出境安全评估的自评模板结构、方法体系与申报准备

## 评估模板结构

根据《数据出境安全评估办法》，自评估报告应包含以下核心章节：

### 1. 数据出境概述
- 数据处理者基本信息（名称、注册地、统一社会信用代码）
- 数据出境的法律依据与场景描述
- 出境链路拓扑图（数据源 → 传输通道 → 境外接收方）

### 2. 数据出境情况
- 数据总量、字段清单、涉及数据主体数量
- 数据分级（重点标注是否涉及重要数据或敏感个人信息）
- 出境频率、传输方式（API/批量/物理介质）
- 数据存储位置（境内 vs 境外，境外存储的具体位置）

### 3. 境外接收方安全评估
- 接收方名称、注册地、数据保护能力证明
- 接收方数据安全管理制度和技术能力评估
- 接收方所在国家/地区的数据保护法律环境评估
- 接收方过往数据安全事件记录（如有）

### 4. 出境目的与必要性
- 数据出境的具体业务目的
- 必要性论证（是否可以不出境或部分不出境？）
- 数据最小化声明（出境的字段是否必要？）

### 5. 风险分析与管控措施
- 出境链路安全风险（传输加密强度、中间人攻击风险）
- 境外存储安全风险（物理安全、访问控制、加密）
- 数据滥用风险（超范围使用、二次传输）
- 法律环境风险（该国政府数据调取权限、法律变更风险）

### 6. 结论与整改计划
- 综合风险等级判定（低/中/高）
- 整改措施清单及时间表
- 持续监控计划

## 评估方法论

- **差距分析法**：对照PIPL + 数据出境安全评估办法 + SCC要求逐项评估
- **成熟度评分**：对接收方的安全能力进行成熟度评分（1-5分），低于3分视为不通过
- **法律环境评分**：基于GDPR充分性认定、CBPR认证、IDF等国际数据保护指标综合评分
- **残余风险等级**：综合技术风险 + 法律风险 + 管理风险三项评分，加权计算

## 监管申报材料包

向网信办申报安全评估时需准备：

1. 数据出境安全评估申报书
2. 数据出境风险自评估报告
3. 数据处理者与境外接收方订立的法律文件
4. 数据安全管理制度与技术措施证明文件
5. 安全评估工作要求的其他材料

申报后监管机构须在7个工作日内决定是否受理，受理后45个工作日内出具评估结论，情况复杂的可适当延长。
