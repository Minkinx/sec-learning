# 风险量化与处置

> 风险处置策略选择、定量分析方法与残余风险接受机制

## 风险处置选项

根据ISO 27001和NIST风险管理框架，风险处置有四种基本策略：

| 策略 | 含义 | 适用场景 |
|------|------|---------|
| **规避（Avoid）** | 取消或避免导致风险的活动 | 风险过高且无法有效降低，如放弃不必要的高危数据处理 |
| **降低（Reduce）** | 通过控制措施减少可能性或影响 | 最常见的策略，实施技术和管理控件的场景 |
| **转移（Transfer）** | 将风险转移给第三方 | 购买网络保险、将高风险处理外包给专业服务商 |
| **接受（Accept）** | 明确接受残余风险 | 风险在容忍度范围内，或降低成本超过预期收益 |

## 定量风险分析（QRA）

### 基础指标

- **SLE（Single Loss Expectancy）**：单次损失期望 = 资产价值（AV）× 损失系数（EF）
- **ARO（Annualized Rate of Occurrence）**：年化发生频率，基于历史数据和行业基准
- **ALE（Annualized Loss Expectancy）** = SLE × ARO

示例：客户数据库资产价值500万，泄露一次损失系数0.3，年化发生概率0.2
- SLE = 500万 × 0.3 = 150万
- ALE = 150万 × 0.2 = 30万/年

### FAIR模型进阶

FAIR模型将风险拆解为六个因素层级：损失事件频率（LEF）× 损失幅度（LM）；其中LEF = 威胁事件频率（TEF）× 脆弱性（Vuln），LM = 一级损失 × 二级损失（响应成本、法律成本、监管罚款、信誉损失等）。

## 风险容忍度阈值

组织应定义明确的风险容忍度（Risk Appetite & Tolerance Thresholds）：

| 风险等级 | 容忍度 | 处置要求 |
|---------|--------|---------|
| 极高 | 零容忍 | 必须立即处置，无例外 |
| 高 | 有限容忍 | 须制定整改计划，在约定时限内降至可接受水平 |
| 中 | 有条件接受 | 可接受但须持续监控，定期评估是否需要处置 |
| 低 | 接受 | 可自行接受，纳入常规监控 |

## 风险处置计划格式

| 字段 | 说明 |
|------|------|
| 风险编号 | RSK-2025-XXX，唯一标识 |
| 风险描述 | 风险场景的简明描述 |
| 当前风险等级 | 基于固有风险的等级（未处置前） |
| 处置策略 | 规避/降低/转移/接受 |
| 具体措施 | 技术/管理/法律层面的具体控制措施 |
| 目标风险等级 | 措施实施后的预期风险等级 |
| 责任部门/人 | 执行部门及责任人 |
| 实施时间表 | 措施的开始和完成日期 |
| 预算估计 | 实施所需的人力、技术和资金成本 |
| 验收标准 | 措施有效性的判定标准 |

## 残余风险接受

- **普通残余风险**：管理层签字接受
- **高残余风险**：须经安全委员会集体决议并记录
- **残余风险复审**：每半年复审一次已接受的风险，确认条件未发生重大变化
- **风险接受备忘录**：正式文档记录接受理由、有效期和复审日期
