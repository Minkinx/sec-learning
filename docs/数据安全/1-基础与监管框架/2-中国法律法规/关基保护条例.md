# 关键信息基础设施安全保护条例

> 《关键信息基础设施安全保护条例》（国务院令第745号）于2021年7月30日公布，2021年9月1日与《数据安全法》同步施行。该条例是《网络安全法》在关键信息基础设施领域的落地细则，与等级保护制度共同构成我国网络安全的"双基"制度体系。

## 立法背景与定位

| 制度 | 法律依据 | 核心内容 | 与CII关系 |
|------|---------|---------|----------|
| 等级保护（等保2.0） | 网络安全法第21条 | 所有网络系统的安全基线 | CII必须满足等保三级+ |
| CII保护 | 网络安全法第31条+本条例 | 关键行业的特殊保护要求 | 在等保基础上额外加强 |
| 数据安全 | 数据安全法 | 数据处理活动的安全保护 | CIIO的数据处理需同时满足 |

**三者关系**：等保是"基线"，CII保护是在基线之上的"加强层"，数据安全则贯穿其中。

## CII认定标准与流程

### 认定范围（八大行业+其他）

| 行业领域 | 典型CII范围 |
|---------|------------|
| 公共通信和信息服务 | 电信网、互联网骨干网、数据中心、CDN |
| 能源 | 电力调度系统、油气管道监控、核电控制系统 |
| 交通 | 航空订票、铁路调度、港口管理、高速公路监控 |
| 水利 | 水利工程监测、水资源调度 |
| 金融 | 支付清算、证券交易、银行核心系统 |
| 公共服务 | 社保、医保、水电燃气收费 |
| 电子政务 | 政务云平台、人口/法人/信用数据库 |
| 国防科技工业 | 军工科研生产管理系统 |

### 认定流程

```
行业主管部门制定认定规则
        ↓
    公开发布认定规则
        ↓
运营者根据规则自行评估
        ↓
运营者向主管部门申报
        ↓
主管部门审核确认 → 通知运营者
        ↓
    报公安部备案
        ↓
定期复审（至少每3年）→ 动态调整
```

### 认定参考标准

通常满足以下条件之一即可认定为CII：

1. **影响规模**：系统中断运行可能影响1000万以上人口
2. **业务关键性**：系统承载国家核心业务或跨省/全国性业务
3. **替代难度**：系统无可替代的备用方案
4. **关联影响**：系统故障可能引发其他关键行业的连锁故障
5. **数据敏感性**：系统处理大量重要数据或个人信息

## 安全保护义务详表

| 义务类别 | 具体要求 | 实施要点 |
|---------|---------|---------|
| **组织建设** | 设立专门安全管理机构 | 机构负责人为CISO或同级；机构应独立于IT运维部门 |
| **负责人任命** | 明确主要负责人和安全管理负责人 | 安全负责人需具备相关资质和背景审查 |
| **人员管理** | 关键岗位背景审查 | 审查范围：有无犯罪记录、境外复杂关系、信息安全违规史 |
| **教育培训** | 每年至少一次安全培训 | 培训覆盖全员，含钓鱼邮件演练、安全意识考核 |
| **制度流程** | 建立安全保护制度和操作规程 | 包含：访问控制、变更管理、密码管理、数据备份 |
| **应急预案** | 制定网络安全事件应急预案 | 每年至少演练一次，覆盖勒索软件、DDoS、数据泄露场景 |
| **技术防护** | 落实等级保护三级以上要求 | 包含：边界防护、入侵检测、主机安全、应用安全、数据安全 |
| **供应链安全** | 优先采购安全可信产品和服务 | 采购清单中应优先选入"网络关键设备和网络安全专用产品目录" |
| **安全检测** | 每年至少一次安全检测评估 | 须由具备资质的检测评估机构实施 |
| **数据安全** | 数据分类分级、重要数据保护 | 数据本地化存储，出境须安全评估 |

## CII安全检测评估

### 评估流程

```
运营者制定评估计划（年度）
        ↓
委托有资质测评机构
        ↓
文档审查 → 现场访谈 → 技术测试 → 渗透测试
        ↓
评估报告编制（含整改建议）
        ↓
评估报告报送主管部门和公安机关
        ↓
整改跟踪 → 闭环管理
```

### 评估资质要求
- 测评机构须具备网络安全等级保护测评资质
- 涉及国家秘密的CII，须具备涉密资质
- 评估团队须具备相关行业背景知识

## 安全审查要求

CII运营者采购网络产品和服务，涉及下列情形之一的，须通过国家安全审查：

| 情形 | 说明 |
|------|------|
| 产品和服务涉及国家秘密 | 须通过涉密审查 |
| 采购方为主要数据处理者 | 可能影响国家安全的 |
| 采购核心网络设备、高性能计算机等 | 涉及关键设备 |
| 采购方为主要CII运营者 | 直接影响CII安全稳定运行的 |

## 法律责任

| 违规行为 | 对企业的处罚 | 对个人的处罚 |
|---------|------------|------------|
| 未履行安全保护义务 | 责令改正+警告 | 5万-10万元罚款 |
| 拒不改正/造成危害 | 10万-100万元罚款 | 5万-10万元罚款 |
| 情节特别严重 | 1000万-5000万元或上年营收5% | 5万-20万元罚款 |
| 采购不合格产品造成危害 | 采购金额10%以下罚款 | 纪律处分 |
| 发生重大安全事件瞒报 | 50万-500万元罚款 | 10万-20万元罚款 |

## 合规时限要求

| 事项 | 时限要求 |
|------|---------|
| CII认定后安全建设整改 | 按主管部门要求时限完成（通常6-12个月） |
| 安全检测评估频率 | 每年至少一次 |
| 评估报告报送 | 评估完成后30日内 |
| 重大安全事件报告 | 事件发生后2小时内 |
| 安全背景审查 | 关键岗位上岗前完成 |
| 安全培训 | 每年至少一次 |

## 与等保2.0的关系详解

| 对比维度 | 等保2.0 | CII保护 |
|---------|--------|---------|
| 适用范围 | 所有网络系统 | 仅限认定的CII |
| 法律依据 | 网络安全法第21条 | 网络安全法第31条+条例 |
| 保护等级 | 1-5级 | 在等保三级基础上增强 |
| 测评频率 | 三级每年、四级半年 | 每年至少一次 |
| 审查范围 | 安全技术+管理 | 在等保基础上增加供应链、人员背景、应急处置 |
| 监管机构 | 公安机关 | 行业主管部门+公安机关 |
| 出境评估 | 不涉及 | CIIO数据出境须安全评估 |

**总结**：CII运营者应当在已满足等保三级要求的基础上，对照本条例的特殊要求进行额外加固。等保是"及格线"，CII保护是"优秀线"。
