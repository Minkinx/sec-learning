# 商用密码应用安全性评估（密评）

> 商用密码应用安全性评估（简称"密评"）是依据密码法对使用商用密码技术的系统进行全面评估的过程，是数据安全合规的关键组成部分。

## 密评的法律依据

密评的核心法律依据来源于三个层级：

| 层级 | 法规文件 | 关键条款 |
|------|---------|---------|
| 法律 | 《密码法》 | 第27条：关基每年至少一次密评 |
| 行政法规 | 《商用密码管理条例》 | 第38-42条：评估范围与程序 |
| 部门规章 | 《商用密码应用安全性评估管理办法》 | 评估机构管理、评估程序、结果处置 |

**强制性要求：** 以下系统必须开展密评：
- 关键信息基础设施（关基）
- 等保三级及以上信息系统
- 政务信息系统
- 国家政务信息化项目

## 密评标准体系

密评的技术依据是 GM/T 系列标准：

| 标准编号 | 标准名称 | 作用 |
|---------|---------|------|
| GM/T 0054-2018 | 信息系统密码应用基本要求 | 密评的核心评估依据 |
| GM/T 0115-2021 | 信息系统密码应用测评要求 | 测评具体操作指南 |
| GM/T 0116-2021 | 信息系统密码应用测评过程指南 | 测评流程规范 |
| GB/T 39786-2021 | 信息安全技术 信息系统密码应用基本要求 | 国家标准版（升级） |

### GM/T 0054 评估维度

GM/T 0054 从技术和管理的角度定义了四个评估方面：

**物理和环境安全：**
- 身份鉴别（门禁系统密码技术）
- 电子门禁记录数据完整性
- 视频监控数据完整性

**网络和通信安全：**
- 通信双方身份鉴别
- 通信数据完整性
- 通信数据机密性
- 网络边界访问控制

**设备和计算安全：**
- 设备身份鉴别
- 设备运行数据完整性
- 设备重要配置数据完整性
- 设备重要参数机密性

**应用和数据安全：**
- 用户身份鉴别
- 用户访问控制
- 用户数据传输机密性/完整性
- 重要数据存储机密性/完整性
- 不可否认性

### 管理要求

**安全管理制度：**
- 密码工作管理制度
- 密码人员管理制度
- 密码产品采购和使用制度
- 密码应急预案

## 密评流程

密评分为五个阶段：

```
定级 → 备案 → 建设整改 → 安全性评估 → 持续改进
```

### 第一阶段：系统定级

确定信息系统的安全保护等级（等保二级至四级），明确需使用的密码技术类型和强度要求。不同等级对应不同的密码应用要求：

| 等保等级 | 密码要求 | 评估频次 |
|---------|---------|---------|
| 二级 | 推荐性使用密码技术 | 建议评估 |
| 三级 | 必须使用合规密码技术 | 每年一次 |
| 四级 | 使用高强度密码技术 | 每半年一次 |

### 第二阶段：方案备案

编制密码应用方案，内容包括：
- 系统基本情况（网络拓扑、业务功能、数据流）
- 密码应用需求分析
- 密码技术选型（国密算法选择）
- 密码产品选型（通过认证的产品）
- 密钥管理体系设计
- 应急预案

### 第三阶段：建设整改

按照密码应用方案实施建设或改造：
- 部署合规密码产品（密码机、密码卡、VPN设备等）
- 实施密码改造（应用层密码对接、数据库加密、传输加密）
- 建立密钥管理体系
- 完善密码管理制度

### 第四阶段：安全性评估

委托具备资质的商用密码检测机构开展评估：

**评估方式：**
- **文档审查**：密码应用方案、管理制度、运维记录
- **配置核查**：密码产品配置、设备参数
- **工具检测**：密码协议检测、密钥强度检测
- **现场访谈**：密码管理人员访谈、运维人员访谈

**评分方法：**

| 评分项 | 权重 | 评分标准 |
|--------|------|---------|
| 物理和环境安全 | 15% | 各测评点符合/不符合/不适用 |
| 网络和通信安全 | 25% | 各测评点符合/不符合/不适用 |
| 设备和计算安全 | 20% | 各测评点符合/不符合/不适用 |
| 应用和数据安全 | 30% | 各测评点符合/不符合/不适用 |
| 安全管理 | 10% | 管理制度、人员、应急预案 |

**结果判定：**

| 等级 | 要求 | 结论 |
|------|------|------|
| 符合 | 所有指标符合 | 通过 |
| 基本符合 | 部分指标不符合但不影响整体安全 | 需限期整改 |
| 不符合 | 关键指标不符合 | 不通过，须全面整改后重新评估 |

### 第五阶段：持续改进

密评通过后仍需持续关注：
- 年度复评（关基系统每年一次）
- 重大变更后重新评估（系统升级、网络改造等）
- 密码产品更新升级
- 密码管理制度持续优化

## 密评与等保的关系

密评和等保测评既有区别又有联系：

| 对比维度 | 等保测评 | 密评 |
|---------|---------|------|
| 法律依据 | 《网络安全法》 | 《密码法》 |
| 测评范围 | 整体安全 | 密码应用安全 |
| 测评周期 | 二级每两年、三级每年 | 三级以上每年 |
| 测评机构 | 等保测评机构 | 商用密码检测机构 |
| 测评结果 | 等级保护测评报告 | 密评报告 |

**衔接关系：** 等保三级及以上系统需同步开展密评，密评结果作为等保测评中"密码技术"部分的输入。两个测评可协同开展以降低合规成本。

## 密评常见问题

| 问题类型 | 典型表现 | 整改建议 |
|---------|---------|---------|
| 算法不合规 | 使用国际算法（RSA、AES）替代国密 | 替换为SM2/SM3/SM4 |
| 密钥管理缺失 | 密钥明文存储、无密钥轮换机制 | 建立KMS，实现密钥全生命周期管理 |
| 证书不合法 | 使用自签名证书、无SM2证书 | 申请合规CA签发的SM2证书 |
| 通信未加密 | 内部通信未启用TLS | 配置国密SSL VPN，启用SM4加密 |
| 日志不完整 | 无密码操作审计日志 | 配置密码设备审计日志功能 |
