# ISO/IEC 27701

> 隐私信息管理体系（Privacy Information Management System, PIMS）标准，是ISO 27001在隐私信息管理方面的扩展。ISO/IEC 27701:2019 为PII控制者和PII处理者提供隐私保护管理框架，是当前国际上最受认可的隐私管理标准之一。

## 概述

ISO/IEC 27701:2019 是对ISO 27001和ISO 27002在隐私信息管理上的补充，于2019年8月发布。其主要目标包括：

1. 为PII控制者和PII处理者建立隐私保护管理框架
2. 提供与GDPR等隐私法规的映射指南
3. 帮助组织在统一的ISMS基础上扩展隐私保护能力
4. 支持第三方认证的隐私管理体系证明

## PIMS与ISMS的关系

| 维度 | ISMS（ISO 27001） | PIMS（ISO 27701） |
|------|------------------|------------------|
| **定位** | 信息安全管理体系基础 | ISMS在隐私管理方面的扩展 |
| **独立性** | 可独立认证 | **不可独立认证**，须基于27001 |
| **认证条件** | 仅需满足27001要求 | 须同时满足27001+27701要求 |
| **控制域** | 14个控制域（114项控制措施） | 在27001基础上增加隐私相关控制 |
| **适用角色** | 所有组织 | PII控制者和/或PII处理者 |
| **有效期** | 3年+年度监督+换证审核 | 与27001同步 |

**实施路径**：组织须先建立符合27001的ISMS，再扩展PIMS。认证审核时须同时涵盖27001和27701的要求。不可单独申请27701认证而没有27001认证。

## 新增控制措施

### PII控制者的附加控制

| 控制要点 | 说明 | 对应GDPR条款 |
|---------|------|-------------|
| **同意管理** | 获取、记录和管理PII主体的同意 | 第7条（同意条件） |
| **目的约束** | 仅在与PII主体沟通的目的范围内处理 | 第5条(1)(b) |
| **数据最小化** | 仅收集业务必需的PII | 第5条(1)(c) |
| **数据质量** | 确保PII的准确性和及时更新 | 第5条(1)(d) |
| **数据主体权利响应** | 建立流程响应知情权、删除权、更正权等 | 第12-22条 |
| **自动化决策披露** | 告知主体自动化决策的逻辑和后果 | 第22条 |
| **DPIA** | 高风险处理的隐私影响评估 | 第35条 |
| **隐私设计（PbD）** | 在产品/流程设计阶段嵌入隐私保护 | 第25条 |
| **跨境传输** | 确保PII跨境传输的合法性 | 第44-49条 |

### PII处理者的附加控制

| 控制要点 | 说明 | 对应GDPR条款 |
|---------|------|-------------|
| **仅按指示处理** | 仅在控制者授权下处理PII | 第28条(3)(a) |
| **分包管理** | 子处理者的选择和合同约束 | 第28条(2)(4) |
| **跨境传输限制** | 处理者不得随意将PII转移到境外 | 第44-49条 |
| **协助义务** | 协助控制者响应用户权利请求 | 第28条(3)(e) |
| **安全事件通知** | 向控制者及时报告个人数据泄露 | 第33条 |
| **合同保障** | 与控制者的数据处理协议 | 第28条(3) |

### 共同控制者的要求

- 各控制者的责任分配须以书面形式明确
- 应在协议中明确谁是PII主体行使权利时的主要联系人

## 与GDPR的映射

ISO 27701的重要价值在于其附录提供了与GDPR的**双向映射关系**：

| 附录 | 内容 | 用途 |
|------|------|------|
| **附录D** | 27701控制措施 → GDPR条款 | 证明现有控制覆盖哪些GDPR要求 |
| **附录E** | GDPR条款 → 27701控制措施 | 针对GDPR要求找到对应的控制措施 |

这个双向映射实现以下目标：

1. **统一合规**：通过一张SoA（适用性声明）同时覆盖27001和27701的合规要求
2. **效率提升**：避免为GDPR合规建立两套平行的体系
3. **审计整合**：内审和外审时合并审核，降低合规成本
4. **持续合规**：体系化的PDCA确保GDPR合规的可持续性

## 与其他隐私框架的对比

| 维度 | ISO 27701 | APEC CBPR | GDPR |
|------|-----------|-----------|------|
| **性质** | 国际管理标准 | 跨境隐私规则体系 | 法律 |
| **适用范围** | 全球（可认证） | APEC成员国 | 欧盟（全球影响） |
| **认证方式** | 第三方认证（须27001为基础） | 自我评估+认证机构 | 无认证，需证明合规 |
| **管理方法** | 基于ISMS的PDCA | 问责制原则 | 问责制+合规要求 |
| **控制粒度** | 详细（隐私控制措施） | 原则性（9项原则） | 详细（99条条款） |
| **处罚机制** | 无直接处罚（认证撤销） | 无直接处罚 | 全球营收4%或€20M |
| **与27701协同** | — | 附录F提供27701与CBPR映射 | 附录D/E提供映射 |

## 认证过程

### 认证流程

```
阶段一：准备（3-6个月）
├── 建设ISMS并通过27001认证（或已有）
├── 识别27701的适用控制
├── 编写PIMS手册和程序文件
├── 实施隐私影响评估流程
└── 建立数据主体权利响应机制

阶段二：预审（可选）
├── 认证机构进行文档审查
├── 预审识别明显不符合项
└── 提前整改

阶段三：正式认证审核
├── 第一阶段审核：文档审查+认证范围确认
├── 整改第一阶段发现的问题
├── 第二阶段审核：现场审核+控制实施验证
└── 颁发认证证书

阶段四：持续监督（3年有效期）
├── 年度监督审核（每年一次）
├── 重大变更后接受专项审核
└── 3年到期后换证审核
```

### 认证范围灵活设置

| 范围类型 | 说明 | 适用场景 |
|---------|------|---------|
| PII控制者认证 | 仅覆盖控制者角色 | 大多数数据控制型企业 |
| PII处理者认证 | 仅覆盖处理者角色 | 云服务商、数据处理外包商 |
| 同时认证 | 覆盖控制者和处理者 | 同时承担两种角色的组织 |
| 范围限定 | 特定业务线、部门或地域 | 先试点再推广的组织 |

## 实施建议

1. **先建立成熟的ISMS体系，再扩展PIMS**——27701不可能在27001缺失的情况下独立有效
2. **利用DPIA流程与PIMS的风险评估流程整合**，避免重复劳动
3. **将数据主体权利响应流程纳入体系文件**，建立标准化的请求处理SOP
4. **将PIMS融入现有的内部审核和管理评审**，避免"两张皮"现象
5. **充分利用条款映射附录**，在SoA中同时覆盖27001和27701的要求
6. **关注法规更新**，如中国个保法、欧盟GDPR执法动态，及时更新控制措施
7. **培训是关键**：隐私管理需要法务、安全、业务多部门协作，培训覆盖所有相关人员
