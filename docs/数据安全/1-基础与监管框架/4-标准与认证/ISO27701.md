# ISO/IEC 27701

> 隐私信息管理体系（PIMS）标准，是ISO 27001在隐私信息管理方面的扩展。

## 概述

ISO/IEC 27701:2019 是对ISO 27001和ISO 27002在隐私信息管理上的补充，为PII控制者和PII处理者提供隐私保护管理框架。

## PIMS与ISMS的关系

- **扩展而非替代**：27701的认证必须建立在27001的基础上
- 组织须先建立符合27001的ISMS，再扩展PIMS
- 认证审核时须同时满足27001和27701的要求

## 新增控制措施

| 角色 | 控制要点 |
|------|---------|
| PII控制者 | 同意管理、目的约束、最小化、数据质量、数据主体权利响应 |
| PII处理者 | 仅在控制者指示下处理、分包管理、跨境传输限制 |
| 共同控制者 | 责任分配须书面明确 |

## 与GDPR的映射

ISO 27701的重要价值在于其附录提供了与GDPR的**映射关系**：

- 附录D：27701控制措施与GDPR条款的对应
- 附录E：GDPR要求与27701控制措施的对应
- 映射覆盖GDPR的大部分实质性条款

这个映射使得认证组织可以通过一张SoA（适用性声明）同时覆盖27001和27701的合规要求。

## 认证范围

- 认证范围可以限定在特定业务线、部门或地域
- 组织可选择仅申请PII控制者认证、PII处理者认证，或两者都申请
- 证书有效期与27001一致（3年+监督审核）

## 实施建议

1. 先建设成熟的ISMS体系，再扩展PIMS
2. 利用DPIA流程与PIMS的风险评估流程整合
3. 将数据主体权利响应流程纳入体系文件
4. 将PIMS融入现有的内部审核和管理评审
