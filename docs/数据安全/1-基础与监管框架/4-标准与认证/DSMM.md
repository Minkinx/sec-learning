# DSMM

> Data Security Capability Maturity Model（数据安全能力成熟度模型），国家标准GB/T 37988-2019。

## 模型概述

DSMM从四个能力维度评估组织的数据安全能力成熟度，涵盖30个能力域。

## 四个能力维度

| 维度 | 说明 | 评估要点 |
|------|------|---------|
| 组织建设 | 组织架构、角色与职责 | 安全委员会、数据Owner、决策机制 |
| 制度流程 | 安全制度、标准操作规程 | 策略完备性、可执行性、版本管理 |
| 技术工具 | 安全技术支撑 | DLP、加密、脱敏、审计平台 |
| 人员能力 | 人员安全意识与技能 | 培训计划、考核机制、资质认证 |

## 五个成熟度等级

| 等级 | 名称 | 特征 |
|------|------|------|
| L1 | 非正式执行 | 无统一流程，依赖个人经验 |
| L2 | 计划跟踪 | 建立基本流程，可重复执行 |
| L3 | 充分定义 | 流程标准化、文档化、组织级推广 |
| L4 | 量化控制 | 可量化度量，数据驱动改进 |
| L5 | 持续优化 | 自动化运营，持续改进文化 |

## 30个能力域

覆盖数据安全生命周期的各阶段（采集、传输、存储、处理、交换、销毁），每个阶段包含多个能力域。例如数据采集阶段包含数据分类分级、数据采集安全、数据源鉴别等。

## 评估方法

- **文档评审**：检查制度、流程、记录
- **人员访谈**：评估人员能力与意识
- **技术验证**：测试技术工具有效性
- **现场观察**：验证实际操作与制度一致性

## DSMM的价值

- 帮助企业系统化评估数据安全现状
- 明确改进方向和优先级
- 作为第三方评估和行业对标的标准
- 与等保2.0形成互补（等保侧重系统，DSMM侧重数据能力）
