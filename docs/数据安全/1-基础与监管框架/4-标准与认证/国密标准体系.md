# 国密标准体系（GM/T）

> 国密（商用密码）标准体系是中国自主的密码技术标准，覆盖算法、协议、产品、检测、应用等全链条。GM/T系列标准已形成与国际标准对等的完整体系。

## 国密算法标准

国密算法的核心是SM系列算法：

### SM2 椭圆曲线公钥密码算法

SM2是基于椭圆曲线密码学（ECC）的公钥算法体系，于2010年发布，替代RSA算法。

| 特性 | 描述 |
|------|------|
| 标准编号 | GB/T 32918（数字签名算法）、GM/T 0003（整体规范） |
| 算法类型 | 椭圆曲线公钥密码 |
| 密钥长度 | 256位 |
| 安全强度 | 相当于RSA-3072 |
| 功能 | 数字签名、密钥交换、公钥加密 |
| 应用 | 数字证书、电子签章、身份认证 |

**与国际算法对比：** SM2-256与RSA-2048的安全性对比如下：

| 指标 | SM2-256 | RSA-2048 |
|------|---------|----------|
| 密钥长度 | 256位 | 2048位 |
| 签名速度 | 快 | 慢 |
| 证书体积 | 小 | 大 |
| 国际标准对标 | ECDSA/ECIES | RSA |

### SM3 密码杂凑算法

SM3是中国自主的密码哈希算法，于2010年发布。

| 特性 | 描述 |
|------|------|
| 标准编号 | GM/T 0004-2012 |
| 算法类型 | 密码杂凑（哈希）算法 |
| 输出长度 | 256位（32字节） |
| 安全强度 | 128位 |
| 应用 | 完整性校验、数字签名、消息认证 |

**与国际算法对比：** SM3与SHA-256的输出长度相同（256位），安全性相当。SM3在设计上采用了与SHA-256不同的压缩函数结构，具有更好的雪崩效应。

### SM4 分组密码算法

SM4是中国自主的对称分组密码算法，原名SMS4。

| 特性 | 描述 |
|------|------|
| 标准编号 | GB/T 32907（国标）、GM/T 0002（行业标准） |
| 算法类型 | 分组密码 |
| 分组长度 | 128位 |
| 密钥长度 | 128位 |
| 轮数 | 32轮 |
| 工作模式 | ECB、CBC、CFB、OFB、CTR、GCM |
| 应用 | 数据加密、通信加密、存储加密 |

**与国际算法对比：** SM4与AES-128的对比如下：

| 指标 | SM4 | AES-128 |
|------|-----|---------|
| 分组长度 | 128位 | 128位 |
| 密钥长度 | 128位 | 128位 |
| 轮数 | 32轮 | 10轮 |
| 结构 | 非平衡Feistel | SPN |
| 软件实现效率 | 较高 | 高（硬件加速） |

### SM9 标识密码算法

SM9是基于标识的密码算法（Identity-Based Cryptography），于2016年发布。

| 特性 | 描述 |
|------|------|
| 标准编号 | GM/T 0044-2016 |
| 算法类型 | 标识密码（IBC） |
| 密钥长度 | 256位 |
| 功能 | 数字签名、密钥交换、密钥封装 |
| 核心优势 | 无需公钥证书，直接使用用户标识作为公钥 |
| 应用 | 物联设备认证、邮件加密、移动端签名 |

### SM7 对称密码算法

SM7面向非接触式IC卡、RFID等场景，分组长度为128位，适用于资源受限环境。

## GM/T 标准体系

GM/T（密码行业标准）系列已发布200多项标准，涵盖以下类别：

| 标准类别 | 标准数量（截至2025） | 典型标准 |
|---------|-------------------|---------|
| 算法标准 | 10+ | GM/T 0002 SM4、GM/T 0003 SM2、GM/T 0004 SM3 |
| 协议标准 | 30+ | GM/T 0024 SSL VPN、GM/T 0060 TLS协议 |
| 产品标准 | 80+ | GM/T 0017 智能密码钥匙、GM/T 0030 密码机 |
| 检测标准 | 20+ | GM/T 0028 密码模块安全要求、GM/T 0054 测评要求 |
| 应用标准 | 40+ | GM/T 0031 电子签章、GM/T 0078 区块链密码 |
| 管理标准 | 20+ | GM/T 0089 密码服务管理规范 |

## 密码模块安全认证

GM/T 0028（密码模块安全要求）对标FIPS 140-2标准，是密码产品合规认证的核心依据：

| 安全等级 | 要求 | 适用产品 |
|---------|------|---------|
| 一级 | 基础安全要求 | 软件密码模块 |
| 二级 | 物理安全+显式身份 | 商用密码产品 |
| 三级 | 防篡改+身份认证增强 | 高安全级密码产品 |
| 四级 | 全环境安全保护 | 核心/普通密码产品 |

## 国密与国际标准对标

| 国密标准 | 功能 | 对标国际标准 |
|---------|------|-------------|
| SM2签名 | 数字签名 | ECDSA（ISO 14888-3） |
| SM2密钥交换 | 密钥协商 | ECDH（ISO 11770） |
| SM2加密 | 公钥加密 | ECIES |
| SM3 | 哈希 | SHA-256 |
| SM4 | 对称加密 | AES（ISO 18033-3） |
| SM9 | 标识密码 | BPKI/IBC（ISO 11770） |
| GM/T 0028 | 密码模块安全 | FIPS 140-2/3 |

## 商用密码产品认证目录

通过认证的密码产品类型：

- 密码芯片
- 密码模块（PCI-E密码卡）
- 密码机（服务器密码机、金融数据密码机）
- VPN密码网关
- SSL VPN设备
- 数字证书认证系统（CA）
- 电子签章系统
- 动态令牌认证系统
- 密钥管理系统（KMS）
- 数据库加密系统
- 文档加密系统

## 国密改造实施要点

系统国密改造涉及以下层面：

**网络层：**
- TLS协议替换为国密SSL（GM/T 0024）
- 使用SM2证书替代RSA证书
- 密码套件切换为SM2-SM4-SM3

**应用层：**
- 签名验签换用SM2
- 哈希校验换用SM3
- 数据加密换用SM4
- CA系统支持SM2证书签发

**数据库层：**
- 数据库存储加密使用SM4
- 透明加密需支持SM4算法
- 备份加密需支持国密

**管理平台：**
- 密钥管理系统支持SM系列
- 统一密码服务平台（密码资源池）
