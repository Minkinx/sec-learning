# 等保2.0

> 网络安全等级保护制度2.0（等保2.0），于2019年12月1日正式实施，是《网络安全法》框架下的核心制度。等保2.0是对等保1.0（GB/T 22239-2008）的重大升级，将保护范围从传统信息系统扩展到新型IT基础设施。

## 五个保护等级

| 等级 | 名称 | 适用对象 | 备案要求 | 测评频率 |
|------|------|---------|---------|---------|
| **第一级** | 自主保护 | 一般系统（内部办公、普通网站） | 自行保护，不强制备案 | 自愿 |
| **第二级** | 指导保护 | 非重要系统（影响较小） | 公安机关备案 | 建议每2年 |
| **第三级** | 监督保护 | 重要系统（地市级关键系统） | 公安备案 | **每年一次** |
| **第四级** | 强制保护 | 极重要系统（省级/国家级关键系统） | 公安备案 | **每半年一次** |
| **第五级** | 专控保护 | 国家核心系统（国防、尖端科技） | 国家专门控制 | 按国家要求 |

## 等保2.0 vs 等保1.0 关键变化

| 对比维度 | 等保1.0 | 等保2.0 |
|---------|---------|---------|
| **保护对象** | 信息系统 | 信息系统 + 云计算、移动互联网、物联网、工业控制、大数据 |
| **安全要求** | 基本要求 | 通用要求 + **扩展要求**（各新型态） |
| **数据安全** | 简单提及 | **独立安全区域**（数据分类分级、备份恢复、防泄漏） |
| **测评标准** | GB/T 22239-2008 | GB/T 22239-2019 |
| **安全控制** | 10类基本要求 | 10类通用要求 + 各扩展场景要求 |
| **定级要素** | 受侵害客体+侵害程度 | 新增"社会秩序和公共利益"考量 |
| **密码要求** | 部分提及 | 单独章节（配合密码法） |

## 10大类安全要求（通用要求）

| 编号 | 要求类别 | 说明 | 三级典型控制 |
|------|---------|------|------------|
| **1** | 安全物理环境 | 机房的物理安全 | 门禁系统、视频监控、温湿度控制、UPS |
| **2** | 安全通信网络 | 网络架构和通信安全 | 网络分区、访问控制列表、通信加密 |
| **3** | 安全区域边界 | 边界防护和入侵检测 | 防火墙、IDS/IPS、恶意代码检测 |
| **4** | 安全计算环境 | 主机、应用和数据安全 | 主机加固、应用安全、漏洞管理 |
| **5** | 安全管理中心 | 统一安全管理和监控 | SIEM、集中日志审计、策略管理 |
| **6** | 安全管理制度 | 安全策略和制度建设 | 安全策略文件、管理制度、操作规范 |
| **7** | 安全管理机构 | 组织和人员管理 | 安全部门设置、岗位职责、人员配备 |
| **8** | 安全管理人员 | 人员录用和培训 | 背景审查、安全意识培训、外部人员管理 |
| **9** | 安全建设管理 | 系统建设和运维 | 安全方案评审、验收测试、变更管理 |
| **10** | 安全运维管理 | 日常运维和应急 | 巡检、备份、应急演练、漏洞修复 |

## 扩展要求（五大场景）

### 云计算扩展要求

| 扩展项 | 三级要求要点 |
|-------|-------------|
| 虚拟机隔离 | 不同租户虚拟机之间实现网络隔离 |
| 镜像安全 | 镜像必须经过安全检查和签名 |
| 数据隔离 | 云存储资源池的多租户隔离 |
| 虚拟化安全 | Hypervisor安全加固、防逃逸 |
| 云平台审计 | 统一日志管理能力 |

### 移动互联网扩展要求

| 扩展项 | 三级要求要点 |
|-------|-------------|
| 移动应用安全 | 应用来源验证、签名校验 |
| 无线接入安全 | 接入认证、加密传输 |
| 终端安全 | 移动设备管理（MDM） |
| API安全 | 接口认证、防重放 |

### 物联网扩展要求

| 扩展项 | 三级要求要点 |
|-------|-------------|
| 感知层安全 | 终端设备身份认证 |
| 网络传输安全 | 轻量级加密协议 |
| 数据安全 | 采集数据完整性校验 |

### 工业控制系统扩展要求

| 扩展项 | 三级要求要点 |
|-------|-------------|
| 控制网络隔离 | 工控网与办公网物理或逻辑隔离 |
| 协议深度检测 | 工控协议白名单 |
| 控制指令校验 | 防篡改、防重放 |

### 大数据扩展要求

| 扩展项 | 三级要求要点 |
|-------|-------------|
| 数据分类分级 | 建立数据分级管理制度 |
| 数据脱敏 | 非生产环境和共享场景脱敏 |
| 数据溯源 | 数据流转和血缘追踪 |
| 数据防泄漏 | 敏感数据DLP策略 |

## 数据安全扩展要求（三级及以上）

等保2.0在三级及以上等级中特别增加了数据安全扩展要求，反映了对数据安全的重视程度提升：

| 要求项 | 说明 | 实施要点 |
|-------|------|---------|
| **数据分类分级** | 建立数据分类分级管理制度 | 参照数安法要求进行数据级别划分 |
| **数据备份恢复** | 异地实时备份、冗余设计 | 3-2-1备份策略，定期恢复演练 |
| **数据防泄漏** | 对敏感数据实施DLP策略 | 终端DLP+网络DLP+邮件DLP |
| **数据脱敏** | 非生产环境使用脱敏数据 | 静态脱敏（数据导出）+动态脱敏（实时查询） |
| **数据完整性** | 校验机制保护重要数据 | 数字签名、哈希校验、审计日志 |
| **数据保密性** | 重要数据加密存储和传输 | TLS 1.2+、AES-256加密、密钥管理 |

## 分类定级流程

```
确定定级对象
    ↓ （明确系统边界、业务功能、服务范围）
初步定级（运营者自评）
    ↓ （安全保护等级定级报告）
专家评审
    ↓ （评审委员会出具评审意见）
主管部门核准
    ↓ （行业主管部门审核确认）
公安机关备案
    ↓ （提交定级报告和备案表）
定级结果
    ↓ （公安机关出具定级证明）
等保测评 → 安全整改 → 持续监督
```

### 定级要素矩阵

| 受侵害客体 | 一般损害 | 严重损害 | 特别严重损害 |
|-----------|---------|---------|------------|
| 公民、法人合法权益 | 第一级 | 第二级 | 第三级 |
| 社会秩序、公共利益 | 第二级 | 第三级 | 第四级 |
| 国家安全 | 第三级 | 第四级 | 第五级 |

## 等保测评流程

```
第一步：定级备案
├── 编制定级报告
├── 准备备案材料
└── 提交公安机关备案（二级及以上）

第二步：安全建设整改
├── 对照对应等级要求进行差距分析
├── 制定安全建设方案
├── 实施技术和管理整改措施
└── 内部验收

第三步：等级测评
├── 委托具备资质的测评机构
├── 测评机构实施现场测评
├── 出具等级测评报告
└── 报告报送公安机关

第四步：监督检查
├── 公安机关定期检查
├── 年度安全自查报告
├── 重大变更及时报告
└── 接受不定期抽查
```

## 与关基保护的关系

| 维度 | 等保2.0（三级） | CII保护 | 关系 |
|------|---------------|---------|------|
| **适用范围** | 所有三级系统 | 仅限认定的CII | CII必须满足等保三级+ |
| **法律依据** | 网络安全法第21条 | 网络安全法第31条+条例 | 等保是基础，CII是加强 |
| **测评频率** | 每年一次 | 每年一次（含等保和CII专项） | 可合并开展 |
| **保护要求** | 10类通用+扩展 | 在等保基础上增加：供应链、人员背景、应急处置 | 等保是基线 |
| **审查要求** | 无专门审查 | 产品和服务国家安全审查 | CII更严格 |

**实践建议**：
- 被认定为CII的组织，应先确保已满足等保三级要求，再对照CII保护条例进行额外加固
- 等保测评和CII检测评估可委托同一机构合并进行，节约成本
- 数据安全法、等保2.0、CII保护条例三类合规要求应统筹规划、统一实施

## 等保2.0合规路线图

```
短期（3-6个月）：定级备案
├── 完成系统梳理和定级
├── 完成专家评审和主管部门核准
└── 完成公安备案

中期（6-12个月）：建设整改
├── 对照等级标准进行差距分析
├── 实施技术整改（安全设备、加固、加密）
├── 完善管理制度和流程
└── 完成首次等级测评

长期（持续）：持续合规
├── 按测评频率要求定期测评
├── 持续监控和改进
├── 关注标准更新和新威胁应对
└── 安全能力常态化运营
```

## 法律责任

等保2.0与《网络安全法》法律责任联动：

| 违规行为 | 法律责任（网络安全法） |
|---------|---------------------|
| 未履行等保义务 | 责令改正+警告 |
| 拒不改正 | 罚款1万-10万元 |
| 造成危害后果 | 罚款1万-10万元+直接责任人罚款5000-1万元 |
| 严重违规 | 停业整顿+吊销许可 |
| CIIO违规 | 罚款10万-100万元+直接责任人罚款1万-10万元 |
