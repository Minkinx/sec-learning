# 等保2.0

> 网络安全等级保护制度2.0（等保2.0），于2019年12月1日正式实施，是《网络安全法》框架下的核心制度。

## 保护等级

| 等级 | 名称 | 适用对象 | 备案要求 |
|------|------|---------|---------|
| 第一级 | 自主保护 | 一般系统 | 自行保护，不强制备案 |
| 第二级 | 指导保护 | 非重要系统 | 公安机关备案 |
| 第三级 | 监督保护 | 重要系统 | 公安备案、每年测评 |
| 第四级 | 强制保护 | 极重要系统 | 公安备案、半年测评 |
| 第五级 | 专控保护 | 国家核心系统 | 国家专门控制 |

## 分类定级流程

```
确定定级对象 → 初步定级 → 专家评审 → 主管部门核准 → 公安备案
```

## 等保2.0 vs 等保1.0 关键变化

| 维度 | 等保1.0 | 等保2.0 |
|------|---------|---------|
| 保护对象 | 信息系统 | 信息系统 + 云计算、移动互联网、物联网、工业控制、大数据 |
| 安全要求 | 基本要求 | 通用要求 + 扩展要求（各新型态） |
| 数据安全 | 简单提及 | 独立区域（数据分类分级、数据备份恢复、数据防泄漏） |
| 测评标准 | GB/T 22239-2008 | GB/T 22239-2019 |
| 安全控制 | 10类 | 10类 + 扩展要求 |

## 数据安全扩展要求

等保2.0在第三级及以上增加了数据安全扩展要求：

- **数据分类分级**：建立数据分类分级管理制度
- **数据备份恢复**：异地实时备份、冗余设计
- **数据防泄漏**：对敏感数据实施DLP策略
- **数据脱敏**：非生产环境使用脱敏数据
- **数据完整性**：校验机制保护重要数据

## 测评流程

1. 定级备案（向公安机关提交定级报告）
2. 安全建设（按照对应等级要求整改）
3. 等级测评（由具备资质的测评机构进行）
4. 监督检查（公安机关定期检查）
