# PCI DSS

> Payment Card Industry Data Security Standard（支付卡行业数据安全标准），由PCI安全标准委员会制定，适用于所有处理持卡人数据的实体。

## 6大目标与12项要求

| 目标 | 要求 |
|------|------|
| **A. 构建和维护安全网络** | 1. 安装和维护防火墙配置；2. 不使用厂商默认密码 |
| **B. 保护持卡人数据** | 3. 保护存储的持卡人数据；4. 加密传输中的持卡人数据 |
| **C. 维护漏洞管理计划** | 5. 使用并定期更新防病毒软件；6. 开发并维护安全系统和应用 |
| **D. 实施强访问控制** | 7. 按业务需要限制数据访问；8. 分配唯一ID；9. 限制物理访问 |
| **E. 定期监控和测试** | 10. 跟踪和监控网络访问；11. 定期测试安全系统和流程 |
| **F. 维护信息安全策略** | 12. 维护针对所有人员的信息安全策略 |

## CDE（Cardholder Data Environment）

持卡人数据环境包括持有、处理、传输持卡人数据或敏感认证数据的系统组件、人员和流程。CDE边界的准确定义是PCI合规的第一步。

## 商户等级

| 等级 | 标准 | 验证要求 |
|------|------|---------|
| 1级 | >600万笔交易/年 | 年度ROC + QSA现场评估 + ASV季度扫描 |
| 2级 | 100万-600万笔/年 | 年度SAQ + ASV季度扫描 |
| 3级 | 2万-100万笔在线交易/年 | 年度SAQ + ASV季度扫描 |
| 4级 | <2万笔在线交易/年 | 年度SAQ + ASV季度扫描 |

## SAQ类型

SAQ A / A-EP / B / B-IP / C / C-VT / D适用于不同场景的交易处理方式。

## PCI DSS 4.0 新趋势

- 从"合规检查"转向**持续安全验证**
- 引入更多灵活的控制方法（自定义控制 vs. 规定控制）
- 强调角色与责任的明确划分
- 增加对多因素认证的要求范围
- 2024年3月31日：旧版本退役，4.0全面生效
