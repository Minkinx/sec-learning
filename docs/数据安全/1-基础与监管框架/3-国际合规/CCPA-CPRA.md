# CCPA / CPRA

> California Consumer Privacy Act（CCPA）于2020年1月1日生效，是美国首部综合性消费者隐私法。California Privacy Rights Act（CPRA）于2023年1月1日生效，是对CCPA的重大修正与扩展。理解CCPA/CPRA对于处理加州居民个人数据的企业具有重要的合规意义。

## CCPA适用范围

适用以下条件之一且在加州开展业务的营利性实体：

| 标准 | 说明 |
|------|------|
| 年总收入 | 超过2500万美元 |
| 个人信息规模 | 年度买卖或分享10万条及以上加州居民个人信息（CPRA从"5万"提升至"10万"） |
| 收入依赖度 | 年收入50%以上来自出售个人信息 |

**适用范围扩展（CPRA）**：
- 新增"共享（Sharing）"概念，覆盖跨上下文行为广告的信息共享
- 扩大对承包商（Contractor）的约束
- 企业有义务确保服务提供商和承包商同样遵守CPRA要求

## 消费者权利详述

| 权利 | CCPA版本 | CPRA扩展内容 | 响应时限 |
|------|---------|-------------|---------|
| **知情权** | 了解收集、使用、共享的个人信息类别和目的 | 新增"共享"概念下的知情权，细化数据类别分类 | 45天内响应（可延期） |
| **删除权** | 要求企业删除已收集的个人信息 | 整体保留，明确例外情形（完成交易、安全、科研等） | 45天内响应 |
| **选择退出权** | 拒绝"出售"个人信息给第三方 | 扩展为"出售或共享"的退出权；新增"限制敏感个人信息使用" | 立即生效 |
| **非歧视权** | 行使权利不影响服务质量和价格 | 保留，新增"忠诚度计划"的合规指导 | — |
| **更正权（新增）** | — | 允许消费者要求更正不准确的个人信息 | 45天内响应 |
| **自动化决策知情权（新增）** | — | 了解自动化决策的运作方式及拒绝自动化决策的权利 | — |
| **数据可携带权** | 获取个人信息的电子副本 | 明确要求便捷、可互操作的格式 | 45天内响应 |

## 企业义务详表

| 义务类别 | 具体要求 | CPRA修订要点 |
|---------|---------|-------------|
| **隐私通知** | 采集点即通知，说明信息类别和目的 | 新增"留存期限"披露要求 |
| **隐私政策** | 公开透明描述数据处理行为 | 至少每12个月更新一次 |
| **响应消费者请求** | 在规定时限内回复权利请求 | 建立简化请求机制 |
| **DPIA** | 数据处理影响评估 | **CPRA新增**：高风险处理的DPIA要求 |
| **最小化原则** | 收集目的变更须获得新同意 | **CPRA新增**：目的约束和数据最小化 |
| **记录保留** | 保留权利请求记录至少24个月 | 保留 |
| **安全措施** | 实施合理的安全保护措施 | 更强调技术和组织措施 |
| **供应商管理** | 与服务提供商/承包商签订协议 | **CPRA新增**：对承包商的同等约束 |

## CPRA重大变更一览

| 变更项 | CCPA | CPRA |
|-------|------|------|
| 个人数据定义 | 宽泛定义 | 增加"敏感个人信息（SPI）"分类 |
| 敏感个人信息（SPI） | 未区分 | 精确地理位置、种族、宗教、社保号、基因数据、生物识别、健康信息、性取向 |
| 出售 | 出售个人信息 | 出售+共享（cross-context behavioral advertising） |
| 执法机构 | 加州总检察长（AG） | **新增**加州隐私保护局（CPPA），专职执法 |
| 处罚上限 | 每次违规$2,500（非故意）/$7,500（故意） | 同上，但CPPA可独立执法 |
| 数据留存 | 未明确 | 要求披露信息保留期限或判断标准 |
| 自动化决策 | 未涉及 | 消费者有权知情和拒绝 |
| Dark patterns | 未涉及 | **禁止**通过诱导性设计阻碍用户行使权利 |

## 与GDPR和PIPL的对比

| 维度 | CCPA/CPRA | GDPR | PIPL（中国个保法） |
|------|-----------|------|-------------------|
| **同意模式** | Opt-out（选择退出） | Opt-in（选择加入，严格同意） | Opt-in（同意+单独同意） |
| **适用范围** | 加州消费者（属地） | 欧盟数据主体（属人+属地） | 境内+境外处理境内自然人信息 |
| **处罚上限** | 每次违规$7,500 | 全球年营收4%或€20M较高者 | 最高5000万元或上年营收5% |
| **被遗忘权** | 有限删除权（有例外） | 更广泛的删除权 | 删除权（符合条件时） |
| **数据可携带** | 电子格式 | 明确的结构化、通用、可互操作格式 | 条件性可携带 |
| **自动化决策** | CPRA新增知情权 | 第22条：拒绝自动化决策 | 有权拒绝自动化决策 |
| **DPIA要求** | 仅高风险处理（CPRA） | 高风险处理必须DPIA | 高风险活动须事前影响评估 |
| **执法机构** | AG + CPPA | 各成员国DPA | 国家网信办+相关部门 |
| **跨境传输** | 未专门规定 | 充分性认定/SCCs/BCR | 安全评估/标准合同/认证 |

## 企业合规建议

1. **确定适用范围**：评估企业是否满足CCPA/CPRA的准入标准
2. **建立DSAR机制**：搭建消费者权利请求提交通道和内部处理流程
3. **数据映射**：梳理个人信息的收集、使用、共享和出售情况
4. **更新隐私政策**：确保覆盖CPRA新增的披露要求（SPI、共享、留存期限）
5. **DPIA流程**：对高风险处理活动实施数据保护影响评估
6. **供应商管理**：与服务提供商和承包商签订符合CPRA要求的数据处理协议
7. **关注SPI**：识别和标记敏感个人信息，实施使用限制
8. **建立合规文化**：定期培训、内部审计、与法务团队密切协作
