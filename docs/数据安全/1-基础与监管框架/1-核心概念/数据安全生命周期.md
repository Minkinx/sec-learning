# 数据安全生命周期

> 数据安全生命周期管理（Data Security Lifecycle Management, DSLM）贯穿数据从创建到销毁的全过程。不同行业和监管框架对生命周期阶段的划分略有差异，但核心思想一致：数据在不同阶段面临不同的风险，需要差异化的安全控制。

## 三大主流生命周期模型对比

| 维度 | NIST SP 800-122 | 微软 DGPC 模型 | ISO/DSMM |
|------|----------------|---------------|----------|
| 阶段数 | 3阶段 | 6阶段 | 6阶段 |
| 划分方式 | 输入-处理-输出 | 采集-传输-存储-使用-共享-销毁 | 采集-传输-存储-处理-交换-销毁 |
| 特点 | 简洁概括，适合战略层沟通 | 覆盖完整，实践指导性强 | 与DSMM评估映射，适合成熟度评估 |
| 适用范围 | 隐私保护、PII管理 | 企业通用数据治理 | 中国数据安全标准体系 |
| 风险侧重 | 隐私泄露风险 | 全链路控制风险 | 能力成熟度评估 |

## 六大阶段详述

### 1. 采集（Collect）

数据采集阶段的风险主要集中在源头——未经授权采集、超范围采集、采集过程中的泄露。

**安全控制措施：**

| 控制项 | 说明 | 实施建议 |
|-------|------|---------|
| 数据最小化 | 仅采集业务必需的数据字段 | 建立采集清单审批机制，定期审查冗余字段 |
| 采集授权 | 明确数据来源的合法性和授权范围 | 获取用户同意（个保法要求）、签订数据采购协议 |
| 采集通道安全 | 防止采集过程中的数据泄露 | 采集接口启用HTTPS、源端认证、日志记录 |
| 数据源鉴别 | 验证数据来源的真实性 | 数字签名、来源IP白名单、数据完整性校验 |
| 分类分级打标 | 采集时即进行初步分级标识 | 自动化分类引擎、元数据标注、敏感数据自动识别 |

### 2. 传输（Transmit）

数据传输跨越网络边界时面临拦截、篡改、中间人攻击等风险。

| 控制项 | 说明 | 实施建议 |
|-------|------|---------|
| 传输加密 | 确保数据在传输过程中不可读 | TLS 1.2+、VPN隧道、IPSec |
| 完整性校验 | 检测传输中的数据篡改 | HMAC、数字签名、Checksum |
| 通道隔离 | 生产数据与测试数据的传输通道隔离 | VLAN分段、传输网络ACL策略 |
| API安全 | 保障接口调用的安全 | API网关认证、限流、参数校验、敏感字段脱敏 |
| 文件传输安全 | 保障文件级传输安全 | SFTP/FTPS替代FTP、文件加密后传输、传输日志审计 |

### 3. 存储（Store）

数据存储是静态数据最集中的阶段，核心风险是存储介质泄露、越权访问。

| 控制项 | 说明 | 实施建议 |
|-------|------|---------|
| 静态加密 | 存储介质层面的数据加密 | AES-256、TDE（透明数据加密）、云KMS |
| 访问控制 | 最小权限原则，基于角色的访问 | RBAC/ABAC、数据库ACL、文件系统权限 |
| 数据脱敏 | 非生产环境使用去标识化数据 | 动态脱敏（实时查询）、静态脱敏（导出） |
| 备份与容灾 | 防止数据丢失 | 3-2-1备份策略、异地灾备、备份加密 |
| 存储介质管理 | 物理介质的安全管理 | 介质分级标识、出入库登记、定期盘点 |

### 4. 使用（Use）

数据在使用阶段面临最大风险——内部人员越权查询、数据导出、屏幕拍照、AI辅助开发中的数据泄露等。

| 控制项 | 说明 | 实施建议 |
|-------|------|---------|
| 动态脱敏 | 根据用户权限实时脱敏查询结果 | 数据库动态脱敏网关、应用层脱敏 |
| DLP | 防止终端侧数据泄露 | 终端DLP、网络DLP、邮件DLP、USB管控 |
| 数据水印 | 可追溯泄露源 | 明水印、暗水印（数字水印）、屏幕水印 |
| 审计与监控 | 全量记录数据访问行为 | 数据库审计、文件审计、用户行为分析（UBA） |
| 沙箱/安全屋 | 高敏感数据在受控环境中使用 | 隐私计算、联邦学习、可信执行环境（TEE） |

### 5. 共享（Share）

数据共享涉及跨组织、跨系统的数据流动，是数据安全的薄弱环节。

| 控制项 | 说明 | 实施建议 |
|-------|------|---------|
| 接口鉴权 | 保障共享API的安全性 | OAuth 2.0、API Token、JWT、双向TLS |
| 数据水印 | 外发数据嵌入可追溯标识 | 文件水印、PDF水印、图片水印 |
| 合作伙伴评估 | 对接收方进行安全评估 | 签署NDA/DPA、安全资质审查、定期巡检 |
| 共享范围控制 | 最小化共享数据范围 | 按需提供、字段级筛选、数据脱敏后共享 |
| 合规审查 | 确保共享符合监管要求 | 数据出境评估（网信办）、PIA评估、共享协议 |

### 6. 销毁（Destroy）

数据销毁是生命周期中容易被忽视但合规要求严格的环节。

| 控制项 | 说明 | 实施建议 |
|-------|------|---------|
| 逻辑销毁 | 使数据不可读取，但介质可用 | 覆写（DoD 5220.22-M标准）、磁盘擦除工具 |
| 物理销毁 | 破坏存储介质 | 消磁、粉碎、高温熔毁、化学腐蚀 |
| 密钥销毁 | 销毁加密密钥使密文不可恢复 | KMS密钥删除、HSM密钥销毁 |
| 证书吊销 | 撤销数据相关数字证书 | CRL/OCSP、CA吊销 |
| 销毁证明 | 可审计的销毁记录 | 销毁流程记录、销毁证书、第三方见证 |

## 生命周期治理实施路线

```
第一阶段：摸底与评估（1-2个月）
├── 数据资产盘点 → 明确数据分布
├── 数据流转测绘（Data Flow Map）→ 识别数据流向
└── 现有安全控制差距分析

第二阶段：基线建设（3-6个月）
├── 按阶段建立安全控制基线
├── 数据分级策略与各阶段控制映射
└── 技术工具选型与部署

第三阶段：运营优化（持续）
├── 自动化策略执行（策略即代码）
├── 定期安全审计与攻防演练
└── 成熟度评估与持续改进
```

## 数据生命周期 vs. 系统生命周期

| 对比维度 | 数据生命周期 | 系统生命周期 |
|---------|-------------|-------------|
| 关注对象 | 数据记录本身 | 基础设施（服务器、DB、应用） |
| 生命周期长度 | 跨系统、跨组织长期存在 | 单个系统的规划-部署-退役 |
| 迁移特性 | 数据可在系统间迁移流转 | 系统通常固定运行环境 |
| 下线处理 | 数据迁移/归档/销毁 | 系统下线+数据合规处置 |
| 治理责任 | 数据Owner（业务部门） | 系统Owner（IT/运维部门） |

**关键教训**：系统下线前必须确保其中数据的合规处置。许多企业在废弃旧系统时仅做了系统注销，未对数据进行分类处置，导致大量敏感数据滞留在废弃介质中，形成"数据僵尸"风险。

## 实践建议

1. **在每个阶段建立对应的安全控制基线**，不同级别的数据在不同阶段采取差异化策略
2. **建立数据流转图谱（Data Flow Map）**，识别生命周期各阶段的风险暴露点
3. **将数据安全生命周期与DevOps流水线集成**，实现"安全左移"
4. **定期进行生命周期安全审计**，重点关注使用和共享阶段的异常行为
5. **关注AI带来的新风险**：大模型训练数据合规、RAG场景中的数据泄露、AI Agent的权限边界
