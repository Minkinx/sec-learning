# 数据安全生命周期

> 数据安全生命周期管理（Data Security Lifecycle Management, DSLM）贯穿数据从创建到销毁的全过程。

## 六大阶段

| 阶段 | 说明 | 安全要点 |
|------|------|----------|
| 采集（Collect） | 数据的产生与获取 | 数据最小化原则、采集授权、同意获取 |
| 传输（Transmit） | 数据在系统间的流动 | TLS/SSL加密、VPN隧道、API安全 |
| 存储（Store） | 数据的持久化保存 | 静态加密、访问控制、数据脱敏 |
| 使用（Use） | 数据的处理与分析 | 动态脱敏、数据防泄漏（DLP）、审计 |
| 共享（Share） | 数据对外提供与交换 | 接口鉴权、数据水印、合作伙伴安全评估 |
| 销毁（Destroy） | 数据的安全删除 | 覆写消磁、物理销毁、证书吊销 |

## 关键框架：NIST SP 800-122

NIST SP 800-122 将数据生命周期简化为三个阶段：

- **输入**：数据的收集与创建
- **处理**：数据的存储、使用、共享
- **输出**：数据的归档与销毁

## 数据生命周期 vs. 系统生命周期

- 数据生命周期关注"数据本身"——一个数据记录可以在多个系统中流转
- 系统生命周期关注"基础设施"——服务器、数据库、应用的运作周期
- 实践中需要将两者结合：系统下线前必须确保其中数据的合规处置

## 实践建议

- 在每个阶段建立对应的安全控制基线
- 对数据进行分级，不同级别的数据在不同阶段采取差异化策略
- 建立数据流转图谱（Data Flow Map），识别生命周期各阶段的风险暴露点
