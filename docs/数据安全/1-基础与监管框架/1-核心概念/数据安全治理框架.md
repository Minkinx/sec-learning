# 数据安全治理框架

> 数据安全治理（Data Security Governance, DSG）是将安全策略、流程与组织架构融为一体的治理体系。

## Gartner数据安全治理框架

Gartner提出的DSG框架包含三个核心维度：

- **业务与安全对齐**：安全策略支撑业务目标，而非阻碍业务
- **数据资产梳理**：识别数据资产，建立数据目录与分类分级
- **风险与合规**：基于监管要求进行风险识别与合规评估

## DCAP（Data-Centric Audit & Protection）

DCAP是Gartner提出的以数据为中心的安全方法论，关键能力包括：

- **数据发现与分类**：自动扫描、识别、打标
- **审计与监控**：细粒度记录数据访问行为
- **保护与控制**：加密、脱敏、访问控制

## 三大支柱

| 支柱 | 内容 |
|------|------|
| 人员（People） | 数据安全委员会、数据Owner、数据管理员 |
| 流程（Process） | 分类分级流程、事件响应流程、变更管理 |
| 技术（Technology） | DLP、加密、IAM、审计、数据血缘 |

## 数据治理 vs. 数据安全治理

- **数据治理**（Data Governance）：关注数据质量、元数据管理、主数据管理——让数据"可用"
- **数据安全治理**（Data Security Governance）：关注数据保护、合规、风险——让数据"安全可控"
- 两者相辅相成：没有数据治理，安全治理难以落地；没有安全治理，数据治理可能引入风险

## 成熟度评估

数据安全治理的成熟度通常划分为：
1. **初始级**：被动应对，无统一策略
2. **规范级**：建立制度，局部落地
3. **量化级**：可度量、可监控
4. **优化级**：持续改进，自动化运营
