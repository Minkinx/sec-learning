# 数据安全与网络安全边界

> 数据安全与网络安全既有深度交叉，又有本质区别。

## 核心差异

| 维度 | 网络安全 | 数据安全 |
|------|----------|----------|
| 焦点 | 基础设施、网络边界、系统漏洞 | 数据本身的生命周期和价值 |
| 防护对象 | 网络、主机、应用 | 结构化与非结构化数据 |
| 威胁模型 | 入侵、恶意软件、拒绝服务 | 泄露、篡改、违规使用 |
| 合规驱动 | 等保、ISO 27001 | 数安法、个保法、GDPR |
| 核心能力 | 防火墙、IDS/IPS、漏洞扫描 | 数据分类分级、DLP、加密、脱敏 |

## 重叠区域

- **加密技术**：既是网络安全手段，也是数据安全的核心防护措施
- **访问控制**：网络层面的访问控制（NAC）与数据层面的权限管理共同构成纵深防御
- **网络DLP**：部署在网关层，用于检测网络传输中的敏感数据

## 融合趋势

- **零信任架构（Zero Trust）**：不再信任网络边界，每次访问都需要验证身份和权限，本质上是将安全重心从"网络"向"数据"迁移
- **SASE（Secure Access Service Edge）**：将网络能力与安全能力融合，数据安全策略嵌入网络边缘
- **数据安全态势管理（DSPM）**：跨多云环境的数据可见性与风险治理

## 组织架构

不同企业的数据安全归属各不相同：
- 大型企业：通常设立独立的数据安全团队，或隶属于CISO
- 强合规行业（金融、医疗）：数据安全团队与网络安全团队并行
- 数据驱动型企业：数据安全与数据治理（CDO）深度协作
