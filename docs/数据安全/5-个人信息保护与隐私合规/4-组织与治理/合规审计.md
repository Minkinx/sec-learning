# 合规审计

## 审计类型对比

| 维度 | 内部审计 | 外部审计 |
|------|---------|---------|
| 执行主体 | 企业内部审计团队/合规团队 | 第三方专业审计机构 |
| 独立性 | 相对独立但受组织影响 | 完全独立 |
| 频率 | 每半年至一年一次 | 每一年至两年一次 |
| 报告用途 | 内部改进、管理层决策 | 监管报送、合规证明 |
| 深度 | 可覆盖全业务线 | 通常抽样审查 |
| 成本 | 固定成本 | 按项目收费 |

## 审计范围

### 治理层面

- 个人信息保护组织架构和职责划分
- 内部管理制度的建立和执行
- 个人信息保护负责人的履职情况
- PIA执行效率和覆盖完整性
- 培训计划的执行和效果

### 流程层面

- 告知同意流程的合规性
- 个人信息收集、使用、共享、删除的全生命周期管理
- 个人权利请求处理效率和合规性
- 委托处理和第三方管理流程
- 安全事件应急响应和通报流程
- 跨境数据传输审批流程

### 技术层面

- 数据加密和密钥管理措施
- 访问控制和权限管理
- 日志记录和监控审计
- 去标识化和匿名化处理效果
- 数据备份和灾难恢复能力
- 网络安全防护（防火墙、入侵检测等）

### 第三方管理层面

- 委托处理协议内容完整性
- 第三方SDK数据收集透明度
- 数据处理协议（DPA）签署和执行情况
- 第三方安全审计结果审查
- 子处理者管理

## PIPL vs GDPR审计要求

| 对比维度 | PIPL | GDPR |
|---------|------|------|
| 审计强制性 | 第54条：强制性定期审计 | 无明确审计条款，但可通过Art.58要求 |
| 审计频率 | 处理者自行确定 | 未明确规定 |
| 审计标准 | 国家网信办可制定审计指引 | EDPB指引 |
| 外部审计要求 | 监管部门可要求委托专业机构 | 监管机构可命令由合格主体进行审计 |
| 审计报告报送 | 应监管要求报送 | 应监管要求配合提供 |

## 审计执行流程

1. **审计计划阶段**
   - 确定审计范围和目标
   - 组建审计团队（外部+内部）
   - 制定审计计划和时间表
   - 发送审计通知和索取资料清单

2. **现场执行阶段**
   - 召开进点会议
   - 文档审查和流程访谈
   - 技术检测和抽样验证
   - 穿行测试和漏洞扫描

3. **报告阶段**
   - 汇总发现的问题
   - 评估风险等级（高/中/低风险）
   - 编制审计报告
   - 出具整改建议

4. **整改跟踪阶段**
   - 责任部门制定整改方案
   - 明确整改期限
   - 整改进展月度跟踪
   - 整改完成后的复查验证

## 整改要求与SLA

| 风险等级 | 定义 | 整改时限 |
|---------|------|---------|
| 高风险 | 可能导致大规模泄露或严重违规 | 15个工作日 |
| 中风险 | 可能造成有限范围的违规 | 30个工作日 |
| 低风险 | 流程优化或文档完善类问题 | 60个工作日 |
