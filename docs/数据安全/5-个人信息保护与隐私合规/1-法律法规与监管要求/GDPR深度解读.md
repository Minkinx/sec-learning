# GDPR深度解读

## 适用范围

| 维度 | 内容 |
|------|------|
| 实体范围（material scope） | 全部或部分通过自动化方式处理的个人数据，以及非自动化方式处理的结构化档案数据 |
| 地域范围（territorial scope） | 在欧盟境内设立的数据控制者或处理者的处理活动；向欧盟境内数据主体提供商品/服务或监控其行为的境外实体 |

## 控制者（Controller）与处理者（Processor）

| 角色 | 定义 | 核心义务 |
|------|------|---------|
| 控制者 | 单独或与他人共同确定数据处理目的和方式的实体 | 承担主要合规责任，对数据主体权利负责 |
| 处理者 | 代表控制者处理个人数据的实体 | 按控制者指令处理，实施适当安全措施 |
| 共同控制者 | 共同确定处理目的和方式的多个实体 | 以透明方式安排各自责任 |

## 六项合法性基础（Art.6）

1. **同意（Consent）** — 自由给予、具体、知情、明确
2. **合同必要性（Contract）** — 履行合同或应数据主体请求采取措施所必需
3. **法律义务（Legal Obligation）** — 控制者履行法律义务所必需
4. **重大利益（Vital Interests）** — 保护数据主体或他人重大利益
5. **公共利益（Public Interest）** — 执行公共利益任务或行使官方权力
6. **正当利益（Legitimate Interests）** — 控制者或第三方正当利益，需进行平衡测试

## 数据主体八项权利

| 权利 | 对应条款 |
|------|---------|
| 知情权（Right to be Informed） | Art.13-14 |
| 访问权（Right of Access） | Art.15 |
| 更正权（Right to Rectification） | Art.16 |
| 删除权（被遗忘权，Right to Erasure） | Art.17 |
| 限制处理权（Right to Restrict Processing） | Art.18 |
| 数据可携权（Right to Data Portability） | Art.20 |
| 反对权（Right to Object） | Art.21 |
| 自动化决策豁免权（Rights re: Automated Decision-making） | Art.22 |

## DPO制度（Art.37-39）

- 强制任命情形：公共机构、大规模系统性监控、大规模特殊类别数据处理
- DPO资质：数据保护法律知识和实践经验
- 报告层级：直接向最高管理层报告
- 独立性保障：不得因履行职责被解雇或处罚
- 联系方式：必须对外公开并通知监管机构

## 数据泄露通知（Art.33-34）

| 通知义务 | 时限 | 条件 |
|---------|------|------|
| 向监管机构通知 | 72小时内 | 除非泄露不太可能对权利和自由造成风险 |
| 向数据主体通知 | 无延迟 | 可能对权利和自由造成高风险时 |

## 处罚体系

| 类别 | 上限 | 适用情形 |
|------|------|---------|
| 一级处罚 | 1000万欧元或全球年营业额的2%（取高者） | 记录义务、安全措施、通知义务等 |
| 二级处罚 | 2000万欧元或全球年营业额的4%（取高者） | 处理原则、合法性基础、数据主体权利、跨境传输 |

## 跨境传输机制

- **充分性认定（Adequacy Decision）** — 欧盟委员会认定第三国保护水平充分
- **标准合同条款（SCC）** — 欧盟委员会批准的标准化数据保护条款
- **有约束力的公司规则（BCR）** — 经监管机构批准的企业内部数据保护规则
- **其他例外** — 明确同意、合同必要性、重大利益等
