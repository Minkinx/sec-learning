# 个人信息保护审计制度

## 法律依据

《个人信息保护法》第五十四条明确要求：个人信息处理者应当定期对其处理个人信息遵守法律、行政法规的情况进行合规审计。第六十四条赋予监管部门要求处理者委托专业机构进行审计的权力。

## 审计类型

| 类型 | 触发方式 | 要求 |
|------|---------|------|
| 定期自主审计 | 法律强制性要求 | 处理者自行组织或委托专业机构，定期执行 |
| 强制委托审计 | 监管机构要求 | 监管部门发现合规风险或安全事件后要求处理者委托专业机构审计 |
| 专项审计 | 特定场景触发的审计 | 合并重组、引入新产品/服务涉及大规模个人信息处理时 |

## 审计范围

### 治理层面

- 个人信息保护组织架构和人员配置
- 内部管理制度和操作规程
- 个人信息保护负责人（DPO）设立情况
- 个人信息保护影响评估（PIA）执行情况

### 处理活动层面

- 告知同意机制的有效性
- 合法性基础的适用正确性
- 最小必要原则的落实情况
- 个人信息主体权利响应机制
- 委托处理和共享转让的合规性

### 技术措施层面

- 安全技术措施有效性（加密、去标识化、访问控制等）
- 安全事件应急响应机制
- 数据备份和恢复措施
- 日志记录和审计追踪能力

### 第三方管理层面

- 委托处理协议和第三方管理
- 共同处理者的责任划分
- 跨境数据传输合规性
- SDK和API接口的安全管理

## 审计频率建议

| 组织类型 | 建议频率 | 说明 |
|---------|---------|------|
| 关键信息基础设施运营者 | 每年至少一次 | 结合等保测评同步进行 |
| 大型互联网平台 | 每年至少一次 | 覆盖所有核心业务线 |
| 处理100万人以上个人信息 | 每两年至少一次 | 视业务复杂度可增加频率 |
| 其他处理者 | 每三年至少一次 | 根据业务变化及时调整 |
| 发生安全事件后 | 立即开展 | 专项审计 |

## 审计方法

1. **文档审查** — 审查隐私政策、用户协议、数据处理记录、PIA报告等
2. **流程穿行测试** — 从数据采集到删除的全生命周期测试
3. **技术检测** — 加密强度验证、访问控制测试、日志完整性检查
4. **人员访谈** — 与数据保护团队、业务负责人、技术负责人面谈
5. **抽样验证** — 对个人信息处理活动进行抽样合规验证

## 整改要求

- 审计报告出具后30个工作日内制定整改方案
- 整改措施需明确责任人和完成时限
- 重大风险应当在发现后立即采取临时控制措施
- 整改完成后进行复查验证
- 审计报告和整改记录需保存至少三年
