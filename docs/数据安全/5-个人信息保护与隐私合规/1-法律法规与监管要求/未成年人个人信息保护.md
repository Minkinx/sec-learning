# 未成年人个人信息保护

## 年龄阈值对比

| 法域 | 儿童年龄定义 | 法律依据 |
|------|------------|---------|
| 中国 | 不满14周岁 | 《个人信息保护法》第31条、《儿童个人信息网络保护规定》 |
| 欧盟GDPR | 不满16周岁（成员国可调至13周岁） | Art.8 |
| 美国COPPA | 不满13周岁 | COPPA Rule |

## 中国特别保护规则（PIPL第15-17条）

### 核心要求

- **单独同意**：处理不满14周岁未成年人个人信息须取得其监护人的单独同意
- **专门规则**：制定专门的未成年人个人信息处理规则
- **识别机制**：建立用户年龄识别机制
- **监护人授权**：设计有效的监护人授权流程
- **敏感信息加强保护**：未成年人个人信息本身即视为敏感个人信息

### 合规要素

| 要求 | 具体措施 |
|------|---------|
| 年龄识别 | 注册环节通过身份证号验证或实名认证机制识别年龄 |
| 监护人同意 | 短信验证、人脸验证、邮箱确认等多方式验证监护人身份 |
| 同意撤销 | 监护人可随时撤销同意，处理后需删除信息 |
| 信息最小化 | 仅收集实现服务目的所必需的未成年人信息 |
| 安全存储 | 采取加密等更高级别的安全措施 |
| 特殊保护人 | 设置未成年人保护专门岗位或责任人 |

## 数据最小化要求

- 不得收集与未成年人服务无关的信息
- 不得诱导未成年人提供超出必要范围的信息
- 不得强制要求未成年人同意与服务无关的数据处理
- 每次使用地理位置等敏感权限需获得额外确认

## 单独处理规则

| 场景 | 要求 |
|------|------|
| 未成年人信息与成年人信息混存 | 必须分类存储并设置不同的访问权限 |
| 共享给第三方 | 须进行单独的安全评估和监护人授权 |
| 公开披露 | 严格禁止，法律另有规定除外 |
| 自动化决策 | 不得对未成年人进行用户画像和精准营销 |

## 合规落地要点

1. 产品设计层面嵌入年龄验证机制
2. 建立监护人授权管理后台（授权记录、撤销功能）
3. 定期清理超过处理期限的未成年人信息
4. 发生泄露时启动特殊通知流程（同时通知监护人和未成年人）
5. 开发面向未成年人的教育性隐私提示内容
