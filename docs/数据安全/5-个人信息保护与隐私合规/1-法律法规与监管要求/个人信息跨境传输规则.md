# 个人信息跨境传输规则

## 三条法定路径

根据PIPL第38条，个人信息处理者因业务需要确需向境外提供个人信息的，应当具备下列条件之一：

| 路径 | 适用场景 | 法律依据 |
|------|---------|---------|
| 安全评估 | 关键信息基础设施运营者、处理100万人以上个人信息、累计向境外提供10万人以上个人信息或1万人以上敏感个人信息 | 《数据出境安全评估办法》 |
| 标准合同（SCC） | 上述条件以外的个人信息出境活动（非关键信息基础设施运营者、数据量低于阈值） | 《个人信息出境标准合同办法》 |
| 认证 | 通过专业机构进行个人信息保护认证 | 《个人信息保护认证管理办法》 |

## 数据出境安全评估

### 触发条件

- 关键信息基础设施运营者向境外提供个人信息
- 处理100万人以上个人信息的处理者向境外提供
- 自上年1月1日起累计向境外提供10万人以上个人信息
- 自上年1月1日起累计向境外提供1万人以上敏感个人信息

### 评估流程

1. 处理者自评估 → 向省级网信办提交申请
2. 省级网信办完备性审核（5个工作日内）
3. 国家网信办组织安全评估（45个工作日内，可延长）
4. 评估结果出具（通过/不通过/重新评估）

## 标准合同条款

### 中国SCC（国家网信办2023年版）

- 合同模板为强制性范本，不得随意修改核心条款
- 备案要求：合同生效后10个工作日内向省级网信办备案
- 内容要求：明确双方责任、数据主体权利、技术安全措施、违约责任
- 重评估情形：合同目的变化、接收方法律环境重大变化等

### 中国SCC vs 欧盟SCC

| 维度 | 中国SCC | 欧盟SCC |
|------|---------|---------|
| 监管介入 | 需备案，监管机构可介入 | 原则上合同双方自治，监管保留审查权 |
| 合同范本 | 国家网信办发布固定模板 | 欧盟委员会发布，模块化设计 |
| 适用方 | 仅限境内处理者与境外接收方 | C2C/C2P/P2P多种组合 |
| 争议解决 | 中国法律管辖 | 可约定第三国法律 |
| 个人权利救济 | 境内诉讼途径 | 第三方受益权条款 |

## 个人信息保护认证

- 认证机构：中国网络安全审查技术与认证中心（CCRC）等
- 认证标准：GB/T 35273《个人信息安全规范》等
- 认证范围：个人信息处理活动、跨境传输活动
- 有效期：通常为3年，需年度监督

## 例外情形

| 情形 | 说明 |
|------|------|
| 个人自行主动跨境 | 个人主动将其信息传输至境外 |
| 必要跨境合同 | 个人作为合同一方，跨境传输为履行合同所必需 |
| 紧急保护 | 为保护个人生命健康和财产安全紧急需要 |
| 法律规定的其他情形 | 兜底条款 |
