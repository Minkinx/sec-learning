# 个人信息加密存储

## 加密范围

| 数据分级 | 加密要求 | 示例 |
|---------|---------|------|
| 敏感个人信息 | **必须加密** | 身份证、银行卡、生物识别、医疗健康、行踪轨迹 |
| 一般个人信息 | **建议加密** | 姓名、电话、地址、邮箱、教育经历 |
| 公开信息 | 按需加密 | 已自主公开的信息 |
| 匿名化数据 | 可不加密 | 已无法识别特定自然人的信息 |

## 算法选择

### 推荐算法

| 算法 | 密钥长度 | 适用场景 | 合规认可 |
|------|---------|---------|---------|
| AES-256-GCM | 256位 | 数据存储加密（对称加密） | GB/T 32907，国际标准 |
| SM4 | 128位 | 数据存储加密（国密算法） | GB/T 32907-2016，国密要求 |
| ChaCha20-Poly1305 | 256位 | 移动终端场景 | RFC 8439 |
| RSA-3072/4096 | 3072/4096位 | 密钥封装传输（非对称加密） | 密码算法推荐 |

### 不推荐算法

- DES/3DES（已证明不安全）
- MD5用于密码存储（不抗碰撞，应使用bcrypt/scrypt/Argon2）
- SHA1（已证明存在碰撞攻击）
- RC4（存在多个已知漏洞）
- RSA-1024及以下（密钥长度不足）

## 密钥管理策略

### 密钥派生

| 模式 | 描述 | 适用场景 |
|------|------|---------|
| 用户主密钥（User Master Key） | 每个用户独立主密钥 | 金融级应用、密码管理器 |
| 基于令牌化（Tokenization） | 用令牌替代真实数据 | 支付卡数据 |
| 数据密钥加密密钥（KEK） | 层级密钥体系（Envelope Encryption） | 云存储、大规模应用 |

### 密钥生命周期管理

- **生成**：使用密码学安全随机数生成器（CSPRNG）
- **分发**：通过安全通道分发，使用公钥加密传输对称密钥
- **存储**：密钥与数据分离存储，使用HSM或KMS管理
- **轮换**：定期轮换（建议每90-180天），支持多版本共存
- **失效/销毁**：安全删除旧密钥，确保不可恢复
- **备份**：密钥备份需加密存储且与主密钥分离

## 数据库级 vs 应用级加密

| 维度 | 数据库级加密（TDE） | 应用级加密 |
|------|-------------------|-----------|
| 加密粒度 | 整个数据库/表空间 | 特定字段 |
| 性能影响 | 低（透明加密解密） | 依字段数量而定 |
| 灵活性 | 不能按字段差异化加密 | 可按字段自定义策略 |
| 查询能力 | 支持各种查询排序 | 加密字段查询受限 |
| 密钥管理 | 数据库内置KMS | 应用层负责密钥管理 |
| 透明性 | 对应用透明 | 应用需感知加密 |

**推荐策略**：数据库级加密作为底层防线 + 对敏感字段实施应用级逐字段加密

## 加密搜索技术

| 技术 | 描述 | 性能 | 安全性 |
|------|------|------|--------|
| 可搜索加密（SSE） | 支持加密数据的关键词搜索 | 中等 | 高 |
| 保留格式加密（FPE） | 加密后保持数据格式不变 | 高 | 中（不可用于强加密场景） |
| 同态加密（HE） | 可在加密数据上直接计算 | 低（性能开销大） | 高 |
| 安全多方计算（MPC） | 多方协作计算不泄露明文 | 低 | 高 |
| 确定性加密（DET） | 相同明文输出相同密文（支持等值查询） | 高 | 中 |
