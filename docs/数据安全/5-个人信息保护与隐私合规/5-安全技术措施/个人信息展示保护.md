# 个人信息展示保护

## 展示脱敏规则

### 各类信息的脱敏标准

| 信息类型 | 脱敏规则 | 示例（原值→脱敏值） |
|---------|---------|-------------------|
| 身份证号 | 保留前6位和后4位，中间隐藏 | 110101****1234 |
| 手机号 | 保留前3位和后4位，中间隐藏 | 138****5678 |
| 固定电话 | 保留区号和最后4位 | 010-****5678 |
| 电子邮件 | 用户名@域名部分展示 | j***@example.com |
| 银行卡号 | 保留前6位和后4位 | 622848****1234 |
| 家庭住址 | 保留到小区/楼栋级别 | 北京市海淀区**小区**号楼 |
| 姓名 | 保留姓氏，名字替换 | 张** |
| 车牌号 | 保留省份和最后两位 | 京A****8 |
| IP地址 | 隐藏最后一段 | 192.168.1.*** |
| 账号/用户名 | 保留首尾字符 | a***n |

### 脱敏程度分级

| 脱敏等级 | 适用场景 | 脱敏比例 | 示例（手机号） |
|---------|---------|---------|--------------|
| 完全展示 | 本人操作时验证身份需要 | 0% | 13812345678 |
| 部分脱敏 | 一般业务展示 | 50% | 138****5678 |
| 深度脱敏 | 非本人可查看的场景 | 80% | 138******* |
| 完全隐藏 | 内部非必须场景 | 100% | *********** |

## 脱敏实现方案

### 前端脱敏 vs 后端脱敏

| 维度 | 前端脱敏 | 后端脱敏 |
|------|---------|---------|
| 实施位置 | 浏览器/APP端JavaScript处理 | 服务端返回时已完成脱敏 |
| 安全性 | 较低（可通过浏览器调试还原） | 较高（数据从不离开安全区） |
| 网络传输 | 传输完整数据后再脱敏 | 仅传输脱敏后数据 |
| 性能影响 | 影响客户端性能 | 增加服务端计算负担 |
| 缓存/CDN | 缓存中可能暴露完整数据 | 缓存数据即为脱敏后数据 |

**最佳实践**：优先使用后端脱敏，敏感数据不应传输到前端

### 传输过程中的脱敏

- API返回数据时在网关层或服务端完成脱敏
- 日志记录时对原始参数进行脱敏替换
- 使用协议缓冲区（Protobuf）等将脱敏字段标记为"脱敏字段"
- 隐私拦截器中统一实施脱敏逻辑

## 屏幕截图/录屏保护

| 保护措施 | 说明 | 适用场景 |
|---------|------|---------|
| 防截屏机制 | APP内禁止截屏操作（FLAG_SECURE） | 支付界面、敏感信息详情页 |
| 水印技术 | 在屏幕上叠加用户身份水印 | 管理后台、客户数据查询页 |
| 隐私模式 | 隐藏消息预览和详情 | IM应用中开启 |
| 剪贴板保护 | 复制敏感信息时弹出警告 | 复制身份证号、银行卡时 |
| 截屏检测 | 检测到截屏操作时记录日志或触发警告 | 高安全环境 |

## 数据展示访问控制

| 控制维度 | 实施措施 |
|---------|---------|
| **角色分级** | 按岗位职责分配数据查看权限（如客服仅查看必要字段） |
| **场景控制** | 仅在必要时展示完整信息（如用户下单流程查看完整地址） |
| **审批流程** | 查看完整敏感信息需主管审批或双人操作 |
| **操作审计** | 查询/查看敏感数据的行为完整记录 |
| **动态脱敏** | 根据用户权限级别动态决定脱敏程度 |
| **访问次数限制** | 单个用户对敏感信息的访问频次限制 |

## 打印/复制限制

- 包含个人信息的页面打印时强制添加水印
- 批量导出个人信息需安全审批且记录操作日志
- 敏感字段设置禁止复制或复制时自动脱敏
- 下载的CSV/Excel文件中敏感字段自动脱敏或加密
- 对PDF导出设置密码保护和打印限制
- 物理文档的打印份数登记和废纸销毁流程
