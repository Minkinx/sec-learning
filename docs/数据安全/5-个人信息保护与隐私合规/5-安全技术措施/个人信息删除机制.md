# 个人信息删除机制

## 删除触发条件（PIPL第47条）

| 情形 | 说明 | 处理要求 |
|------|------|---------|
| 处理目的已实现或无法实现 | 服务关系终止、功能下线 | 立即启动删除程序 |
| 保存期限已届满 | 超出数据处理目的所需期限 | 到期后及时删除 |
| 个人撤回同意 | 用户行使撤回同意权 | 收到请求后启动 |
| 处理者违反法律法规 | 超范围收集、未告知等违规行为 | 立即停止并删除 |
| 法律法规规定的其他情形 | 监管要求、司法判决等 | 按规定执行 |

## 技术删除实现

### 存储系统删除

| 存储类型 | 删除方法 | 注意事项 |
|---------|---------|---------|
| 关系型数据库（MySQL/PostgreSQL） | DELETE + 数据覆盖 | 需处理外键约束和触发器 |
| NoSQL（MongoDB/Redis） | 文档删除 + 索引清理 | 确保所有副本和分片删除 |
| 对象存储（S3/OSS/COS） | 文件版本删除 + 保留策略更新 | 版本控制的Bucket需删除所有版本 |
| 分布式文件系统 | 文件元数据删除 + 块标记释放 | 确认数据块不再被引用 |
| CDN缓存 | URL失效 + 刷新缓存 | CDN可能有延迟 |

### 备份数据删除挑战

| 挑战 | 解决策略 |
|------|---------|
| 备份保留策略与删除要求冲突 | 定期恢复测试中验证删除成功，备份保留期内标记为不可用 |
| 快速恢复要求 | 保持删除前的快照但加密并销毁密钥（Crypto-shredding） |
| 多地备份 | 确保所有地理位置备份同步删除 |
| 备份格式不可直接修改 | 在下一轮备份周期覆盖旧备份 |

## 删除 vs 匿名化

| 维度 | 物理删除 | 匿名化处理 | 加密粉碎（Crypto-shredding） |
|------|---------|-----------|---------------------------|
| 操作 | 擦除存储介质上的数据 | 不可逆地去除标识 | 销毁加密密钥使数据不可读 |
| 恢复性 | 不可恢复（覆写后） | 不可恢复 | 密钥销毁后不可恢复 |
| 性能 | 高（需大量I/O操作） | 中 | 低（仅需销毁密钥） |
| 适用场景 | 数据库记录删除 | 统计/分析用数据 | 加密存储的大文件 |
| 合规接受 | 完全合规 | 符合匿名化规则则合规 | 需确保密钥完全不可恢复 |
| 审计难度 | 低（删除操作记录） | 需证明匿名化充分 | 需证明密钥销毁彻底 |

**推荐策略**：优先物理删除；不能满足业务需求时选择匿名化；加密存储的数据采用加密粉碎

## 删除确认与通知

### 删除确认流程

1. 删除操作完成后，系统自动生成删除确认报告
2. 删除确认报告包含：删除时间、删除范围、删除方式
3. 涉及个人请求的删除，需向请求人发送确认通知
4. 删除确认通知应通过安全的通信渠道发送
5. 保留删除操作日志（日志中不可包含原始数据）

### 关联通知义务

- 删除信息前，如该信息已被共享给第三方，应通知第三方删除
- 涉及公开发布的信息，应予撤回或删除
- 委托处理场景下，应通知受托人执行删除
- 共同控制者场景，应协调各方同步删除

## 删除操作的合规记录

| 记录要素 | 内容要求 |
|---------|---------|
| 请求来源 | 用户ID/请求工单号 |
| 删除范围 | 涉及的数据类型和数量 |
| 删除时间 | 操作时间戳 |
| 删除方式 | 物理删除/匿名化/加密粉碎 |
| 执行系统 | 涉及的数据存储系统 |
| 执行人/自动触发 | 操作人或触发条件 |
| 复核人 | 删除操作的审批人 |
| 删除后验证 | 验证结果（是否成功） |
| 关联第三方 | 已通知的第三方和处理结果 |
