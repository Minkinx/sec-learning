# 隐私政策撰写规范

## 法规要求的内容要素

### PIPL要求的告知内容

- 处理者的名称和联系方式
- 处理目的、处理方式
- 处理的个人信息种类和保存期限
- 个人行使权利的方式和程序
- 向第三方提供信息的场景
- 跨境传输的情况（如适用）
- 自动化决策情况说明
- 投诉和举报渠道

### GDPR要求的告知内容（Art.13-14）

- 控制者及代表人身份与联系方式
- DPO联系方式
- 处理目的与合法性基础
- 正当利益目的（如适用）
- 数据接收方或类别
- 跨境传输意图及保障措施
- 数据保留期限或确定标准
- 数据主体权利列表
- 撤回同意的权利
- 向监管机构投诉的权利
- 合同或法律要求的提供义务及不提供的后果
- 自动化决策的存在、逻辑及后果

## 撰写原则

| 原则 | 要求 | 实施建议 |
|------|------|---------|
| **透明性** | 信息处理活动全面披露 | 覆盖全生命周期，不隐瞒关键处理行为 |
| **清晰性** | 使用清晰简明的语言 | 避免法律术语堆砌，使用层级标题 |
| **可访问性** | 方便用户随时获取 | 固定入口、弹窗提醒、下载保存功能 |
| **分层性** | 重要信息突出显示 | 采用分层设计，首层强制阅读 |
| **及时性** | 变更后及时更新 | 重大变更触发重新同意流程 |

## 分层通知设计

### 第一层：概要通知

- 放置在产品首屏或注册页
- 突出核心信息：收集什么、为什么收集、如何控制
- 字数控制在200字以内
- 设置"查看完整隐私政策"链接

### 第二层：完整隐私政策

- 分章节详尽说明（建议8-12个章节）
- 每章以简短摘要开头
- 使用信息图或表格增强可读性
- 提供PDF下载版本

## 典型章节结构

| 章节 | 内容 |
|------|------|
| 引言 | 政策概述、更新日期、适用范围 |
| 信息收集 | 收集的信息类型、来源、方式、目的 |
| 信息使用 | 使用目的、合法性基础、自动化决策 |
| 信息共享 | 第三方共享、委托处理、跨境传输 |
| 信息存储 | 存储地点、存储期限、安全措施 |
| 个人权利 | 权利说明、行使方式、响应时限 |
| Cookie与同类技术 | 技术类型、用途、偏好管理 |
| 未成年人保护 | 年龄门槛、监护人同意、专门规则 |
| 政策更新 | 变更通知方式、生效日期 |
| 联系方式 | DPO/个人信息保护负责人、投诉渠道 |

## 多语言版本管理

- 中文版本为法律效力版本（境内业务）
- 英文及多语言版本供国际用户参考
- 各语言版本需实质内容一致
- 翻译差异以中文版本为准
- 针对不同法域用户的额外披露要求
