# 隐私政策版本管理

## 版本编号体系

### 推荐格式

```
主版本号.次版本号.修订号 YYYY-MM-DD
```

| 版本级别 | 变更类型 | 示例 |
|---------|---------|------|
| 主版本 | 重大结构调整、法律依据变更、跨境传输新增 | v2.0.0 |
| 次版本 | 新增信息类型、新增处理目的、新增第三方 | v1.3.0 |
| 修订号 | 措辞优化、格式调整、非实质性内容更新 | v1.0.2 |

### 版本号管理实践

- 每次发布正式变更前分配新版本号
- 内部审阅版本使用 "-rc" 后缀（如 v1.2.0-rc1）
- 版本号需在文件头部和文件名中体现
- 版本发布需经过法务审批

## 变更日志要求

### 基本要素

| 信息项 | 说明 |
|-------|------|
| 版本号 | 新版本标识 |
| 发布日期 | 生效日期 |
| 变更类型 | 新增/修改/删除 |
| 变更位置 | 章节和具体条款 |
| 变更摘要 | 变的内容描述（非法律专业用语） |
| 变更原因 | 法律法规变化/业务变化/监管要求 |
| 审阅人/审批人 | 法务审核记录 |

### 变更日志格式示例

```
## v1.3.0 (2024-06-15)

### 新增
- 新增第4.2节"第三方SDK信息收集说明"，增加广告SDK披露
- 新增第6.3节"自动化决策拒绝权利"说明

### 修改
- 修改第2.1节信息收集范围：增加设备标识符（OAID）收集说明
- 修改第8.1节数据保存期限：由3年调整为5年

### 删除
- 删除第5.3节中失效的合作方信息
- 删除已停止服务的功能相关的信息收集说明
```

## 通知机制

| 变更类型 | 有无溯及力 | 用户通知方式 | 是否需要重新同意 |
|---------|-----------|-------------|----------------|
| 实质性不利变更 | 无溯及力（新版本生效后适用） | 弹窗+站内信+邮件 | 是 |
| 一般性变更 | 无溯及力 | 站内信/公告/推送 | 需视情况 |
| 措辞优化 | 无溯及力 | 版本历史说明 | 否 |
| 业务关系更名 | 不适用 | 公告通知 | 否 |
| 法律依据更新 | 不适用 | 弹窗+公告 | 可能需重新同意 |

## 重大变更识别标准

符合以下任一条件视为重大变更：

- 处理个人信息的目的、类型、方式发生实质性变化
- 新增向第三方提供信息的情形
- 新增跨境数据传输
- 变更个人信息主体权利行使方式
- 数据保留期限延长
- 引入新的自动化决策场景
- 变更个人信息保护负责人或DPO联系方式
- 因法律法规变化需要调整核心条款

## 档案管理

- 存档所有历史版本的隐私政策（电子版+PDF签署版）
- 建立版本对照表（version mapping matrix）
- 保存每个版本的发布时间和用户同意记录
- 历史版本保留期限 = 业务存续期 + 3年
- 使用文件命名规范：`隐私政策_v{主版本}.{次版本}.{修订号}_YYYYMMDD.pdf`
- 建议使用git或文档管理平台进行版本控制
