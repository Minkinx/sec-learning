# 自动化决策与算法治理

## 定义与范围

### PIPL第24条界定的自动化决策

通过计算机程序自动分析、评估个人行为习惯、兴趣爱好或者经济、健康、信用状况等，并进行决策的活动。包括：

- **用户画像（Profiling）**：对个人特征的系统性评估
- **个性化推荐**：基于画像的信息推送、商品推荐
- **信用评分**：自动化的信用风险评估
- **招聘筛选**：基于算法的候选人匹配

### GDPR第22条规定的自动化决策

仅基于自动化处理（包括用户画像）对个人作出的决策，且该决策对个人产生法律效力或类似重大影响。

## 核心合规要求

| 要求 | PIPL | GDPR |
|------|------|------|
| 透明告知 | 第24条：告知使用自动化决策并说明决策逻辑 | Art.13-14：告知存在自动化决策及逻辑 |
| 拒绝权利 | 第24条：个人有权拒绝仅通过自动化决策作出决定 | Art.22：有权不受自动化决策约束 |
| 人工干预权 | 第24条：保证决策的透明度，提供不针对个人特征的选项 | Art.22(3)：保障人工干预权 |
| 公平对待 | 不得在价格等交易条件上实行不合理的差别待遇 | 禁止歧视性结果 |
| 影响评估 | 通过PIA评估自动化决策风险 | DPIA强制要求 |

## 算法备案制度

### 境内算法备案

- **法律依据**：《互联网信息服务算法推荐管理规定》
- **备案范围**：具有舆论属性或社会动员能力的算法推荐服务
- **备案内容**：算法类型、应用场景、算法机理、数据使用等
- **备案流程**：上线前向网信办提交备案材料 → 审核通过后获取备案编号
- **备案变更**：算法参数重大调整需重新备案

### 备案信息要素

| 信息项 | 说明 |
|-------|------|
| 算法主体信息 | 服务提供者名称、统一社会信用代码 |
| 算法类型 | 生成合成、排序精选、检索过滤、调度决策等 |
| 应用场景 | 新闻推送、搜索排序、调度派单等 |
| 数据使用情况 | 训练数据来源、标签体系 |
| 算法机理 | 模型结构、评估指标 |
| 风险防控 | 防歧视机制、干预机制、投诉渠道 |

## 透明度义务

### 告知内容

- 存在自动化决策的事实（显著方式告知）
- 自动化决策的基本逻辑和主要运行机制
- 自动化决策可能产生的结果和影响
- 提供了哪些不针对个人特征的选项

### 算法说明要求

- 用通俗易懂的语言描述算法逻辑
- 不可使用纯技术性描述
- 需包含关键参数和权重的一般性说明
- 需说明人工干预的触发条件和方式

## 算法审计

- 定期对算法的公平性、准确性进行审计
- 审计内容包括：歧视风险评估、透明度评估、结果偏差检测
- 发现歧视性结果的应立即停止使用并整改
- 审计报告需保存备查
