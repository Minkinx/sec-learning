# 合法性基础

## 六项合法性基础对比（PIPL第13条 vs GDPR Art.6）

| 合法性基础 | PIPL是否认可 | GDPR是否认可 | 适用要点 |
|-----------|-------------|-------------|---------|
| 个人同意 | 是 | 是 | PIPL要求"单独同意"场景更多 |
| 订立或履行合同所必需 | 是 | 是 | 需证明合同目的的非必要性 |
| 履行法定职责或义务 | 是 | 是 | 需明确法律、法规的具体条款 |
| 应对突发公共卫生事件 | 是 | 不单独列明 | 适用"重大利益"或"公共利益" |
| 为实施公共利益 | 是 | 是 | 新闻报道、舆论监督等 |
| 正当利益（Legitimate Interests） | 不单独列明 | 是 | GDPR独有的平衡测试 |
| 法律法规规定的其他情形 | 是（兜底） | 不适用 | 国内法兜底条款 |

## 合法性基础选择决策树

### Step 1：是否有法律明确规定要求处理？

```
是 → 法定职责/法律义务
否 → 继续 Step 2
```

### Step 2：处理是否为履行合同所必需？

```
是 → 合同必要性（需保存合同关系证据）
否 → 继续 Step 3
```

### Step 3：处理是否涉及重大公众利益或紧急情况？

```
是 → 公共利益/重大利益
否 → 继续 Step 4
```

### Step 4：是否属于GDPR管辖下的正当利益场景？

```
是 → 进行LIA平衡测试
否 → 依赖同意作为合法性基础
```

## 正当利益平衡测试（LIA）

GDPR场景下适用，需完成三要素评估：

| 评估要素 | 内容 |
|---------|------|
| **目的测试** | 是否追求合法的正当利益（直接营销、网络安全等） |
| **必要性测试** | 处理是否为达成该利益所必需，是否存在更少侵入性方式 |
| **平衡测试** | 处理对数据主体的影响与数据主体合理预期的利益权衡 |

### LIA文档化要求

- 书面记录平衡测试过程和结果
- 保持测试文档以供监管机构核查
- 重大业务变更时重新进行LIA

## 书面证明要求

- 选择的合法性基础需有充分的书面记录
- 合同场景需保留合同原件或副本
- 法定职责需明确引用的法律条款
- 同意场景需保存完整的同意记录
- PIPL要求处理者对其处理活动承担举证责任

## 常见错误

| 错误场景 | 问题 | 正确做法 |
|---------|------|---------|
| 默认使用同意作为唯一基础 | 增加撤回风险和运营负担 | 优先评估其他合法性基础 |
| 合同必要性范围扩大化 | 将改善体验推广为合同必要 | 严格限定核心功能 |
| 未保留文档记录 | 无法证明合法性基础适用正确 | 建立文档化制度 |
| 混合基础未明确区分 | 多个基础混用导致合规模糊 | 按处理目的分别确定基础 |
