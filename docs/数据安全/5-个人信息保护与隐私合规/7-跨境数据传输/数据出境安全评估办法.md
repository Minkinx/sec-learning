# 数据出境安全评估办法

## 法律依据

《数据出境安全评估办法》（国家网信办2022年7月发布，2022年9月1日生效），是PIPL第38条关于信息安全评估路径的具体实施细则。

## 触发条件（需同时满足或之一）

| 触发条件 | 说明 |
|---------|------|
| 关键信息基础设施运营者（CIIO） | 无论数据量大小，只要CIIO向境外提供个人信息 |
| 处理100万人以上个人信息 | 处理者处理个人信息总量达到100万人 |
| 累计向境外提供10万人以上个人信息 | 自上年1月1日起累计 |
| 累计向境外提供1万人以上敏感个人信息 | 自上年1月1日起累计 |
| 国家网信办规定的其他情形 | 兜底条款 |

### 触发条件判断流程图

```
是否为CIIO？
  ├─ 是 → 必须进行安全评估
  └─ 否 → 处理个人信息总量是否≥100万人？
              ├─ 是 → 累计向境外提供≥1万人敏感个人信息？
              │          ├─ 是 → 必须进行安全评估
              │          └─ 否 → 累计向境外提供≥10万人个人信息？
              │                    ├─ 是 → 必须进行安全评估
              │                    └─ 否 → 可选择SCC或认证
              └─ 否 → 累计向境外提供≥1万人敏感个人信息？
                         ├─ 是 → 必须进行安全评估
                         └─ 否 → 累计向境外提供≥10万人个人信息？
                                   ├─ 是 → 必须进行安全评估
                                   └─ 否 → 可选择SCC或认证
```

## 申请流程

### 第一阶段：自评估

1. 处理者自行开展数据出境风险自评估
2. 自评估报告需包括数据处理目的、范围、方式、境外接收方的保护能力等
3. 自评估报告有效期3年，处理目的变化或法律变化时需重新自评估

### 第二阶段：申报

1. 向所在地省级网信办提交申请材料
2. 材料包括：安全评估申请表、自评估报告、数据处理协议、数据出境说明等

### 第三阶段：审核

| 环节 | 时限 | 负责部门 |
|------|------|---------|
| 完备性审核 | 5个工作日内 | 省级网信办 |
| 安全评估 | 45个工作日内（可延长） | 国家网信办 |
| 结果送达 | 评估完成后15个工作日内 | 省级网信办 |

### 第四阶段：结果执行

- **通过**：按申报方案开展数据出境活动
- **不通过**：不得出境，需整改后重新申报
- **补正**：补充材料后继续评估

## 评估考量因素

| 评估维度 | 评估要点 |
|---------|---------|
| 数据出境目的 | 目的合法性、正当性、必要性 |
| 数据范围 | 出境数据类型和数量是否最小必要 |
| 数据处理协议 | 双方权利义务约定是否充分 |
| 接收方保护能力 | 法律环境、技术能力、管理制度 |
| 出境风险 | 数据被泄露、篡改、滥用的风险 |
| 在境内保留 | 境内留存的数据是否仍有意义 |
| 个人权益影响 | 对个人信息权益的潜在影响 |
| 整改措施 | 已采取或拟采取的风险缓解措施 |

## 有效期与重新评估

| 事项 | 要求 |
|------|------|
| 有效期 | 安全评估结果有效期为2年 |
| 重新申请 | 有效期届满前60个工作日内提出 |
| 重新评估触发 | 处理目的变化、数据类型增加、接收方法律变化、发生安全事件等 |
| 报告变更 | 未改变出境目的的较小变更可申请变更 |
