# 典型场景合规

## 场景一：跨境HR数据共享

### 场景描述

跨国企业将境内员工的人事信息（姓名、薪资、绩效、社保等）传输至境外总部进行人力资源管理。

### 合规分析

| 维度 | 分析 |
|------|------|
| 出境数据类型 | 敏感个人信息（薪资）、一般个人信息（姓名、职位、社保） |
| 数据量 | 视公司规模而定 |
| 处理目的 | 人力资源管理、全球员工信息系统 |
| 合法性基础 | 劳动合同的必要性？需审慎评估（HR数据通常不被认为"合同必要"范围） |
| 同意要求 | 需要员工的单独同意（敏感个人信息+跨境传输双重条件） |

### 合规路径

1. 进行数据出境风险自评估
2. 根据数据量选择：安全评估（>10万人）或SCC（<10万人）
3. 签署标准合同条款（出境方 → 境外总部接收方）
4. 获取员工的单独同意
5. 更新隐私政策中的跨境传输说明
6. 实施技术保护措施（传输加密、访问控制）

## 场景二：境外子公司数据访问

### 场景描述

境内母公司的数据分析平台需要允许境外子公司的工程师远程访问境内存储的用户数据。

### 合规分析

| 问题 | 解答 |
|------|------|
| 这是否构成"出境"？ | 是，境外人员访问境内服务器上的数据属于数据出境 |
| 是否可以仅通过限制访问规避？ | 需要采取访问控制+数据脱敏 |
| 合同主体如何确定？ | 境内母公司与境外子公司分别作为处理者和接收方 |

### 合规路径

1. 评估定期远程访问的工作必要性
2. 对非必要字段进行脱敏处理（去标识化）
3. 签署内部数据共享协议（含SCC条款）
4. 实施最小权限访问控制
5. 记录所有数据访问日志并定期审计
6. 如数据量达到阈值则选择安全评估路径

## 场景三：云服务数据本地化

### 场景描述

境内企业使用境外云服务提供商（如AWS国际站、Azure国际站）的基础设施服务，数据存储在境外服务器。

### 合规要点

| 要素 | 要求 |
|------|------|
| 数据存储位置 | 优先选择中国境内区域（AWS中国、Azure中国等） |
| 合同安排 | 与云服务商签署DPA，明确数据存储限制 |
| 运营实体 | 使用境内实体运营的云服务（如光环新网运营AWS中国） |
| 技术隔离 | 确保租户数据与境外数据物理或逻辑隔离 |
| 数据传输 | 境内服务间的通信不发生跨境流动 |

### 注意事项

- 使用国际版云服务默认数据可能存储在境外节点
- 切换到中国区云服务并确认数据驻留策略
- 云服务商的技术支持人员如有境外访问权限，需评估

## 场景四：SaaS多租户服务

### 场景描述

境内SaaS服务商向海外客户提供服务，需将海外客户数据存储在境内或第三国。

### 双向合规分析

| 方向 | 法律要求 |
|------|---------|
| 数据从境外流入境内 | 遵守境外法域法律（如GDPR），中国法律对入境无限制 |
| 境内存储和再出境 | 若数据中包含境内个人数据再出境→需评估合规路径 |
| 仅处理境外数据 | 若完全不涉及境内个人，则不受PIPL限制 |

### 合规路径

1. 明确数据处理范围：区分境内用户数据和境外用户数据
2. 对境外用户数据遵循其所在法域的法律（如GDPR SCC）
3. 对境内用户数据按PIPL执行合规管理
4. 建立数据分类存储策略（按用户来源分区存储）

## 场景五：国际合作研究

### 场景描述

境内科研机构与境外机构合作开展医学研究，需要共享匿名化或脱敏的医学数据。

### 合规要点

| 要素 | 要求 |
|------|------|
| 数据敏感性 | 医疗健康数据为敏感个人信息 |
| 匿名化可行性 | 需确认匿名化程度是否达到法定标准 |
| 知情同意 | 需获得患者/受试者的知情同意 |
| 伦理审查 | 需通过伦理委员会审批 |
| PIA要求 | 必须进行PIA，高风险需安全评估 |

## 决策流程图

```
识别数据出境场景
    ↓
评估是否涉及境内个人数据
    ↓ 是
是否属于CIIO或数据量达到阈值？
├─ 是 → 安全评估路径（向省级网信办申报）
└─ 否 → SCC or 认证路径
        ↓
确认具体场景类型 → 执行相应的合规操作
    ↓
实施技术保护措施（加密、脱敏、访问控制等）
    ↓
完成合规记录存档（PIA、SCC、自评估报告等）
    ↓
定期复核合规状态并更新相关文件
```
