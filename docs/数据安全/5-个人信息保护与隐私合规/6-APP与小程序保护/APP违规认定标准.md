# APP违规认定标准

## 常见违规行为分类

根据国家网信办、工信部、公安部等多部门的专项整治行动通报，APP个人信息保护违规行为可归纳为以下主要类型：

### 违规收集个人信息

| 违规类型 | 具体表现 | 处罚案例 |
|---------|---------|---------|
| 未告知收集目的 | 隐私政策未说明收集某一信息的目的 | 众多中小型APP被通报 |
| 超范围收集 | 收集与服务无关的信息（如计算器APP读取联系人） | 多次专项通报中的主要问题 |
| 强制收集 | 以不同意收集为由拒绝提供服务 | 社交类、工具类APP高频违规 |
| 频率过高 | 非必要的频繁读取位置/相册 | 地图导航类、生活服务类 |
| 诱导收集 | "领红包"等诱导方式收集无关信息 | 电商类APP通报案例 |

### 违规使用个人信息

- 未经授权将个人信息用于营销推广
- 超出用户授权范围的数据分析和画像
- 默认为"个性化推荐"且用户无法关闭
- 使用个人信息进行大数据杀熟

### 违规共享与传输

| 违规类型 | 具体表现 |
|---------|---------|
| 未告知第三方共享 | 隐私政策未披露共享第三方的类型和目的 |
| 未经同意共享敏感信息 | 位置、通信录等未获单独同意即共享 |
| SDK超范围收集 | 未对第三方SDK的收集行为进行约束 |
| 跨境传输未告知 | 向境外服务器传输个人数据未告知用户 |

### 不尊重用户权利

- 删除/注销账户入口难找或流程复杂
- 拒绝提供信息查阅和复制功能
- 不提供关闭个性化推荐选项
- 撤回同意后仍继续处理数据
- 设置不合理的注销条件（如要求上传身份证）

## APP违规认定主要依据

| 法规/标准 | 核心内容 |
|----------|---------|
| 《个人信息保护法》 | 告知同意、最小必要、权利保障等 |
| APP违法违规收集使用个人信息行为认定方法 | 9种认定情形（国家网信办） |
| GB/T 35273-2020 个人信息安全规范 | 个人信息处理具体要求和最佳实践 |
| 《常见类型移动互联网应用程序必要个人信息范围规定》 | 39类APP的必要个人信息范围 |
| CAMAS（APP安全认证） | 针对APP个人信息保护的认证标准 |

## 公安部（MSA）APP安全检测标准

| 检测项目 | 标准要求 |
|---------|---------|
| 隐私政策检测 | 是否有独立政策、是否覆盖所有处理活动 |
| 告知同意检测 | 首次使用是否弹窗告知、是否默认同意 |
| 权限使用检测 | 权限调用是否与功能一致、是否预授权 |
| 数据采集检测 | 是否超范围采集、是否加密传输 |
| 第三方SDK检测 | SDK是否独立合规、是否持续读取数据 |
| 注销功能检测 | 注销入口是否明显、流程是否简便 |
| 数据删除检测 | 注销账户后数据是否彻底删除 |

## 整改时限与程序

| 违规程度 | 整改期限 | 监管措施 |
|---------|---------|---------|
| 一般违规 | 15个工作日 | 通报、约谈 |
| 严重违规 | 限期整改（视情况） | 下架、暂停更新 |
| 拒不整改 | 持续监管 | 行政处罚、关闭账号 |
| 重大安全事件 | 立即停止 | 启动安全评估、暂停服务 |

## 整改流程

1. 收到通报后24小时内确认接收
2. 组建整改团队（法务+技术+产品）
3. 排查问题根因并制定整改方案
4. 按期限完成整改并提交整改报告
5. 等待相关部门复核确认
6. 如涉及下架则申请重新上架
