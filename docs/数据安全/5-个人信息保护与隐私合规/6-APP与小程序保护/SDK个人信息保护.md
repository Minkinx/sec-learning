# SDK个人信息保护

## SDK数据收集风险概述

第三方SDK在APP中广泛使用（统计、广告、登录、支付、推送等），但SDK的数据收集行为常常超出APP开发者的控制范围，成为个人信息保护的灰色地带。

### 常见SDK合规风险

| 风险类型 | 具体表现 | 典型案例 |
|---------|---------|---------|
| 超范围收集 | SDK收集与自身功能无关的信息（设备信息、应用列表等） | 第三方广告SDK常收集设备指纹 |
| 未告知收集 | APP未在隐私政策中披露SDK采集行为 | 推送SDK收集设备标识未披露 |
| 私自共享数据 | SDK未经允许将数据传输至境外服务器 | 分析SDK将数据回传至海外 |
| 持续后台采集 | 退出APP后SDK仍在采集数据 | 某些广告SDK常驻后台 |
| 权限不合理 | SDK要求非必需的敏感权限 | 某个推送SDK要求读取通讯录 |

## SDK尽职调查

### 选型阶段评估

| 评估维度 | 评估内容 | 评估方法 |
|---------|---------|---------|
| 数据收集范围 | SDK收集了哪些个人信息 | 审查SDK隐私政策和文档 |
| 数据使用目的 | SDK如何使用收集的数据 | 获取SDK数据处理说明 |
| 数据存储位置 | 数据存储在哪个区域 | 确认数据中心所在地 |
| 数据共享情况 | SDK是否共享数据给第三方 | 审查SDK的第三方合作方列表 |
| 安全能力 | SDK的安全防护措施 | 索取安全审计报告或认证 |
| 合规资质 | SDK提供商的合规背书 | 检查ISO 27701、SOC2等认证 |

### 合同要求

- 要求SDK提供商签署数据处理协议（DPA）
- 明确SDK的数据处理目的和范围
- 禁止SDK超出约定范围使用数据
- 规定SDK的安全防护标准和事件通报义务
- 约束SDK的子处理者管理
- 明确数据删除和归还的约定

## 数据共享协议

### DPA核心条款

| 条款 | 内容 |
|------|------|
| 处理目的 | 仅用于提供SDK功能必需的处理 |
| 数据范围 | 明确列举SDK可以处理的数据字段 |
| 处理限制 | 禁止SDK将数据用于自身商业目的（广告、画像等） |
| 安全措施 | SDK须采取的技术和组织安全措施标准 |
| 子处理者 | SDK更换子处理者需提前通知并获得同意 |
| 事件通报 | 安全事件发生后24-48小时内通报APP开发者 |
| 审计权 | APP开发者有权审计SDK的数据处理活动 |
| 删除义务 | 合同终止后SDK须删除所有数据的义务 |
| 赔偿责任 | 因SDK违规导致的损失由SDK提供商承担 |

## SDK更新管理

| 管理环节 | 要求 |
|---------|------|
| 更新通知 | SDK新版本发布前提前通知APP开发者 |
| 变更评估 | 评估SDK新版本的数据处理行为变化 |
| 隐私政策同步 | 更新APP隐私政策以反映SDK的新数据处理行为 |
| 合规模块测试 | 集成新版本后测试权限调用、数据采集行为 |
| 紧急更新 | 安全漏洞修复版本需优先处理 |

## SDK数据收集的告知同意

### 告知义务

- APP隐私政策中逐项披露集成的SDK信息
- 披露内容包括：SDK名称、SDK提供方、收集目的、收集字段、共享方式
- 建议使用表格形式呈现SDK清单

### SDK清单示例

| SDK名称 | 提供方 | 收集信息 | 使用目的 | 隐私政策链接 |
|---------|--------|---------|---------|-------------|
| 微信OpenSDK | 腾讯 | 设备标识、网络状态 | 微信登录、分享 | https://... |
| 阿里推送SDK | 阿里巴巴 | 设备标识、应用列表 | 推送通知 | https://... |
| 百度地图SDK | 百度 | 位置信息、设备信息 | 地图显示、定位 | https://... |

## SDK行为监控

- 使用网络抓包工具监控SDK的数据外传行为
- 使用权限检测工具检查SDK的权限调用记录
- 定期对比SDK的实际行为与隐私政策声明的差异
- 建立SDK异常行为预警机制
- 部署移动端隐私检测SDK实现实时监控
