# 权限管理合规

## 权限分类体系

### Android权限分类

| 权限级别 | 定义 | 示例 | 授权方式 |
|---------|------|------|---------|
| 正常权限（Normal） | 对用户隐私风险较低的权限 | 网络访问、蓝牙、NFC | 安装时自动授予 |
| 危险权限（Dangerous） | 可能访问用户私密数据的权限 | 位置、联系人、相机、麦克风 | 运行时动态请求 |
| 特殊权限（Special） | 系统级敏感权限 | 系统设置修改、无障碍服务 | 系统设置手动授权 |

### iOS权限分类

| 权限类型 | 示例 | 授权方式 |
|---------|------|---------|
| 敏感权限 | 位置、相机、麦克风、相册 | 首次使用弹窗授权 |
| 基础权限 | 通知、网络 | 安装后首次启动询问 |
| 推送权限 | 推送通知 | 授权后系统级管理 |

## 权限请求合规要求

### 请求时机

| 场景 | 合规要求 | 做法 |
|------|---------|------|
| APP启动时 | 不得启动即索权 | 遵循"即用即申请"原则 |
| 功能触发时 | 合理时机 | 仅当用户实际使用某项需要该权限的功能时请求 |
| 后台运行时 | 需有合理场景 | 导航APP后台位置权限 |
| 未使用状态下 | 禁止 | 不得在未使用相关功能时请求权限 |

### 弹窗设计合规要求

| 要素 | 要求 | 示例 |
|------|------|------|
| 权限用途说明 | 明确说明为何需要该权限 | "需要定位权限以显示你的位置" |
| 弹窗层级 | 先说明用途，再请求授权 | 自定义说明弹窗 → 系统授权弹窗 |
| 拒绝处理 | 拒绝时不得反复弹窗 | 首次拒绝 → 提示可通过设置开启 |
| 拒绝后果 | 说明不可用功能 | "定位功能不可用，无法显示地图" |
| 选项公正 | "永远允许"和"拒绝"选项平等 | 避免"仅在使用中"和"始终允许"等误导选项 |

### iOS的Purpose Strings

iOS 14+要求为每个权限提供详细的用途描述字符串，例如：

```
NSLocationWhenInUseUsageDescription - "此应用需要访问您的位置，以便为您提供附近的商家推荐。"
NSCameraUsageDescription - "此应用需要相机权限，用于拍摄和上传头像照片。"
```

## 非必要权限拒绝处理

| 权限 | 是否为核心功能必需 |
|------|-----------------|
| 位置（精确） | 导航类：必需/阅读类：非必需 |
| 相机 | 社交拍照类：必需/工具类：非必需 |
| 联系人 | 社交推荐类：可用非必需/工具类：非必需 |
| 麦克风 | 语音类：必需/其他：非必需 |
| 相册 | 头像上传类：非必需（可用系统选择替代） |
| 运动健身 | 健身类：必需/其他：非必需 |
| 日历 | 日历管理类：必需/其他：非必需 |

**合规要点**：非核心功能权限被拒绝后，APP不应闪退或停止运行，仅禁用对应功能

## 权限声明要求

- APP内需展示权限使用说明清单
- 清单需列明权限类型、用途、数据类型
- 支持用户查看权限使用记录（iOS 14+已系统支持）
- 系统设置的权限管理入口需清晰

## 权限撤销机制

| 平台 | 撤销方式 | 用户路径 |
|------|---------|---------|
| Android | 系统设置中手动关闭 | 设置 → 应用管理 → 权限管理 |
| iOS | 系统设置中手动关闭 | 设置 → 隐私 → 对应权限 |
| APP内 | 建议内置权限管理入口 | 我的 → 设置 → 权限管理 |

**注意**：PIPL要求撤回同意应当如同给予同意一样简便，APP内应提供"权限管理"入口，引导用户一键跳转到系统设置或直接在APP内开关权限。
