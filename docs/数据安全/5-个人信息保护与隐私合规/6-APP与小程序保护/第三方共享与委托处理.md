# 第三方共享与委托处理

## 基本概念区分

| 维度 | 委托处理（Processor） | 第三方共享（Third-party Sharing） | 共同控制（Joint Controller） |
|------|---------------------|-------------------------------|---------------------------|
| 决定权 | 委托方决定目的和方式 | 双方各自独立决定 | 共同决定目的和方式 |
| 告知义务 | 委托方履行告知 | 双方均有告知义务 | 共同安排告知义务 |
| 同意要求 | 委托方获取同意 | 需获单独同意（PIPL第23条） | 需获同意 |
| 法律责任 | 委托方承担主要责任 | 各自对违规行为负责 | 连带责任 |
| 典型场景 | 云服务、数据分析服务商 | 广告合作方、业务合作方 | 联合营销、数据合作项目 |

## 委托处理合规要求

### 委托合同要点（PIPL第21条）

| 条款 | 内容要求 |
|------|---------|
| 处理目的 | 明确约定受托人处理个人信息的目的 |
| 处理期限 | 约定合作期限和数据处理的起止时间 |
| 处理方式 | 明确数据处理的技术方式和手段 |
| 信息类型 | 列举委托处理的个人信息种类 |
| 保护措施 | 受托人应采取的安全技术措施要求 |
| 禁止行为 | 禁止受托人超出委托范围处理数据 |
| 转委托 | 受托人转委托需获得委托方同意 |
| 协助义务 | 受托人应协助委托方响应权利请求 |
| 合同终止 | 终止后删除或归还数据 |

### 受托人监督义务

- 定期对受托人的数据处理行为进行安全评估
- 要求受托人提供合规审计报告
- 监督受托人是否超出授权范围使用数据
- 确认受托人的安全防护措施持续有效

## 第三方共享合规要求（PIPL第23条）

### 告知内容

- 接收方的名称和联系方式
- 处理目的、处理方式
- 个人信息的种类
- 共享的必要性说明

### 单独同意要求

- 向第三方提供个人信息前，必须获得个人的单独同意
- 不得与用户协议、隐私政策一揽子捆绑同意
- 每次向不同第三方提供均需获取单独同意
- 同意记录需保存备查

### 第三方信息变更

- 第三方更改原有处理目的和方式的，应重新取得个人同意
- 共享协议更改也需重新获得同意
- 新增第三方类型时需补充告知

## 数据处理协议（DPA）

### 必备条款

| 条款 | PIPL要求 | GDPR要求（Art.28） |
|------|---------|-------------------|
| 数据处理范围 | 明确约定数据类型和目的 | 文档化处理指令 |
| 保密义务 | 约定保密要求 | 约束受托人保密 |
| 安全措施 | 约定技术要求 | 列举安全措施 |
| 子处理者 | 转委托须同意 | 须授权，提前通知 |
| 数据删除 | 终止后删除/归还 | 终止后删除/归还 |
| 审计权限 | 委托方有权审计 | 提供审计所需信息 |
| 跨境传输 | 应约定 | 需符合跨境规则 |
| 数据泄露通报 | 应及时通报 | 协助通知义务 |

## 子处理者管理

### 管理流程

1. 受托人（Processor）需事先获委托方授权方的子处理者名单
2. 添加子处理者需提前通知并获取委托方同意
3. 委托方可要求查看子处理者的安全能力证明
4. 子处理者的违约行为由主受托人承担完全责任
5. 委托方有权反对委托方使用不合格的子处理者

### 风险控制

- 与子处理者签订与本处理同等级的DPA
- 评估子处理者的数据安全能力（安全认证、审计报告）
- 限制子处理者的数据处理范围仅限于提供服务所需
- 确保子处理者遵守与委托方相同的安全标准
