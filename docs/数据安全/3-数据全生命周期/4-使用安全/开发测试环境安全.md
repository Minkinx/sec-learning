# 开发测试环境安全

## 概述

开发测试环境是数据泄露的高发区。使用生产数据做开发和测试虽能保证数据真实性，但会放大敏感数据暴露面。开发测试环境安全的核心在于用最小风险的数据支撑功能开发和测试工作。

## 合成数据生成 vs 数据子集

| 维度 | 合成数据 (Synthetic Data) | 数据子集 (Data Subsetting) |
|------|---------------------------|---------------------------|
| 数据来源 | 统计模型/生成式 AI 生成 | 从生产数据抽取子集 |
| 真实性 | 统计分布相似，非真实用户 | 真实数据子集 |
| 隐私风险 | 低（非真实用户数据） | 高（含真实信息） |
| 数据质量 | 取决于生成模型质量 | 保持原始数据特性 |
| 异常覆盖 | 可控制生成异常数据 | 取决于子集选择策略 |
| 成本 | 高（模型训练/部署） | 中（ETL 管道） |
| 适用场景 | 模型训练、大规模测试 | 功能测试、集成测试 |

### 合成数据生成技术

| 技术 | 说明 | 工具 |
|------|------|------|
| GAN (Generative Adversarial Network) | 生成对抗网络生成高保真数据 | CTGAN, DP-GAN |
| VAE (Variational Autoencoder) | 变分自编码器 | TVAE, Synthcity |
| 差分隐私合成 | DP-SGD 训练生成模型 | Google DP, OpenDP |
| 基于规则的生成 | 按业务规则生成 | Mockaroo, DataFaker |

### 数据子集策略

```yaml
子集选择策略:
  随机抽样: 按比例随机抽取 (风险: 覆盖不全)
  分层抽样: 按业务类别分层抽取 (覆盖各场景)
  条件过滤: 按条件筛选 (如: 近 30 天活跃用户)
  行级子集: 限制数据行数 (用于功能测试)
  列级子集: 仅包含测试必需字段 (减少敏感字段)
```

## TDM (Test Data Management)

TDM 是开发测试数据的全生命周期管理框架：

### TDM 核心流程

```
生产库 → 数据抽取 → 脱敏/匿名 → 子集化 → TDM 资产库
                                                   ↓
开发环境 ←──────────────────────────────────────────┘
测试环境 ←── 自动布署（API 调用或 CI/CD 触发）────┘
性能测试 ←── 性能数据集（保留数据分布特征）─────┘
```

### TDM 平台功能

| 功能 | 说明 | 实现方式 |
|------|------|----------|
| 数据发现 | 自动识别敏感字段和数据关系 | Data Discovery + ML |
| 脱敏处理 | 内置多种脱敏算法 | 掩码/置换/加密 |
| 关系保持 | 跨表外键关系保持 | Referential Integrity Engine |
| 子集化 | 灵活的切片/取样策略 | 配置化规则引擎 |
| 自动化部署 | CI/CD 集成自动刷新测试数据 | API + CLI |
| 数据版本管理 | 可回滚到指定数据版本 | Git-like 版本控制 |
| 合规审计 | 全流程记录 | 审计日志 |

## 非生产环境数据脱敏

### 脱敏策略矩阵

| 数据类型 | 脱敏策略 | 开发环境 | 测试环境 | 性能测试 |
|----------|----------|----------|----------|----------|
| 姓名 | 随机姓名替换 | 是 | 是 | 随机替换 |
| 手机号 | 随机生成 | 是 | 是 | 格式保持 |
| 身份证 | FPE 加密 | 是 | 是 | FPE |
| 地址 | 截断到城市 | 是 | 是 | 范围泛化 |
| 邮箱 | 随机别名 | 是 | 是 | 格式保持 |
| 金额 | 随机偏移 ±5% | 是 | 可选 | 分布保持 |
| 日期 | 相对偏移 | 是 | 是 | 偏移保持间隔 |

### 脱敏管道架构

```
生产数据库
  ↓
数据抽取 (CDC / Batch)
  ↓
脱敏引擎:
  ├── 确定性加密 (FPE, AES-GCM-SIV)
  ├── 随机置换 (Shuffling)
  ├── 统计扰动 (DP noise injection)
  └── 泛化处理 (Generalization)
  ↓
数据子集化:
  ├── 行级切片 (WHERE 条件)
  ├── 列级选择 (仅必需字段)  
  └── 关系保持 (外键一致性)
  ↓
TDM 资产存储 (脱敏数据湖)
```

## 数据刷新策略

### 刷新频率决策

```yaml
刷新频率:
  开发环境:
    - 频率: 每周一次
    - 触发: 周五晚自动刷新
    - 数据量: 一周的变更数据 + 全量子集
  
  测试环境:
    - 频率: 每次大版本发布前
    - 触发: CI/CD pipeline
    - 数据量: 根据测试用例需求
  
  性能测试:
    - 频率: 每季度或按需
    - 数据量: 生产级别数据量 (模拟)
    - 特殊要求: 保留数据分布特征
```

### 增量刷新 vs 全量刷新

| 维度 | 增量刷新 | 全量刷新 |
|------|----------|----------|
| 时间 | 分钟级 | 小时级 |
| 数据一致性 | CDC 同步 | 快照恢复 |
| 资源消耗 | 低 | 高 |
| 回滚复杂度 | 高（需处理冲突） | 低（快照回滚） |
| 适用场景 | 频繁小变更 | 版本发布/大变更 |

## 数据库克隆安全

### 克隆技术

| 技术 | 原理 | 速度 | 存储消耗 |
|------|------|------|----------|
| Snapshot Clone | ZFS/Btrfs 快照 | 秒级 | 仅变更数据 |
| Thin Clone | 共享 + Copy-on-Write | 秒级 | 按需分配 |
| Full Clone | 完整数据复制 | 分钟级 | 完整大小 |
| DBMS Clone | 数据库原生克隆 (Oracle/MSSQL) | 分钟级 | 仅变更数据 |

### 克隆后安全措施

```bash
# 克隆后进行安全加固
1. 变更数据库密码
2. 禁用生产环境访问权限
3. 删除（脱敏）敏感数据
4. 启用 DDM 脱敏策略
5. 启用审计日志
6. 限制网络访问
7. 设置数据过期时间
8. 启动自动清理脚本

# 示例：克隆后脱敏脚本
# psql -d cloned_db -c "UPDATE users SET ssn = 'MASKED-' || id;"
# psql -d cloned_db -c "UPDATE users SET phone = 'MASKED-' || id;"
```
