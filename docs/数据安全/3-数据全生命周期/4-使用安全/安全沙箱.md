# 安全沙箱

## 概述

安全沙箱为数据分析提供隔离的执行环境，确保数据在受控范围内被安全使用。在隐私合规要求日益严格的背景下，沙箱技术成为平衡数据价值释放与安全保护的关键基础设施。

## 沙箱类型

### 数据沙箱 (Data Sandbox)

数据沙箱针对特定数据集提供受限访问环境：

| 特性 | 说明 |
|------|------|
| 数据集范围 | 预先划定的数据子集（表/行/列级） |
| 访问控制 | 基于角色的数据权限 |
| 操作限制 | 禁止导出、禁止 Join 非授权表 |
| 环境隔离 | 数据库 Schema 级别隔离 |
| 典型用途 | 分析师查询特定数据集 |

### 研究沙箱 (Research Sandbox)

研究沙箱为数据科学团队提供灵活的分析环境：

```
[Analytica Workspace] ←→ [Researcher Sandbox]
     │                           │
     │  ┌─────────────────────┐  │
     │  │ 脱敏数据子集          │  │
     │  │ Python/R/Notebook    │  │
     │  │ 预装分析工具包        │  │
     │  │ 预装隐私保护库        │  │
     │  └─────────────────────┘  │
```

### 分析沙箱 (Analytics Sandbox)

| 维度 | 数据沙箱 | 研究沙箱 | 分析沙箱 |
|------|----------|----------|----------|
| 用户 | 业务分析师 | 数据科学家 | 外部合作伙伴 |
| 数据范围 | 预定义数据集 | 宽表数据集 | 受限视图 |
| 计算能力 | 标准 SQL | Spark/Python | SQL + 受限 Python |
| 导出控制 | 禁止 | 审批制 | 脱敏后审批 |
| 审计粒度 | 查询级 | 代码级 | 操作级 |

## 隔离技术

### 环境隔离层级

| 层级 | 技术 | 隔离程度 | 适用场景 |
|------|------|----------|----------|
| 数据库 Schema | 不同 Schema 空间 | 弱 | 数据沙箱 |
| 虚拟机 | VirtualBox/VMware | 强 | 研究沙箱 |
| 容器 | Docker + 资源限制 | 中 | 分析沙箱 |
| Kubernetes | Namespace + Network Policy | 中 | 企业级沙箱 |
| TEE | Intel SGX/AMD SEV | 最强 | 高敏数据 |
| 裸机 | 物理隔离 | 最强 | 最高安全 |

### 容器级隔离最佳实践

```yaml
# Docker 沙箱安全配置
security_opt:
  - no-new-privileges:true   # 禁止提权
  - seccomp:default.json      # 系统调用过滤

cap_drop:
  - ALL                        # 移除所有内核能力

read_only: true                # 只读根文件系统

tmpfs:
  - /tmp:size=100M,noexec,nosuid  # 临时文件无执行权限

ulimits:
  - nofile=1024                  # 文件描述符限制
  - nproc=100                    # 进程数限制
```

## 数据进出控制 (Ingress/Egress)

### Ingress 控制

```
数据进入沙箱前的检查:
  1. 数据范围确认（是否在授权范围内）
  2. 数据脱敏状态检查（是否仍需脱敏）
  3. 数据量限制（是否超出配额）
  4. 数据使用期限检查（是否在有效期内）
  
实现:
  - 数据预览（先查看样本再决定全量导入）
  - 自动数据脱敏（Ingress 管道集成 DDM）
  - 数据分类标记（预置 sensitivity label）
```

### Egress 控制

```
数据导出沙箱的控制机制:
  1. 导出审批: 多人审批 + 理由说明
  2. 数据量限制: 单次/日/周导出上限
  3. 脱敏检查: 自动识别敏感字段，强制脱敏
  4. 水印注入: 自动嵌入接收方水印
  5. 记录审计: 全量导出日志

自动脱敏规则:
  导出格式      脱敏操作
  CSV         自动遮蔽 SSN/Phone/Email
  Excel       单元格级脱敏
  JSON        字段级脱敏 + 截断
  PDF         页面级水印 + 文档水印
```

## TEE-Based 沙箱

TEE (Trusted Execution Environment) 沙箱提供硬件级隔离：

```
数据源 ──→ TEE Sandbox (Enclave)
            ├── Enclave A (Analyst 1)
            ├── Enclave B (Analyst 2)
            └── Enclave C (External Partner)
               └── 数据在 Enclave 内解密 + 计算
                   → 仅输出聚合结果
                   → 原始数据不出 Enclave
```

### TEE 沙箱优势

| 优势 | 说明 |
|------|------|
| Hardened Isolation | CPU 级内存隔离，OS/VM/管理程序不可见 |
| 远程证明 (Remote Attestation) | 验证 Enclave 是否被篡改 |
| 代码完整性 | Enclave 代码被测量和签名 |
| 数据机密性 | 内存加密，外部无法读取 |
| 适用场景 | 隐私计算、多方数据联合分析 |

## 隐私保护分析场景

### 典型用例

```yaml
医疗研究:
  数据: 患者病历 (PHI 数据)
  沙箱策略:
    - 仅允许聚合查询 (COUNT/AVG/SUM)
    - 禁止导出个体级数据
    - 查询结果 ddifferential privacy 扰动
    - 所有查询记录审计

金融风控:
  数据: 交易记录 + 用户画像
  沙箱策略:
    - 仅允许特定模型训练
    - 模型导出需审批
    - 含模型反转攻击检测
    - 训练数据不出沙箱
```
