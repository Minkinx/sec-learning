# 数据水印技术

## 概述

数据水印技术通过在数据中嵌入不可见的标识信息，实现数据泄露溯源和版权保护。在数据共享和数据交换场景中，水印技术是追责的关键技术手段，能够在不破坏数据可用性的前提下提供事后追踪能力。

## 可见水印 vs 不可见水印

| 维度 | 可见水印 (Visible) | 不可见水印 (Invisible) |
|------|--------------------|----------------------|
| 感知性 | 肉眼可识别 | 嵌入在数据内部 |
| 鲁棒性 | 低（可被裁剪） | 高（不易移除） |
| 目标 | 威慑 / 版权声明 | 隐藏式溯源追踪 |
| 对数据影响 | 较大（遮挡美学） | 较小（统计扰动） |
| 应用场景 | 图片/视频版权标记 | 数据泄露溯源 |
| 举证难度 | 低（直接可见） | 高（需算法提取） |
| 反规避 | 容易（截图裁剪） | 困难（需深度分析） |

## 数字水印技术 (多媒体)

### LSB (Least Significant Bit)

LSB 水印通过修改图像/音频/视频像素的最低有效位来嵌入信息：

```
像素原始值:  10101100  11010010  01111001
嵌入水印后:  10101101  11010011  01111000
              ↑LSB翻转  ↑LSB翻转  ↑LSB翻转
```

**特点**：
- 嵌入容量大（每像素 1-3 位）
- 对图像质量影响极小
- **弱点**：易被压缩/去噪破坏
- **增强**：使用扩频技术分散嵌入

### DCT (Discrete Cosine Transform)

DCT 域水印在频域嵌入信息，鲁棒性更强：

```
嵌入流程:
1. 将图像分块 (8×8 或 16×16)
2. 对每个块执行 DCT 变换
3. 在中频系数中嵌入水印位
4. 执行逆 DCT 恢复图像

提取流程:
1. 对图像分块并执行 DCT
2. 从中频系数提取水印位
3. 与原始水印比对
```

**JPEG 友好**：DCT 域水印在 JPEG 压缩后仍可提取。

### Spread Spectrum (扩频水印)

将水印信号扩展至整个频域，类似通信领域的 DSSS：

| 参数 | 说明 | 典型值 |
|------|------|--------|
| Chip Rate | 每个信息位对应的 chip 数 | 1000 chips/bit |
| Carrier Frequency | 嵌入频带范围 | 中频段 (DCT 系数) |
| Pseudorandom Sequence | PN 码序列 | Gold code 或 m-sequence |
| Detection Threshold | 提取检测门限 | 3σ (99.7% 置信度) |

## 文本水印技术

### Entity Swapping

在文本数据的结构化字段中嵌入水印，通过同义词/同义表达替换：

```yaml
原始文本: "张三参加了2024年5月在北京举行的安全会议"
水印版: 
  方案 1 (同义词替换): "张三先生参与了2024年5月于首都北京举办的安全会议"
  方案 2 (标点替换): "张三、参加了2024年5月在北京举行的安全会议"
```

| 替换类型 | 示例 | 水印容量 |
|----------|------|----------|
| 同义词 | 参加→参与, 和→与, 在→于 | 0.5-1 bit/词 |
| 标点替换 | 逗号→顿号, 句号→分号 | 0.3 bit/句 |
| 字序调换 | 左右→右左 (地名/人名) | 1 bit/处 |
| 格式调整 | 空格数、缩进 | 0.5 bit/行 |

### Syntax Structure Watermarking

通过调整句子结构嵌入信息：

```
原始: 主语 + 谓语 + 宾语式
水印: 状语 + 谓语 + 主语 + 宾语式

示例:
原始: "安全团队检测到了数据泄露"
水印: "经过持续监控，数据泄露被安全团队检测到"
```

### 结构化数据水印

在数据库记录中嵌入水印，适用于数据泄露溯源：

```
ID     Name     Phone           Email                Dept
101    张三     138****0000      zhangs@example.com   IT
102    李四     139****1111      lisi@example.com     HR
103    王五     137****2222      wangw@example.com    Finance

水印嵌入策略:
  - 数值列: 最后一位 ±1 (修改 5% 记录)
  - 字符串列: 末尾空白差异
  - 日期列: 毫秒部分调整
  - Email: 用户名别名 (gmail + alias)
```

## 数据泄露溯源

### 溯源流程

```
1. 数据分发前为每个接收方嵌入唯一水印
2. 当疑似泄露数据出现时，提取水印
3. 匹配水印与分发记录 → 确定泄露来源
4. 提供法律证据

水印分配:
  接收方 A: UserID=101 → 水印 W101
  接收方 B: UserID=102 → 水印 W102
  接收方 C: UserID=103 → 水印 W103
  
泄露数据中提取到: W102 → 追溯到 B
```

### 水印容量与鲁棒性平衡

| 需求 | 策略 | 权衡 |
|------|------|------|
| 大量接收方 | 使用较长水印序列 | 数据修改更多 |
| 高鲁棒性 | 冗余编码 (Reed-Solomon) | 嵌入容量降低 |
| 低检测率 | 使用更强的嵌入算法 | 性能开销 |
| 数据质量 | 最小化修改 | 水印可能被清洗 |
