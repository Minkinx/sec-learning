# 脱敏采集技术

## 概述

脱敏采集是指在数据进入采集管道时即对敏感信息进行遮蔽或转换，确保数据在后续存储和处理环节中降低敏感度。脱敏时机和方式的选择直接影响安全效果与业务可用性。

## 采集前脱敏 vs 采集后脱敏

| 维度 | 采集前脱敏 (Pre-collection) | 采集后脱敏 (Post-collection) |
|------|-----------------------------|------------------------------|
| 处理位置 | 客户端 / 采集端点 | 服务端采集管道 |
| 优势 | 原始数据不进入网络 | 可保留原始数据用于审计 |
| 风险 | 不可逆，数据质量受损 | 原始数据暴露窗口 |
| 适用场景 | 第三方数据采集 | 内部数据采集系统 |
| GDPR 合规 | 强 (data protection by design) | 需额外控制措施 |

## 采集点动态脱敏 (Dynamic Masking at Collection Point)

在 API Gateway 或采集代理层实施动态脱敏：

```
Client → API Gateway (动态脱敏) → Internal Service (脱敏后数据)
                                → Audit Log (原始数据, 受限访问)
```

实现方案：
- **基于字段注解**：在 API 定义中标注敏感字段，gateway 自动脱敏
- **配置驱动**：通过脱敏规则引擎（如 OpenPolicyAgent）动态应用策略
- **上下文感知**：根据调用方角色动态调整脱敏强度

## 可逆脱敏 vs 不可逆脱敏

### 不可逆脱敏 (Irreversible)

| 技术 | 说明 | 适用类型 |
|------|------|----------|
| 哈希 (SHA-256 + Salt) | 单向散列 | ID、用户名 |
| 截断 (Truncation) | 保留部分字符 | 信用卡号前 6 后 4 |
| 令牌化 (Tokenization) | 替换为无意义 token | 支付卡号 (PCI DSS) |
| 泛化 (Generalization) | 降低精度 | 年龄→年龄段，地址→城市 |

### 可逆脱敏 (Reversible)

| 技术 | 说明 | 安全要求 |
|------|------|----------|
| 格式保持加密 (FPE) | 密文保持原格式 | 密钥管理必须完善 |
| 确定性加密 | 相同明文→相同密文 | 易受频率分析攻击 |
| 可调加密 (Tweakable Encryption) | 加入 tweak 参数 | 防频率分析 |

## Format-Preserving Encryption (FPE) 在采集管道中的应用

FPE 确保加密后的数据格式与原始数据一致（如信用卡号 16 位数字），在不破坏下游系统格式要求的前提下实现脱敏：

```
原始: 6222-1234-5678-9012
FPE:  4398-7654-3210-8765 (仍为 16 位数字, 符合 Luhn 算法)
```

### FPE 算法选型

| 算法 | 标准 | 特点 |
|------|------|------|
| FF1 | NIST SP 800-38G | 支持任意 alphabet, 推荐使用 |
| FF3-1 | NIST SP 800-38Gr1 | 更高效，但最小明文长度限制 |
| VAES3 | 非标准 | 仅用于 legacy 系统迁移 |

### 采集管道中的 FPE 部署

```
采集端点 → FPE 加密 → 传输 → 存储 (密文)
                        ↓
                  密钥服务器 (HSM/KMS)
```

- 加密在采集代理层执行，密钥从 KMS 获取
- 使用 per-field 独立密钥防止关联分析
- 加密上下文中记录算法版本和 key ID 以便轮换

## 参考标准

- PCI DSS Requirement 3.4 (Render PAN unreadable)
- ISO/IEC 27038 (Redaction of sensitive information)
