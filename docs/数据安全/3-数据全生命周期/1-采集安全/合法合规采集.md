# 合法合规采集

## 概述

数据采集的合法合规是数据安全治理的基石。在全球监管趋严的背景下，企业必须在业务需求与合规要求之间取得平衡，确保数据采集活动具备合法的法律基础。

## 告知同意要求

根据《个人信息保护法》和 GDPR，告知同意需满足以下要素：

| 要素 | 说明 | 检查要点 |
|------|------|----------|
| 自主性 | 用户自愿作出，不受胁迫 | 拒绝不影响基本服务 |
| 具体性 | 针对特定处理目的 | 禁止一揽子授权 |
| 知情性 | 充分了解处理方式与范围 | 隐私政策清晰易懂 |
| 明确性 | 通过 affirmative action 作出 | 禁止默认勾选 |

## 最小必要原则

- **目的关联性**：采集的数据必须与处理目的直接相关，禁止为了"将来可能需要"而过度采集。
- **数据最小化**：仅采集实现特定目的所必需的最少数据类型和数量。
- **存储期限最小化**：设置数据保留期限，到期自动删除或匿名化。
- **定期审查机制**：每半年审查一次采集字段的必要性，移除不再必需的字段。

## Legitimate Interest 平衡测试

当企业以 legitimate interest 作为处理基础时，需完成三要素测试：

1. **目的测试（Purpose Test）**：处理目的是否 legitimate？
2. **必要性测试（Necessity Test）**：该处理是否为实现目的所必要？
3. **平衡测试（Balancing Test）**：处理对用户权利的影响是否与利益相称？

## Consent Management Platform (CMP)

CMP 是管理用户同意状态的核心基础设施：

- **功能要求**：偏好采集、同意记录存取、实时更新、多语言支持
- **技术架构**：通常以 Cookie Banner / Preference Center 形式嵌入
- **合规记录**：每次同意变更均需记录时间戳、用户标识、同意范围
- **主流方案**：OneTrust、Cookiebot、TrustArc

## 采集范围文档化

通过 ROPA（Record of Processing Activities）管理采集活动：

```
采集活动 ID: DPIA-2024-001
处理目的: 用户行为分析用于产品优化
采集字段: user_id, session_id, page_path, event_type
法律基础: 同意 (Article 6(1)(a))
保留期限: 26 个月
数据流向: Web Server → Kafka → Data Lake → Analytics DB
```

## Audit Trail 要求

- 记录每次采集请求的元数据（时间、来源、scope）
- 同意状态的变更历史需保留至少 3 年
- 定期进行合规审计，保留审计报告
- 向监管机构证明合规的能力（accountability）
