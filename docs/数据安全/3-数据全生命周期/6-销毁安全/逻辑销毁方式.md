# 逻辑销毁方式

## 概述

逻辑销毁通过软件方式使存储数据不可恢复，适用于需要保留介质继续使用或无法物理销毁的场景。与物理销毁相比，逻辑销毁更具成本效益和灵活性，但需要更高的技术验证标准。

## 覆写 (Overwriting)

### DoD 5220.22-M 标准

DoD 5220.22-M 定义了三种覆写方案：

```yaml
DoD 5220.22-M 三遍覆写:
  Pass 1: 写 0x00 (全零)
  Pass 2: 写 0xFF (全一)  
  Pass 3: 写 随机字符
  验证: 最后一次覆写后验证

安全性考量:
  - 对于现代 HDD (高密度, PRM 技术), 3 遍覆写已经充分
  - 单次覆写对于现代 HDD 理论上已足够
  - "数据剩余"在实验室条件下也无法恢复
```

### Gutmann 算法

Gutmann 算法（Peter Gutmann, 1996）定义了 35 遍覆写：

```
Pass 1-4:    随机字符
Pass 5-31:   特定编码模式 (MFM/RLL 编码相关)
Pass 32-35:  随机字符

实际应用:
  - 35 遍覆写对现代 HDD **过度保守**
  - 现代 HDD 的高密度使得前几遍覆写已足够
  - 除非超高安全要求，推荐使用 3-7 遍覆写
```

### 覆写方案比较

| 标准 | 次数 | 模式 | 适用场景 | 验证 |
|------|------|------|----------|------|
| 单次覆写 | 1 | 全零/随机 | 低敏感度, 内部重用 | 读校验 |
| NIST SP 800-88 Clear | 1 | 全介质全零/全一 | 一般敏感度 | 读校验 |
| DoD 5220.22-M (3 pass) | 3 | 0x00 → 0xFF → Random | 中敏感度 | Pass 3 后校验 |
| DoD 5220.22-M (7 pass) | 7 | 交替 0x00/0xFF + Random | 高敏感度 | 完成校验 |
| Gutmann (35 pass) | 35 | 随机 + 模式序列 | 极高标准 | 完成校验 |
| NIST SP 800-88 Purge | 1 + crypto | 覆写 + 加密擦除 | 现代 HDD 推荐 | 加密密钥清除 |

## Cryptographic Erasure (加密擦除)

加密擦除通过销毁加密密钥使数据不可解密，无需覆写所有数据。

### 加密擦除原理

```
加密存储架构:
  数据: AES-256-GCM 加密存储
  密钥: DEK (Data Encryption Key) 加密后存储
  主密钥: KEK (Key Encryption Key) — 存储在 KMS/HSM
  
加密擦除流程:
  1. 定位数据加密时使用的 DEK/KEK
  2. 销毁 KEK (在 HSM/KMS 中删除)
  3. DEK 被加密保护，KEK 移除后 DEK 无法解密
  4. 所有加密数据变为不可恢复的密文
  
关键优势:
  - 瞬时完成 (毫秒级)
  - 无论数据量多大，时间恒定
  - 不依赖介质类型 (HDD/SSD/云存储适用)
```

### 加密擦除 vs 覆写

| 维度 | 加密擦除 | 覆写 |
|------|----------|------|
| 时间 | 毫秒级 | 小时级 (TB 级数据) |
| 介质影响 | 无 | 缩短 HDD/SSD 寿命 |
| 数据量影响 | O(1) 恒定时间 | O(n) 线性时间 |
| 验证难度 | 验证密钥已销毁 | 需要读校验 |
| 认可度 | NIST SP 800-88 认可 | 广泛认可 |
| 适用云环境 | 是 | 通常不可行 |
| 前提条件 | 数据加密存储 | 直接访问介质 |

### 加密擦除的挑战

| 挑战 | 说明 | 缓解措施 |
|------|------|----------|
| 密钥副本 | 密钥可能被备份 | 追踪所有密钥副本 |
| 密钥层级 | KEK/DEK 多层架构 | 确保销毁最根密钥 |
| DEK 持久化 | DEK 与数据共存 | 毁掉 DEK 的密钥包装器 |
| KMS 备份还原 | 密钥从备份中恢复 | 删除 KMS 备份 |
| 合规接受度 | 部分法规不认可 | 确认当地法规要求 |

## Secure Erase / NVMe Format

### ATA Secure Erase

ATA Secure Erase 是 HDD 硬件原生支持的安全擦除指令：

```
流程:
  1. 设置 HDD 密码 (Security Set Password)
  2. 发送 Secure Erase 命令
  3. HDD 内部触发数据擦除机制
  4. 自动清除密码
  
实现:
  hdparm --user-master u --security-set-pass p /dev/sdX
  hdparm --user-master u --security-erase p /dev/sdX

验证:
  hdparm -I /dev/sdX | grep "not frozen"
  # 确认 Security: enabled → not enabled
```

### NVMe Format / Sanitize

NVMe SSD 支持更高级的格式和清理命令：

| 命令 | 类型 | 说明 | 时间 |
|------|------|------|------|
| NVMe Format | NVM Format | 逻辑块格式化 | 秒级 |
| NVMe Sanitize - Block Erase | Sanitize | 全块物理擦除 | 秒级 |
| NVMe Sanitize - Overwrite | Sanitize | 全芯片覆写 | 分级 (1-5 min) |
| NVMe Sanitize - Crypto Erase | Sanitize | 加密擦除 (SSD 内置加密) | 毫秒级 |

```bash
# NVMe Sanitize (Block Erase)
nvme sanitize /dev/nvme0 -a 0x02 --no-dealloc

# NVMe Format
nvme format /dev/nvme0 --lbaf=0 --ses=2

# 查看 sanitize 状态
nvme sanitize-log /dev/nvme0
```

### Secure Erase 注意事项

| 注意点 | 说明 |
|--------|------|
| 硬盘冻结 (Frozen) | BIOS/OS 可能锁定 Security 状态，需热插拔或进睡眠 |
| SSD 特殊性 | ATA Secure Erase 对 SSD 可能不充分（OP 保留空间） |
| NVMe namespace | 需确认所有 namespace 已被清理 |
| 密码清除 | Secure Erase 完成后自动清除密码 |
| SED (自加密硬盘) | 执行 Crypto Erase 而非 Secure Erase |

## Block-Level vs File-Level

| 维度 | Block-Level | File-Level |
|------|------------|------------|
| 覆写范围 | 整个存储设备/分区 | 单个文件/目录 |
| 速度 | 与存储容量相关 | 与文件大小相关 |
| 有效性 | 高（覆写所有扇区） | 中（可能遗留临时文件/碎片） |
| 元数据残留 | 无 | 文件名/时间戳可能保留 |
| 适用场景 | 设备退役/重用 | 文件安全删除 |
| 工具 | dd, shred -z (block), hdparm | shred, srm, wipe |

### File-Level 安全删除

```bash
# shred 安全删除文件 (覆写 + 截断)
shred -v -z -n 3 sensitive_file.dat

# 参数说明:
# -v: 显示进度
# -z: 最后一遍以零覆盖（隐藏覆写痕迹）
# -n 3: 3 遍随机覆写
# -u: 删除后截断并移除文件
```

### Block-Level 示例

```bash
# 全盘覆写
dd if=/dev/urandom of=/dev/sdX bs=1M status=progress

# NIST SP 800-88 Clear (单次全零覆写)
dd if=/dev/zero of=/dev/sdX bs=1M status=progress

# 验证覆写
dd if=/dev/sdX bs=1M status=progress | xxd | head -20
```

## 软件工具对比

| 工具 | 平台 | 覆写标准 | 加密擦除 | 校验 | 适用场景 |
|------|------|----------|----------|------|----------|
| shred | Linux | 可配置 (默认 3 pass) | 否 | 否 | 文件级安全删除 |
| dd | Linux/Unix | N/A (底层写) | 否 | 否 | 块级覆写 |
| hdparm | Linux | ATA Secure Erase | 否 | 否 | HDD 安全擦除 |
| nvme-cli | Linux | NVMe Format/Sanitize | 是 (Crypto Erase) | 是 | NVMe SSD 擦除 |
| srm | Linux/macOS | Gutmann/DoD/US DoE | 否 | 否 | 文件级安全删除 |
| DBAN (Darik's Boot) | Bootable | 多种标准 | 否 | 是 | 启动级全盘擦除 |
| Eraser | Windows | 多种标准 | 否 | 否 | Windows 文件/文件夹 |
| Disk Utility | macOS | 覆写 (1-7 pass) | 否 | 是 | macOS 磁盘工具 |
