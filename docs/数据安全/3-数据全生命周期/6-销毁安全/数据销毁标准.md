# 数据销毁标准

## 概述

数据销毁是数据全生命周期的最终环节。合规的数据销毁不仅需要选择正确的销毁技术，还需要遵循标准化流程、保留销毁证据链、满足监管审计要求。NIST SP 800-88 是目前最广泛采用的数据销毁标准。

## NIST SP 800-88 标准 (Clear/Purge/Destroy)

NIST SP 800-88 Rev.1 将数据销毁分为三个层级：

| 层级 | 英文术语 | 中文 | 说明 | 目标 |
|------|----------|------|------|------|
| 1 | Clear | 清除 | 通过软件方式覆写数据 | 防止简单数据恢复 |
| 2 | Purge | 净化 | 通过硬件或软件方式使数据不可恢复 | 防止实验室级恢复 |
| 3 | Destroy | 销毁 | 物理破坏存储介质 | 完全物理破坏 |

### Clear (清除)

Clear 级别对存储介质进行逻辑覆写，防止使用常见文件恢复工具恢复数据：

```
Clear 的适用条件:
  - 介质在同一组织内部重用
  - 数据敏感度较低
  - 具备覆写能力且不影响介质寿命
  - 需配合验证流程

不适用场景:
  - 介质将离开组织控制
  - 数据涉及商业秘密或国家安全
  - 合规要求 Purge 或 Destroy
```

### Purge (净化)

Purge 级别确保即使使用实验室级技术（如电子显微镜、磁力显微镜）也无法恢复数据：

```
Purge 实现方式:
  1. 多轮次覆写 (如 DoD 5220.22-M)
  2. Cryptographic Erasure (加密擦除)
  3. 安全擦除指令 (ATA Secure Erase / NVMe Format)
  4. 消磁 (Degaussing)
```

### Destroy (销毁)

Destroy 级别是物理销毁介质，完全不可逆：

```
物理销毁方法:
  1. 粉碎 (Shredding) — 介质粒度 < 2mm
  2. 焚烧 (Incineration) — 1400°C+ 
  3. 熔融 (Melting) — 金属/半导体
  4. 化学溶解 (Chemical Dissolution) — 酸解
```

## ISO 27001 销毁要求

ISO 27001:2022 Annex A 对数据销毁的要求：

### A.8.10 信息移除

```
控制项: 当不再需要时, 应移除信息系统中的信息和资产。

实施指南:
  a) 制定信息保留和销毁策略
  b) 对各类信息分类定义保留期限
  c) 实施安全的销毁程序
  d) 在资产转让或报废时确保数据清除
  e) 保留销毁记录作为合规证据
  f) 定期审查销毁策略的有效性
```

### A.8.11 物理介质安全处置

```
控制项: 使用正式的处置程序安全处置不再需要的介质。

实施要求:
  - 介质处置前获得授权
  - 建立介质分类与处置方法映射表
  - 外包处置需签订保密协议
  - 获取处置完成证书
  - 保留处置审计跟踪
```

## 数据清理决策矩阵

| 介质类型 | 清除 (Clear) | 净化 (Purge) | 销毁 (Destroy) |
|----------|-------------|-------------|----------------|
| HDD (磁介质) | 单次全盘覆写 | ATA Secure Erase + 多次覆写 | 粉碎 / 消磁 |
| SSD (NAND) | NVMe Format (增强) | NVMe Sanitize + Block Erase | 粉碎 / 焚烧 |
| 磁带 | 覆写 | 消磁 | 粉碎 / 焚烧 |
| 光盘 (CD/DVD) | — | 数据不可删除特性 | 粉碎 |
| RAM | 断电清除 | — | 物理破坏 |
| 云存储 (逻辑) | 对象删除 + 验证 | 加密擦除 + 资源终止 | — |
| 纸张 | — | — | 粉碎 (4mm × 40mm) |
| 软盘 | 覆写 | 消磁 | 粉碎 |

### 介质敏感度矩阵

```yaml
低敏感度数据 (Internal):
  Clear 级别足够
  - 单次覆写 (全零或随机数据)
  - 验证: 抽样校验

中敏感度数据 (Confidential):
  Purge 级别
  - ATA Secure Erase 或多次覆写 (3 passes)
  - 验证: 全量校验

高敏感度数据 (Restricted):
  Destroy 级别
  - 物理粉碎 / 消磁 / 焚烧
  - 验证: 链式监护记录
```

## 合规认证 (Certification)

销毁完成后需出具合规认证文件：

### Destruction Certificate

```
[组织名称] 数据销毁认证证书

证书编号: DC-2024-00152
日期: 2024-05-15

介质清单:
  - HDD x 50 (序列号: WD-202405-001 ~ WD-202405-050)
  - SSD x 20 (序列号: SA-202405-001 ~ SA-202405-020)
  - 磁带 x 30 (标签: TAPE-202405-001 ~ TAPE-202405-030)

销毁方法: 
  - HDD/SSD: 工业粉碎机 (粒度 < 1mm)
  - 磁带: 消磁 + 焚烧

见证人: [姓名], [职位]
销毁人: [姓名], [职位]
审核人: [姓名], [职位]

签名: _________________  日期: _______________
```

### 审计追踪要求

| 记录项 | 说明 | 保留期限 |
|--------|------|----------|
| 资产处置申请 | 销毁原因和授权 | 5 年 |
| 介质清单 | 序列号、类型、容量 | 5 年 |
| 销毁方法 | 使用技术和工具 | 5 年 |
| 执行记录 | 时间、地点、执行人 | 5 年 |
| 验证结果 | 销毁验证测试报告 | 5 年 |
| 证书 | 第三方/内部签发的销毁证书 | 永久 |
