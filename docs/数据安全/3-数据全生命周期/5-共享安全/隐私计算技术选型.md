# 隐私计算技术选型

## 概述

隐私计算（Privacy-Preserving Computation）是一组在保护数据隐私的前提下实现数据价值流通的技术集合。联邦学习（FL）、安全多方计算（MPC）、可信执行环境（TEE）、同态加密（HE）各有优劣。技术选型需要综合安全模型、性能、数据特性和部署环境等因素。

## FL / MPC / TEE / HE 多维对比

| 维度 | 联邦学习 (FL) | 安全多方计算 (MPC) | 可信执行环境 (TEE) | 同态加密 (HE) |
|------|---------------|-------------------|-------------------|---------------|
| 安全模型 | 原始数据不出本地 | 密码学保证 | 硬件信任根 | 密码学保证 |
| 信任假设 | 诚实参与者 | Honest/Dishonest Majority | 信任 CPU 厂商 | 无需信任第三方 |
| 计算精度 | 模型参数精度 | 精确计算 | 精确计算 | 近似精度 |
| 性能 | 中（通信密集） | 差（高通信/计算） | 优（接近明文） | 差（计算开销高） |
| 数据规模 | 大（分布式） | 小（结构化数据） | 中到大型 | 小到中型 |
| 安全类型 | 梯度安全 | 输入安全 | 执行安全 | 数据安全 |
| 部署复杂度 | 中 | 高 | 中（需硬件支持） | 高 |

## 决策树

### 技术选型决策流程

```
问题: 需要在保护隐私的前提下联合计算数据吗？
│
├── 数据参与方 > 2？
│   ├── 是 → 数据规模大？ 
│   │   ├── 是 → 联邦学习 (FL)
│   │   └── 否 → 参与方可信任？
│   │       ├── 是 → MPC (Honest Majority)
│   │       └── 否 → MPC (Dishonest Majority)
│   │
│   └── 数据参与方 = 2？
│       └── 计算类型？
│           ├── 统计分析 → MPC
│           ├── 机器学习 → FL (vertical FL)
│           ├── 精确查询 → MPC / TEE
│           └── 简单聚合 → 差分隐私 + MPC
│
├── 是否需要计算结果可验证？
│   ├── 是 → MPC (公平性) / TEE (远程证明)
│   └── 否 → FL
│
└── 性能要求高？
    ├── 是 → TEE (接近明文性能)
    └── 否 → HE (全同态加密)
```

## 用例映射

| 业务场景 | 推荐技术 | 说明 |
|----------|----------|------|
| 联合信贷风控 | 联邦学习 + MPC | 银行间联合建模，MPC 用于特征工程 |
| 医疗数据联合分析 | MPC + TEE | 医院间统计分析，TEE 处理高敏数据 |
| 广告 CTI 优化 | FL + 差分隐私 | 多方数据训练广告模型 |
| 黑名单查询 | MPC (PSI/OPRF) | 隐私集合求交，不暴露输入 |
| 供应链 KPI 计算 | HE | 无需交互的单向加密计算 |
| 基因数据分析 | TEE | 大规模基因数据处理 |
| 隐私保护 SQL | MPC (SPDZ) | 多方 SQL 联合查询 |
| 密码货币交易 | MPC (Threshold ECDSA) | 门限签名 |

### 多技术组合方案

```yaml
复杂场景: 多家医院联合训练疾病预测模型

技术组合:
  1. 联邦学习 (FL): 各医院本地训练，仅共享梯度
  2. 安全聚合 (Secure Aggregation): MPC 协议聚合梯度
  3. 差分隐私 (DP): 在梯度中添加噪声防止梯度泄露
  4. TEE: 聚合服务器部署在 TEE 中增强安全

优势:
  - FL 解决数据不出本地
  - Secure Aggregation 解决聚合者可见各参与方梯度的问题
  - DP 防止梯度推断攻击
  - TEE 保护聚合服务器完整性
```

## 技术成熟度评估

| 技术 | 学术界成熟度 | 工业界成熟度 | 标准制定 | 硬件支持 |
|------|-------------|-------------|----------|----------|
| FL | 成熟 | 较成熟 | IEEE 标准进行中 | 通用 CPU |
| MPC | 成熟 | 初期商业化 | 无统一标准 | Intel IPP-Crypto |
| TEE | 较成熟 | 较成熟 | GlobalPlatform TEE | Intel SGX/AMD SEV/ARM TZ |
| HE | 持续突破 | 早期商业化 | IBM HELib/Microsoft SEAL | FPGA 加速试验中 |

### 建议选型策略

```
短期 (0-6 个月):
  选择成熟度高的技术: FL + 差分隐私
  风险: 低
  验证：MVP 快速验证业务价值

中期 (6-18 个月):
  引入 MPC 补充精确计算场景
  风险: 中
  验证：PoC 评估性能瓶颈

长期 (18+ 个月):
  关注 HE 进展 (CKKS 方案)
  风险: 取决于 HE 性能突破
  评估：硬件加速成熟度
```
