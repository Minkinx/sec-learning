# 联邦学习详解

## 概述

联邦学习 (Federated Learning, FL) 是一种分布式机器学习范式，多个参与方在不共享原始数据的前提下协作训练模型。FL 的核心思想是"数据不动模型动"，在数据本地化的同时实现模型能力的聚合。

## 横向/纵向/迁移联邦学习

### 横向联邦学习 (Horizontal FL / Sample-Partitioned)

```
参与方 A (城市 A)   参与方 B (城市 B)     参与方 N (城市 N)
    │                    │                     │
  ┌─┴─┐               ┌─┴─┐                ┌─┴─┐
  │用户ID: 1-1000     │用户ID: 1001-2000   │用户ID: 2001-3000
  │特质: 年龄/收入     │特质: 年龄/收入      │特质: 年龄/收入
  │标签: 是否逾期      │标签: 是否逾期       │标签: 是否逾期
  └───┘               └───┘                └───┘
```

**适用场景**：数据特征重叠多，用户重叠少（如不同城市的同类型银行）。

### 纵向联邦学习 (Vertical FL / Feature-Partitioned)

```
参与方 A (银行)             参与方 B (电商)
  │                            │
┌─┴─┐                      ┌─┴─┐
│用户ID                     │用户ID
│收入/资产负债              │消费记录/用户画像
│标签: 是否逾期             │ (无标签)
└───┘                      └───┘
      ↘                  ↙
   [实体对齐] (PSI)
        ↓
   [联邦模型训练]
```

**适用场景**：用户重叠多，数据特征重叠少。

### 迁移联邦学习 (Transfer FL / Federated Transfer Learning)

两个参与方样本和特征重叠都很少时，利用迁移学习能力实现知识迁移：

```
参与方 A (中国金融机构)     参与方 B (欧洲电商)
  │                            │
  │ 特征空间: 信用特征         │ 特征空间: 消费特征
  │ 样本空间: 中国用户         │ 样本空间: 欧洲用户
  │ 标签: 有标签              │ 标签: 无标签
  │                            │
  └─────── 联邦迁移学习 ───────┘
```

## Secure Aggregation 协议

### Secure Aggregation 流程

```
参与方 1: w1 ──→ 加密 ──→ Enc(w1)
参与方 2: w2 ──→ 加密 ──→ Enc(w2)
参与方 3: w3 ──→ 加密 ──→ Enc(w3)
                          ↓
                 聚合服务器: Enc(w1)+Enc(w2)+Enc(w3)
                      ↓
                  解密 → w_global
```

### Bonneh 等人 Secure Aggregation 协议 (CCS'17)

| 阶段 | 步骤 | 通信量 |
|------|------|--------|
| 1. Key Agreement | 参与方两两 Diffie-Hellman 密钥交换 | O(n²) |
| 2. Share Generation | 生成密钥分享掩码 | O(n) |
| 3. Masking | 掩码模型参数 | O(n) |
| 4. Unmasking | 聚合后恢复 | O(n) |
| 5. Verification | 验证聚合结果 | O(n) |

**安全属性**：
- 服务器不知道任何参与方的原始参数
- 最多 n-2 个参与方合谋也无法获取其他方参数
- 容错能力：允许部分参与方掉线

## 梯度泄露攻击

### 攻击原理

攻击者从共享的梯度中恢复训练数据：

```
输入: 梯度 ∇w = ∂L/∂w (共享的梯度)
方法: 构建虚梯度，优化虚数据使得虚梯度与真实梯度接近
目标: 恢复训练数据 x 和标签 y

攻击算法 (Deep Leakage from Gradients):
  1. 初始化虚数据 x' 和虚标签 y'
  2. 计算虚梯度 ∇w' = ∂L(x', y')/∂w
  3. 优化 min ||∇w - ∇w'||² 更新 x', y'
  4. 迭代直至收敛
```

### 防御措施

| 防御方法 | 说明 | 有效性 | 模型精度影响 |
|----------|------|--------|-------------|
| Gradient Clipping | 裁剪梯度最大值 | 中 | 小 |
| Differential Privacy | 加噪梯度 | 强 | 中到高 |
| Gradient Compression | 梯度压缩/稀疏化 | 弱 | 小 |
| Secure Aggregation | 加密聚合梯度 | 强 | 无 |
| Soteria | 在训练中扰动表示 | 中 | 小 |
| 层级自适应保护 | 仅保护关键层梯度 | 中 | 可调 |

## 差分隐私在 FL 中的应用

### DP-FL 架构

```
局部差分隐私 (Local DP):
  每个参与方在本地梯度和参数上添加噪声
  累积隐私预算 ε_local

全局差分隐私 (Global DP):
  聚合服务器在最终的全局模型上添加噪声
  隐私预算 ε_global = ∑ε_local

推荐实践: 组合使用
  每个参与方: 使用 Local DP (小噪声)
  聚合服务器: 使用 Global DP (残余噪声)
```

### 隐私预算分配

| 训练轮次 | Privacy Budget (ε) | 噪声级别 σ |
|----------|-------------------|------------|
| Round 1-50 | 0.2/round | 高 |
| Round 51-200 | 0.5/round | 中 |
| Round 201-1000 | 1.0/round | 低 |
| **Total** | **ε_total = 10.0** | — |

**关键参数** (DP-SGD):
- `C` = clipping threshold (梯度裁剪阈值)
- `σ` = noise multiplier (噪声系数)
- `ε` = privacy budget (隐私预算)
- `δ` = failure probability (失败概率, 通常 ≤ 1/n)

## FedAvg / FedProx 算法

### FedAvg (Federated Averaging)

```
算法: FedAvg (McMahan et al., 2017)

初始化: 全局模型 w₀
每轮 t:
  1. 选择 K 个参与方 (比例 C)
  2. 对每个参与方 k:
     wᵗᵏ = LocalTrain(wₜ, Dₖ, E epochs)
  3. 聚合: wₜ₊₁ = ∑((nₖ/n) × wᵗᵏ)
  4. 分发 wₜ₊₁ 给所有参与方
```

| 超参数 | 说明 | 典型值 |
|--------|------|--------|
| C | 参与比例 | 0.1 (10%) |
| E | 本地训练轮次 | 1-5 |
| B | 本地 batch size | 10-64 |
| η | 学习率 | 0.01-0.1 |

### FedProx (Li et al., 2020)

FedProx 解决 FedAvg 在 heterogeneous 数据上的不稳定问题：

```
FedProx 改进:
  Loss = Loss_original + (μ/2) × ||w_local - w_global||²

  μ: proximal term 系数 (0.001 ~ 0.1)
  - 防止本地模型偏离全局模型太多
  - 允许部分参与方本地训练更多轮次
  - 容忍系统异构性 (不同设备算力不同)
```

### 算法对比

| 算法 | 数据异构性 | 收敛速度 | 通信效率 | 实现复杂度 |
|------|-----------|----------|----------|-----------|
| FedAvg | 敏感 | 快 | 高 | 低 |
| FedProx | 鲁棒 | 中 | 高 | 低 |
| FedNova | 鲁棒 | 中 | 中 | 中 |
| Scaffold | 鲁棒 | 快 | 中 | 高 |
