# 大数据平台安全

## 概述

大数据平台（Hadoop/Spark/Kafka/Flink）处理海量数据，面临着与传统数据库不同的安全挑战。分布式架构、多用户共享、复杂的数据管道使得安全控制更加困难。需从认证、授权、加密、审计四个维度建立全面防护。

## Hadoop 安全架构

### 核心组件安全

| 组件 | 安全配置 | 关键防护 |
|------|----------|----------|
| HDFS | Kerberos + HDFS ACL + Transparent Encryption | 网络分区, 数据块加密 |
| YARN | Kerberos + Linux Container Executor | 资源隔离, 用户映射 |
| Hive | Sentry/Ranger + Hive Authorization | 列级访问控制 |
| HBase | Kerberos + Cell-Level ACL | 行/列族级权限 |
| ZooKeeper | SASL + ACL + mTLS | 配置安全, 选举安全 |

### Kerberos 认证

```
Client ─── AS-REQ ──→ KDC (Authentication Server)
KDC ──── AS-REP ───→ Client (TGT)
Client ─── TGS-REQ ──→ KDC (Ticket Granting Server)
KDC ──── TGS-REP ───→ Client (Service Ticket)
Client ─── AP-REQ ───→ Service (HDFS/YARN/etc)
Service ── Verify ──→ KDC
```

### HDFS Transparent Encryption

```
HDFS 加密密钥架构:
  KMS (Key Management Server)
    └── 加密区 (Encryption Zone) 密钥
         └── 数据加密密钥 (DEK)
              └── 文件数据块加密 (AES-CTR)

加密流程:
  客户端请求读文件
  客户端向 KMS 请求 EZK（加密区密钥）
  KMS 验证权限后返回 DEK
  客户端使用 DEK 解密数据块
  DEK 加密传输, HDFS DataNode 不接触明文密钥
```

## Apache Ranger / Apache Sentry

### Ranger 授权架构

```
Ranger Admin
  ├── Policy Engine (策略定义)
  ├── Audit Server (审计日志)
  └── User/Group Sync (LDAP/AD)

Ranger Plugin (嵌入各组件)
  ├── HDFS Plugin
  ├── Hive Plugin
  ├── HBase Plugin
  ├── Kafka Plugin
  └── Solr Plugin
```

### Ranger vs Sentry 对比

| 特性 | Apache Ranger | Apache Sentry |
|------|---------------|---------------|
| 授权模型 | RBAC + Attribute-Based | RBAC |
| 列级控制 | 支持 | 支持 |
| Row-Level Filter | 支持 | 有限支持 |
| Data Masking | 支持 | 不支持 |
| Audit Integration | 原生审计服务 | 需额外集成 |
| 策略管理 | Web UI + REST API | CLI + Configuration |
| 组件覆盖 | 广泛 (HDFS/Hive/HBase/Kafka/Solr/Storm) | Hive/HBase/Impala |
| 活跃度 | 活跃 | 退役 (Apache Attic) |

### Ranger 策略示例

```json
{
  "service": "hive_dev",
  "name": "mask_ssn_column",
  "policyItems": [{
    "users": ["analyst_user"],
    "accesses": [{"type": "select"}],
    "dataMaskInfo": {
      "dataMaskType": "MASK_SHOW_LAST_4",
      "valueExpr": "ssn"
    }
  }, {
    "users": ["admin"],
    "accesses": [{"type": "select"}],
    "dataMaskInfo": {
      "dataMaskType": "CUSTOM",
      "maskedValue": "ssn"
    }
  }]
}
```

## Spark 安全

| 安全维度 | 配置 | 说明 |
|----------|------|------|
| RPC 加密 | `spark.authenticate=true` | 所有 RPC 通信加密 |
| RPC 密钥 | `spark.authenticate.secret` | 共享密钥或 Kerberos |
| SSL | `spark.ssl.enabled=true` | Web UI/History Server SSL |
| 动态资源分配 | `spark.dynamicAllocation.enabled=true` | 防止资源滥用 |
| AC 访问控制 | `spark.acls.enable=true` | 作业级访问控制 |
| 事件日志 | `spark.eventLog.enabled=true` | 审计日志 |
| 网络隔离 | `spark.network.crypto.enabled=true` | Shuffle 加密 |

### Spark RPC 加密配置

```yaml
spark:
  authenticate: true
  authenticate.secret: <secret-or-kerberos>
  network.crypto.enabled: true
  network.crypto.saslFallback: false
  ssl:
    enabled: true
    keyStore: /etc/security/spark.jks
    keyStorePassword: <password>
    trustStore: /etc/security/spark.truststore
    trustStorePassword: <password>
```

## Kafka 安全

### Kafka 安全协议

| 协议 | 认证 | 加密 | 说明 |
|------|------|------|------|
| PLAINTEXT | 无 | 无 | 仅开发环境 |
| SSL | SSL Cert | TLS | 双向认证 + 传输加密 |
| SASL_PLAINTEXT | SASL | 无 | 需配合网络层加密 |
| SASL_SSL | SASL + SSL Cert | TLS | 推荐生产配置 |

### SASL 认证机制对比

| 机制 | 说明 | 适用场景 |
|------|------|----------|
| GSSAPI | Kerberos 认证 | 已建 Kerberos 环境 |
| PLAIN | 用户名/密码 | 简单认证场景 |
| SCRAM-SHA-256/512 | 加盐哈希认证 | 推荐替代 PLAIN |
| OAUTHBEARER | OAuth 2.0 令牌 | 云原生/K8s 环境 |

### Kafka 授权 ACL

```bash
# 生产者授权
kafka-acls.sh --authorizer-properties \
  --add \
  --allow-principal User:producer_app \
  --operation Write \
  --operation Create \
  --topic sensitive-topic

# 消费者授权  
kafka-acls.sh --authorizer-properties \
  --add \
  --allow-principal User:consumer_app \
  --operation Read \
  --topic sensitive-topic \
  --group analytics-group
```

## 列级脱敏与审计日志

### 列级脱敏 (Ranger + Hive)

| 脱敏类型 | 生效方式 | 示例策略 |
|----------|----------|----------|
| MASK_NULL | 替换为 NULL | ssn → NULL |
| MASK_SHOW_LAST_4 | 仅显示最后 4 位 | 123-45-6789 → xxx-xx-6789 |
| MASK_HASH | SHA-256 哈希 | 123-45-6789 → a3f1b2c8... |
| CUSTOM | 自定义格式 | email → j***@example.com |

### 审计日志集成

```yaml
审计日志流:
  大数据组件 → Ranger Audits (Solr) → Elasticsearch → Kibana Dashboard
            → Apache Atlas (数据血缘)
            → SIEM (Splunk/QRadar)
  
  审计内容:
    - 用户访问了哪些表/列
    - 执行的查询语句
    - 返回的结果行数
    - 查询时间/来源 IP
    - 敏感列覆盖情况
```
