# 文件传输安全

## 概述

文件传输是企业数据交换中不可忽视的安全环节。从内部系统间的大文件传输到与合作伙伴的敏感数据交换，选择合适的安全传输协议和保护机制至关重要。

## SFTP vs FTPS vs HTTPS 对比

| 特性 | SFTP (SSH) | FTPS (TLS) | HTTPS |
|------|------------|------------|-------|
| 基础协议 | SSH (port 22) | FTP + TLS (port 990/21) | HTTP + TLS (port 443) |
| 认证方式 | 用户名/密码 + SSH Key | 用户名/密码 + X.509 证书 | 多种（Basic/OAuth/Client Cert） |
| 防火墙友好 | 是（单端口） | 否（需动态端口） | 是（单端口） |
| 目录浏览 | 支持 | 支持 | 需 WebDAV |
| 断点续传 | 支持 | 支持 | 部分支持 |
| 命令行工具 | OpenSSH、lftp | lftp、curl | curl、wget |
| 自动化 | 易 (SSH Key + cron) | 中 (需证书管理) | 易 (标准 HTTP) |
| 推荐场景 | 服务器间文件同步 | Legacy 系统迁移 | Web 应用文件上传 |

## Managed File Transfer (MFT) 平台

MFT 平台为企业提供集中化、可审计的文件传输管理能力：

| 功能 | 说明 | 安全价值 |
|------|------|----------|
| 集中策略管理 | 统一的传输策略配置 | 确保全局一致性 |
| 自动加密 | 传输中/存储中自动加密 | 减少人为错误 |
| 审计日志 | 全链路传输记录 | 满足合规审计 |
| 工作流引擎 | 审批流 + 传输任务 | 防止未授权传输 |
| 协议转换 | 自动适配多种传输协议 | 兼容性+安全性 |
| 异常告警 | 传输失败/异常模式检测 | 快速响应 |

### 主流 MFT 方案

- **商业**：IBM Sterling File Gateway、Axway MFT、GoAnywhere MFT
- **开源**：Apache NiFi、Rsync (加强版)、FileZilla Pro (Limited)

## PGP 加密

PGP (Pretty Good Privacy) 在文件传输中提供端到端的机密性和完整性：

### PGP 操作模式

```
加密流程:
明文文件 → 压缩 → 对称加密 (Session Key) → RSA/OAEP 加密 Session Key → PGP 文件
                                                                   ↓
                                                             接收方公钥

解密流程:
PGP 文件 → 解密 Session Key (RSA/OAEP) → 对称解密 → 解压 → 明文文件
```

### PGP 密钥管理

| 密钥类型 | 生命周期 | 安全要求 |
|----------|----------|----------|
| 主密钥 (Master Key) | 长期使用 (2-5 年) | 离线存储, 多因子保护 |
| 子密钥 (Sub Key) | 定期轮换 (6-12 个月) | 在线使用, 每日签名 |
| 会话密钥 (Session Key) | 单次使用 | 随机生成, 每次唯一 |

**PGP 安全建议**：
- 使用 RSA 4096 bit 或 ECC (Curve25519)
- 指纹验证（带外确认公钥指纹）
- 密钥到期自动轮换
- 私钥加密存储（强密码保护）

## 文件完整性校验

| 校验方式 | 算法 | 说明 |
|----------|------|------|
| 传输校验 | TCP Checksum | 基础差错检测，非安全 |
| 完整性校验 | SHA-256 Hash | 文件级完整性验证 |
| 签名校验 | ECDSA/RSA Signature | 完整性 + 发送方认证 |
| 块校验 | Merkle Tree | 大文件增量校验 |
| 传输重试 | 分块校验重传 | 断点续传支持 |

### 文件传输验证流程

```bash
# 发送方：生成文件摘要并签名
sha256sum important_data.csv > important_data.csv.sha256
gpg --detach-sign --armor important_data.csv.sha256

# 接收方：验证签名和完整性
gpg --verify important_data.csv.sha256.asc important_data.csv.sha256
sha256sum -c important_data.csv.sha256
```

## 传输审计日志

文件传输审计日志应包含以下关键字段：

| 字段 | 说明 | 示例 |
|------|------|------|
| 传输 ID | 唯一标识 | TXFR-20240515-001 |
| 文件名 | 传输的文件名 | report_2024Q1.csv |
| 文件大小 | 字节数 | 152345678 |
| 文件哈希 | SHA-256 | a3f1b2c8... |
| 发送方 | 用户/系统标识 | user@source-system |
| 接收方 | 用户/系统标识 | partner@destination |
| 协议 | 传输协议 | SFTP |
| 时间戳 | 开始/结束时间 | 2024-05-15T10:30:00Z |
| 状态 | 成功/失败/重试 | SUCCESS |
| 加密类型 | 加密方式 | PGP + TLS |

## AS2/AS4 协议

AS2 (Applicability Statement 2) 和 AS4 是 EDI 领域的安全传输协议：

| 特性 | AS2 | AS4 |
|------|-----|-----|
| 传输层 | HTTP/S | Web Services |
| 加密 | S/MIME | S/MIME + WS-Security |
| 回执 | MDN (Message Disposition Notification) | Receipt (WS-Reliability) |
| 适用行业 | 零售、制造业 | 电子商务、政务 |
| 安全性 | 签名 + 加密 + 不可否认性 | 签名 + 加密 + 可靠消息 |

AS2 的核心安全机制：
- **不可否认性**：发送方签名 + 接收方 MDN 回执签名
- **消息完整性**：S/MIME 数字签名
- **机密性**：S/MIME 加密
- **重放保护**：唯一消息 ID + 时间戳
