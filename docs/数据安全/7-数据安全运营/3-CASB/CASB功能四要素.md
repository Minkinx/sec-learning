# CASB 功能四要素

## 概述

云访问安全代理（Cloud Access Security Broker, CASB）是在用户和云服务之间执行安全策略的中间平台。Gartner 定义了 CASB 的四大功能支柱：可视性（Visibility）、数据安全（Data Security）、威胁防护（Threat Protection）和合规性（Compliance）。CASB 可部署为 Forward Proxy（正向代理，位于用户端）或 Reverse Proxy（反向代理，位于云服务端），也可通过 API 模式（API 直接与云服务集成）实现。

## 部署模式对比

| 部署模式 | 位置 | 流量路径 | 阻断能力 | 适用场景 |
|----------|------|----------|----------|----------|
| Forward Proxy（正向代理） | 靠近用户侧 | 用户 → CASB → 云服务 | 实时阻断 | 控制用户访问流量 |
| Reverse Proxy（反向代理） | 靠近云服务侧 | 用户 → CASB（替代云服务） | 实时阻断 | 保护自托管应用迁移到云 |
| API 模式 | 云服务内 | CASB 通过 API 读取云服务 | 检测后处置（非实时） | 已存储数据扫描和持续监控 |
| Log Collection | 日志汇聚 | 收集云服务日志分析 | 无阻断 | 纯审计场景 |

## 可视性（Visibility）

- **影子 IT 发现**：自动识别企业内部使用的未经批准的云服务，覆盖数千种 SaaS 应用。通过流量分析（SSL 证书指纹、DNS 查询、HTTP 特征、SNI 字段）识别应用身份，支持超过 30,000 种云应用的标识。
- **应用目录与风险评分**：维护包含风险评分、合规认证、数据中心位置、安全功能等属性的云应用数据库。评分维度包括：应用安全功能（MFA/加密/日志）、合规认证（SOC2/ISO27001/HIPAA）、数据驻留位置、历史安全事件、用户评价。
- **用户使用分析**：展示每个云应用的用户数量、活跃度、数据上传下载量、使用频率分布，辅助决策是否批准使用、是否需要管控。
- **数据共享可视性**：检测云存储中公开共享的链接、外部协作用户、过度授权的服务账号。

## 数据安全（Data Security）

CASB 的数据安全能力涵盖：

| 能力 | 说明 | 实现方式 | 典型用例 |
|------|------|----------|----------|
| 内容检测与 DLP | 检测上传下载流量中的敏感数据 | 正则/指纹/ML 分类 | 阻止信用卡号上传到个人云盘 |
| 加密与密钥管理 | 实现数据托管加密 | BYOK/客户管理的加密密钥 | 云端敏感数据的额外加密层 |
| 条件访问控制 | 基于数据敏感度和用户上下文实施访问策略 | 结合 IDP 的 SAML/OIDC 断言 | 仅允许受管设备访问 L4 数据 |
| 数据共享权限审计 | 检查云存储中共享权限配置 | API 定期扫描 + 实时告警 | 发现 Google Drive 公开链接 |
| 数据保留与删除 | 按策略自动删除过期数据 | API 批量操作 | 满足数据最小化原则 |

## 威胁防护（Threat Protection）

- **UEBA 集成**：基于用户行为的异常检测，识别账号劫持、内部威胁和恶意应用授权。从登录行为（失败次数/地理位置）、访问行为（数据量/时间段）、文件行为（下载/删除/共享）三个维度构建用户画像。
- **恶意软件检测**：对上传云存储的文件进行恶意软件扫描（沙箱检测 + AI 分析），防止通过云应用作为恶意软件分发渠道。检测范围覆盖：Office 宏病毒、可执行文件、PDF 嵌入式脚本。
- **异常行为分析**：检测非正常访问模式，包括：
  - 短时间内大量下载（数据窃取）
  - 从异常地理位置登录（账号劫持）
  - 异常时间访问（内部威胁）
  - 删除大量共享文件（恶意破坏）
  - 异常 OAuth 应用授权（恶意应用授权）

## 合规性（Compliance）

| 合规维度 | CASB 提供的功能 | 支持的标准 |
|----------|-----------------|------------|
| 数据驻留 | 展示数据所在区域，限制数据跨区域流动 | GDPR 数据本地化要求 |
| 审计日志 | 保留云应用操作的全量审计日志 | SOC2/PCI-DSS/GDPR |
| 合规报告 | 一键生成云服务使用合规报告 | SOC2/ISO 27001/HIPAA |
| 策略模板 | 预配置的行业合规策略模板 | PCI-DSS/HIPAA/等保 |
| 数据治理 | 数据分类、保留、删除的全周期管理 | PIPL/GDPR 数据最小化 |
| 第三方访问控制 | 管理 OAuth 应用对云数据的授权 | SOX/CCPA/CPRA |

## 四要素关联

四要素相互支撑形成闭环：

```
    可视性 (Visibility)
         │
         ▼
    数据安全 (Data Security)
         │
         ▼
    威胁防护 (Threat Protection)
         │
         ▼
    合规性 (Compliance)
         │
         └────→ 输出合规证据和审计报告
                    │
                    ▼
               反馈到可视性（新发现的风险驱动新一轮循环）
```

- **可视性**是基础：只有知道哪些云服务在使用、谁在使用、如何使用，才能进行保护
- **数据安全**是核心：对发现的数据实施 DLP、加密、访问控制等保护措施
- **威胁防护**是升华：在数据保护基础上，检测和防御针对云服务的攻击行为
- **合规性**是保障：确保整个流程可审计、可证明，满足法规要求
