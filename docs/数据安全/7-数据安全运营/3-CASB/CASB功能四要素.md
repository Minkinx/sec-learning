# CASB 功能四要素

## 概述

云访问安全代理（Cloud Access Security Broker, CASB）是在用户和云服务之间执行安全策略的中间平台。Gartner 定义了 CASB 的四大功能支柱：可视性、数据安全、威胁防护和合规性。

## 可视性（Visibility）

- **影子 IT 发现**：自动识别企业内部使用的未经批准的云服务，覆盖数千种 SaaS 应用，通过流量分析（SSL 证书指纹/DNS 查询/HTTP 特征）识别应用身份。
- **应用目录**：维护包含风险评分、合规认证、数据中心位置等属性的云应用数据库，帮助安全团队评估每个应用的业务风险和合规状态。
- **用户使用分析**：展示每个云应用的用户数量、活跃度、数据上传下载量，辅助决策是否批准使用。

## 数据安全（Data Security）

CASB 的数据安全能力涵盖：内容检测与 DLP（支持正则表达式/指纹/关键字匹配，覆盖上传和下载流量）、加密（通过 Bring Your Own Key 实现数据托管加密）、访问控制（基于数据敏感级别的条件访问）、以及数据共享权限审计（如 Google Drive 中的公共链接检测）。

## 威胁防护（Threat Protection）

- **UEBA 集成**：基于用户行为的异常检测，识别账号劫持、内部威胁和恶意应用授权。
- **恶意软件检测**：对上传云存储的文件进行恶意软件扫描（沙箱检测 + AI 分析），防止通过云应用作为恶意软件分发渠道。
- **异常行为分析**：检测非正常访问模式（如短时间内大量下载、从异常地理位置登录、非常规时间访问）。

## 合规性（Compliance）

CASB 帮助企业满足云使用层面的合规要求：生成云服务使用的合规报告（GDPR/SOC2/PCI-DSS）；展示数据驻留情况确保符合本地化要求；通过预配置策略模板快速对齐行业监管标准；保留云应用操作的审计日志，满足溯源要求。

## 四要素关联

四要素相互支撑形成闭环：可视性发现云服务使用现状 -> 数据安全保护云中数据 -> 威胁防护检测异常行为 -> 合规性确保整个流程符合监管要求。
