# 影子 IT 发现

## 概述

影子 IT（Shadow IT）指未经企业 IT 部门批准而使用的技术资源、设备、软件或服务。在云服务场景下，员工使用未经审核的 SaaS 应用存储和处理企业数据，是数据泄露的主要风险来源之一。根据 Gartner 报告，企业实际使用的云服务数量通常是 IT 部门知道的 10-15 倍，影子 IT 导致的数据泄露事件占比超过 25%。

## 发现方法

| 方法 | 原理 | 覆盖范围 | 部署复杂度 | 优缺点 |
|------|------|----------|-----------|--------|
| 网络流量分析 | 分析防火墙/代理日志中的目标域名和应用特征 | 公司网络内所有流量 | 低（利用现有基础设施） | 覆盖广；无法检测加密流量以外的应用；VPN 外流量不可见 |
| API 集成 | 通过 OAuth/API 与云应用目录集成获取应用授权列表 | 已授权 OAuth 应用 | 中（需 OAuth 令牌管理） | 精度高；仅覆盖授权应用 |
| 终端代理 | 终端 Agent 采集 DNS/进程/网络连接信息 | 受管端点 | 高（需部署客户端） | 覆盖端侧全流量；部署成本高 |
| DNS 日志分析 | 分析 DNS 查询日志识别未知域名 | 公司网络内所有 DNS 查询 | 低（DNS 日志已存在） | 轻量但仍需检查 TLS SNI 补充 |
| SSL 证书指纹 | 检查 TLS 握手阶段的证书指纹 | 加密流量（不解密） | 中（需专门的流量分析设备） | 不解密即可识别应用；部分应用可能指纹变化 |
| 用户调查 | 匿名问卷收集员工使用的应用 | 主观视角 | 低 | 只能作为补充手段，员工可能隐瞒 |

## 风险评估

发现影子 IT 后需要进行风险评估，核心评估维度：

| 评估维度 | 权重（参考） | 评估方法 | 评分标准 |
|----------|-------------|----------|----------|
| 数据敏感性 | 25% | 检查员工放入该应用的数据类型 | 无 PII=0 分，含 PII=5 分，含财务/健康=10 分 |
| 用户规模 | 15% | 使用该应用的员工数量 | <10人=2分，10-100=5分，>100=10分 |
| 访问控制强度 | 20% | SSO/MFA/密码策略 | MFA+SSO=1分，仅密码=5分，无认证=10分 |
| 合规认证状态 | 20% | SOC2/ISO27001/HIPAA 等 | 有合规=0分，无合规=7分，有违规记录=10分 |
| 数据驻留位置 | 10% | 数据中心所在司法管辖区 | 境内=1分，允许的跨境区域=5分，不允许区域=10分 |
| 历史安全事件 | 10% | 过去 12 个月的已知安全事件 | 无事件=0分，有通报事件=10分 |

## 风险评分模型

综合上述维度建立风险评分模型，得分范围 0-100：

```
风险评分 = Σ(维度得分 × 维度权重)
         = 数据敏感性×25% + 用户规模×15% + 
           访问控制×20% + 合规状态×20% +
           数据驻留×10% + 历史事件×10%
```

按阈值分级处置：
| 风险等级 | 评分范围 | 处置策略 |
|----------|----------|----------|
| 低风险 | 0-30 分 | 观察，可纳入 IT 管理清单 |
| 中风险 | 31-60 分 | 整改（要求增加安全控制）或替换为经批准的替代应用 |
| 高风险 | 61-100 分 | 立即阻断（在网关层阻止访问），通知用户并迁移数据 |

## 处置工作流

针对影子 IT 的发现结果，建立标准处置流程：

1. **发现与告警**：影子 IT 发现引擎检测到新的未授权云服务，生成告警。
2. **评估**：安全团队评估应用的风险评分（自动 + 人工复核），确定风险评估等级。
3. **决策**：判定应用的处理方式：
   - **批准（Approve）**：将应用纳入 IT 管理，配置 SSO/DLP，定期审计。
   - **整改（Remediate）**：要求业务部门在使用前增加所需安全控制（如启用 MFA），复查后批准。
   - **阻断（Block）**：在网关/代理层阻止对所有实例的访问，通知用户禁止使用。
   - **迁移（Migrate）**：将已存储的数据迁移至企业批准的替代应用。
4. **执行**：落地决策，自动或手动配置防火墙/SWG/CASB 策略。
5. **验证**：24-48 小时后复查确保阻断生效或整改已完成。对阻断的应用，持续监控是否出现绕过行为。
6. **用户沟通**：通知相关人员决策原因和替代方案。

## 持续运营

影子 IT 发现不是一次性项目，需要持续运营：

- **每周**：检查新增影子 IT 应用，评估风险，执行处置工作流
- **每月**：生成影子 IT 趋势报告，展示风险变化趋势和各部门风险分布
- **每季度**：审查应用目录，清理不再使用的应用，更新风险评分数据
- **每年**：审计全量云服务使用情况，更新影子 IT 策略和应用白名单
