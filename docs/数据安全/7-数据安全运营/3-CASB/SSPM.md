# SSPM

## 概述

SaaS 安全态势管理（SaaS Security Posture Management, SSPM）专注于检测和修复 SaaS 应用中的错误配置和安全风险，补充了 CASB 在配置安全层面的不足。SSPM 通过 API 持续监控 SaaS 应用的安全配置状态，自动发现配置漂移、不合规设置和安全弱点。SSPM 是云安全态势管理（CSPM）在 SaaS 层的延伸，两者共同构成完整的云安全态势视图。

## SSPM vs CASB vs CSPM

| 维度 | SSPM | CASB | CSPM |
|------|------|------|------|
| 聚焦层 | SaaS 应用配置 | 用户与云服务间的数据流 | IaaS/PaaS 基础设施 |
| 核心能力 | 配置检查、合规基准、自动修复 | 数据安全、威胁防护、可视性 | 基础设施配置检查、合规 |
| 集成方式 | API（只读+配置写入） | 代理/API/日志 | API（云服务商 API） |
| 典型覆盖 | M365/Google Workspace/Salesforce/Slack | 数千种 SaaS 应用 | AWS/Azure/GCP |
| 输出 | 配置评分、修复建议 | 数据流风险、用户行为告警 | 基础设施风险、合规评分 |
| 互补关系 | CASB 发现影子 IT，SSPM 确保已批准应用安全 | SSPM 提供配置层面的深度检查 | SSPM 和 CSPM 在 IaC 安全交汇 |

## 配置检查维度

SSPM 覆盖的核心配置维度，按风险域划分：

### 身份与访问管理

| 检查项 | 风险等级 | 检查方法 | 修复建议 |
|--------|----------|----------|----------|
| SSO 配置完整性 | 高 | 验证 SAML/OIDC SSO 是否已配置并强制使用 | 配置并强制 SSO 登录 |
| MFA 启用率 | 高 | 统计未启用 MFA 的用户占比 | 对所有用户强制 MFA |
| 服务账号权限 | 中 | 检查 API Token 和 Service Account 权限范围 | 遵循最小权限原则 |
| 外部用户共享策略 | 中 | 验证外部共享的设置限制 | 限制为仅特定域或组织成员 |
| 密码策略（复杂度/过期时间/历史） | 中 | 检查密码策略强度配置 | 按 NIST SP 800-63 更新密码策略 |
| 休眠账号 | 中 | 检测 90 天以上未登录的账号 | 禁用并归档 |
| 管理员角色数量 | 中 | 统计拥有管理员权限的用户数 | 减少超权限管理员至最少 |

### 数据存储安全

| 检查项 | 风险等级 | 检查方法 | 修复建议 |
|--------|----------|----------|----------|
| 默认分享链接权限 | 高 | 检测默认共享链接的权限设置 | 设为"仅特定人员"而非"组织内所有人" |
| 外部来宾访问设置 | 高 | 检查外部来宾账户权限范围 | 限制来宾仅访问所需资源 |
| 数据保留策略 | 中 | 验证数据保留期配置 | 按数据分类设置保留期，到期自动删除 |
| 公开共享检测 | 高 | 扫描所有链接公开的文件/文档 | 发现后自动关闭并通知所有者 |
| 第三方应用数据访问 | 中 | 审计 OAuth 授权范围 | 清单定期审查并撤销不需要的授权 |

### 网络与端点

| 检查项 | 风险等级 | 说明 |
|--------|----------|------|
| IP 白名单配置 | 高 | 是否配置了受信任 IP 范围，仅允许从企业网络访问 |
| API 端点暴露 | 中 | 不需要对外暴露的 API 端点是否已限制访问 |
| OAuth 授权范围 | 高 | 已授权的 OAuth 应用是否拥有超出必要的权限范围 |
| 条件访问策略 | 中 | 是否配置了基于设备合规状态的条件访问 |

### 日志与监控

| 检查项 | 风险等级 | 检查方法 | 合规映射 |
|--------|----------|----------|----------|
| 审计日志启用 | 高 | 验证审计日志配置已开启 | SOC2/GDPR/PIPL |
| 日志保留时长 | 高 | 检查日志保留期限配置 | 最低 90 天（GDPR）、1年（SOX） |
| 管理员操作告警 | 中 | 检查敏感操作告警规则是否配置 | 最佳实践 |
| 登录失败告警 | 中 | 检查多次登录失败告警是否开启 | SOC2 |
| 数据导出告警 | 高 | 检查批量数据导出告警规则 | 最佳实践 |

## CIS SaaS 基准

CIS（Center for Internet Security）发布了适用于主流 SaaS 应用的配置基准。SSPM 工具应支持映射企业配置至 CIS 基准，量化合规评分：

| CIS 基准 | 覆盖检查项数 | 最新版本 | 适用范围 |
|----------|-------------|----------|----------|
| CIS Microsoft 365 Foundations Benchmark | 120+ 项 | v2.0 | Microsoft 365 全租户 |
| CIS Google Workspace Foundations Benchmark | 80+ 项 | v2.0 | Google Workspace |
| CIS Salesforce Benchmark | 50+ 项 | v1.0 | Salesforce CRM |
| CIS Zoom Benchmark | 30+ 项 | v1.0 | Zoom 视频会议 |

合规评分计算公式：
```
合规评分 = (通过的检查项数 / 总检查项数) × 100%
```

目标：初始 > 70%，持续运行 > 90%。

## 自动修复

SSPM 的修复流程应尽可能自动化：

| 自动化程度 | 修复方式 | 适用场景 | 风险 |
|-----------|----------|----------|------|
| 全自动修复 | SSPM 通过 API 直接修改配置 | 明确无害的配置：关闭公共分享链接、禁用旧版协议 | 需配置自动修复范围白名单 |
| 半自动修复 | 生成修复建议，管理员确认后执行 | 需要业务评估的变更：移除管理员权限 | 需人工审批环节 |
| 手动修复 | 生成工单，分配给对应管理员 | 需要协调多方的变更：OAuth 授权变更 | 修复周期长 |

**推荐自动修复的配置项：**
- 自动关闭公共分享链接
- 禁用不合规的旧版认证协议（IMAP/POP3 不强制 MFA）
- 强制开启审计日志记录
- 自动禁用休眠超过 90 天的账号

**需要审批的配置变更：**
- 修改管理员权限分配
- 删除服务账号
- 修改数据保留策略（涉及数据删除）

## CSPM 集成

SSPM 与 CSPM（云安全态势管理）协同形成完整的云安全态势管理视图：

```
                ┌─────────────────────────────────┐
                │     统一云安全态势管理平台         │
                ├─────────────────────────────────┤
                │                                 │
    ┌───────────▼────────────┐  ┌───────────▼────┐
    │      SSPM (SaaS层)     │  │ CSPM (IaaS层)  │
    │  ┌──────────────────┐  │  │ ┌────────────┐ │
    │  │ M365 配置检查     │  │  │ │ AWS Config │ │
    │  │ Google WS 检查    │  │  │ │ Azure Pol.│ │
    │  │ Salesforce 检查   │  │  │ │ GCP         │ │
    │  │ Slack 检查        │  │  │ │ 合规规则    │ │
    │  └──────────────────┘  │  │ └────────────┘ │
    └────────────────────────┘  └────────────────┘
                │                         │
                └──────────┬──────────────┘
                           │
                   ┌───────▼────────┐
                   │ 统一合规评分    │
                   │ 统一修复跟踪    │
                   │ 统一仪表盘      │
                   └────────────────┘
```

两者共同输出统一的安全态势评分和整改建议，帮助安全团队从全局视角管理云安全。
