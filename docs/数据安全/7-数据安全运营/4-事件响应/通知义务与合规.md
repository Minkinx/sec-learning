# 通知义务与合规

## 法律法规通知时限对比

| 法规 | 通知时限 | 通知对象 | 主要要求 |
|------|----------|----------|----------|
| 中国《个人信息保护法》第57条 | **立即** 采取补救措施 + **72小时内** | 监管机构（网信办/行业主管） | 个人信息泄露、篡改、丢失时履行通知义务 |
| 欧盟 GDPR 第33条 | **72小时内** | 监管机构（DPA） | 个人数据泄露可能导致风险时通知；可延迟通知需提供理由 |
| 美国 CCPA/CPRA | **30天** 整改期 | 用户 | 30天内可补救则避免处罚 |
| 美国 SEC（上市公司） | **4个工作日** | SEC | 2023年12月生效，认定重大网络安全事件即须披露 |
| 韩国 PIPA | **立即**（知悉后） | 韩国个人信息保护委员会 | 超过1万名用户数据泄露须立即通知 |
| 新加坡 PDPA | **尽快** | 个人数据保护委员会 | 造成重大损失或大量数据泄露时须通知 |

## 通知内容标准

根据 PIPL 和 GDPR 的通行要求，数据泄露通知应包含以下要素：

1. **事件描述**：泄露发生时间和范围、发现时间、事件性质
2. **数据类型**：涉及的个人信息类别、数据字段、记录数量
3. **影响评估**：已造成或可能造成的影响、风险评估结论
4. **处置措施**：已采取的应急措施、后续响应计划
5. **联系人信息**：数据保护官（DPO）或安全负责人的联系方式

## 通知对象分类

- **监管机构通知**：向主管部门提交正式报告，格式和内容需符合法定要求，PIPL 要求通知履行个人信息保护职责的部门。
- **受影响个体通知**：通过邮件/短信/电话/公告等方式通知数据主体，语言需简明易懂，说明泄露影响和建议采取的自我保护措施。
- **合作伙伴通知**：当泄露涉及合作方数据或通过合作方渠道泄露时及时通知。
- **媒体与公众**：大规模泄露事件可能需要发布公开声明。

## 跨境通知

当数据泄露涉及多国/多司法管辖区时，需要协调不同法规的通知义务。建议做法：1）最早的通告时限优先（72小时内）；2）统一通知内容模板，按各辖区要求微调；3）指定跨境协调律师（满足律师-客户特权保护）；4）与各监管机构保持透明沟通。

## 通知模板

企业应预先准备通知模板，包含可填补的变量字段。模板应经法务审核，按场景（监管机构/个人/媒体）分版本。通知发出前需经事件响应指挥组（含法务和公关）审批。
