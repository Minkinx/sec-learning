# 泄露追溯技术

## 概述

数据泄露追溯是在事件发生后，通过技术手段定位数据泄露的源头、路径和责任人。追溯能力直接影响事件定责、合规报告和法律诉讼的证据效力。

## 数字水印技术

水印技术通过在数据中嵌入不可见标识，便于泄露后追溯到源头。

- **数字水印（Digital Watermarking）**：在文档/图片/视频中嵌入不可见的数字标识，泄露后通过提取水印识别泄露源。需要抵抗常见变换操作（截图/重编码/格式转换）。
- **结构水印（Structural Watermarking）**：在数据库字段中添加微量噪声或伪记录作为标识，泄露的数据库记录可通过唯一性模式追溯到具体的数据集版本（如员工编号末尾的校验位偏移）。
- **行为水印（Behavioral Watermarking）**：在数据的生成或分发过程中植入访问时间偏移、请求排列顺序等行为特征，通过日志关联确定泄露渠道。

## 蜜标技术

蜜标（Honeytoken）和蜜罐（Honeypot）是主动防御与追溯结合的技术手段：

- **蜜标**：在真实数据中插入虚假但无法区分的记录（如假身份证号、假银行账号），当蜜标数据在未授权渠道出现时，确认数据泄露发生并追溯泄露源。
- **数据库蜜标**：在数据库表中插入虚假客户记录，记录内容具有唯一的特征码，便于追踪。
- **文档蜜标**：在文档中嵌入独特短语或段落，不同分发渠道的内容略有差异（Canary Tokens）。

## 日志链重建

- **时间线重建**：将分散在多个系统的日志（AD 认证日志、数据库审计日志、VPN 日志、DLP 事件）按时间对齐，重建事件全貌。
- **访问路径追溯**：沿着用户访问链追溯（谁->通过什么系统->访问了什么数据->做了什么操作->数据流向哪里）。
- **关联分析**：将账号活动、网络流、文件操作多源关联，补全碎片化的证据链。

## 用户行为时间线

基于多源日志构建用户活动的时间线视图：登录/注销时间、访问的数据对象、命令执行记录、文件传输记录、异常行为标记。时间线视图帮助分析师快速理解事件全貌，定位关键行为节点。

## 溯源定责

溯源的最终目标是归因（Attribution），需要区分：直接责任人（执行泄露操作的用户账号）、间接责任人（权限授予者、系统管理员）、辅助因素（缺失的安全控制）。归因结果需要形成法律可采信的证据链，包括时间戳、日志来源、操作者身份等要素。
