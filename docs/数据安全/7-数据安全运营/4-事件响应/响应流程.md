# 响应流程

## NIST 事件响应生命周期

基于 NIST SP 800-61 的事件响应框架，数据安全事件响应分为以下六个阶段：

### 1. 准备（Preparation）
- 建立数据安全事件响应团队（DSIRT），明确角色和职责
- 制定数据安全事件响应手册（Playbook），覆盖常见场景
- 准备取证工具链（内存取证/磁盘取证/网络取证）
- 定期演练（桌面推演 + 红蓝对抗，至少每季度一次）

### 2. 检测与分析（Detection & Analysis）
- 从 SIEM/SOAR/DLP/UEBA 平台接收告警并确认事件真实性
- 收集事件上下文信息：涉及数据类型、受影响系统、发生时间窗口、用户身份
- 初步确定事件严重等级，决定是否需要启动正式响应流程

### 3. 遏制（Containment）
- 短期遏制：隔离受影响系统、撤销受影响账号权限、阻断网络连接
- 长期遏制：部署临时安全控制（额外监控/加严策略），防止事件扩大
- 取证保全：对受影响系统进行快照和内存镜像，保留日志证据

### 4. 根除（Eradication）
- 确认并消除根本原因（修复漏洞/清除恶意软件/重置凭据）
- 确认数据泄露范围（哪些数据、多少记录、哪些用户受影响）
- 修补安全控制缺口，防止同类事件再次发生

### 5. 恢复（Recovery）
- 逐步恢复受影响系统至正常运行状态
- 加强监控策略，重点观察恢复后的系统行为
- 验证修复措施有效性后再下线应急控制

### 6. 事后复盘（Post-Incident）
- 组织全流程复盘会议，输出事件报告

## 通信计划

事件响应中的通信需覆盖多个利益相关方：内部管理层（定期汇报进展）、法务部门（合规评估和外部沟通）、监管机构（按法律要求履行报告义务）、公关团队（对外声明和媒体应对）、受影响客户（通知义务）、合作伙伴（协同处置）。通信模板和审批流程应在准备阶段完成。

## 工具链

SIEM 提供告警和日志聚合；SOAR 实现剧本自动化和工单编排；案件管理平台（如 TheHive/SOC Prime）跟踪事件处置全程；取证工具（Volatility/Autopsy/FTK）进行深度分析。工具链的预集成（Pre-Integration）是缩短 MTTD/MTTR 的关键。
