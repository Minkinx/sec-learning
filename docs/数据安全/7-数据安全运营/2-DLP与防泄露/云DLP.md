# 云 DLP

## 概述

随着企业业务上云，数据分布在 SaaS、PaaS、IaaS 等多种云环境中，传统网络 DLP 无法覆盖云端数据流动。云 DLP 通过 API 集成和代理部署，弥合云环境与数据安全之间的鸿沟。云 DLP 的核心能力在于：在数据上传到云服务之前、存储在云中期间、以及从云服务下载时，对数据进行持续的检测和保护。

## 部署模式

| 模式 | 工作原理 | 阻断能力 | 延迟影响 | 适用场景 |
|------|----------|----------|----------|----------|
| API 模式 | 通过云服务商 API 扫描已存储数据 | 检测后处置（非实时阻断） | 低（异步扫描） | 已有数据的批量扫描和持续监控 |
| Inline Proxy | 正向代理串联数据流 | 实时检测与阻断 | 中（新增代理跳数） | 实时控制上传和下载流量 |
| Reverse Proxy | 反向代理部署在云服务前 | 实时检测与阻断 | 中（新增代理跳数） | 保护自托管到云的应用 |
| Agent 模式 | 在云工作负载上部署代理 | 实时检测 | 中-低（本地决策） | IaaS 场景云主机保护 |
| Cloud-Native | 使用云服务商原生 DLP 服务 | 视服务而定 | 低（原生集成） | 单一云环境、简化运维 |

## CASB 集成

云 DLP 通常与云访问安全代理（CASB）深度集成，形成完整的云数据保护方案：

| 集成能力 | 说明 | 实现方式 |
|----------|------|----------|
| 影子 IT 发现 | 通过 CASB 发现未授权的云应用使用 | 网络流量 + API 发现 |
| 策略同步 | DLP 策略通过 CASB 统一推送到各云应用 | 标准策略语言 + 适配器转换 |
| 事件关联 | 云 DLP 事件与 CASB 日志关联分析 | 统一事件格式 + SIEM 集成 |
| 用户上下文 | 从 CASB 获取用户身份和设备信息丰富 DLP 决策 | SSO 集成 + 设备姿态评估 |
| 访问控制 | 结合数据敏感度和用户上下文实施条件访问 | CASB 反向代理 + 条件访问策略 |

## 多云策略一致性

在多云环境下保持 DLP 策略一致性是核心挑战。策略定义应遵循"一次定义、多端执行"的原则：

| 步骤 | 内容 | 工具支持 |
|------|------|----------|
| 策略定义 | 使用统一策略描述语言定义 DLP 规则 | 自定义 DSL 或 YAML 策略模板 |
| 策略转换 | 将统一策略转换为各云平台原生格式 | 策略转换适配器/引擎 |
| 策略分发 | 通过 CI/CD pipeline 分发到各云环境 | Terraform/CloudFormation/Pulumi |
| 一致性验证 | 定期验证各环境策略一致性 | 自动化策略对比工具 |
| 冲突检测 | 检测并解决不同环境之间的策略冲突 | 策略分析引擎 |

## 数据驻留控制

云 DLP 需要结合数据驻留要求实施区域级管控：

| 控制机制 | 实现方式 | 适用范围 |
|----------|----------|----------|
| AWS SCP (Service Control Policy) | 使用 aws:RequestedRegion 条件键限制 S3 跨区域复制 | AWS 全组织 |
| Azure Policy | 使用 allowedLocations 策略限制资源创建区域 | Azure 全环境 |
| GCP Organization Policy | 配置 resourcemanager.allowedLocations 限制 | GCP 全组织 |
| 标签策略 | 对数据打标并基于标签控制跨区域流动 | 跨云环境 |
| 实时告警 | 检测到数据移动到不允许区域时触发告警或阻断 | 全环境 |

## 云原生 DLP 服务对比

| 服务 | 核心能力 | 覆盖范围 | 定价 | 适用场景 |
|------|----------|----------|------|----------|
| AWS Macie | S3 桶自动发现、ML 敏感数据分类、仪表盘 | S3 + 支持 Athena 查询的数据库 | 按数据量（$0.10/GB） | AWS 环境敏感数据发现与分类 |
| Microsoft Purview | 统一数据治理与 DLP、Teams/Exchange/SPO/Endpoint | M365 全家桶 + Azure + 第三方 | E5 许可 | Microsoft 365 生态企业 |
| Google DLP | 内容检测 API、模板化分类、去标识化、风险分析 | GCS + BigQuery + 任意 API | 按处理量（$0.01-0.05/unit） | GCP 环境或仅需 API 检测 |
| Amazon DataZone | 数据目录 + 治理 + 数据共享控制 | AWS 数据湖 | 按资产数 | 数据湖场景 |
| Oracle Data Safe | 数据库安全 + 敏感数据发现 + 脱敏 + 审计 | Oracle 数据库（含 OCI） | 按实例数 | Oracle 数据库云场景 |

## 最佳实践

1. **评估先行**：上云前先评估数据敏感度和合规要求，确定哪些数据可以上云、上哪个云、存放在哪个区域。
2. **服务商原生服务优先**：优先使用云服务商提供的原生 DLP 服务，降低部署和运维复杂度。
3. **统一策略管理**：多云环境使用统一策略管理平台（如 CSPM/CNAPP），避免策略碎片化。
4. **分级保护**：根据数据分类分级结果应用不同强度的云 DLP 控制，L4 数据实施最严格管控。
5. **定期审计**：每季度审计云 DLP 覆盖率和策略有效性，根据业务变化调整策略。
