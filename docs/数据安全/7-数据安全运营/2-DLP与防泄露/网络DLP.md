# 网络 DLP

## 概述

网络数据防泄露（Network DLP）在数据流经网络时进行检测与阻断，监控邮件、Web 流量、即时通讯、文件传输等通道。作为企业 DLP 架构的流量层防线，网络 DLP 覆盖所有经过网络边界的数据流。

## 部署模式

- **Inline（在线模式）**：串联部署在网络路径上，实现实时阻断能力，延迟要求严格（通常需低于 1ms），需要考虑高可用性和故障切换策略（bypass 机制）。
- **Out-of-Band（旁路模式）**：通过 TAP/SPAN 端口镜像流量进行分析，无法实时阻断但零延迟影响，适合纯监控审计场景。

## 协议覆盖

| 协议/通道 | 监控内容 | 典型配置 |
|-----------|----------|----------|
| SMTP | 邮件正文和附件 | 集成邮件安全网关，支持 TLS 解密 |
| HTTP/HTTPS | Web 上传、网页邮件 | 与 SWG/代理集成，透明代理 SSL 解密 |
| SMB | 文件共享协议 | 监测文件复制到非授权共享 |
| IM（Teams/Slack） | 即时通讯文件传输 | API 集成或流量解密 |
| FTP/SFTP | 文件传输协议 | 深度内容检测 + 协议合规 |

## 内容检测技术

网络 DLP 的内容检测引擎包括：正则表达式（卡号/身份证号等模式匹配）、精确哈希指纹（对文档进行 SHA256 哈希比对）、模糊指纹（基于向量化的相似度匹配）、ML 分类（训练内容分类模型判断文档类型）。

## TLS 解密挑战

随着 HTTPS 成为默认协议，网络 DLP 面临 TLS 解密的两难：需要部署可信根证书并配置堡垒浏览器实现中间人解密，但会引发隐私合规风险（如 GDPR 禁止监控员工个人流量）。解决方案是采用选择性解密策略，仅对已知 SaaS 域名的流量进行解密分析。

## 集成架构

网络 DLP 需与企业网络基础设施集成：Secure Web Gateway（SWG）提供 HTTP/HTTPS 流量通道；Email Security Gateway 提供邮件通道；CASB 提供 SaaS 流量通道；NDR（网络检测与响应）提供异常行为关联。
