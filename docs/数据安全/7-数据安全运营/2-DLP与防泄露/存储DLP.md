# 存储 DLP

## 概述

存储 DLP（Data-at-Rest Discovery）专注于发现和分类已存储在各类存储系统中的敏感数据，是企业 DLP 架构中数据发现与梳理的核心能力。

## 扫描范围

存储 DLP 的覆盖范围包括：

- **文件共享**：Windows 文件服务器（SMB 共享）、NFS 共享、NAS 设备
- **数据库**：结构化数据存储中的敏感字段（JDBC/ODBC 连接扫描）
- **SharePoint/OneDrive**：通过 Microsoft Graph API 进行在线扫描
- **云存储**：S3 Bucket、Blob Storage、Google Cloud Storage
- **企业内部 Wiki/文档库**：Confluence、SharePoint Online

## 内容分类与标签

扫描引擎根据内容识别结果对文件进行分类标记，分类方法包括：

- **精确识别**：正则表达式匹配（身份证号/信用卡号/护照号）、数据格式校验（Luhn 算法）
- **上下文识别**：基于文件元数据（创建者/部门/文件路径）结合内容判定敏感等级
- **ML 分类**：基于 NLP 的文档内容理解，识别合同、简历、财务报告等文档类型
- **自定义分类器**：支持企业特有的业务数据识别（如客户编号、产品代号）

## 处理流程

发现敏感数据后的处理工作流：

1. **自动分类**：扫描发现并自动标记为对应的敏感等级（L1-L4 或内部/机密/绝密）
2. **告警通知**：通知数据所有者确认数据合理性
3. **处置动作**：基于策略自动执行修复（加密/隔离/删除/匿名化）
4. **验证闭环**：重新扫描确认处置已生效

## 修复自动化

常见修复策略及其适用场景：

| 动作 | 技术实现 | 适用场景 |
|------|----------|----------|
| 隔离 | ACL 移除 + 移动到隔离区 | 过期或无人认领的敏感文件 |
| 加密 | EFS/BitLocker/KMS 加密 | 需要保留的业务敏感文件 |
| 删除 | 安全擦除（DoD 5220.22-M） | 确认不再需要的副本 |
| 匿名化 | 数据脱敏后保留 | 测试/开发环境中的生产数据 |

## 陈旧数据管理

大部分数据泄露涉及过时或无人认领的敏感数据。建立陈旧数据生命周期管理策略：90 天以上未访问的数据标记为冷数据，180 天以上触发归档或删除。定期执行数据清理（Data Sanitization），将存储 DLP 发现的结果与 CMDB 关联，识别 orphaned 数据。
