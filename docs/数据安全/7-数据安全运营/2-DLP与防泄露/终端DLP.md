# 终端 DLP

## 概述

终端 DLP 部署在用户终端设备上，监控和控制数据在终端上的使用行为。作为 DLP 架构中最贴近用户的防线，终端 DLP 能够覆盖离线场景和加密通道场景。

## 管控通道

终端 DLP 覆盖的主要数据通道包括：

- **USB 存储控制**：基于设备白名单（VID/PID）或用户角色的 USB 访问策略，支持只读/写入/完全阻断模式。
- **屏幕截取控制**：拦截 Print Screen 键、截图软件（Snipaste/QQ截图）、录屏软件调用，部分方案支持屏幕水印。
- **剪贴板控制**：监控剪贴板内容，基于敏感内容判定是否允许粘贴到外部应用。
- **打印控制**：基于打印机类型（网络打印机/虚拟 PDF）和数据内容的管控，支持打印水印叠加。
- **文件操作监控**：监控本地文件复制/移动/重命名操作，防止文件脱离受控目录。

## 策略执行模型

| 动作 | 说明 | 适用场景 |
|------|------|----------|
| Block | 直接阻断操作 | 高敏感数据外传 |
| Warn | 弹出警告后允许继续 | 中等风险场景 |
| Allow | 允许操作并记录审计 | 低风险操作审计 |
| Encrypt | 自动加密后允许 | 文件外发场景 |

## 终端代理架构

终端 DLP 以 Agent 形式运行，技术要求包括：低资源占用（CPU < 5% / 内存 < 200MB）、支持离线策略缓存（用户在离线时仍可执行策略）、防卸载保护（需要管理员密码卸载）。主流方案通过内核驱动（minifilter/kext）实现文件系统层拦截。

## 平台支持

| 操作系统 | 支持能力 | 注意事项 |
|----------|----------|----------|
| Windows | 全功能（minifilter 驱动） | 支持文件系统、注册表、进程 |
| macOS | 有限（Endpoint Security Framework） | 需要 TCC 权限授权 |
| Linux | 基础（inotify/fanotify） | 主要支持文件监控 |

## 绕过检测

终端 DLP 需防范的绕过手段：关闭代理进程、使用虚拟化环境、通过加密压缩绕过内容检测、使用系统 API 绕过文件系统钩子。高级终端 DLP 应具备进程完整性校验、行为异常检测、以及与 EDR 联动的防绕过能力。
