# 终端 DLP

## 概述

终端 DLP 部署在用户终端设备上，监控和控制数据在终端上的使用行为。作为 DLP 架构中最贴近用户的防线，终端 DLP 能够覆盖离线场景和加密通道场景——即使终端不在企业网络内（远程办公），仍能执行安全策略。终端 DLP 弥补了网络 DLP 在天花板上的盲区：无法监控的加密 P2P 通信、离线 USB 拷贝、内部网络数据转发等。

## 终端代理架构

终端 DLP 以 Agent 形式运行在用户终端上，技术要求涵盖多个层面：

| 架构组件 | 功能 | 实现技术 |
|----------|------|----------|
| 文件系统驱动 | 拦截文件读/写/复制/移动/删除操作 | Windows: minifilter driver; macOS: Endpoint Security Framework; Linux: fanotify |
| 网络层驱动 | 监控终端网络通信（HTTP/SMTP/FTP） | Windows: WFP (Windows Filtering Platform); LSP (Layered Service Provider) |
| 剪贴板监控 | 监控复制粘贴操作内容 | Windows: SetClipboardViewer API hook |
| 屏幕截取控制 | 拦截截图操作，支持水印叠加 | 进程监控 + 系统 API hook |
| 打印监控 | 监控打印队列和打印内容 | Windows: Print Spooler API hook |
| 设备管控 | USB/蓝牙/光驱等外设的访问控制 | Windows: PnP 设备回调; macOS: IOKit |
| 离线策略缓存 | 终端离线时本地执行策略 | 本地加密存储 + 在线策略同步 |

**核心性能要求：**
- CPU 占用率：< 5%（空闲时 < 1%）
- 内存占用：< 200MB
- 磁盘占用：< 1GB（含缓存和日志）
- 系统启动影响：< 3 秒延迟

## 管控通道

终端 DLP 覆盖的主要数据通道包括：

| 通道 | 监控粒度 | 管控方式 | 绕过难度 |
|------|----------|----------|----------|
| USB 存储控制 | 设备 VID/PID/序列号级别 | 白名单/黑名单/只读/加密 | 中（可尝试 Mount 绕过） |
| 蓝牙传输 | 配对设备类型 | 禁止/仅允许白名单设备 | 低（不支持大规模传输） |
| 剪贴板 | 内容文本 + 来源/目标应用 | 阻断/警告/审计 | 低（驱替 API 直接拦截） |
| 打印 | 打印机类型 + 文档内容 | 禁止打印/水印叠加/审批 | 低（截屏打印绕过） |
| 屏幕截取 | 截图工具进程 + 快捷键 | 拦截/空白/水印 | 中（手机翻拍绕过） |
| 文件操作 | 文件路径 + 目标路径 | 禁止复制/审计追踪 | 中（重命名/压缩绕过） |
| 网络通信 | 目标 IP/域名 + 内容 | 阻断/审计 | 高（加密隧道绕过） |

## 策略执行模型

| 动作 | 说明 | 用户提示 | 适用场景 |
|------|------|----------|----------|
| Block | 直接阻断操作，用户无法执行 | 弹出"操作被安全策略禁止"提示 | 高敏感数据外传（如 L4 数据） |
| Warn | 弹出警告后用户可选择继续或取消 | 弹窗提示数据包含敏感信息 | 中等风险场景（如 L3 数据） |
| Justify | 用户输入理由后可继续 | 弹窗提示输入合规理由并记录审计 | 业务需要但需审计留痕 |
| Allow with Audit | 允许操作并记录审计日志 | 后台记录，用户无感知 | 低风险操作审计（如 L1/L2 数据） |
| Encrypt | 自动加密后允许 | 用户无感知，文件自动加密 | 文件外发/移动介质场景 |
| Quarantine | 隔离文件至安全沙箱 | 文件被自动移至加密沙箱 | 疑似恶意文件/未知类型 |

## 平台支持能力

| 操作系统 | 支持能力 | 驱动模式 | 注意事项 |
|----------|----------|----------|----------|
| Windows 10/11 | 全功能 | minifilter 驱动 | 支持文件系统、注册表、进程、网络全栈拦截 |
| Windows Server | 基础功能（无 GUI 交互） | minifilter 驱动 | 终端 DLP 通常不部署在服务器 |
| macOS (Intel/Apple Silicon) | 有限（Endpoint Security Framework） | 用户态框架 | 需要 TCC 权限授权，Apple Silicon 系统安全限制更多 |
| Linux (Ubuntu/CentOS) | 基础（inotify/fanotify） | 文件系统监控 | 主要支持文件监控，屏幕控制不支持 |
| ChromeOS / Android | 有限（企业 MDM） | MDM API | 需走 EMM/MDM 策略通道 |
| iOS/iPadOS | 有限（MDM + 应用沙箱） | MDM + 应用级 | 系统封闭，DLP 能力远弱于其他平台 |

## 绕过检测与防御

高级终端 DLP 应具备以下防绕过能力：

| 绕过手段 | 检测方法 | 防御措施 |
|----------|----------|----------|
| 关闭 Agent 进程 | 进程存活监控 + 心跳检测 | 管理员密码保护，系统服务方式运行 |
| 卸载 Agent | 防卸载保护 + 注册表/文件完整性校验 | 需管理员审批方可卸载 |
| 虚拟机逃逸 | 检测虚拟化环境标志 | 阻止在虚拟机内运行高敏感操作 |
| 加密压缩绕过 | 调用栈检测 + 高熵文件检测 | 对高熵文件触发深度内容检测 |
| 系统 API 绕过 | 内核驱动级文件系统拦截 | minifilter 驱动级别比用户态 Hook 更难绕过 |
| DLL 注入禁用 | DLL 签名校验 + 进程保护 | 仅允许签名的安全模块加载 |
| 调试进程绕过 | 反调试技术 + 进程句柄保护 | 检测调试器后阻止敏感操作 |

## 最佳实践

1. **分层部署**：对高敏部门（财务/法务/HR）启用全量终端 DLP 管控，普通部门启用轻量审计模式。
2. **用户沟通**：部署前须通知用户终端 DLP 的监控范围和目的，避免隐私争议，建议在员工手册中明确。
3. **离线策略**：确保用户在离线状态下策略仍可执行，策略缓存定期同步（默认每 24 小时更新一次）。
4. **异常行为关联**：终端 DLP 事件应实时上传至 SIEM 平台，与 UEBA 和网络 DLP 事件进行关联分析。
5. **EDR 联动**：与端点检测与响应（EDR）平台联动，当 EDR 检测到恶意行为时自动将终端 DLP 策略切换为严格模式。
