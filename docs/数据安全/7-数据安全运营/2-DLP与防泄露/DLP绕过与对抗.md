# DLP 绕过与对抗

## 常见绕过技术

攻击者和内部用户采用多种方法绕过 DLP 检测，了解这些技术是制定有效对抗策略的前提：

- **加密与压缩**：使用 ZIP/RAR 加密压缩后外传，密码通过带外通道传递。DLP 在无解密密钥时无法检测内容。
- **文件分段与重命名**：将文件分割成多个小片段传输，或将敏感文件重命名为非敏感扩展名（如将 .xlsx 改为 .txt）。
- **图像化绕过**：截取文档截图并保存为图片格式，常规文本 DLP 引擎无法识别图片中的文字内容。
- **URL 编码与 Base64 编码**：将敏感数据编码后分段传输，绕过关键字和正则匹配。
- **分块传输**：将数据分拆为多个小块分段发送，规避基于数据量阈值触发的策略。
- **隐写术**：将数据嵌入图片或音视频文件的冗余位中，检测难度极高。

## DLP 对抗策略

针对上述绕过手段，建立多层对抗防御体系：

| 绕过方式 | 对抗措施 |
|----------|----------|
| 加密压缩 | 根据文件名和熵检测加密文件特征，阻断加密文件外传 |
| 文件分段 | 连接跟踪与会话重建，合并分段行为进行内容检测 |
| 图像绕过 | OCR 文字识别提取图片文字内容，纳入检测范围 |
| 编码绕过 | 解码器链自动识别常见编码格式并解码检测 |
| 分块传输 | 流量重组 + 基于时间窗口的聚合分析 |

## 策略加固

DLP 策略需要针对绕过手段持续加固：1）在策略中增加对加密文件（高熵文件头）的传输阻断；2）对未知文件类型启用深度内容分析；3）配置输出通道的白名单（仅允许经批准的云存储和邮件域名）；4）对批量操作设置二次确认或审批流程。

## 用户意识防范

部分 DLP 绕过属于用户无意识行为（如员工将文件转存到个人网盘以远程办公）。通过安全培训和认知教育降低无意识合规风险。同时建立"无惩罚举报"文化，鼓励用户在发现不便时主动联系安全团队优化策略，而非采取规避手段。

## 纵深防御

DLP 作为数据安全防线之一，无法做到 100% 防护。构建纵深防御体系：网络 DLP + 终端 DLP + 云 DLP 协同工作，辅以 UEBA 行为异常检测和数据访问审计，即使单层被绕过也能被其他层次捕获。
