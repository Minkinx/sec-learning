# 数据安全态势感知

## 定义与框架

数据安全态势感知（Data Security Situational Awareness）参考 NIST SP 800-160 中态势感知的定义，包含感知（Perception）、理解（Comprehension）、预测（Projection）三个层次。在数据安全领域，具体体现为对数据资产、访问行为、威胁态势和合规状态的全面可视与智能决策。

## 架构设计

数据安全态势感知平台通常采用分层架构设计：

| 层级 | 功能 | 技术组件 | 数据源 |
|------|------|----------|--------|
| 数据采集层 | 多源日志和事件收集 | Fluentd/Logstash/Kafka | 数据库审计、DLP、IAM、网络流 |
| 存储与计算层 | 海量数据存储与实时分析 | ClickHouse/Elasticsearch/Spark Streaming | 归一化后的结构化事件数据 |
| 分析引擎层 | 关联规则、ML模型、UEBA | Splunk ES/QRadar/自研引擎 | 基线模型、规则库、威胁情报 |
| 展示与决策层 | 可视化大屏、报表、告警 | Grafana/Kibana/自研Portal | 聚合指标、KPI、风险评分 |

## 关键指标体系

态势感知平台应构建自上而下的 KPI 体系：

- **覆盖度指标**：受监控的数据资产覆盖率、已分类数据的比例、审计覆盖的系统数量、日志源完整性评分
- **风险指标**：高风险用户数、未修复的漏洞数、已暴露的敏感数据量（记录数/存储量）、攻击面暴露评分
- **响应指标**：MTTD（平均检测时间）、MTTR（平均响应时间）、告警疲劳度（告警量/分析师人数）、误报率
- **合规指标**：不合规配置数量、审计日志完整性、数据留存合规率、法规映射覆盖率

## 数据资产清单映射

态势感知的基础是维护实时更新的数据资产清单，需要与 CMDB 和扫描工具集成，自动发现以下维度：

| 维度 | 描述 | 自动发现来源 |
|------|------|-------------|
| 数据源类型 | 数据库/文件/云存储/大数据平台 | CMDB + 扫描器 |
| 地理位置 | 数据中心/云区域 | 云服务商标签 |
| 所有者 | 业务负责人/数据管家 | 企业目录（AD/LDAP）|
| 分类标签 | L1-L4 敏感度分级 | 数据分类工具输出 |
| 最近访问时间 | 最后访问时间戳 | 审计日志解析 |
| 安全控制状态 | 加密/脱敏/访问控制状态 | 安全策略检查引擎 |

## 风险热力图

风险热力图以矩阵形式展示风险分布，横轴为数据敏感度（从公开到高度敏感），纵轴为风险等级（从低到严重），气泡大小代表受影响的数据量。该视图帮助管理层快速定位高风险区域，指导安全投入优先级。风险评分公式示例：

```
风险评分 = 敏感度权重 × 暴露因子 × (1 - 控制有效性)
```

其中敏感度权重根据分类分级结果赋值（L4=10, L3=7, L2=4, L1=1），暴露因子基于可访问该数据的用户数和网络可达性计算。

## 实时监控与历史分析

- **实时监控**：关注正在发生的异常事件（大量数据下载、非工作时间批量访问），触发阈值即告警。数据刷新延迟要求在分钟级以内。采用流处理引擎（如 Flink/Spark Streaming）实现秒级延迟的事件关联。
- **历史分析**：基于时间窗口（24h/7d/30d）的聚合趋势分析，发现长期异常模式（权限蔓延、持续的数据外流），使用 ClickHouse/Druid 等 OLAP 引擎处理。历史分析还应包括同比/环比基准对比，识别逐步恶化的趋势。

## 关联分析场景

| 分析场景 | 输入数据源 | 检测规则示例 | 响应动作 |
|----------|-----------|-------------|----------|
| 失陷账号数据窃取 | VPN 日志 + 数据库审计 + DLP | 非常规登录后触发大量数据库查询并伴随 DLP 事件 | 锁定账号 + 阻断会话 |
| 内部威胁数据囤积 | 文件服务器审计 + USB 控制 | 单用户频繁复制大量文件到可移动介质 | 触发审批流程 + 录屏取证 |
| 合规配置漂移 | CSPM + 配置管理库 | 数据库公网访问权限突然开启 | 自动回滚 + 告警通知 |
| 横向移动检测 | AD 认证 + 网络流 + 数据访问 | 非管理员账号尝试访问多个数据库服务器 | 隔离端点 + 事件升级 |

## SOC/NOC 集成

数据安全态势感知应融入企业安全运营体系：

- **与 SOC 集成**：对接 SIEM/SOAR 平台实现事件流转，告警自动创建工单，同步 Playbook 执行状态
- **与 NOC 集成**：结合网络可视性发现数据外传路径，识别异常带宽消耗和 C2 通信
- **输出产物**：每日安全日报（前日事件摘要）、每周风险趋势报告（风险评分变化）、每月合规报告（法规匹配度）、季度管理层报告（战略级视图）
