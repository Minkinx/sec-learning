# 数据安全态势感知

## 定义与框架

数据安全态势感知（Data Security Situational Awareness）参考 NIST SP 800-160 中态势感知的定义，包含感知（Perception）、理解（Comprehension）、预测（Projection）三个层次。在数据安全领域，具体体现为对数据资产、访问行为、威胁态势和合规状态的全面可视与智能决策。

## 关键指标体系

态势感知平台应构建自上而下的 KPI 体系：

- **覆盖度指标**：受监控的数据资产覆盖率、已分类数据的比例、审计覆盖的系统数量
- **风险指标**：高风险用户数、未修复的漏洞数、已暴露的敏感数据量（记录数/存储量）
- **响应指标**：MTTD（平均检测时间）、MTTR（平均响应时间）、告警疲劳度（告警量/分析师人数）
- **合规指标**：不合规配置数量、审计日志完整性、数据留存合规率

## 数据资产清单映射

态势感知的基础是维护实时更新的数据资产清单，需要与 CMDB 和扫描工具集成，自动发现以下维度：数据源类型（数据库/文件/云存储）、地理位置、所有者、分类标签、最近访问时间、安全控制状态。

## 风险热力图

风险热力图以矩阵形式展示风险分布，横轴为数据敏感度（从公开到高度敏感），纵轴为风险等级（从低到严重），气泡大小代表受影响的数据量。该视图帮助管理层快速定位高风险区域，指导安全投入优先级。

## 实时监控与历史分析

- **实时监控**：关注正在发生的异常事件（大量数据下载、非工作时间批量访问），触发阈值即告警。数据刷新延迟要求在分钟级以内。
- **历史分析**：基于时间窗口（24h/7d/30d）的聚合趋势分析，发现长期异常模式（权限蔓延、持续的数据外流），使用 Spark/ClickHouse 等 OLAP 引擎处理。

## SOC/NOC 集成

数据安全态势感知应融入企业安全运营体系：与 SOC 平台的工单系统对接实现事件流转，与 NOC 的网络可视性结合发现数据外传路径。典型输出包括每日安全日报、每周风险趋势报告、每月合规报告。
