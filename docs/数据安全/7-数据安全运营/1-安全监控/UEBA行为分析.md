# UEBA 行为分析

## 定义与原理

用户与实体行为分析（User and Entity Behavior Analytics, UEBA）通过建立主体（用户、设备、应用、数据）的正常行为基线，利用统计和机器学习算法识别偏离基线的异常行为，从而检测内部威胁、账号接管和数据泄露。

## 基线建立

UEBA 的基线分为三个层次：

- **历史基线**：基于主体自身的历史行为数据（过去 30/90 天的访问模式），周期性更新以反映正常变化。
- **同群组基线**：将相似角色（如同一部门、同一职级）的用户聚合建模，检测偏离同群组的行为（如普通员工访问高管权限）。
- **时间序列基线**：考虑时间周期性（工作日 vs 周末、上班时段 vs 非工作时间），消除周期性因素的干扰。

## 异常检测算法

| 方法 | 技术栈 | 适用场景 |
|------|--------|----------|
| 统计方法 | Z-Score、MAD、分位数 | 单一维度的简单异常（如下载量暴增） |
| 传统 ML | Isolation Forest、One-Class SVM | 多维特征的异常（时间+地点+频率+数据量） |
| 深度学习 | Autoencoder、LSTM | 序列行为异常（时序模式偏离） |
| 图分析 | 图神经网络（GNN） | 访问关系异常（实体之间权限传播分析） |

## 数据源与实体类型

UEBA 引擎整合的信号包括：身份认证日志（SSO/VPN）、数据访问日志、DLP 事件、邮件元数据、终端行为数据。核心实体类型及分析维度：

- **用户**：登录时间/地点、访问数据量、访问频次、失败登录次数
- **设备**：操作系统/补丁版本、已装软件、网络连接目标
- **应用**：API 调用频率、数据返回量、错误率
- **数据**：分类标签、访问者身份、最近修改时间

## 告警研判

UEBA 告警需经过三级研判流程：自动降噪（过滤已知误报）-> 分析师调查（关联上下文数据）-> 升级处置（确认恶意行为后移交 IR 团队）。常用工具对比：Splunk UBA（机器学习算法丰富）、Microsoft UEBA（与 Azure AD/Defender 深度集成）、Exabeam（基于时间线的行为分析引擎）。
