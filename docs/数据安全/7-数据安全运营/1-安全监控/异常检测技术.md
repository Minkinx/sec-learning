# 异常检测技术

## 检测范式对比

数据安全场景中的异常检测可分为规则驱动和机器学习驱动两大范式：

| 维度 | 规则驱动 | 机器学习 |
|------|----------|----------|
| 可解释性 | 高（规则逻辑透明，可直接审计） | 中低（模型黑盒，需 SHAP/LIME 解释） |
| 维护成本 | 高（人工维护规则库，需领域专家持续更新） | 中（需要数据标注与训练 pipeline） |
| 未知攻击 | 无法检测（仅匹配已知模式） | 具备检测能力（发现孤立异常） |
| 误报控制 | 规则可精确调整，FP 可控 | 需要阈值调优，存在概念漂移 |
| 部署速度 | 快（即写即用，经验驱动） | 慢（需要训练周期和数据积累） |
| 适应变化 | 低（规则需要人工更新响应环境变化） | 中高（模型可定期增量学习） |

实践中通常采用混合模式：规则引擎处理已知威胁的快速检测，ML 模型发现未知和变异威胁，两者互补形成完整检测体系。

## 异常检测方法分类

### 统计方法

基于数据分布的统计学特征判断异常值：

| 方法 | 原理 | 适用数据类型 | 优缺点 |
|------|------|-------------|--------|
| Z-Score | 计算数据点与均值的标准差距离 | 正态分布数据 | 简单有效，但对非正态分布不鲁棒 |
| 修正 Z-Score (MAD) | 使用中位数和绝对中位差 | 含离群点的数据 | 对异常值更鲁棒（Robust） |
| 箱线图/四分位距 | 基于 Q1-1.5×IQR 和 Q3+1.5×IQR | 任意分布 | 可视化直观，阈值的统计意义明确 |
| Grubbs 检验 | 单变量离群点显著性检验 | 近似正态分布 | 有统计学显著性支撑 |

### 机器学习方法

- **监督学习**：需要高质量标注数据集，适用于已知攻击模式检测（如恶意软件流量特征），在数据安全领域标注成本较高。常用算法包括 XGBoost、Random Forest、SVM。
- **无监督学习**：自动发现异常簇群，适用于基线未知的场景（如新业务系统接入）。典型算法：DBSCAN（密度聚类）、Autoencoder 重构误差检测、GMM（高斯混合模型）。
- **半监督学习**：仅标注正常行为数据，检测偏离正常模式的异常，最大程度降低标注需求，是数据安全领域的主流选择。典型方法：One-Class SVM、Isolation Forest。

### 深度学习与序列分析

| 方法 | 原理 | 适用场景 | 训练要求 |
|------|------|----------|----------|
| Autoencoder | 压缩-重构误差检测 | 多维特征异常 | 仅需正常样本训练 |
| LSTM/GRU | 时序预测误差异常 | 序列日志异常（如 API 调用序列） | 需时序数据 |
| Transformers | 自注意力机制捕捉长程依赖 | 复杂行为序列异常 | 数据量大、计算资源高 |
| GAN (生成对抗) | 生成器与判别器对抗训练 | 图像/流量异常检测 | 训练不稳定 |

## 特征工程

数据访问场景的核心特征维度：

| 特征类别 | 具体特征 | 特征描述 | 示例值 |
|----------|----------|----------|--------|
| 时间特征 | 访问时间段编码 | 将访问时间编码为一天中的时段 | 上班时段(0)、非上班(1) |
| 时间特征 | 访问间隔 | 两次连续请求的时间间隔 | 平均间隔 3.2 秒 |
| 空间特征 | 源 IP 地理位置 | 解析 IP 对应的城市/国家 | 北京、中国 |
| 空间特征 | Geo-Velocity | 短时间内的地理位置移动速度 | 15 分钟内北京→上海(不可达) |
| 量级特征 | 查询返回行数 | 单次查询返回的记录数 | 50,000 行 |
| 量级特征 | 传输数据量 | 单位时间下载总量 | 250 MB/小时 |
| 频率特征 | 访问频次 | 单位时间 API 调用次数 | 500 次/分钟 |
| 频率特征 | 失败率 | 失败请求占比 | 35% |
| 行为序列 | 操作序列模式 | 操作的排列组合顺序 | SELECT→SELECT→EXPORT |
| 同群组对比 | 群组偏差分数 | 对同群组行为均值的偏离程度 | 2.5 倍标准差 |

## 误报管理

误报是异常检测的最大挑战，业界采用分层处理策略：

1. **自动化规则过滤**：白名单机制 + 阈值放宽，对已知的合法业务模式（如定时任务、ETL 作业）建立例外白名单。
2. **上下文关联降噪**：整合 CMDB 数据判定业务合理性，如访问来源是否为已知运维终端的 IP 段，行为是否为审批通过的变更窗口。
3. **分析师反馈闭环**：FP 标记后自动调整模型权重，将分析师确认的误报信号反馈到训练 pipeline 更新模型。
4. **告警聚合与去重**：将相同实体、相同模式的连续告警聚合成一个事件，减少告警风暴。采用时间窗聚合（如 10 分钟内同类告警合并）。

## 基准测试

数据安全异常检测的常用基准数据集包括：

| 数据集 | 来源 | 规模 | 适用场景 | 特点 |
|--------|------|------|----------|------|
| LANL 内部威胁 | 洛斯阿拉莫斯国家实验室 | 约 1.2 亿条日志 | 内部威胁检测 | 真实网络环境数据 |
| CERT Insider Threat | 卡内基梅隆大学 CERT | 约 3200 万条 | 内部威胁基准 | 合成数据但含真实场景 |
| AWS S3 Access Logs | 自构建 | 视业务规模 | 云存储异常检测 | 需自行构建基线 |
| CIC-IDS2017 | 加拿大网络安全研究所 | 约 300 万条 | 网络入侵检测 | 含多种攻击流量 |
| HDFS Logs | 公开基准 | 1100 万+ 条 | 系统日志异常 | 时序异常检测基准 |

评价指标应重点考察不平衡数据集下的 Precision-Recall 曲线（PR-AUC），避免使用准确率（Accuracy）造成误导。在数据安全场景中，Precision（精确率）= TP/(TP+FP) 和 Recall（召回率）= TP/(TP+FN) 的 F1-Score 是核心评价指标。
