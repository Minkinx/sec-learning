# 异常检测技术

## 检测范式对比

数据安全场景中的异常检测可分为规则驱动和机器学习驱动两大范式：

| 维度 | 规则驱动 | 机器学习 |
|------|----------|----------|
| 可解释性 | 高（规则逻辑透明） | 中低（模型黑盒） |
| 维护成本 | 高（人工维护规则库） | 中（数据标注与训练） |
| 未知攻击 | 无法检测 | 具备检测能力 |
| 误报控制 | 规则可精确调整 | 需要阈值调优 |
| 部署速度 | 快（即写即用） | 慢（需要训练周期） |

## 学习方法选择

- **监督学习**：需要高质量标注数据集，适用于已知攻击模式检测（如恶意软件流量特征），在数据安全领域标注成本较高。
- **无监督学习**：自动发现异常簇群，适用于基线未知的场景（如新业务系统接入），典型算法包括 DBSCAN、Autoencoder。
- **半监督学习**：仅标注正常行为数据，检测偏离正常模式的异常，最大程度降低标注需求，是数据安全领域的主流选择。

## 特征工程

数据访问场景的核心特征维度：

- **时间特征**：访问时间段（上班/非上班）、访问时长、两次访问间隔、相对于历史平均时间的偏差
- **地理位置特征**：源 IP 地理位置、Geo-Velocity（不可达距离的短时登录）、VPN 状态
- **数据量特征**：单次查询返回行数、单位时间内下载总量、同比变化率
- **频率特征**：单位时间访问次数、失败率、API 调用频率分布
- **同群组对比特征**：与同角色用户的行为差异度、与历史自身行为偏差

## 误报管理

误报是异常检测的最大挑战，业界采用分层处理策略：1）自动化规则过滤（白名单 + 阈值放宽）；2）上下文关联降噪（整合 CMDB 数据判定业务合理性）；3）分析师反馈闭环（FP 标记后自动调整模型权重）。

## 基准测试

数据安全异常检测的常用基准数据集包括：LANL 内部威胁数据集、CERT Insider Threat 数据集、AWS S3 Access Logs（自构建）。评价指标应重点考察不平衡数据集下的 Precision-Recall 曲线，避免使用准确率（Accuracy）造成误导。
