# 业务维度分类

> 从业务视角对数据进行分类，是数据分类分级的基础步骤。

## 常见的业务分类维度

### 按数据来源
- **客户数据**：用户个人信息、交易记录、行为数据
- **财务数据**：财务报表、审计记录、预算信息
- **人力资源数据**：员工档案、薪酬信息、绩效评估
- **研发数据**：源代码、设计文档、技术专利
- **供应链数据**：供应商信息、采购记录、物流数据

### 按数据用途
- **业务运营数据**：支撑日常业务运转
- **分析决策数据**：用于商业智能、报表分析
- **合规监管数据**：满足监管报送要求
- **产品研发数据**：产品设计与迭代

### 按数据结构
- **结构化数据**：数据库表、CSV文件
- **半结构化数据**：JSON、XML、日志文件
- **非结构化数据**：文档、图片、音视频

## 分类方法实施

1. 业务部门牵头，数据治理团队配合
2. 梳理业务流程，识别各环节涉及的数据
3. 建立数据分类体系表
4. 持续维护和更新分类体系

## 最佳实践

- 分类标准应兼顾业务理解和技术实现
- 分类体系不宜过细（建议5-8个一级分类）
- 定期复盘，确保分类与业务实际一致
- 建立"分类-业务场景"映射表，便于运营人员理解
- 初期可采用"标签"方式，避免陷入分类粒度争论
