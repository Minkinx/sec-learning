# 分类标准制定流程

> 建立科学、可执行的分类标准体系是数据安全治理的基石。

## 制定步骤

1. **调研阶段**：盘点现有数据资产，整理业务线数据清单，收集相关监管法规要求
2. **设计阶段**：确定分类维度（业务维度+敏感维度双轴），定义分类层级（建议不超过四级），制定分类命名规范
3. **评审阶段**：组织业务部门、法务合规、安全团队、数据治理四方联合评审
4. **发布阶段**：经数据安全委员会或 CIO 审批后正式发布，嵌入数据安全管理制度
5. **培训阶段**：对数据所有者、数据使用者、安全运营人员开展分类标准培训
6. **推行阶段**：选取试点数据域，跑通分类落地全流程，再全面推行

## 利益相关方参与

| 角色 | 职责 | 参与节点 |
|------|------|---------|
| 法务/合规部 | 提供法规解读，确保分类满足监管要求 | 全程参与 |
| 业务部门 | 定义业务数据语义，确认分类合理性 | 调研、评审 |
| 安全团队 | 提供技术可行性意见，输出保护策略 | 设计、评审、推行 |
| 数据治理 | 维护数据目录，组织分类落地 | 全程主导 |
| IT/平台团队 | 技术实现自动化标定和分类标记 | 设计、推行 |

## 与监管要求对标

分类标准必须能够映射到主要法规的数据分类要求：
- 《数据安全法》第21条：核心数据、重要数据、一般数据三级
- 《个人信息保护法》：一般个人信息、敏感个人信息
- 行业监管：金融（C1/C2/C3）、医疗（电子病历分级）、电信（用户个人信息保护）
- 跨境场景：GDPR、CCPA 等域外法规对应映射

建议制作"法规-分类"交叉对照表，确保分类标准兼容国内外合规要求。

## 审批与治理流程

- 牵头部门起草分类标准 V1.0
- 数据安全工作组（虚拟组织）初审，提出修订意见
- 数据安全委员会（高管参与）终审批准
- 标准发布后，纳入公司内控体系，变更须走变更管理流程
- 标准修订频率：小幅修订按需，重大修订每年一次

## 定期评审周期

- **季度**：检查新上线业务是否纳入分类体系，新增数据字段是否完成分类
- **年度**：全面复审分类标准是否与业务现状和法规变化匹配
- **触发式**：法规更新、重大安全事故、新业务线成立时立即启动专项评审
