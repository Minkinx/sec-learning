# 数据标定自动化

> 利用自动化手段提升数据分类标定的效率和准确性。

## 自动化 vs 人工标定

| 维度 | 自动化标定 | 人工标定 |
|------|-----------|----------|
| 速度 | 百万级字段/小时 | 数百字段/人天 |
| 准确率 | 85%-95%（取决于数据质量） | 95%+（取决于人员经验） |
| 一致性 | 高，规则统一 | 低，不同人员标准不一 |
| 覆盖范围 | 全量数据 | 抽样/重点数据 |
| 适用场景 | 结构化数据、常规文本 | 复杂语义、特殊业务数据 |
| 成本 | 一次性工具投入 + 持续模型维护 | 持续人力投入 |

**建议策略**：自动化初筛 + 人工复核纠偏，关键数据域的最终分类结果必须人工确认。

## 基于机器学习的分类方法

### NLP 非结构化数据分类
- **文本分类模型**：基于 BERT/RoBERTa 等预训练模型微调，对文档、邮件、聊天记录进行内容分类
- **实体识别**：NER 技术提取文本中的敏感实体（身份证号、手机号、银行卡号等）
- **主题建模**：LDA 等算法识别大批量文档的主题分布，辅助标签生成
- 典型流程：文档预处理 -> 向量化 -> 模型推理 -> 置信度评分 -> 人工审核低置信度结果

### 模式匹配与正则表达式
- 适用于结构化数据字段分类（如数据库列名、API 返回字段）
- 基于字段名语义（如 `id_card`、`phone`）+ 数据模式（正则验证）+ 数据分布特征综合判断
- 可结合数据字典和历史标注数据进行有监督学习

## 工具能力对比

| 能力项 | 商业工具（如 BigID、Varonis） | 开源工具（如 Amundsen、DataHub） | 自研方案 |
|-------|------------------------------|--------------------------------|---------|
| 开箱即用 | 高 | 中 | 低 |
| 定制能力 | 中 | 高 | 最高 |
| 合规模板 | 丰富 | 需自行构建 | 完全可控 |
| 支持成本 | 商业支持 | 社区支持 | 自维护 |
| 采购周期 | 3-6个月 | 1-2周部署 | 取决于团队规模 |

## 准确率指标

- **精确率（Precision）**：标定为敏感的字段中，真正敏感的比例。低精确率意味着误报多，影响业务使用
- **召回率（Recall）**：真正敏感的字段中，被正确标定的比例。低召回率意味着漏报多，存在合规风险
- **F1-Score**：精确率与召回率的调和平均，综合评价分类效果
- 建议目标：精确率 > 90%，召回率 > 95%（合规场景偏向高召回率）

## 误报/漏报处理

1. 建立误报/漏报反馈通道，业务人员可对自动标定结果进行"打标"修正
2. 修正数据回流至训练集，定期（月度）更新分类模型
3. 严重漏报（如核心数据未被识别）应触发安全事件流程
4. 统计误报率趋势，作为模型优化和工具选型的关键指标
