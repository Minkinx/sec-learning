# 核心数据

> 数据安全领域的最高保护等级，一旦泄露或破坏将造成重大国家安全损害。

## 法律定义与范围

《数据安全法》第21条规定，关系国家安全、国民经济命脉、重要民生、重大公共利益的数据属于国家核心数据，实行更加严格的管理制度。具体范围包括但不限于：

- **国家安全领域**：国防情报、军事设施数据、涉密地理信息
- **关键基础设施**：核心生产控制系统数据、电网拓扑、水利枢纽运行数据
- **重大科研项目**：国家科技重大专项成果、核心技术专利群
- **敏感公共数据**：大规模人口基因数据、传染性疾病监测数据

## 核心数据的识别标准

1. 国家安全风险评估——数据泄露是否可能被境外势力利用
2. 经济运行影响评估——数据破坏是否导致重要行业瘫痪
3. 公共安全影响评估——数据滥用是否引发大规模公共事件
4. 不可替代性评估——数据的唯一性和恢复难度

## 处理要求

| 控制项 | 要求 |
|--------|------|
| 访问控制 | 最小权限原则，多因素认证，逐次审批 |
| 存储加密 | 国密 SM4/SM2 加密，密钥分片管理 |
| 传输加密 | 国密 SSL/TLS，专线传输 |
| 审计日志 | 全量操作日志，实时告警，不可篡改存储 |
| 备份恢复 | 异地灾备，加密备份，定期恢复演练 |
| 数据处理 | 禁止出境；境内处理须在安全可控环境 |
| 人员管理 | 背景审查，安全培训，签订保密协议 |

## 加密标准

- **存储加密**：必须采用国家密码管理局认定的商用密码算法（SM2/SM3/SM4）
- **传输加密**：国密 TLS 1.3 + 证书双向认证
- **密钥管理**：硬件安全模块（HSM）托管，密钥生命周期管理，定期轮换

## 事件通报机制

核心数据发生安全事件时，须：
1. 30分钟内启动应急响应，通报数据安全负责人
2. 2小时内向行业主管监管部门初步报告
3. 24小时内提交书面事件报告
4. 配合国家安全机关展开调查，保留现场证据

## 日常管理

- 定期（半年）开展核心数据安全风险评估
- 每季度进行核心数据权限审查和回收
- 核心数据处理系统须通过等保三级以上测评
- 建立核心数据资产管理台账，动态更新
