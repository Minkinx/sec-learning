# 重要数据

> 介于核心数据与一般数据之间的保护等级，泄漏或破坏将造成严重损害。

## 法律定义

《数据安全法》将重要数据定义为"一旦遭到篡改、破坏、泄露或者非法获取、非法利用，可能危害国家安全、经济运行、社会稳定、公共健康和安全等的数据"。与核心数据的区别在于损害程度——核心数据造成"重大"损害，重要数据造成"严重"损害。

## 重要数据识别指南

参照《重要数据识别指南》（征求意见稿），重要数据的识别维度包括：

1. **经济运行**：未公开的宏观经济统计数据、重点行业产能数据、关键原材料储备数据
2. **社会稳定**：大规模人口流动数据、敏感舆情数据、重要物资供应链数据
3. **公共健康**：疫情防控数据、药品不良反应监测数据
4. **科技发展**：国家科技计划项目关键技术参数、核心技术领域论文数据
5. **国家安全**：地理测绘数据（超精度）、海洋环境监测数据

## 行业特定重要数据目录

| 行业 | 重要数据示例 | 参考依据 |
|------|-------------|---------|
| 金融 | 反洗钱大额交易数据、征信基础数据 | 《金融数据安全 数据安全分级指南》（JR/T 0197） |
| 能源 | 石油储备数据、电网实时负荷数据 | 《电力行业数据分类分级指南》 |
| 电信 | 基础电信网络拓扑、大规模用户通信信令数据 | 《基础电信企业数据分类分级方法》 |
| 交通 | 铁路调度数据、民航航路数据 | 行业主管部门规定 |
| 医疗 | 大规模基因测序数据、突发公共卫生事件监测数据 | 《健康医疗大数据安全管理办法》 |
| 工业 | 核心生产工艺参数、工业控制系统配置 | 《工业数据安全管理办法》 |

## 安全保护措施

- 遵循"数据分类分级->风险评估->安全措施"的阶梯式管理
- 重要数据处理系统须通过等保二级以上测评（推荐等保三级）
- 访问须基于 RBAC + ABAC 双重控制，留痕审计
- 数据脱敏或匿名化后方可进入测试/开发环境

## 跨境传输限制

- 重要数据原则上应在境内存储和处理
- 确需出境的，须开展数据出境安全评估，通过国家网信部门组织的安全评估后方可出境
- 评估内容包括：出境目的合法正当必要性、风险评估、接收方保障能力、合同约束力等
- 建议定期（年度）重新评估出境场景的合规性

## 重要数据目录维护

- 建立重要数据目录，明确数据名称、所属系统、责任部门、保护等级
- 定期（每半年）更新，重大业务变更后即时更新
- 重要数据目录本身属于敏感信息，应限制访问
