# 分类分级工具选型

> 选择合适的工具能够大幅提升分类分级工作的效率和质量。

## 供应商格局

| 类别 | 代表厂商/项目 | 定位 |
|------|-------------|------|
| 数据安全平台 | BigID、Varonis、Securiti | 全生命周期数据安全管理，含分类分级 |
| 数据目录平台 | Collibra、Alation、DataHub | 数据治理和目录，含轻量分类分级 |
| 数据库安全 | Imperata、Guardium | 数据库活动监控 + 数据发现分类 |
| 云原生 | AWS Macie、阿里云SDDP、GCP DLP | 云上数据的自动发现和分类 |
| 开源 | DataHub、Amundsen、Apache Atlas | 数据治理基础能力，需定制开发分类能力 |

## 评估标准

### 准确率
- 字段级分类精确率和召回率（核心指标）
- 对复合字段（如地址=省+市+区+街道）的识别能力
- 对上下文敏感的识别（如 "password" 字段名 vs "password_reset_token" 的区分）
- 建议采购前进行 PoC 验证，使用真实数据评估准确率

### 可扩展性
- 支持的数据源类型（关系型数据库、NoSQL、对象存储、大数据平台、SaaS 应用）
- 扫描性能和并发能力（百万字段/小时）
- 是否支持分布式部署、横向扩展
- 是否支持自定义分类规则和插件机制

### 合规报告
- 是否内置中国法规模板（《数据安全法》《个保法》《等保》）
- 是否支持 GDPR、CCPA 等多法规报告
- 报告格式是否可定制导出（PDF、Excel、API）

### 集成能力
- 是否提供标准 API 对接 DLP、CASB、SIEM 等安全系统
- 是否支持与 CMDB、数据目录平台的双向集成
- 是否支持 SSO、RBAC、审计日志

## 商业方案 vs 开源方案

| 维度 | 商业方案 | 开源方案 |
|------|---------|---------|
| 功能完整度 | 开箱即用 | 需二次开发 |
| 实施周期 | 2-4周部署 | 1-3个月定制 |
| 合规模板 | 预置 | 需自行构建 |
| 支持服务 | 原厂/代理商 | 社区/自维护 |
| 许可成本 | 按数据量/连接器计费 | 免费（运维人力成本） |
| 推荐场景 | 合规要求紧迫、团队规模有限 | 有数据工程团队、定制需求强 |

## RFI 流程

1. **需求梳理**：明确数据源种类、数据量级、合规要求、集成要求
2. **初筛**：基于公开资料和行业报告筛选 3-5 家候选供应商
3. **发出 RFI**：以标准化问卷形式获取厂商功能、部署、支持信息
4. **方案演示**：要求厂商针对真实场景进行演示
5. **PoC 验证**：在隔离环境中使用真实数据测试（至少2周）
6. **综合评分**：从功能、性能、合规、成本、生态 5 个维度打分

## PoC 方法论

- **数据集**：选取包含结构化、半结构化、非结构化数据的真实样本
- **测试用例**：至少覆盖 10 种常见敏感数据类型
- **评估指标**：精确率 > 90%、召回率 > 95%、吞吐量达标
- **否决条件**：核心数据漏识别率 > 5%、无法对接关键数据源
