# 数据资产测绘

> 构建数据流转的可视化视图，理清数据从哪里来、到哪里去。

## 数据流映射

数据流映射（Data Flow Mapping）是数据资产测绘的核心环节，目标是绘制数据在组织内的完整流转路径：

1. **数据入口识别**：确定数据从何处进入组织（用户注册页面、API 接口、文件上传、第三方数据导入等）
2. **处理节点识别**：数据经过哪些系统和中间件（ETL、消息队列、API Gateway、微服务等）
3. **存储节点识别**：数据持久化在哪些位置（数据库、对象存储、缓存、搜索引擎等）
4. **出口识别**：数据流向何处（第三方、用户展示、报表导出、跨境传输等）

## 数据血缘

数据血缘（Data Lineage）追踪数据从源头到消费端的完整链路：
- **字段级血缘**：追踪单个数据字段的来源和变换过程（如用户姓名从注册表 -> ETL转换 -> 数据仓库维度表）
- **表级血缘**：追踪数据表之间的依赖关系（上游表 -> 下游表）
- **系统级血缘**：追踪跨系统的数据流向（CRM -> 数据湖 -> BI 报表）

血缘分析的关键价值：
- 当发现敏感数据时，可快速追溯影响范围
- 当上游数据发生变更时，可评估下游影响
- 合规审计时，可清晰证明数据处理的合法性

## 关系映射

数据资产之间的关系包括：
- **依赖关系**：系统A依赖数据库B，报表C依赖数据集D
- **复制关系**：生产库数据同步至只读副本、灾备站点
- **转换关系**：原始数据经ETL转为分析数据
- **共享关系**：数据向第三方/合作伙伴开放

建议使用图数据库（如 Neo4j）存储关系，支持复杂的关联查询和路径分析。

## 可视化方案

| 方案 | 适用场景 | 代表工具 |
|------|---------|---------|
| 节点-连线图 | 数据流、系统依赖 | d3.js、Cytoscape.js、DataHub |
| 桑基图 | 数据流向和流量 | ECharts、Tableau |
| 矩阵图 | 系统间数据共享矩阵 | Excel、BI 工具 |
| 拓扑图 | 网络层数据流 | NetFlow 分析工具 |

## 常用工具

- **DataHub**（LinkedIn 开源）：提供数据发现、血缘和治理能力
- **Apache Atlas**：大数据生态的数据治理和血缘追踪
- **Alation**：商业数据目录，自动解析血缘
- **自研方案**：解析 SQL 日志 + ETL 脚本实现血缘推导，适用于定制化场景
- **云原生方案**：AWS Glue Data Catalog、阿里云 DataWorks

## 测绘工作的组织建议

1. 测绘不是一次性工作，应随系统变更持续更新
2. 优先覆盖核心业务系统和存有重要数据的系统
3. 将血缘数据嵌入数据开发流程（CI/CD 中集成血缘采集）
4. 建立"数据流-系统-责任人"的关联关系，便于问题溯源
