# 与 DLP/CASB 的联动

> 将分类分级结果与安全控制层联动，实现基于数据标签的自动化防护。

## 集成模式

数据分类分级平台与 DLP/CASB 的集成主要有三种模式：

1. **标签传递模式**：分类分级平台将数据标签（如"PII-敏感"、"重要数据"、"核心数据"）通过 API 或文件同步至 DLP/CASB，由后者基于标签执行控制策略
2. **策略联动模式**：分类分级结果变化时，自动通知 DLP/CASB 更新对应的保护策略（如某数据升级为核心数据，自动启用加密和阻断外发）
3. **事件闭环模式**：DLP/CASB 检测到数据泄露事件后，将事件上下文回传分类分级平台，触发数据资产重新评估和策略调整

## 策略同步工作流

```
数据分类分级系统                    DLP/CASB
      |                                |
  识别新敏感数据                     监控数据流动
      |                                |
  打标分类分级标签   --(API/文件)-->  加载标签信息
      |                                |
  标签变更通知     --(webhook)-->     更新策略规则
      |                                |
                                    执行保护动作:
                                    - 告警/阻断/加密
      |                                |
                                   上报事件信息
      |                      <--(API)--  
  触发资产重新评估
```

## 基于分类标签的自动化保护

| 数据标签 | DLP 策略 | CASB 策略 |
|---------|---------|----------|
| 核心数据 | 禁止外发、禁止复制、强制加密 | 禁止云存储上传、禁止外部共享 |
| 重要数据 | 外发审批、自动加密、水印标记 | 限制外部协作、审计所有操作 |
| 敏感PII | 脱敏外发、阻断未授权邮件 | 检测SaaS应用中的PII存储 |
| 一般内部 | 审计记录、提示标签 | 允许协作但记录日志 |
| 公开数据 | 不限制 | 不限制 |

## 安全事件富化

DLP/CASB 检测到的数据安全事件，可通过集成实现自动富化：

- **资产信息富化**：事件中的文件/数据属于哪个系统、哪个部门、什么业务？
- **分级信息富化**：泄露的数据是什么级别（核心/重要/一般）？
- **影响范围富化**：该数据资产关联多少个下游系统？

富化后的信息可大幅提升事件响应的精准度和效率：
- 一般数据泄露 -> 常规工单流程
- 重要数据泄露 -> 紧急工单 + 通知部门负责人
- 核心数据泄露 -> 安全事件应急响应 + 通报管理层

## SOAR 编排集成

数据安全事件处理可进一步与 SOAR 平台编排：

1. **检测**：DLP 告警触发
2. **富化**：自动查询数据资产地图，获取数据分级和资产信息
3. **响应**：根据分级自动执行 playbook（如核心数据泄露 -> 立即阻断 + 通知 + 取证）
4. **恢复**：事件处置后，触发数据资产重新评估流程

## 实施建议

- 优先实现标签传递，这是联动的基础
- 分阶段推进：先告警同步，再阻断联动，最后自动化编排
- 建立标签规范，确保分类分级系统和 DLP/CASB 使用统一的标签体系
- 定期验证联动有效性，确保策略同步未因系统变更而中断
