# 差分隐私 (Differential Privacy)

## ε-Differential Privacy 定义

**差分隐私 (Differential Privacy, DP)** 由 Cynthia Dwork 等人提出，是当前最严格的隐私保护定义之一。其核心思想是：在数据集 D 和其相邻数据集 D'（仅相差一条记录）上运行同一随机化算法 M，输出结果的分布差异受到 ε 的约束：

$$\\Pr[M(D) \\in S] \\leq e^\\varepsilon \\cdot \\Pr[M(D') \\in S]$$

参数 **ε (epsilon)** 控制隐私预算——ε 越小，隐私保护越强，但数据效用越低。ε=0 意味着完全保护隐私但输出不包含任何有用信息；ε 在 0.1 到 10 之间是常见选择范围。

## 核心机制

### 1. Laplace 机制 (Laplace Mechanism)
用于数值型查询。向查询结果添加服从 Laplace 分布的随机噪声，噪声尺度为 $\\Delta f / \\varepsilon$，其中 $\\Delta f$ 是查询的全局敏感度 (global sensitivity)。

- **适用**：计数、求和、均值等数值聚合查询。
- **噪声量**：与敏感度成正比，与 ε 成反比。

### 2. 指数机制 (Exponential Mechanism)
用于分类/非数值型查询。以概率选择输出结果，概率与效用函数值成指数关系。

- **适用**：排序、推荐、最佳答案选择等场景。
- **优势**：可以处理任意类型的输出空间。

## Local DP vs Central DP

| 维度 | Central DP | Local DP (LDP) |
|------|-----------|----------------|
| 信任模型 | 信任数据收集者 | 不信任数据收集者 |
| 噪声添加位置 | 聚合服务端 | 客户端（数据产生端） |
| 噪声量 | 较小（聚合后加噪） | 较大（单条数据加噪） |
| 数据效用 | 较高 | 较低 |
| 典型应用 | 美国人口普查 | Apple、Google、Microsoft 的遥测收集 |

## ε Budget 管理

ε budget 管理是 DP 系统设计的核心挑战：

- **组合定理 (Composition Theorem)** — 对同一数据集进行 n 次 DP 查询，总隐私损失为 $\\sum \\varepsilon_i$（顺序组合）或 $\\varepsilon \\cdot \\sqrt{n}$（高级组合）。
- **隐私会计 (Privacy Accounting)** — Rényi DP (RDP) 和 z-CDP 提供更精确的跟踪方法。
- **预算分配策略** — 为不同类型的查询分配独立的 ε budget，关键查询分配更多 budget。

## 实际应用

### Apple
Apple 在 iOS/macOS 中使用 Local DP 收集用户使用数据（表情符号使用、Safari 默认设置等），ε 约为 1-8。

### 美国 Census Bureau
2020 年人口普查使用 Central DP 发布汇总数据，使用 2010 Census 的"重构攻击"为驱动因素，总 ε budget 约为 16.73。

### Google
RAPPOR (Randomized Aggregatable Privacy-Preserving Ordinal Response) 系统使用 Local DP 收集 Chrome 使用数据。

**总结**：差分隐私提供了可量化的隐私保证，是当前隐私保护领域的黄金标准，尤其适合统计数据发布和联邦聚合场景。
