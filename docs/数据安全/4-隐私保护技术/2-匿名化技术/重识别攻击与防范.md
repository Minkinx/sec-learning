# 重识别攻击与防范

## 概述

**重识别攻击 (Re-identification Attack)** 是指攻击者通过将匿名化或脱敏后的数据集与外部辅助信息进行关联分析，重新识别出数据主体身份的技术手段。随着互联网数据的爆炸式增长和计算能力的提升，重识别攻击的成功率大幅提高，成为数据发布和共享场景下的核心安全威胁。

## 重识别攻击类型

### 1. 链接攻击 (Linkage Attack)

将脱敏数据集与外部可获取的辅助数据进行关联，通过 quasi-identifier（准标识符）匹配实现个体重新识别。这是最经典也是最早被系统性研究的重识别攻击方式。

**攻击流程**：
```
脱敏数据集 (含 ZIP、DOB、Gender)
         ↓
与外部数据集 (如 Voter Registry) 在 quasi-identifier 上做 JOIN
         ↓
匹配成功 → 建立 {个体身份 → 敏感信息} 映射
```

- **经典案例**：Latanya Sweeney 使用 Massachusetts 政府发布的医院数据（仅去除了姓名和 SSN）与 voter registry 通过 ZIP+出生日期+性别进行链接，成功识别出时任州长 William Weld 的医疗记录。该攻击直接推动了 k-anonymity 模型的提出。

### 2. 单出攻击 (Singling-Out Attack)

攻击者能够从数据集中将某个特定个体的记录与其他人区分开来，即使无法直接获取该个体的身份信息。单出攻击不需要链接外部数据，仅通过数据集内部的唯一性即可实施。

**判定条件**：若数据集中的某条记录具有唯一的属性组合（或属性组合在数据集中出现的次数极少），则该记录可以被"单出"。

**示例**：一个包含 {年龄, 职业, 邮编} 的数据集，其中"35岁, 核物理学家, 100081"的组合仅出现一次，攻击者可确定该记录指向特定个体。

### 3. 推断攻击 (Inference Attack)

攻击者通过分析数据集中的统计模式或相关性，推断出未直接公开的敏感属性值。

**子类型**：
| 类型 | 方法 | 示例 |
|------|------|------|
| 属性推断 | 利用属性间的统计相关性 | 根据邮编+职业推断收入水平 |
| 成员推断 | 判断某个个体是否在数据集中 | 确定某人是否在医疗数据库中 |
| 模型推断 | 从 ML 模型输出反推训练数据 | 从模型 API 提取训练数据特征 |

**成员推断攻击**尤其危险——即使个体的具体数据未被泄露，仅"某人存在于某数据集中"这一事实本身就可能造成隐私侵犯（如某人在 HIV 患者数据集中）。

### 4. 差分攻击 (Differencing Attack)

通过对同一数据集进行多次查询，从查询结果的细微差异推断个体信息。

```
查询 A: SELECT COUNT(*) FROM 医疗表 WHERE 疾病 = 'HIV'          → 100
查询 B: SELECT COUNT(*) FROM 医疗表 WHERE 疾病 = 'HIV' AND 姓名 != '张三' → 99
推断: 张三的疾病为 HIV
```

- **防御方法**：差分隐私通过向查询结果添加校准噪声来掩盖单条记录的影响。
- **防御强度**：ε 参数越小，隐私保护越强（通常 ε=1 为强保护，ε=10 为弱保护）。

### 5. 辅助数据攻击 (Auxiliary Data Attack)

利用日益丰富的外部数据源增强重识别能力：

| 辅助数据源 | 数据内容 | 对重识别的贡献 |
|-----------|----------|---------------|
| 社交媒体 | 姓名、位置、兴趣、社交关系 | 高（多维度交叉验证） |
| 公共记录 | 选民登记、房产信息、法庭记录 | 高（准标识符匹配） |
| 商业数据 | 消费记录、信用报告 | 中-高 |
| 数据泄露 | 已泄露的账号数据 | 极高 |
| 开放数据 | 政府开放数据集 | 中 |

## 真实案例分析

### Massachusetts 医疗数据泄露 (1997)

| 维度 | 详情 |
|------|------|
| 发布数据 | 马萨诸塞州政府发布的医院出院记录，约 13.5 万条 |
| 去除字段 | 姓名、具体地址、SSN |
| 保留字段 | ZIP code、出生日期、性别、诊断代码 |
| 辅助数据 | Cambridge 市选民登记册（公开可获取） |
| 攻击方法 | 以 {ZIP, DOB, Gender} 为 quasi-identifier 进行链接 |
| 攻击结果 | 成功识别时任州长 Weld 的医疗记录（该组合在选民册中唯一匹配） |
| 影响 | 直接催生了 k-anonymity 隐私保护模型 |

### AOL 搜索日志泄露 (2006)

AOL 发布了约 65 万用户的 2000 万条匿名化搜索查询记录，仅用数字编号替代了用户名。**纽约时报**记者通过分析搜索关键词的历史序列——包括姓名搜索、地址查询、兴趣词等上下文信息——成功识别出多位具体用户。其中最著名的案例是编号 4417749 的用户被确认为 Georgia 州的 Thelma Arnold 老妇人。

**教训**：简单的用户 ID 替换不足以保护隐私——搜索查询本身包含大量可用于识别的语义信息。

### Netflix Prize 数据集 (2007)

Netflix 发布了约 50 万用户的匿名化电影评分数据，每条记录仅包含 {用户编号, 电影, 评分, 日期}。**Narayanan 和 Shmatikov** 使用 IMDb 作为辅助数据集，通过评分模式匹配（用户对相同电影的评分对比），成功识别出部分 Netflix 用户，并推断其政治倾向、性取向等敏感信息。

**教训**：即使用户 ID 被替换，独特的评分模式足以作为"行为指纹"进行重识别。

### 医疗基因组数据重识别 (2013)

Gymrek 等人证明，公开的基因组数据（如 1000 Genomes Project）与公共记录（姓氏推断）结合，可以重新识别参与者的身份。攻击者通过分析 Y-STR 标记推断姓氏，辅以年龄、地理位置等信息锁定具体个体。

## 攻击面分析

| 攻击面 | 风险描述 | 风险等级 | 防御难度 |
|--------|----------|----------|----------|
| Quasi-identifier 组合 | 少量属性即可唯一标识大部分个体 | 高 | 中 |
| 罕见特征 | 稀有属性组合大幅增加识别概率 | 高 | 高 |
| 时序数据 | 时间戳序列提供额外识别维度 | 中-高 | 高 |
| 高维数据 | "维度诅咒"使高维数据几乎天然唯一 | 高 | 极高 |
| 外部数据交叉 | 互联网数据极大丰富辅助数据来源 | 持续增长 | 中 |
| 行为模式 | 评分、浏览、消费等行为的独特模式 | 高 | 高 |

## 缓解策略

### 技术防御措施

1. **数据最小化** — 仅收集和发布满足业务需求的最少数据，从源头降低风险。
2. **强匿名化处理** — 采用 k-anonymity + l-diversity + t-closeness 组合模型。
3. **差分隐私** — 对查询结果添加校准噪声，提供严格的数学隐私保证。
4. **发布前审计** — 自动评估重识别风险后再发布数据集。
5. **隐私预算控制** — 限制发布查询的次数和精度（特别是在统计数据库场景）。

### 流程与管理措施

6. **数据使用协议** — 通过法律手段约束数据接收方的使用行为。
7. **访问控制** — 实施最小权限原则，限制数据访问范围。
8. **数据沙箱** — 在受控环境中提供数据访问，禁止导出原始数据。
9. **定期风险评估** — 建立持续的风险评估机制。

### 技术防御矩阵

| 攻击类型 | k-Anonymity | l-Diversity | 差分隐私 | 访问控制 | 数据最小化 |
|----------|-------------|-------------|----------|----------|-----------|
| 链接攻击 | 防御 | 防御 | 防御 | 部分防御 | 预防 |
| 单出攻击 | 防御 | 防御 | 防御 | 不适用 | 预防 |
| 推断攻击 | 部分防御 | 部分防御 | 防御 | 不适用 | 预防 |
| 差分攻击 | 不防御 | 不防御 | 防御 | 不适用 | 不适用 |
| 辅助数据攻击 | 部分防御 | 部分防御 | 防御 | 不适用 | 预防 |

## 去匿名化风险评分 (De-anonymization Risk Scoring)

建议建立系统化的风险评估框架：

```
Risk Score = α × 数据敏感度 + β × 辅助数据可用性 + γ × 属性唯一性 + δ × 攻击者动机
```

### 评分维度详解

| 维度 | 低分 (0-2) | 中分 (3-5) | 高分 (6-8) | 极高 (9-10) |
|------|-----------|-----------|-----------|------------|
| 数据敏感度 | 公开信息 | 联系方式 | 医疗/金融数据 | 生物识别/基因数据 |
| 辅助数据可用性 | 无公开辅助数据 | 少量付费数据 | 政府开放数据 | 社交媒体+商业数据 |
| 属性唯一性 | 所有属性泛化 | 部分泛化 | 原始粒度 | 含罕见/高维属性 |
| 攻击者动机 | 无明确动机 | 一般好奇 | 经济利益 | 政治/恶意目的 |

### 实施建议

- 设定阈值（如总分 ≥ 25）触发额外的保护措施或限制发布
- 每季度对数据资产进行一次风险评分
- 评分结果纳入数据治理委员会的审批流程
- 高风险数据集需进行 Data Protection Impact Assessment (DPIA)
