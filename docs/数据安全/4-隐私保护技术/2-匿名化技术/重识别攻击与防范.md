# 重识别攻击与防范

## 重识别攻击类型

### 1. 链接攻击 (Linkage Attack)
将脱敏数据集与外部可获取的辅助数据进行关联，通过 quasi-identifier 匹配实现个体重新识别。

- **经典案例**：Sweeney 使用 Massachusetts 政府发布的医院数据（仅去除了姓名、SSN）与 voter registry 通过 ZIP+出生日期+性别进行链接，成功识别出时任州长 Weld 的医疗记录。

### 2. 差分攻击 (Differencing Attack)
通过对同一数据集进行多次查询（如"总人数"和"排除某人后的总人数"），从查询结果的差异推断个体信息。

- **防御方法**：差分隐私通过添加噪声来掩盖单条记录的影响。

### 3. 辅助数据攻击 (Auxiliary Data Attack)
利用外部数据源（社交媒体、公共记录、商业数据）增强重识别能力。随着互联网数据的爆炸式增长，辅助数据的可用性大幅提升了攻击成功率。

## 真实案例

### AOL 搜索日志泄露 (2006)
AOL 发布了约 65 万用户的 2000 万条匿名化搜索查询记录，仅用用户编号替代了用户名。**纽约时报**通过分析搜索关键词（如姓名、地址、兴趣）成功识别出具体用户。

### Netflix Prize 数据集 (2007)
Netflix 发布了约 50 万用户的匿名化电影评分数据，仅保留用户编号和评分记录。**Narayanan 和 Shmatikov** 使用 IMDb 作为辅助数据，通过评分模式匹配成功识别出部分用户。

### Massachusetts 医疗数据 (1997)
前面提到的 Sweeney 攻击案例，直接推动了 k-anonymity 模型的提出。

## 攻击面分析

| 攻击面 | 风险描述 | 风险等级 |
|--------|----------|----------|
| Quasi-identifier 组合 | 少量属性即可唯一标识大部分个体 | 高 |
| 罕见特征 | 数据集中罕见的属性组合大幅增加识别概率 | 高 |
| 时序数据 | 时间戳序列提供了额外的识别信息 | 中-高 |
| 高维数据 | "维度诅咒"使得高维数据几乎天然唯一 | 高 |
| 外部数据交叉 | 互联网数据极大丰富了辅助数据来源 | 持续增长 |

## 缓解策略

1. **数据最小化** — 仅收集和发布满足业务需求的最少数据。
2. **隐私预算控制** — 约束发布查询的次数和精度。
3. **强匿名化处理** — 使用差分隐私或 k-anonymity 组合模型。
4. **发布前审计** — 自动评估重识别风险后再发布。
5. **数据使用协议** — 通过法律手段约束数据使用行为。

## 去匿名化风险评分

建议建立 **去匿名化风险评分 (De-anonymization Risk Score)** 框架：

```
Risk Score = f(数据敏感度, 辅助数据可用性, 属性唯一性, 攻击者动机)
```

定期对数据集进行风险评分，设定阈值触发额外的保护措施或限制发布。风险评分应作为数据治理流程的一部分持续运行。
