# l-Diversity

## 从 k-Anonymity 到 l-Diversity

k-anonymity 无法抵御同质性攻击和背景知识攻击。**l-diversity** 由 Machanavajjhala 等人提出，要求每个等价类中敏感属性至少包含 l 个"well-represented"的值。即在满足 k-anonymity 的基础上，进一步提升等价类内敏感属性的多样性。

## l-Diversity 的三种形式

### 1. Distinct l-Diversity
最简单形式：每个等价类中敏感属性的不同取值数量至少为 l。

- **优势**：实现简单，计算成本低。
- **劣势**：不考虑值的分布频率。一个等价类中即使有 l 个不同值，若某个值占主导地位（90%+），隐私保护仍然不足。

### 2. Entropy l-Diversity
更严格的形式：每个等价类的敏感属性熵值至少为 log(l)。

- **公式**：$-\\sum_{s \\in S} p(s) \\log p(s) \\geq \\log(l)$
- **优势**：考虑了敏感值的分布均匀性。
- **劣势**：熵值阈值难以直观解释，实际部署中参数选择困难。

### 3. Recursive (c,l)-Diversity
要求等价类中出现频率最高的敏感值不超过其他值出现频率之和的 c 倍，且至少有 l 个不同的敏感值。

- **优势**：对分布偏斜敏感，能较好地检测主导值问题。
- **劣势**：参数 c 和 l 的选择需要领域知识和反复调优。

## 局限性

### 偏斜攻击 (Skewness Attack)
当敏感属性的整体分布本身偏斜时，即使满足 l-diversity，攻击者也能获得有意义的推断。例如，某疾病数据集中 99% 的记录为"阴性"，即使等价类包含"阳性"和"阴性"两个值，攻击者仍可高置信度推断个体为"阴性"。

### 相似性攻击 (Similarity Attack)
如果等价类中的敏感值在语义上相似（如薪资全部在 50k-55k 之间），即使有不同的取值，攻击者仍能精确推断。l-diversity 不考虑敏感值之间的语义距离。

## 演进

为克服 l-diversity 的局限性，后续提出了：

- **t-Closeness** — 要求等价类中敏感属性的分布与整体分布的距离不超过 t。
- **(α,k)-Anonymity** — 在 k-anonymity 基础上，限制任何等价类中敏感值出现频率不超过 α。

**核心建议**：l-diversity 是 k-anonymity 的重要补充，但需结合实际数据分布选择合适的变体，并与 t-closeness 等高级模型组合使用。
