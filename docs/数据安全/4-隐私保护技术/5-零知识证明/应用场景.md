# zk-SNARKs / zk-STARKs 应用场景

## 身份验证

零知识证明在身份验证领域的核心价值是 **"证明你知道而无须说出你知道什么"**。

### 年龄验证
- **场景**：用户需要证明自己年满 18 岁但不暴露具体出生日期。
- **实现**：政府签发一个包含出生日期的签名数字凭证，用户通过 zk-proof 证明 "birth_date ≤ 2008-05-26"，输出 age ≥ 18。
- **优势**：服务端无需收集用户生日信息，降低数据泄露风险。

### KYC 合规证明
- **场景**：用户在多个金融平台开户，每个平台都要求完整的 KYC 资料。
- **实现**：用户完成一次 KYC 审核后获得签名凭证，后续使用 zk-proof 向其他平台证明 "某个受信任的 KYC 审核方已验证了我的身份"。
- **优势**：减少重复提交敏感 KYC 文档，降低个人信息泄露面。

## 合规证明

### 信用评分证明
- **场景**：用户向贷款平台证明自己的信用评分 ≥ 700，但不暴露确切分数。
- **实现**：信用机构签发加密信用报告，用户生成 zk-proof 证明评分满足阈值要求。

### 收入证明
- **场景**：租客向房东证明月收入 ≥ 特定金额。
- **实现**：雇主签名工资记录，租客生成 zk-proof 证明收入满足要求。

### 零知识健康证明
- **场景**：入境旅客向海关证明已接种指定疫苗，但不暴露完整医疗记录。
- **实现**：医疗机构签发疫苗凭证，旅客生成选择性的 zk-proof 仅披露必要信息。

## 可验证计算 (Verifiable Computation)

### 可信数据聚合
- **场景**：多个数据源提供数据，第三方聚合计算统计结果。
- **实现**：每个数据源签署自己的数据，聚合者使用 zk-proof 证明"所有数据源的签名有效且聚合计算正确"。

### 去中心化预言机
- **场景**：区块链需要从链下获取数据，但需确保数据未被篡改。
- **实现**：预言机节点生成 zk-proof 证明数据来源于可信源且未被篡改。

## 去中心化身份 (DID)

**去中心化身份 (Decentralized Identity, DID)** 是零知识证明的重要应用领域：

- **DID Document** — 用户持有私钥控制自己的数字身份，无需中心化认证机构。
- **Verifiable Credential (VC)** — 发行方（政府、学校、雇主）签发经过签名的数字凭证。
- **Zero-Knowledge Presentation** — 持有者有选择性地披露 VC 中的部分信息，使用 zk-proof 证明其余属性的合规性。

### 典型的 DID + ZK 流程

```
Issuer → 签发 VC (签名) → Holder
Holder → 生成 Presentation (zk-proof) → Verifier
Verifier → 验证签名 + zk-proof → 合规或拒绝
```

## 凭证展示 (Credential Presentation)

### Selective Disclosure
从包含多属性的凭证中选择性地公开部分属性。例如：凭证包含 {姓名, 年龄, 国籍, 地址}，仅向酒吧出示 {年龄 ≥ 21} 的证明。

### BBS+ Signatures
BBS+ 签名支持在多消息签名上进行选择性地披露，与 zk-proof 配合使用可以高效实现凭证的选择性展示。该方案已被纳入 W3C Verifiable Credentials 标准。

**总结**：零知识证明在不同行业中有着广泛的应用前景，从金融合规到数字身份，从隐私交易到数据验证。随着证明效率的提升和开发工具的成熟，zk-proof 正从学术研究走向大规模生产部署。
