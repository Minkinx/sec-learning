# 脱敏效果评估

## 评估维度

脱敏效果评估需在三个相互制约的维度间取得平衡：

1. **隐私保护程度 (Privacy Protection Degree)** — 脱敏数据抵抗重识别攻击的能力。
2. **数据可用性 (Data Utility)** — 脱敏数据在分析、测试中的实用价值。
3. **性能 (Performance)** — 脱敏过程的计算开销和延迟。

这三个维度构成"隐私-效用-性能三角"，优化其中一个维度通常以牺牲另外两个为代价。

## 重识别风险评估指标

### 1. 记录链接成功率 (Record Linkage Success Rate)
模拟攻击者将脱敏数据与外部数据集链接的能力。成功率越低说明保护效果越好。

### 2. 属性推断准确率 (Attribute Inference Accuracy)
评估攻击者能否根据脱敏数据准确推断未公开属性的值。准确率接近随机猜测为理想。

### 3. 唯一性比率 (Uniqueness Ratio)
脱敏后数据集中唯一记录的比例。唯一性越高，重识别风险越大。

### 4. k-anonymity 达标率
评估脱敏数据集满足 k-anonymity 的记录比例。k 值越大，隐私保护越强。

## 信息损失度量

### 通用度量指标
- **IL1 (Information Loss 1)** — 基于分类精度的信息损失。
- **IL2** — 基于统计分布差异的信息损失。
- **DM (Discernibility Metric)** — 度量泛化后等价类的大小分布。

### 统计特征保留度
| 指标 | 描述 | 理想值 |
|------|------|--------|
| 均值偏差 | 脱敏前后的均值差异 | ≤5% |
| 方差保持率 | 脱敏前后的方差比率 | ≥90% |
| 相关性偏差 | 脱敏前后的列间相关系数变化 | ≤0.1 |
| 分布一致性 | KS 检验 p 值 | ≥0.05 |

## k-anonymity 验证

验证脱敏数据集是否满足 k-anonymity 的步骤：

1. **识别 quasi-identifier** — 确定可被攻击者用于链接的属性集合。
2. **统计等价类** — 按 quasi-identifier 分组，统计每组记录数。
3. **检查最小值** — 若所有组的大小均 ≥ k，则数据集满足 k-anonymity。
4. **覆盖率分析** — 计算满足 k-anonymity 的记录占总记录的比例。

## 基准测试方法

建立脱敏效果基准测试的推荐流程：

```bash
1. 准备原始数据集和对应的脱敏数据集
2. 运行重识别攻击模拟（链接攻击、差分攻击）
3. 计算隐私度量指标（k-anonymity、l-diversity）
4. 计算可用性度量指标（IL1、IL2、统计特征保持率）
5. 生成评估报告，标记未达标的脱敏策略
```

**最佳实践**：将脱敏效果评估工具集成到 CI pipeline 中，每次脱敏策略变更后自动执行评估并对比基线。
