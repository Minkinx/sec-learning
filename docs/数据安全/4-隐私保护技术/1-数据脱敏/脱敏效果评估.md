# 脱敏效果评估

## 评估维度概述

脱敏效果评估需在三个相互制约的维度间取得平衡，构成"隐私-效用-性能三角"：

1. **隐私保护程度 (Privacy Protection Degree)** — 脱敏数据抵抗重识别攻击的能力。
2. **数据可用性 (Data Utility)** — 脱敏数据在分析、测试和建模中的实用价值。
3. **性能开销 (Performance Overhead)** — 脱敏过程的计算资源消耗和时间延迟。

优化其中一个维度通常以牺牲另外两个为代价。评估的目标不是追求单一维度的最优，而是在业务需求场景下找到可接受的最佳平衡点。

```
                   隐私保护
                     /\
                    /  \
                   /    \
                  / 最佳 \
                 /  平衡  \
                /   点    \
               /__________\
          数据可用性 ------ 性能开销
```

## 隐私保护度量指标

### 重识别风险评估指标

| 指标 | 定义 | 计算方法 | 理想值 |
|------|------|----------|--------|
| 记录链接成功率 | 模拟攻击者成功将脱敏记录与外部数据链接的比例 | 成功链接数 / 总记录数 | <5% |
| 属性推断准确率 | 攻击者正确推断敏感属性的比例 | 正确推断数 / 总尝试数 | 接近随机基线 |
| 唯一性比率 | 脱敏后数据集中唯一记录的比例 | 唯一记录数 / 总记录数 | <1% |
| k-anonymity 达标率 | 满足 k-anonymity 的记录比例 | 达标记录数 / 总记录数 | 100% |
| l-diversity 达标率 | 满足 l-diversity 的等价类比例 | 达标等价类数 / 总等价类数 | 100% |

### 量化风险评估模型

**Marketer 模型**：假设攻击者拥有与目标数据集相同的准标识符属性，评估在最优攻击策略下的重识别概率。

```
P(重识别 | 脱敏数据集 D, 辅助数据 A) = 
    Σ 对于D中每条记录r，若A中有唯一匹配则视为成功 / |D|
```

** Prosecutor 模型**：假设攻击者知道目标个体在数据集中，攻击者利用辅助数据中的目标记录尝试在脱敏数据集中找到匹配。

### 具体计算示例

假设脱敏后的数据集 D 包含 10000 条记录，辅助数据集 A 包含 8000 条记录，共有 4 个准标识符属性。评估流程：

1. 对 D 按准标识符分组，计算每组的记录数（等价类大小）
2. 对 A 中的每条记录，尝试在 D 中找到准标识符匹配
3. 若 A 中的一条记录在 D 中匹配到唯一一条记录，则视为成功链接
4. 统计成功链接数，计算链接成功率

| 场景 | 等价类最小大小 | 唯一记录数 | 链接成功率 | 风险评估 |
|------|--------------|-----------|-----------|----------|
| 未脱敏 | 1 | 10000 | 100% | 极高 |
| k=2 匿名化 | 2 | 1200 | 35% | 高 |
| k=5 匿名化 | 5 | 80 | 5% | 中 |
| k=10 匿名化 | 10 | 0 | <1% | 低 |

## 数据可用性度量

### 信息损失度量指标

#### 通用度量指标

| 指标 | 缩写 | 计算方法 | 说明 |
|------|------|----------|------|
| Information Loss 1 | IL1 | 基于分类精度的信息损失 | 值域泛化程度 |
| Information Loss 2 | IL2 | 基于统计分布差异的信息损失 | 泛化后熵的损失 |
| Discernibility Metric | DM | 度量等价类的大小分布 | DM = Σ|E_i|² |
| Global Certainty Penalty | GCP | 全局确定性惩罚 | 1 - (泛化精度/原始精度) |

#### DM 实例计算

考虑一个满足 k=5 匿名化的数据集，等价类分布为：{5, 6, 5, 8, 7, 5, 10, 5}。

```
DM = 5² + 6² + 5² + 8² + 7² + 5² + 10² + 5²
   = 25 + 36 + 25 + 64 + 49 + 25 + 100 + 25
   = 349
```

DM 值越小，等价类越均匀，信息损失越低。

### 统计特征保留度

| 指标 | 描述 | 计算方法 | 理想值 |
|------|------|----------|--------|
| 均值偏差 | 脱敏前后的均值差异 | \|μ_原始 - μ_脱敏\| / μ_原始 × 100% | ≤5% |
| 方差保持率 | 脱敏前后的方差比率 | σ²_脱敏 / σ²_原始 × 100% | ≥90% |
| 相关性偏差 | 脱敏前后的列间相关系数变化 | \|ρ_原始 - ρ_脱敏\| | ≤0.1 |
| 分布一致性 | KS 检验 p 值 | KS 双样本检验 | ≥0.05 |
| 排序保持率 | 脱敏前后值排序的一致性 | Kendall's τ | ≥0.85 |

### 分类器效用评估

对于用于 ML 的数据集，评估脱敏后数据训练出的模型性能：

```
效用保持率 = 脱敏数据训练模型的 Accuracy / 原始数据训练模型的 Accuracy × 100%
```

| 场景 | 原始 Accuracy | 脱敏后 Accuracy | 保持率 |
|------|-------------|----------------|--------|
| 弱脱敏 | 92% | 88% | 95.7% |
| 中等脱敏 | 92% | 82% | 89.1% |
| 强脱敏 | 92% | 68% | 73.9% |

一般要求效用保持率 ≥ 85%。

## 评估方法论

### 定量评估流程

推荐采用以下系统化的评估流程：

```
阶段 1: 基线测量
  ├── 分析原始数据的统计特征
  ├── 测量准标识符的唯一性
  └── 评估原始数据的重识别风险

阶段 2: 脱敏后测量
  ├── 计算隐私保护指标
  ├── 计算信息损失指标
  └── 执行模拟攻击测试

阶段 3: 综合评分
  ├── 加权综合评分
  ├── 与业务需求对标
  └── 输出评估报告

阶段 4: 持续监控
  ├── 纳入 CI/CD pipeline
  ├── 定期重新评估
  └── 策略调优迭代
```

### 合成评分模型

建议采用加权综合评分方法，根据业务场景调整权重：

```
综合评分 = W₁ × 隐私保护分 + W₂ × 数据可用性分 + W₃ × 性能分

典型权重分配:
- 高安全场景: W₁=0.6, W₂=0.3, W₃=0.1
- 数据分析场景: W₁=0.3, W₂=0.5, W₃=0.2
- 均衡场景: W₁=0.4, W₂=0.4, W₃=0.2
```

```yaml
# .gitlab-ci.yml 脱敏效果评估 Stage
mask_validation:
  stage: validate
  script:
    - pip install anonymeter pandas
    - anonymeter evaluate \
        --original ./data/raw.csv \
        --masked ./data/masked.csv \
        --quasi-idents age,zip,gender \
        --sensitive disease
    - anonymeter report --format html --output ./reports/
  artifacts:
    paths:
      - ./reports/mask-validation.html
    expire_in: 30 days
```

## 合规性评估

脱敏效果是否满足具体合规要求：

| 合规标准 | 对脱敏的要求 | 验证方法 |
|----------|-------------|----------|
| GDPR Art. 4(5) | 匿名化后数据不属于个人数据 | 重识别风险评估 |
| HIPAA Privacy Rule | Expert Determination 或 Safe Harbor | Safe Harbor: 移除 18 种标识符 |
| PCI DSS Req. 3.4 | PAN 不可读存储 | 验证掩码/令牌化/加密 |
| PIPL (个人信息保护法) | 匿名化后不可识别 | 重识别风险评估 |
| GB/T 35273 | 去标识化处理效果评估 | 发布前脱敏效果评估报告 |
